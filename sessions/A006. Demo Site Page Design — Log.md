## What was built

A cloud **design / planning** session — deliverables are documents, not application code. It designs the public website the demo install ships with (a pretend-nonprofit site over the `demo:reset` data) and drafts the local build session that will author it.

### Phase 1 — Org identity + page list
- Chose the fictional org: **Greenline Community Trust**, a community land/parks trust. Picked because it maps cleanly onto every entity `RandomDataGenerator` already produces (events = volunteer days/fundraisers; donations = gifts; memberships = "Friends of Greenline"; news posts = trail-log updates; products = a small store). Voice: warm, plain-spoken, signed-by-a-person (Ellen Park, ED). Visual feel: outdoors/parks/community sample-image categories.
- Finalized the 7-entry nav: six curated pages (`home`, `events`, `donate`, `about`, `contact`, `news`) + the reused `/demo` dark-side page. Slugs aligned to the generator's URL shapes (posts live under `news/…`).

### Phase 2 — Per-page composition + copy
- Specced each page band-by-band: layout type/columns/background/padding, exact widget **handles**, config values, `query_config` data contracts, and realistic copy (mission, giving levels, board bios, headers). Handles + config/query schemas verified by reading `app/Widgets/` (Hero, EventsListing, EventCalendar, EventDescription, EventRegistration, DonationForm, ThreeBuckets, RecentDonations, BoardMembers, TextBlock, Image, WebForm, MapEmbed, BlogListing, BlogPager, Nav, Logo, SocialSharing).
- Built the data-alignment table: which widgets read generated data vs. what the build session must seed beyond the generator.

### Phase 3 — Nav, chrome, `/demo` integration, reset wiring
- Chrome is global (not per-page); spec instructs the build to use the production chrome defaults at build time (E16 may land first).
- `/demo` integration: clip the single page from `sessions/public website/demo.json` into the demo bundle verbatim. Kept deliberately simple.
- Reset wiring designed precisely against the real code path: `demo:reset` → `migrate:fresh --seed` → `DemoBaselineSeeder` (wipe + `RandomDataGenerator->generate`) → **add** a `ContentImporter::import()` step (after CRM generate, super-admin auth active) loading a deployed runtime copy of the bundle (`import_pages` + `replace_duplicate_pages` + `import_navigation` + `import_collections`). Placing it in `DemoBaselineSeeder` covers both full and `--soft` resets.

### Phase 4 — Deliverables
- `sessions/public website/demo-site-pages-spec.md` — the design spec (canonical, build-ready).
- `sessions/demo-site-pages-build.PROMPT-DRAFT.md` — the drafted local build-session prompt (no session number; user slots it).
