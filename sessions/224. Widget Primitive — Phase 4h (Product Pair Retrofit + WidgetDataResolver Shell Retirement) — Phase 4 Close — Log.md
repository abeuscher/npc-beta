### Notable findings

- **First nested DTO projection landed without extending the contract grammar.** The decision (in the session prompt) was to fix the sub-row shape (`['id', 'label', 'amount']`) inside the projector and treat `prices` as a single boolean in the flat `fields` whitelist. This matches the YAGNI of contract-grammar generalization: until a second nested case demands it (e.g. a future widget that wants different sub-row fields than ProductCarousel/ProductDisplay), the recursive `fields: ['prices' => [...]]` declaration stays deferred. The right time to formalize is when a second case lands; the contract layer can absorb that extension without breaking existing nested projections.
- **List-shaped + single-row dispatch for the same model in one beat.** ProductCarousel (`MANY`) and ProductDisplay (`ONE`) both target the `'product'` model. The cardinality-first dispatch added in 223 absorbed the new arm cleanly — `resolveSystemModel()` matches on cardinality, then on model. No special casing required. This is the architectural beat that propagates forward into every Phase 5 record-detail-sidebar widget that wants a parent record + a list of children.
- **Two ProductDisplay widgets on one page hit one query.** Per-request slug-keyed cache `'product:one:' . $slug` (mirror of 223's event-side cache). Confirmed by the resolver's batching design — same DataContract filters → same cache key → one fetch.
- **`ProductCarousel`'s `widget_type_collections` API field was already empty.** Asserted in the seeder test that survived the migration. No `WidgetTypeSeeder` re-run was required this session — neither Product widget changed its `collections()` declaration (both already `[]` from the parent default).
- **Stale comment-only `WidgetDataResolver` references caught by the Phase 1 audit.** Two PHPDoc comments in `DemoDataService.php` and `WidgetContentTypeProjector.php` referenced `WidgetDataResolver` only in their docstrings — not as call sites. Pre-existing artifacts of an earlier architecture; updated in-session to drop the stale reference because they would otherwise rot in future grep audits and confuse readers.
- **`-d memory_limit=2G` override worked single-pass for the ninth consecutive session.** The pre-existing mid-suite OOM (Spatie GD image processing) is fully sidestepped. Whether to bake into project tooling remains a separate question.
- **`FakeImportCsvGeneratorTest` flake observed for the seventh consecutive Phase 4 session.** Stochastic full-suite-only; passes in isolation. Pre-existing latent global-state leak in fake-faker isolation between consecutive seeded runs. Not widget-primitive-track work.
