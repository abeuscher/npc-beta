# 050. Roadmap Planning & Prioritisation

A planning-only session. No code is written. The output is a revised `session-outlines.md` that reflects the current state of the project, Al's notes from testing, and agreed priorities for the next phase of work.

---

## Before doing anything else

Read the following files in full:

- `docs/schema.md` — current database state
- `sessions/session-outlines.md` — the existing roadmap
- `sessions/042. Codebase Audit — Fields, Schema, Permissions & Help Coverage — Log.md`
- `sessions/049. Codebase Audit — Fields, Schema, Permissions & Help Coverage — Log.md`

Take notes. Build a clear picture of:
- What models, features, and systems are fully built
- What is partially built or has known gaps (noted in logs as deferred)
- What is planned but not yet started (session stubs in outlines)

Summarise your understanding before asking for Al's notes.

---

## Session flow

**Step 1 — Orient.** Read the files above. Summarise the current feature state in a short structured list: Built / Partial / Planned. Confirm you are ready for Al's input.

**Step 2 — Receive Al's notes.** Al will paste or describe additions, corrections, and priority changes from his own testing notes. Listen, ask clarifying questions if anything is ambiguous, and do not begin rewriting until you have the full picture.

**Step 3 — Discuss grouping and ordering.** Work through the combined list together. The goal is a roadmap ordered by dependency and value — not arbitrary. Raise any sequencing constraints you notice (e.g. Member Portal before Household self-service, Stripe Webhooks needs payload spec first).

**Step 4 — Rewrite `session-outlines.md`.** Once ordering is agreed, rewrite the future sessions section of `session-outlines.md` to reflect the new roadmap. Keep the Completed Sessions table unchanged. Stubs that already have good descriptions keep them; new items get a one-paragraph description written during the session.

---

## Out of scope

- Writing any code
- Designing implementation details for specific sessions (save that for the session itself)
- Committing to timelines or estimates
