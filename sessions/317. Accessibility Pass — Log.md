## What was built

### Audit half — three sweeps

- **Aria sweep** (mechanical) across public render, page-builder Vue editor,
  admin/Filament custom markup, and the portal. Surface inventory produced a
  findings table grouped by surface (A = public, B = page builder, C = admin +
  portal). Two inbox items the prompt folded in were verified **already
  resolved at s315** and required no work: the inline-formatting-toolbar
  `title`/`aria-label` (present + correct) and the page-builder ellipsis-menu
  parity (header actions auto-render via `<x-filament-panels::page>`, confirmed
  in `EditPage::getHeaderActions()`).
- **Usability sweep** (judgment) over operator-facing controls. The two named
  inbox items were verified: the columns-primitive alignment labels (real —
  owner deferred, see below) and the ellipsis-menu parity (already resolved).
  One agent-flagged "blocking" finding (ellipsis menu missing on the builder
  edit view) was a **false positive** — disproved by reading the code.
- **Lighthouse sweep** (tooling) via the headless `lighthouse` CLI against the
  Playwright Chromium, public pages + admin login. Baseline captured (first
  baseline for this project). `/news` 404s locally (no seeded blog index on the
  dev box — not auditable, not a finding). The recurring console error on every
  page is a stale local widget-bundle 404, an environment state, not a defect.

### Remediation half — four iterations on `session-317/1`

- **/1 — Aria (public render).** Skip-to-content link + `id="main"` on the
  public layout (focus-revealed `.skip-link` rule in `_base.scss`); logo link
  always carries an accessible name (img `alt` falls back to app name, or
  decorative `""` when wordmark text is shown); ProductCarousel prev/next
  `aria-label`; MapEmbed + VideoEmbed iframe `title`; MapEmbed activate overlay
  becomes a keyboard-operable `role="button"` (Enter/Space); SocialSharing
  icon links/buttons `aria-label`. `npm run build`.
- **/2 — Aria (page-builder editor).** Inspector leaf controls
  (text/url/textarea/number/select) carry `aria-label` from their field label
  (the visible `<label>` is a sibling with no `for`); InspectorTabs gets
  `role="tablist"`/`role="tab"`/`aria-selected`; icon-only buttons gain
  `aria-label` (InspectorHeader rename/reset/delete, ButtonListField +
  RepeaterField move/remove, ColorPicker compact trigger, LayoutRegion selector,
  PreviewCanvas viewport presets + `aria-pressed`); ButtonListField per-row
  inputs labelled; the redundant hover layout-handle marked `aria-hidden` (its
  action is keyboard-reachable via the selector button). `npm run build`.
- **/3 — Aria (admin + portal).** DesignSystem page tabs get
  `role="tablist"`/`role="tab"`/`aria-selected`; per-field `@error` spans across
  PortalSignup / reset-password / EventRegistration guest / invitation-set-
  password gain `id`s + conditional `aria-describedby`/`aria-invalid` on their
  inputs; PortalChangePassword + invitation password-confirm wire
  `aria-describedby` to their JS "Passwords do not match" alert. Incidental bug
  fixed: a duplicate `class` attribute on PortalChangePassword's match-error div.
  PortalLogin / PortalForgotPassword / PortalContactEdit were already adequate
  (labelled inputs + top-of-form `role="alert"` summary) and left unchanged.
  Blade-only, no build.
- **/4 — Tooling.** Added `@axe-core/playwright` + `tests/e2e/accessibility.spec.ts`,
  a standing public-surface guard: axe-core on home + blog index, WCAG 2 A/AA,
  asserting zero violations. `color-contrast` disabled (out of scope, Design
  System track); the admin Filament shell is deliberately not scanned (stock-
  framework violations this project doesn't author). Both pages green on the
  isolated e2e stack.

  item remains. B7 PreviewRegion drag handle (mouse affordance; aria-label
  present). C4 admin `aria-dialog-name` + positive `tabindex` (stock Filament
  markup, not project-authored).
