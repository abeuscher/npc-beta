## What was built

### Phase 1 — Bug fixes

**Bug 1 — Header/logo not showing in theme**
- Root cause: `_header` and `_footer` system pages were created by the seeder without any widgets. ChromeRenderer returned null, falling through to a legacy component that depended on retired SiteSettings.
- Fix: Updated `SystemPageSeeder` to add `site_header` and `site_footer` widgets to the chrome pages on seed. Re-ran the seeder to fix existing installs.

**Bug 2 — Blog post publish dates**
- `Page::scopePublished()` only checked `status = 'published'`. Posts with future `published_at` dates were visible.
- Fix: Added `published_at <= now()` condition to the scope (null published_at still allowed for backwards compat). All public controllers, sitemap, and widget data resolver use this scope.
- Tests: 2 new tests — future-dated post returns 404 on direct URL, future-dated post excluded from listing query.

**Bug 3 — Unpublished page links from admin**
- The "View page" link in EditPage and EditPost always rendered a clickable URL regardless of page status.
- Fix: Draft pages now show a disabled, grayed-out link with a native `title` tooltip ("Page not published" / "Post not published"). No Livewire round-trip, no side effects on other page elements.

**Bug 4 — Convert forms to Tailwind**
- Already done — no Pico CSS remnants found anywhere in the codebase. No changes needed.

**Bug 5 — Event date UX**
- Replaced the start/end DateTimePicker pair with a decomposed UI: DatePicker + hour/minute/AM-PM dropdowns for start time, plus hours/minutes/all-day duration controls.
- Start Time and Duration fieldsets display side by side (7/5 column split).
- `ends_at` is computed from `starts_at + duration` in `EventResource::computeEndsAt()`, called from both CreateEvent and EditEvent mutators.
- EditEvent decomposes stored `starts_at`/`ends_at` back into the virtual fields on load.
- Restored Filament's built-in JS date picker (removed project-wide `->native()` override).
- Custom CSS (`tight-grid` class) for compact flex layout on the Start Time fieldset.
- Event form restructured: Event Details, Location Info (collapsed), and Online Meeting Info (collapsed) in a left column; Registration Details in a right column.

**Bug 6 — CMS Settings breakpoints layout**
- Changed the breakpoint repeater from `->columns(1)` to `->grid(5)` for horizontal tiling.

**Bug 7 — Horizon endpoint access control**
- Published `config/horizon.php`.
- Added `Horizon::auth()` gate in AuthServiceProvider — checks `horizon_enabled` site setting (default: disabled) and requires super_admin role.
- Added "Enable Horizon dashboard" toggle to General Settings (super_admin only).
- Tests: 4 new tests — disabled returns 403, non-super-admin returns 403, enabled + super_admin returns 200, unauthenticated returns 403.

**Bug 8 — Widget inspector auto-open**
- `PageBuilder::createBlock()` now calls `$this->selectBlock($newBlock->id)` after creating a new widget, which dispatches the `block-selected` event and opens the inspector.

### Additional changes

- **Checkbox/radio visibility** — Added CSS override for checkbox and radio ring color (#333) in light mode.
- **Admin CSS** — Added flex-based layout override for the Start Time fieldset to allow precise field sizing without grid quantization.
