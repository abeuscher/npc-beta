## Bug fixed during session

**Blog prefix false-positive validation error** ‚Äî the `blog_prefix` validation rule in `CmsSettingsPage` was checking `CmsPage::where('slug', $value)->exists()` without excluding the blog index page itself. Since the blog index page's slug equals the current prefix, saving any other setting without changing the prefix always triggered the conflict error. Fixed by excluding the current prefix from the slug conflict check:
```php
CmsPage::where('slug', $value)->where('slug', '!=', $currentPrefix)->exists()
```

## Deferred decisions

**Navigation UI restructure (session 032):** During manual testing it became clear that a flat table with a numeric `sort_order` field is not usable for editors. The decision was made to replace `NavigationItemResource` with a menu-first `NavigationMenuResource` backed by a new `NavigationMenu` model, using Filament Repeaters with drag-and-drop reordering. Full spec written as session 032.
