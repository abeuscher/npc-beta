# 055. Quill Fix, Page Layout & Event Date Simplification — Log

## What was built

### Workstream 1 — Quill Editor Overflow Fix

The Quill editor was horizontally overflowing its Filament panel. Root cause: the `wire:ignore` wrapper div had no overflow constraint, allowing the Quill toolbar's fixed-width structure to escape the container. The fix adds `overflow-hidden` to the wrapper, which clips overflow correctly. This creates a CSS block formatting context (BFC) that has a side-effect of clipping the `.ql-container` bottom border. Compensated by adding `border-b border-[#ccc]` to the wrapper, restoring the visual border. `overflow-x: hidden` was tried and rejected — per the CSS spec, setting `overflow-x` without `overflow-y` promotes `overflow-y` to `auto`, producing the same clipping. Final fix in `resources/views/forms/components/quill-editor.blade.php`.

### Workstream 2 — Page & Event Edit Layout

**PageResource:** Entire form wrapped in `Group::make()->columns(3)->columnSpanFull()`. Left `columnSpan(2)` contains title/slug section, Page Builder, SEO, and Custom Fields. Right `columnSpan(1)` is a new `Section::make('Settings')` containing Published toggle, Publish Date, and Tags. Old `Publication` section removed.

**EventResource:** Same `columns(3)` / `columnSpan(2)` / `columnSpan(1)` pattern. Right column holds `Section::make('Settings')` (slug, status, landing page placeholder, tags) and `Section::make('Registration Details')`.

Post-session addition: Title, Description QuillEditor, `starts_at`, and `ends_at` date pickers were wrapped into a single `Section::make('Main Info')` on the left column.

### Workstream 3 — Event Date Simplification

Removed multi-date support entirely. Each event now has exactly one start time and one optional end time stored directly on `events`.

**Migration `2026_03_21_000001_simplify_event_dates`:** Adds `starts_at` (timestamp nullable) and `ends_at` (timestamp nullable) to `events`; drops `event_dates` table.

**Migration `2026_03_21_210001_make_events_starts_at_not_null`:** Makes `starts_at` NOT NULL (backfills any null rows with `now()` before altering). `starts_at` is now required at both the form and database levels. `EventFactory` updated to default `starts_at` to `now()->addDays(7)`.

**Deleted:** `app/Models/EventDate.php`, `app/Filament/Resources/EventResource/RelationManagers/EventDatesRelationManager.php`, `database/factories/EventDateFactory.php`.

**Updated:** `Event` model (`$fillable`, `$casts`, `scopeUpcoming`), `EditEvent.php` (reminder visibility and action loop simplified), `EventReminder.php` (constructor changed from `EventDate` to `Event`), `WidgetDataResolver.php`, `SendEventReminders` command, `PageController.php`.

**Widgets:** `event-dates.blade.php` rewritten to read `$event->starts_at`/`$event->ends_at` directly. `event-description.blade.php` updated to render a formatted date string above the description — same-day format: `January 6th, 7:30pm – 10pm`; spanning days: `January 6th, 7:30pm – January 7th, 10pm`; no end: `January 6th, 7:30pm`. On-the-hour times drop `:00`.

`createLandingPageForEvent` updated: removes `event_dates` widget from auto-created landing pages; new pages get `event_description` and `event_registration` only.

**schema.md** updated: `event_dates` table section removed, `starts_at` and `ends_at` added to `events`.

### Additional — Debug Generator

`DashboardDebugGeneratorWidget` updated: events are now generated with realistic forward-spaced dates. Each event starts 3–7 days after the previous, start times on the quarter-hour between 9am–8pm, duration 30 min–3 hrs in 15-min steps.

### Additional — Events List

- `starts_at` column added to the events table view (formatted `M j, Y g:ia`)
- Default sort changed to `starts_at` ascending (soonest upcoming first)
- `ListEvents.php` gains `$showPastEvents` flag and `getTableQuery()` override that hides past events by default
- Ellipsis menu (`⋮`) added to the list header with a "Show past events / Hide past events" toggle — same pattern as the Contacts list
