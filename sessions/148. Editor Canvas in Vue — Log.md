## What was built

### Phase 1 ‚Äî Preview canvas components

- `useViewport.ts` composable ‚Äî reactive zoom state with Desktop (1920), Tablet (1024), Mobile (375) presets; `computeZoom()` and `setViewport()` methods
- `useLibraryLoader.ts` composable ‚Äî `loadLibs()` loads widget CSS/JS from `window.__widgetLibs` manifest (idempotent); `reinitAlpine()` destroys and re-initialises Alpine trees, Swiper instances, and Chart.js instances inside a scope element (ported from `preview-manager.js`)
- `PreviewRegion.vue` ‚Äî renders widget `preview_html` via `v-html` with `pointer-events: none`; transparent overlay div captures clicks for selection; `--selected` class adds indigo outline
- `PreviewCanvas.vue` ‚Äî viewport toggle buttons with SVG icons; CSS zoom container matching existing Blade approach; ResizeObserver for responsive zoom; loads libs on mount; watches preview HTML changes and re-inits Alpine; prevents link clicks and form submits in preview scope; empty state message

### Phase 2 ‚Äî Editor toolbar and event bridge

- `EditorToolbar.vue` ‚Äî block count, Edit/Handles mode toggle, "Save as Template" button (visible when blocks > 0), "+ Add Block" button; toolbar buttons dispatch browser `CustomEvent`s (`open-widget-picker`, `open-save-template-modal`)
- Blade event bridge ‚Äî added `x-on:open-widget-picker.window` and `x-on:open-save-template-modal.window` Alpine listeners on the page-builder root div to forward Vue CustomEvents to Livewire methods (`openAddModal()`, `openSaveTemplateModal()`)
- `PageBuilder.php` ‚Äî added `$this->js("window.dispatchEvent(new CustomEvent('widget-tree-updated'))")` after `createBlock`, `deleteBlock`, `copyBlock`, `updateOrder`, `onMoveToMain`, `onMoveToColumn`; added `template-saved` event after `saveAsTemplate`

### Phase 3 ‚Äî Wired into App.vue

- `App.vue` ‚Äî replaced PoC layout with `EditorToolbar` + `PreviewCanvas` (edit mode) / `BlockListPoc` (handles mode); event bridge listeners for `widget-tree-updated` and `template-saved` with cleanup in `onUnmounted`
- `stores/editor.ts` ‚Äî added `replaceTree()` action (accepts `{ widgets, required_libs }`); added `reloadTree()` action that fetches fresh tree from the API (used by the event bridge signal); both exposed in store return
