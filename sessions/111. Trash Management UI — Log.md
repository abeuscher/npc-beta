## What was built

### Phase 1 — Soft-delete support on each resource

Applied the standard Filament soft-delete pattern to all 9 resources with `SoftDeletes`:

- **ContactResource**, **PageResource**, **PostResource**, **OrganizationResource**, **MembershipResource**, **FormResource**, **CollectionResource**, **NoteResource**, **CampaignResource**

Per resource:
1. Added `getEloquentQuery()` override with `->withoutGlobalScopes([SoftDeletingScope::class])` so trashed records are included in the base query
2. `TrashedFilter` — already present on all 9 (no changes needed)
3. Gated `ForceDeleteAction` and `ForceDeleteBulkAction` to `super_admin` role on all resources (previously only ContactResource had this gate)
4. Updated `canForceDelete()` on all resources to check `hasRole('super_admin')`

Updated 5 relation managers with full soft-delete support (TrashedFilter, RestoreAction, ForceDeleteAction gated to super_admin, bulk equivalents):
- CollectionItemsRelationManager
- NotesRelationManager (Contact)
- NotesRelationManager (Organization)
- MembershipsRelationManager
- FormSubmissionsRelationManager (also replaced `delete_form_submission` permission gate with `super_admin` for force-delete)

### Phase 2 — Policy gates

Added `restore`, `restoreAny`, `forceDelete`, and `forceDeleteAny` methods to all 9 policies:
- ContactPolicy, PagePolicy, PostPolicy, OrganizationPolicy, MembershipPolicy, NotePolicy, CampaignPolicy, CollectionPolicy, CollectionItemPolicy

Restore gated to the resource's `delete` permission. Force-delete gated to `super_admin` role.

### Help docs

Added a "Deleted Records and Trash" section to 9 help docs explaining the TrashedFilter, restore workflow, and super-admin-only force-delete:
- contacts.md, cms-pages.md, posts.md, organizations.md, memberships.md, forms.md, collection-manager.md, notes.md, campaigns.md

Each section includes resource-specific notes (e.g., cascade warnings for forms/collections, system page protection for pages, org link clearing for organizations).
