## Process rules for every session

Base-prompt rules apply. Session-specific:

- **Zero coverage loss is the invariant.** A test that fails only under `--parallel` is an *isolation defect to fix*, never a test to delete, skip, or move to a serial-only carve-out without explicit user sign-off. Fixing shared state is the job.
- **Root-cause isolation, don't paper over.** Prefer making a test create/own its own paths/fixtures over global serialization. A serial-only escape hatch for a genuinely unparallelizable test is allowed only with a one-line logged rationale and user awareness.
- **Don't collide with the drift-guard session.** The build-pipeline tests' real-artifact writes are owned there; coordinate the isolation approach rather than re-solving it differently here.
- **Verify objectively, yourself.** `--parallel` green with the identical test count, run repeatedly to prove order-independence (run it ≥2× / shuffled). Report before/after wall time.
- **Run the fast suite (parallel) + Playwright (sequential, never concurrent with Pest) when complete.** Green, count unchanged, every delta explained.
