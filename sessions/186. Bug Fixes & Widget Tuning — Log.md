### Phase 2 — Widget tuning & inline-JS extraction

Established a reusable pattern for pulling inline `x-data="{...}"` out of widget templates:

- Each widget's Alpine component moves to a standalone `script.js` next to `template.blade.php` and attaches a factory to `window.NPWidgets.<name>`.
- The template calls it via `x-data="NPWidgets.<name>()"`.
- The file is registered under `assets['js']` in the widget definition, so `AssetBuildService` bundles it into the hashed `public-widgets-{hash}.js` build.
- Per-instance config flows through the existing `<script x-ref="listingData" type="application/json">` data island.

Applied to four widgets:

- `app/Widgets/BlogListing/` — script.js + template + definition.
- `app/Widgets/EventsListing/` — script.js + template + definition.
- `app/Widgets/BarChart/` — script.js + template + definition.
- `app/Widgets/EventCalendar/` — script.js + template + definition. Also converted from a bare `<script>` IIFE to an Alpine component so `WidgetPreviewRenderer` (which strips `<script>` tags) can now initialise it in the admin preview.

Supporting changes:

- `app/Providers/Filament/AdminPanelProvider.php`: new `panels::head.end` render hook loads the hashed public widget JS bundle into the admin panel, so the page-builder preview can resolve `NPWidgets.*` factories.
- `gap` moved from PHP-interpolated `spaceBetween:` into the JSON data island for blog/events listings; the four gap assertions in `tests/Feature/ContentListingWidgetsTest.php` updated to match the new shape.

Bundle size moved from 438 B → 6.5 KB after extraction (expected — that's the widget Alpine code landing in the bundle).

### Phase 3 — Page-context token substitution in widget config fields

Made `{{title}}`, `{{date}}`, `{{excerpt}}`, `{{author}}`, `{{starts_at}}`, `{{location}}` usable in any widget's `text` or `richtext` config field, without per-widget wiring.

- `app/Services/PageContextTokens.php`: added optional `$escapeHtml` parameter to `substitute()`. Richtext callers get HTML-escaped values (safe for `{!! !!}` output); text callers get raw values (Blade escapes on output).
- `app/Services/WidgetRenderer.php`: after image processing, walks the widget's config schema and substitutes tokens in every `text` / `richtext` field. Resolves the owning page from `PageContext::currentPage` when it's a real DB record, else falls back to `$pw->page`. The `exists` check is required because Laravel's container auto-injects an empty `Page` when `PageContext` is resolved without a binding.
- Tests added for the new `$escapeHtml` branch, and a feature test in `tests/Feature/HeroWidgetTest.php` that hydrates a Hero with `{{title}}` / `{{date}}` and asserts the rendered output.
