## Process rules for every session

Base-prompt process rules apply in full (`sessions/template-base-prompt.md`). Session-specific notes:

- **Reproduce before you fix; observe, don't assert.** Drive the real browser at a mobile viewport (Playwright against the live local site, read-only — never the DB-resetting specs) to see exactly how the hamburger and footer columns break before touching code. Each fix is proven by re-driving the same surface, not by reasoning the code now looks right. This is the explicit 332/333 lesson.

- **Front-end builds:** site-header/footer Blade + `resources/scss/**` → `npm run build`; Nav-widget SCSS/JS (`app/Widgets/Nav/**`) → `docker compose exec app php artisan build:public`. Rebuild the touched surface before re-driving the browser.

The breakpoint single-sourcing precursor (the 576/768/… triplication) is also that session's; only touch it if the nav fix unavoidably crosses it, and flag if so.
