- **Tests for new behaviour.** Each in-session fix gets a Pest test. The walk itself produces no tests (it's documentation).
- **When implementation is complete on each iteration**, run the fast test suite: `docker compose exec app php artisan test --exclude-group=slow`. Target: 281 close baseline + any new audit tests.
- **Front-end builds.** If a public-flip fix adds a confirmation dialog in Filament, it's likely Filament-native (no SCSS/JS build needed). If a fix touches public-side widget visibility indicators, `build:public` may be required.
- **Playwright run** — judgment-driven. If a fix changes a confirmation flow in the admin UI, a small spec for the flow is worth adding.
- **Pause for manual testing** — judgment-driven. The data-classification doc itself is objectively verifiable (the doc exists and is coherent); fixes may need user verification.
