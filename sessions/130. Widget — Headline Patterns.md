# 129. Widget — Headline Patterns

Add a `headline` widget type: a configurable heading element with alignment, heading level, optional subtitle, and decorative styling options. Used for section dividers, page titles, and visual breaks between content sections.

---

## Design decisions (resolved before starting)

- This is a pure config-driven widget — no collections, no data sources. The admin types a heading and picks styling options.
- Categorised as `content` with `allowed_page_types: null`.
- Decorative styles are a fixed set of options (not arbitrary CSS): underline, overline, centered rule, none. These are implemented as CSS classes, not inline styles.
- Heading level (h1–h6) is configurable for semantic correctness.

### Config schema

| Key | Label | Type | Default | Notes |
|---|---|---|---|---|
| heading | Heading Text | text | '' | |
| subtitle | Subtitle | text | '' | Optional smaller text below heading |
| level | Heading Level | select | 'h2' | Options: h1, h2, h3, h4, h5, h6 |
| alignment | Alignment | select | 'center' | Options: left, center, right |
| decoration | Decoration | select | 'none' | Options: none, underline, overline, rule |
| decoration_color | Use Primary Color | toggle | true | When true, decoration uses site primary colour; when false, uses neutral gray |

### Template

Renders the configured heading level element with the heading text, optional subtitle below in a smaller muted style, text alignment, and decorative treatment. Decoration styles:

- **none** — plain heading
- **underline** — short horizontal line below the heading
- **overline** — short horizontal line above the heading
- **rule** — full-width horizontal rule above and below, heading centred between them

All decorations use CSS pseudo-elements or border styling. Tailwind utilities for layout, minimal custom CSS for decorations.
