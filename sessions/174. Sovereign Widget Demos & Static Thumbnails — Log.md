## What was built

### Phase 1 — Relocate four demo seeders; delete `ProductDemoSeeder`

- `database/seeders/CarouselDemoSeeder.php` → `app/Widgets/Carousel/DemoSeeder.php` (namespace `App\Widgets\Carousel`).
- `database/seeders/ChartDemoSeeder.php` → `app/Widgets/BarChart/DemoSeeder.php`.
- `database/seeders/LogoGardenDemoSeeder.php` → `app/Widgets/LogoGarden/DemoSeeder.php`.
- `database/seeders/BoardMembersDemoSeeder.php` → `app/Widgets/BoardMembers/DemoSeeder.php`.
- `database/seeders/ProductDemoSeeder.php` deleted outright — it was domain-model seeding, not widget-specific demo data. `ProductCarousel` and `ProductDisplay` now render their own disconnected state when there are zero products, which is the honest render-state signal.
- `DashboardDebugGeneratorWidget::seedWidgetCollections()` rewritten to iterate `WidgetRegistry::all()` and call each widget's `demoSeeder()` — no hard-coded seeder list.
- Test updates: `LogoGardenWidgetTest` / `BoardMembersThreeBucketsTest` point at the new FQCNs; the feedback-string assertion was relaxed to `toStartWith('Widget demo collections seeded:')` since the message is now built from the iterated widget labels. The `ProductDemoSeeder creates products with prices and images` test was removed from `ProductCarouselTest` per the design decision.

### Phase 2 — `demoSeeder()` contract on the widget base class

- Added `WidgetDefinition::demoSeeder(): ?string` (default `null`).
- Overrode on six widgets: `CarouselDefinition`, `BarChartDefinition`, `LogoGardenDefinition`, `BoardMembersDefinition`, `EventCalendarDefinition`, `DonationFormDefinition`.

### Phase 3 — Two new demo seeders for domain-model-backed widgets

- `app/Widgets/EventCalendar/DemoSeeder.php` — creates three published upcoming events (`demo-event-1`, `demo-event-2`, `demo-event-3`) at known offsets. Idempotent via `updateOrCreate` on slug.
- `app/Widgets/DonationForm/DemoSeeder.php` — creates one `Fund` (code `demo-fund`) and one `Campaign` (`Spring Annual Appeal`). Idempotent.

### Phase 4 — Dev-only demo route

- `routes/dev.php` (new): one route group behind `DevRoutesMiddleware`, `GET /dev/widgets/{handle}` → `App\Http\Controllers\Dev\WidgetDemoController@show`.
- `routes/web.php` conditionally requires `dev.php` only when `! App::environment('production')`.
- `app/Http/Middleware/DevRoutesMiddleware.php` — belt-and-braces 404 at request time in production.
- `app/Http/Controllers/Dev/WidgetDemoController.php` — resolves the widget definition via the registry, runs `demoSeeder()` if declared (idempotent), builds a transient `PageWidget` whose config composes `defaults() + demoConfig() + collection_handle override`, composes appearance via `AppearanceStyleComposer::compose()` on the transient PageWidget, renders through the normal `WidgetRenderer` pipeline, returns the minimal demo view.
- `resources/views/dev/widget-demo.blade.php` — 800×500 body, same widget asset bundle (`public/build/widgets/manifest.json`) the live site uses, rendered HTML wrapped in `<div class="np-widget-demo__wrap">` carrying the composed `inline_style`.

### Phase 5 — Playwright script + Artisan manifest command

- `app/Console/Commands/WidgetsManifestJsonCommand.php` — `php artisan widgets:manifest-json` emits `WidgetRegistry::manifests()` as JSON on stdout.
- `scripts/generate-thumbnails.js` — standalone Node (ESM, because the project's `package.json` declares `"type": "module"`). Reads the widget list via `docker compose exec app php artisan widgets:manifest-json`, iterates, captures each at 800×500 via Playwright, writes `app/Widgets/{PascalName}/thumbnails/static.png`. CLI flags: `--widget=handle`, `--all`, `--base-url=...`. Falls back to the global `npm root -g` install of Playwright when the local resolver doesn't find it.

- `EventCalendar` (seeded events rendered in month view).

### Mid-session additions beyond the written prompt

- **`WidgetDefinition::demoConfig(): array`** (default `[]`) — added when `TextBlock` needed lorem ipsum for a legible thumbnail. Dev controller merges it on top of `defaults()`.
- **`WidgetDefinition::demoAppearanceConfig(): array`** (default `[]`) — added to let widgets declare demo-only appearance (TextBlock padding, Hero gradient). Dev controller feeds it into the transient `PageWidget.appearance_config` and composes through `AppearanceStyleComposer` exactly like the live pipeline.
- **Debugbar disabled on the demo controller** — `Barryvdh\Debugbar\Facades\Debugbar::disable()` at the top of `show()` so the debug toolbar doesn't leak into thumbnails.
- **Hero / TextBlock / VideoEmbed overrides** — `demoConfig()` + `demoAppearanceConfig()` authored on each to produce the thumbnails the user specified (lorem block with padding, dark-to-light-blue gradient with "Welcome" H1, YouTube Gizmo embed).

Both new contract methods are symmetric with `demoSeeder()` (sovereign, live on the definition), CI-validated by `WidgetManifestTest`, and documented.

## Production deployment notes

None beyond standard `migrate`. Dev-only routes, middleware, controller, and thumbnail binaries do not affect production behavior — `routes/dev.php` is not required in production, and `DevRoutesMiddleware` aborts 404 even if it were.

## Deferred decisions

- **Animated thumbnails (originally Stage 4.5 Phase 2).** Deferred to post-Beta 1 after a value-vs-cost discussion during the session. Cost includes a new per-widget interaction contract, ffmpeg host dependency, multi-variant widget design (`logo_garden` alone has three motion modes), repo bloat from MP4 files, and per-widget re-record maintenance when behavior changes. Value is only realized in the Stage 6 widget browser UI, which itself is not Beta-1-critical and can convey "what this widget is" adequately via static + a preset count chip. The `scripts/generate-thumbnails.js` script is positioned to grow an `--animated` flag when we revisit. `session-outlines.md` was updated to reflect the deferral.
- **Preset work pulled forward.** In place of animated thumbnails, the Stage 5 preset sub-arc was formally defined and the next session (175) scoped to pilot the preset inspector UI against Hero. Per-widget preset authoring will be batched across sessions 176+.
- **`DemoDataService` retirement.** Still wired into the builder preview fallback path. Not touched this session; cleanup remains a separate future session.
- **Retina / `deviceScaleFactor: 2` captures.** Not explored. If ever wanted, it's a single Playwright context option.
