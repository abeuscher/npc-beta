## What was built

### Phase 1 — Nginx conf + route + controller

- **`docker/nginx/prod.conf`** — third mTLS-gated location block added immediately after `/api/logs`, with per-location `fastcgi_read_timeout 600;` override so nginx does not kill the upstream connection at the default 300s while a backup is running. Comment on the mTLS-gated routes updated to name all three endpoints.
- **`docker/nginx/default.conf`** — same location block added for parity (the v2.1.0 `/api/logs` precedent had both default + prod gates; matched the convention here).
- **`routes/api.php`** — `Route::middleware(['throttle:6,1'])->post('/backup/trigger', [BackupController::class, 'trigger'])`. Tighter throttle than `/api/health` and `/api/logs`' 60-rpm because backups are expensive.
- **New `app/Http/Controllers/Api/Fleet/BackupController.php`** with a single `trigger(Request)` action:
  - `@set_time_limit(600)` defensively at the top of the action.
  - Captures `$startTime = microtime(true)` and the pre-call success-record timestamp via `Storage::disk('local')->get('fleet/last-backup-at')` (the latter used both for the post-call mtime cross-check AND as the `last_backup_at` value in failed-envelope responses, so the operator sees the last *known* successful backup rather than just `null` when the trigger fails).
  - Runs `Artisan::call('backup:run')` inside `try { ... } catch (Throwable $e) { ... }`.
  - **On exit-0 + post-call mtime ≥ start:** returns success envelope `{contract_version: '2.2.0', status: 'success', last_backup_at: <ISO>, duration_ms: <int>, message: null}`.
  - **On exit-0 + integrity-guard fires:** downgrades to failed envelope with `message: 'backup:run exited cleanly but success record was not updated'`.
  - **On non-zero exit:** failed envelope; `message` source = `Artisan::output()` (sanitised).
  - **On caught exception:** failed envelope; `message` source = `\Throwable::getMessage()` (same sanitisation).
  - All branches return HTTP 200 (mirrors `/api/health`'s pattern of returning 200 with degraded-status envelopes).
  - `duration_ms = (int) ((microtime(true) - $startTime) * 1000)` regardless of branch.
- **`HealthController::CONTRACT_VERSION` bumped 2.1.0 → 2.2.0.** The canonical field FM polls; bumps on every contract version change. Existing `HealthEndpointTest` assertion updated to match.

## Findings register

- **`request_terminate_timeout` is unset on the upstream `php:8.4-fpm` image and defaults to `0` (no ceiling).** Verified by grep across the repo (no `www.conf` override, no Dockerfile injection). Future PHP-FPM config tuning that introduces a non-zero `request_terminate_timeout` must remember the `/api/backup/trigger` 600s ceiling — bump in concert. Documented in the spec doc.
- **GET on a POST-only Laravel route falls through to the public-site page-slug catchall, not 405.** Specifically: `GET /api/backup/trigger` resolves through `web.php`'s `Route::get('/{path}', ...)` catchall, which tries to look up a Page model by slug, and returns 404 with a Laravel-format JSON body (`exception: "Symfony\\Component\\HttpKernel\\Exception\\NotFoundHttpException"`). The Symfony `MethodNotAllowedHttpException` path that produces 405 doesn't fire because the URL matches a different (non-API) route entirely. Spec doc framing updated: FM consumers should rely on "the backup pipeline does not run for non-POST requests" rather than "GET returns 405."
- **`HealthController::CONTRACT_VERSION` bumps with every contract version change** (canonical FM polling field). Easy to forget — should be a checklist item on every future boundary-touching session. Captured in the 263 close as a process reminder; consider promoting to a CLAUDE.md or template-base-prompt note if a future session forgets it.
- **`Artisan::output()` is the right error-message source on non-zero `Artisan::call`.** `Artisan::call()` returns exit code, not a message. `Artisan::output()` returns whatever the artisan command emitted to stdout/stderr. Multi-line output is common (Spatie Backup logs each step); the sanitisation pipeline's newline-collapse handles this.
- **Mockery facade pattern (not `Artisan::fake()`) is the codebase convention** for mocking artisan calls. Existing test suite uses `DB::shouldReceive` / `Redis::shouldReceive`; the new tests use `Artisan::shouldReceive('call')` / `Artisan::shouldReceive('output')` to match.
