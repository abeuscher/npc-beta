## Security considerations

CSS-delivery scoping plus a DB column-drop with a one-shot data migration. The relocation is a data-preservation operation — the byte-faithfulness guard was written before the migration and proves the default template's stored colours land identically; the migration logs (does not silently discard) any non-default divergent templates. Token delivery is `.np-site`-scoped and verified by real-browser computed values not to leak into the Filament admin. No auth, portal, data-exposure, or Fleet surface touched.
