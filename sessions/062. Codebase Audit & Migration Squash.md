# 062. Codebase Audit & Migration Squash

Follows the pattern established in session 049. Full audit of fields, schema, permissions, and help coverage against the current state of the codebase. Additional scope beyond session 049: squash the migration history using `php artisan schema:dump`, delete the superseded migration files, and verify a clean `migrate:fresh --seed` on both the local Docker environment and the production dev server.

---

## Before writing any code

Read the following files:

- Every file in `app/Filament/Resources/` — all resource edit forms
- Every file in `app/Filament/Pages/` — all custom admin pages
- Every file in `app/Models/` — all models, their `$fillable`, `$casts`, and soft delete status
- Every migration in `database/migrations/` — sort by date, focus on anything added since session 049
- Every file in `resources/docs/` — all help articles and their registered routes
- `docs/schema.md` — the running schema document

Take notes as you read. Cross-reference against the session 049 log to avoid re-auditing what was already resolved.

## Workstream 1 — Edit page field audit

Same methodology as sessions 042 and 049. For every Filament resource and admin page that has an edit form, compare the fields shown against the database columns and the model's `$fillable`.

Focus on what is new since session 049:

- **Page types** — `system` type added. Confirm `type` appears in `$fillable` and `$casts` (or an enum cast). Confirm system page editor UI (locked slug, locked type, suppressed delete) is correctly applied.
- **SiteSettings added in sessions 056–061** — `portal_prefix`, `system_prefix`, `system_page_content_reset_password`, `system_page_content_email_verify`, and any other new keys. Confirm all are seeded and surfaced in `GeneralSettingsPage`.
- **PageWidget** model — `is_required` or similar flag added to support required widget suppression. Confirm it is in `$fillable` and `$casts`.
- **WidgetType** model — confirm any `is_pinned` or required-widget map mechanism is accounted for.
- **PortalAccount** model — confirm all columns from sessions 056–060 are in `$fillable` and `$casts`.

For each discrepancy: deliberate omission (document with a comment) or gap to fix.

## Workstream 9 — Migration squash

**This workstream is new and is the primary additional scope of this audit.**

1. Run `php artisan schema:dump` inside the Docker container. This writes a `database/schema/pgsql-schema.sql` (or similar) file that replaces all existing migrations with a single authoritative schema snapshot.

2. Delete all migration files in `database/migrations/` that predate the squash dump. Confirm the dump file alone is sufficient to restore the schema from scratch.

3. Run `php artisan migrate:fresh --seed` inside the Docker container and confirm it completes cleanly with no errors or warnings.

4. SSH into the production dev server and repeat `migrate:fresh --seed` there. Confirm the same clean result.

5. Document the squash date and any gotchas in a brief note at the top of `docs/schema.md`.

**Important**: take a database backup before running `migrate:fresh` on production. Coordinate with the user on timing.
