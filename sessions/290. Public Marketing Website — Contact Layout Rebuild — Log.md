- Restored the contact email-fallback to a real `mailto:` link with `text.link_color: #ffffff`. **G22 resolved.** Mechanism verified end-to-end (HTML wrapper carries the var, compiled CSS consumes it, page loads that CSS).

## Security considerations

No new security-sensitive runtime code beyond the `link_color` composer path, which is `HEX_PATTERN`-validated and structurally identical to the existing `text.color` emission ‚Äî no new surface. The HtmlSanitizer's inline-style stripping (G22) was confirmed correct and left unchanged; the link-color feature deliberately routes around it via the trusted server-composed wrapper style rather than re-admitting inline style, so the XSS posture is preserved. The substantive security work was forward-looking: the form-notification feature (291) was fully threat-modeled and pre-registered in `docs/security-forms.md` with dispositions, the recipient invariant ("submission data must never influence the recipient set or any mail header") was defined as the load-bearing control, and the 291 prompt binds the implementer to recipient-invariant / token-allowlist / escaping tests as acceptance criteria.
