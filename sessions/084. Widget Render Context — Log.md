## What was built

### Phase 1 — Build PageContext

- Created `App\Services\PageContext` with eager properties (`currentPage`, `currentUser`), lazy memoized stream methods (`posts()`, `pages()`, `upcomingEvents()`, `collection()`), and per-key lazy record methods (`event()`, `product()`, `form()`).
- `pages()` added at user request alongside `posts()` — returns published non-post pages ordered by title, intended for nav and search widgets.
- All stream and record methods are memoized; streams cache the full result and apply `take($limit)` in memory so the DB is only hit once per stream per request.

### Phase 2 — Refactor PageController

- `PageController::renderPage()` constructs `PageContext` and calls `View::share('pageContext', $pageContext)` before the widget loop.
- Removed the `$collectionData`, `$eventData`, and `$productData` resolution blocks entirely. `Blade::render()` now passes only `['config' => $config]`.
- Client-mode widget block simplified: window variable injection removed (no longer populated); code block appended directly.
- `PostController::renderPage()` discovered during testing to be an identical copy of the old controller — applied the same refactor.

### Phase 3 — Pilot widget updates

- **`text_block`** — regression confirmed; no template changes required.
- **`blog_pager`** — replaced `$currentPage` with `$pageContext->currentPage`; removed debug output block; fixed boundary wrapping by switching from raw DB queries to a collection-index approach using the memoized `$pageContext->posts()`; corrected prev/next column directions (left = newer, right = older); added `<small>` author name and friendly publish date beneath each title link; two-column layout using `blog-pager__next` / `blog-pager__prev` spans.
- **`blog_listing`** — switched from `$blog_posts` array (WidgetDataResolver) to `$pageContext->posts()` Eloquent collection; template updated to model property access; limit cast from string config before passing to `posts()`.
- **`WidgetTypeSeeder`** — cleared `collections` on `blog_listing`; added `limit` to `blog_listing` config schema; updated `blog_pager` template to plain `@include('widgets.blog-pager')`.
- **`PostResource`** — changed default sort from `updated_at` to `published_at` descending, and confirmed publish date is visible and sortable in the admin list view.

## Security checklist — verified

- [x] `PageContext` streams return only published/public records — no draft, private, or admin-only data is reachable from a widget template.
- [x] `currentUser` is read from the portal guard — it cannot be spoofed or overridden by widget config.
- [x] `PageContext` has no setter methods — it is constructed once and immutable after that.
- [x] No widget template can reach outside `$pageContext` and `$config` to access unscoped model data.
