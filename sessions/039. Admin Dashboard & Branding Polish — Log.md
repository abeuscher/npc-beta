## What was built

### General Settings — Primary colour picker

Added a `ColorPicker` field to the Admin Panel section of General Settings. Saves to `site_settings` under the key `admin_primary_color`. Loaded via `AppServiceProvider` into `config('site.admin_primary_color')`. `AdminPanelProvider` now calls `Color::hex($primaryColor)` instead of the hard-coded `Color::Amber`, so the panel accent colour updates on next page load after saving. Default is `#f59e0b` (amber) if the setting has not been saved.

### General Settings — Integrations section

Added a new "Integrations" section to General Settings with two plain `TextInput` fields:
- **Stripe API Key** → saved to `stripe_api_key`, loaded as `config('services.stripe.key')`
- **QuickBooks API Key** → saved to `quickbooks_api_key`, loaded as `config('services.quickbooks.key')`

No functionality is wired beyond storage. These keys exist so the Integration Status dashboard widget has something to check.

### Dashboard restructure

Created a custom `App\Filament\Pages\Dashboard` extending Filament's default Dashboard and returning `2` from `getColumns()`. This gives the dashboard a 2-column grid.

Removed `Widgets\AccountWidget` and `Widgets\FilamentInfoWidget` from `AdminPanelProvider`.

Four widgets now occupy the dashboard in a 2×2 layout:

| Sort | Widget | Column |
|------|--------|--------|
| 1 | DashboardWelcomeWidget | Left |
| 2 | DashboardHelpPlaceholderWidget | Right |
| 3 | DashboardQuickActionsWidget | Left |
| 4 | DashboardIntegrationStatusWidget | Right |

**DashboardWelcomeWidget** (existing — updated):
- `canView()` changed from checking for non-empty `dashboard_welcome` to always returning `true`
- Blade view updated to show a soft italic placeholder when the welcome message is empty
- Sort changed from `-10` to `1`

**DashboardHelpPlaceholderWidget** (new):
- Disabled search input (plain HTML) with "Help articles coming soon." placeholder text
- Sort: 2

**DashboardQuickActionsWidget** (new):
- Two Filament-styled buttons: New Blog Post (`/admin/posts/create`) and New Event (`/admin/events/create`)
- Sort: 3

**DashboardIntegrationStatusWidget** (new):
- Reads `config()` for MailChimp, Resend, Stripe, and QuickBooks keys
- Shows a "Connected" row for each key that is non-empty
- Shows "No integrations configured yet." when all keys are empty
- Sort: 4

### Help doc

Updated `resources/docs/dashboard.md` to accurately describe the new 2×2 layout and each of the four panels.
