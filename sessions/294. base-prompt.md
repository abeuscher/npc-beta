## Process rules for every session

Base-prompt process rules apply in full. Session-specific notes:

- **Container query, not media query.** This is the load-bearing E7-stub decision. The collapse must fire off the *element's container*, not the browser viewport, so the page-builder preview (simulated viewport via `container-type: inline-size`) collapses correctly alongside the live site. **Open the session by verifying the public side also establishes a containment context** for `.page-layout` (a media query would only work on the public site and break the preview; a container query needs an ancestor `container-type`). If the public side has no container context, that is an **Open question to resolve at session start** — the session prompt carries the two candidate resolutions; do not silently pick the one that breaks the preview.
- **Minimal `--layout-cols` only.** Per the agreed decision: only `grid-template-columns` becomes a custom property (so the container query can override it with no `!important` and no renderer-side breakpoint logic). The other ~7 grid inline properties stay exactly as emitted today. Do **not** convert them; do **not** reinvent the layout renderer. A broader no-inline conversion is explicitly out of scope and was decided against on readability grounds.
- **`collapse_mobile` must round-trip.** New `layout_config` key, default `true`, in `layout_config` (not `appearance_config`). Required: inspector toggle on the Column Settings tab ("Collapse columns on mobile", default checked) on **both** the Filament and Vue surfaces, the Vue preview honoring it, and **export/import round-trip with a Pest test** — G1 (285) is the precedent for why an un-round-tripped layout-config key is a silent data-loss bug.
- **Swiper blanket, with carve-out escalation.** The six Swiper widgets (`BlogListing`, `BlogPager`, `Carousel`, `EventsListing`, `LogoGarden`, `ProductCarousel`) get one blanket mobile pattern. If a widget's existing Swiper config (autoplay, custom `slidesPerView`/breakpoints) makes the blanket wrong for it, that widget is a surfaced carve-out, not a silent special-case — escalate per the residue protocol.
- **The autonomous loop is the method; the oracle is the stop condition.** Build the audit harness first (5 public pages × the mobile widths → structured violations: page/element horizontal overflow, columns still side-by-side below the collapse threshold, clipped/overflowing text, distorted declared-aspect-ratio media). Then iterate: apply fix → rebuild the public bundle (`build:public`) / `npm run build` as the touched path requires → re-run the audit → repeat until the violation list is empty across all five pages and all widths. The loop must **terminate on the empty oracle or on a bounded iteration budget**, surfacing residue rather than spinning.
- **Residue-escalation protocol — surface, don't guess.** Three things are pre-identified as user-judgment, not agent decisions: (a) the **Demo page's intentional 50/50 split** mobile behavior (collapse stacked, or `collapse_mobile: false`?), (b) the **Pricing à-la-carte 2×2** mobile behavior (1-up or stay 2-up on larger phones?), (c) anything the loop cannot drive to zero without a design call. Set the `collapse_mobile` toggle per each page's evident intent, then surface the set for confirmation at the manual-test handoff. Any *new* weirdness (Vue-mirror divergence, a Swiper carve-out, a public-side containment surprise) escalates the same way.
- **Type size is explicitly NOT this session.** Headings will remain desktop-scaled on mobile after 294 — that is **expected and acceptable** for this session's bar. The objective oracle still enforces "no text overflows its container / no horizontal scroll" (aggressive wrap via overflow guards handles that even at desktop type sizes). "Type *feels* right on mobile" is E5's deliverable, not a 294 defect. Do not tune typography here; do not add a SCSS type clamp (it would be silently overridden by `TypographyCompiler`'s post-bundle inline `<style>` on this very install — see the E5 stub).
- **No contract change.** If the build appears to need a Fleet Manager contract or `/api/health` touch, stop and surface — scope drifted.
- **Run the fast suite when implementation is complete:** `docker compose exec app php artisan test --exclude-group=slow`. Expect the 293 baseline plus the new 294 tests, all green (modulo the pre-existing `AppVersionStampTest` red if not folded into 293).
- **Pause for manual testing only when human judgment is required.** End-of-session: announce the audit oracle is at zero across the five pages, surface the before/after mobile captures and the residue set (the toggle decisions for Demo / Pricing and any carve-outs) for the user's visual + design review, then stop and wait.
