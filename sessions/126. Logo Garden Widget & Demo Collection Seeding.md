# 126. Logo Garden Widget & Demo Collection Seeding

Build a logo garden widget that displays a collection of partner/sponsor logos in static grid, carousel, or flipper mode. Add a "Seed widget collections" button to the debug data generator that populates all widget demo collections (carousel, chart, logo garden) in one click.

---

## Design decisions (resolved before starting)

- The logo garden is collection-backed (`collections: ['logos']`) following the carousel and bar chart pattern.
- Three display modes: **static** (CSS grid), **carousel** (Swiper.js), **flipper** (CSS 3D flip animation that rotates logos on the Y axis, revealing the next logo behind).
- Logos are images (svg, gif, png, jpg, webp) stored via Spatie media library on `CollectionItem` records, same as carousel slides.
- Each logo slide has a solid background colour (configurable) so the flipper's backface-visibility works cleanly without transparency bleed.
- The "Seed widget collections" button on the debug generator runs all demo collection seeders (`CarouselDemoSeeder`, `ChartDemoSeeder`, and the new `LogoGardenDemoSeeder`) in one action. It is a separate button from the existing Generate button ‚Äî it does not depend on the record type dropdown or quantity input.

## Phase 1 ‚Äî Widget type registration

Add a `logo_garden` entry to `WidgetTypeSeeder` with:

- **handle:** `logo_garden`
- **label:** Logo Garden
- **description:** Grid or carousel of partner/sponsor logos from a custom collection.
- **category:** `['content', 'media']`
- **collections:** `['logos']`
- **config_schema:**
  - `collection_handle` ‚Äî select, `options_from: collections`
  - `image_field` ‚Äî select, `options_from: collection_fields:image`, `depends_on: collection_handle`
  - `display_mode` ‚Äî select, options: `static` / `carousel` / `flipper`, default: `static`
  - `show_name` ‚Äî toggle, default: false. When enabled, displays the `name_field` value as a caption below each logo.
  - `name_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`. Used for `title` attribute and optional caption. Only visible when `show_name` is true.
  - `background_color` ‚Äî text, default: `#ffffff`. Applied to the section and individual logo slides.
  - `columns` ‚Äî number, default: 5, advanced. Number of columns in static grid mode.
  - `logo_max_height` ‚Äî number, default: 80, advanced. Max height in px for logo sizing.
  - `carousel_advance` ‚Äî select, advanced, options: `one` / `page` / `smooth`, default: `one`. Only relevant in carousel mode.
  - `carousel_duration` ‚Äî number, advanced, default: 3000. Autoplay interval in ms.
  - `flip_duration` ‚Äî number, advanced, default: 4000. Time between flips in ms (flipper mode).

- **assets:** SCSS at `resources/scss/widgets/_logo-garden.scss`

## Phase 2 ‚Äî Blade template and SCSS

Create `resources/views/widgets/logo-garden.blade.php`:

### Static mode
- CSS grid with configurable column count, responsive (fewer columns at smaller breakpoints).
- Each logo is centred in its cell with `object-fit: contain` and `max-height` applied.
- Each cell has the configured background colour.
- The `title` attribute is set to the name field value.
- If `show_name` is enabled, render the name as small text below the image.

### Carousel mode
- Use Swiper.js (already loaded globally) with Alpine.js init, same pattern as the carousel widget.
- `slides_per_view` derived from the `columns` config (same number of logos visible as static columns).
- `carousel_advance` maps to Swiper config:
  - `one` ‚Äî advance one slide at a time
  - `page` ‚Äî advance by `slidesPerGroup` equal to `slidesPerView`
  - `smooth` ‚Äî use `freeMode` with continuous horizontal scroll, autoplay with no pause
- Loop enabled, autoplay with `carousel_duration` interval.
- Navigation arrows, no pagination dots.

### Flipper mode
- Render logos in a CSS grid (same as static) but each cell is a 3D flip container.
- Each cell contains a front face (current logo) and a back face (next logo in the sequence, wrapping around).
- CSS animation: `rotateY(180deg)` with `backface-visibility: hidden` on both faces.
- Use a staggered CSS animation so all logos don't flip at the same time ‚Äî each cell's animation-delay is offset (e.g., `delay = index * 200ms`).
- The flip repeats on an interval controlled by `flip_duration`.
- Both faces have the configured background colour (solid, no transparency).
- Use Alpine.js `x-data` with `setInterval` to toggle a flip class, cycling through logo pairs.

### SCSS (`resources/scss/widgets/_logo-garden.scss`)
- `.widget-logo-garden` container styles
- Grid layout with CSS custom property for column count (`--logo-columns`)
- Logo cell: centred content, solid background, padding
- Flip animation keyframes and 3D transform styles
- Responsive breakpoints: reduce columns at `$bp-md` and `$bp-sm`

## Phase 3 ‚Äî Demo collection seeder

Create `database/seeders/LogoGardenDemoSeeder.php`:

- Collection handle: `logo-garden-demo`
- Collection name: Logo Garden Demo
- Fields: `name` (text, required), `logo` (image)
- Create 8-10 items with company names (e.g., "Acme Corp", "Globex Industries", "Initech", "Umbrella Corp", etc.)
- No images attached in the seeder (same as carousel demo) ‚Äî the user adds images manually for testing.

## Phase 4 ‚Äî Debug generator: seed widget collections button

Add a "Seed widget collections" button to `DashboardDebugGeneratorWidget`:

- New Livewire method `seedWidgetCollections()` that calls:
  - `CarouselDemoSeeder`
  - `ChartDemoSeeder`
  - `LogoGardenDemoSeeder`
- The button sits alongside the existing Generate and Wipe buttons.
- Uses `Artisan::call('db:seed', ['--class' => $seederClass])` for each seeder.
- Feedback message: "Widget demo collections seeded."
- This button is independent of the record type dropdown ‚Äî it always seeds all widget collections.
