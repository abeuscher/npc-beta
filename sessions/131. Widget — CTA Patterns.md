# 131. Widget — CTA Patterns

Add a `cta_block` widget type: a call-to-action section with heading, body text, and one or two styled buttons. Configurable background colour, button style variants, and layout direction.

---

## Design decisions (resolved before starting)

- Config-driven widget — no collections. The admin fills in text and button details.
- Two buttons supported: a primary CTA and an optional secondary CTA. This covers the common "Sign Up / Learn More" dual-action pattern.
- Button style variants: solid (filled), outline, and pill (rounded). These are CSS class variants.
- Background options: transparent (blends with page), light fill (subtle section break), dark fill (inverse text).
- Categorised as `content` with `allowed_page_types: null`.

### Config schema

| Key | Label | Type | Default | Notes |
|---|---|---|---|---|
| heading | Heading | text | '' | |
| body | Body Text | textarea | '' | |
| primary_text | Primary Button Text | text | '' | Leave empty to hide |
| primary_url | Primary Button URL | url | '' | |
| primary_style | Primary Button Style | select | 'solid' | Options: solid, outline, pill |
| secondary_text | Secondary Button Text | text | '' | Leave empty to hide |
| secondary_url | Secondary Button URL | url | '' | |
| secondary_style | Secondary Button Style | select | 'outline' | Options: solid, outline, pill |
| background | Background | select | 'transparent' | Options: transparent, light, dark |
| alignment | Alignment | select | 'center' | Options: left, center, right |

### Template

Section container with configurable background. Heading, body text, and a button row (one or two buttons depending on config). Buttons use the selected style variant:

- **solid** — filled with primary colour, white text
- **outline** — transparent with primary colour border and text
- **pill** — solid but with fully rounded corners

When background is `dark`, text and outline buttons invert to white. Tailwind utilities. Buttons link to configured URLs via standard anchor tags.
