## What was built

### Fullscreen toggle and scroll indicator

- Added `fullscreen` config toggle to the hero widget. When on, applies `hero--fullscreen` class (min-height: 100vh). When off, height classes (`hero--height-16`, `hero--height-24`, etc.) apply.
- Added `scroll_indicator` config toggle — renders an animated bouncing chevron at the bottom of the hero.
- `min_height` field is hidden in the inspector when fullscreen is on (new `hidden_when` inspector feature).

### Full width and full bleed

- Added `full_width` config toggle (default true) so users can switch between edge-to-edge and contained rendering per instance. PageController reads `config.full_width` and overrides the widget type default.
- Fixed `overlap_nav` (relabelled "Full Bleed") — layout nav wrapper now uses `position: absolute` via `.site-nav-wrapper--overlay` class instead of the old fragile `margin-top: -4.5rem` hack on the hero.
- Added `nav_link_color` and `nav_hover_color` colour picker fields (shown only when Full Bleed is on, grouped side by side). Values flow as CSS variables (`--nav-link-color`, `--nav-hover-color`) on the nav wrapper.

### Semantic markup — no Tailwind in widget templates

- Rewrote the hero template with semantic class names: `.widget--hero`, `.hero-bg`, `.hero-overlay`, `.hero-body`, `.hero-inner`, `.hero-content`, `.hero-ctas`, `.hero-scroll-indicator`.
- All widget styling moved to SCSS (`resources/scss/_custom.scss`). No Tailwind utilities in the hero template.
- CSS custom properties on the widget root for per-instance values (`--hero-bg`, `--hero-overlay`). All actual styling in SCSS rules.
- Config toggles map to modifier classes (`hero--fullscreen`, `hero--overlap-nav`, `hero--pos-center-center`, `hero--height-24`).
- Rewrote buttons partial with semantic classes: `.btn-group`, `.btn`, `.btn--primary`, `.btn--secondary`, `.btn--text`.

### Video background

- Created `video` inspector field type (MP4/WebM upload via Spatie media library). Same upload pipeline as image fields.
- Added `background_video` config field to the hero. Video takes priority over image when both are set.
- Renders as `<video class="hero-video" autoplay muted loop playsinline>` with `object-fit: cover` in SCSS.
- Updated WidgetRenderer and PageBuilderInspector to handle `video` type fields alongside `image`.

### Inspector enhancements

- `hidden_when` — hides a field when a referenced toggle is truthy (Alpine `x-show`).
- `shown_when` — shows a field only when a referenced toggle is truthy.
- `group` — fields with the same group name render side by side in a 2-column grid.
- `type: 'color'` — colour picker field (swatch + text input).
- `type: 'video'` — video file upload field.
- Fixed toggle fields not saving immediately — changed `wire:model` to `wire:model.live` (Livewire 3 deferred binding issue).

### Removed nav hover backgrounds

- Removed `hover:bg-gray-100` from site header nav links (both widget and fallback component).

### SCSS architecture

- Created `_variables.scss` with standard breakpoints (xs through xxl) and container max-widths.
- Created `_layout.scss` with `.site-container` — responsive container class matching Bootstrap/Foundation behavior (90% width on small screens, stepping to fixed max-widths at breakpoints).
- Created `_buttons.scss` with `.btn-group` and `.btn` component styles.
- Refactored `_custom.scss` — removed duplicated styles now in their own partials; added `.site-header` and `.site-nav` component styles.
- Updated `public.scss` to import `_variables`, `_layout`, `_buttons`, `_custom` in order.
- Tailwind imports remain for now (other templates still use utilities). Removal is flagged as a pre-task for the next session.

### Site header widget rewrite

- Removed all Tailwind utilities from `site-header.blade.php`.
- Rewrote with semantic classes: `.site-header`, `.site-header__bar`, `.site-header__logo`, `.site-nav`, `.site-nav__link`, `.site-nav__toggle`, `.site-nav__dropdown`, `.site-nav__dropdown-menu`, `.site-nav__caret`.
- Applied `.site-container` for content width alignment with the hero.

### Documentation

- Created `docs/widget-conventions.md` capturing all architectural decisions for future widget development.
- Updated session 123 stub to reflect partial completion (video background built here).
- Added Tailwind removal pre-task to session outlines for next session.
