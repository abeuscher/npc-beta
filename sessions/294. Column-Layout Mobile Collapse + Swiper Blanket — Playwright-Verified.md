# 294. Column-Layout Mobile Collapse + Swiper Blanket — Playwright-Verified

## Open questions to resolve at session start

- **Public-side containment context for the container query.** The preview side is satisfied (`.widget-preview-scope` has `container-type: inline-size` as of 207). The container query also needs an ancestor `container-type` on the **public** render of `.page-layout`. Verify whether one exists. If not, resolve between: **(a)** add a `container-type: inline-size` (or `normal`→`inline-size`) to the public page-layout wrapper so the *same* container query drives both surfaces (preferred — single mechanism, preview and public identical); **(b)** container query for the preview + a `max-width: $bp-md` media-query fallback for public (two mechanisms — only if (a) causes a containment side-effect on existing layouts). Decide before writing the SCSS; do not pick (b) by default — it reintroduces the preview/public divergence the stub's whole approach exists to avoid.
- **Collapse threshold.** Use `$bp-md` (768px) for parity with the existing `.form-grid` collapse precedent (`_grid.scss`) unless the audit shows a layout that breaks above it — then surface, don't silently introduce a new breakpoint token.
- **`collapse_mobile` default-on for existing rows.** New key absent on every existing layout. Confirm the read path treats absent-or-true as collapse (concrete-value discipline: the renderer/inspector resolves a concrete `true` default, it does not branch on "missing"). No backfill migration; default-on at read.

## Phases

### Phase 1 — The audit oracle

Write a host-run ESM Playwright harness (`scripts/audit-mobile.js`, mirroring `generate-page-screenshots.js`'s `--base-url`/`NODE_PATH` ergonomics). For each of the five public URLs at a set of mobile widths (e.g. 320, 360, 390, 414, 767), run an in-page audit returning a structured violation list:

- **Horizontal overflow** — `documentElement.scrollWidth > innerWidth`, plus per-element offenders (`getBoundingClientRect()` past the viewport right edge).
- **Columns not collapsed** — any `.layout-grid` whose `.layout-column` children are still laid out side-by-side below the collapse threshold while its layout's resolved `collapse_mobile` is true.
- **Clipped / overflowing text** — `scrollWidth > clientWidth` on text containers; broken-word overflow.
- **Distorted declared-aspect-ratio media** — rendered ratio diverging from a declared `aspect_ratio` on Image-widget instances.

Output: machine-readable (JSON to stdout / file) so it is the loop's stop condition. Run on demand, not added to CI. Capture a `before` baseline for the close handoff.

### Phase 2 — Column-layout collapse (the core)

- `PageBlockRenderer::renderLayoutBlock()` — emit `grid-template-columns` as `--layout-cols` (single custom property); leave the rest inline.
- `resources/js/page-builder-vue/components/LayoutRegion.vue` — mirror the same `--layout-cols` so preview matches public.
- SCSS — `.layout-grid { grid-template-columns: var(--layout-cols, 1fr); }` (stylesheet now *owns* the property → the collapse override needs no `!important`). A **container query** at the threshold sets the grid to a single column, **gated on the `collapse_mobile` state** (a class/attribute the renderer + Vue emit, e.g. `.page-layout` carrying `data-collapse-mobile` or a modifier class). Resolve the public containment context per the Open question.
- `layout_config.collapse_mobile` — new bool, default true: inspector toggle on the Column Settings tab ("Collapse columns on mobile", default checked) on **both** Filament and Vue inspectors; Vue preview honors it; `ContentExporter`/`ContentImporter` round-trip it (**G1/285 precedent — a non-round-tripping `layout_config` key is silent data loss**).

### Phase 3 — Swiper blanket + overflow guards

- One blanket mobile pattern across `BlogListing`, `BlogPager`, `Carousel`, `EventsListing`, `LogoGarden`, `ProductCarousel` (collapse per-slide column count / set a mobile `slidesPerView` at the threshold). A widget whose bespoke Swiper config makes the blanket wrong becomes a **surfaced carve-out**, not a silent special-case.
- Global overflow guards (the standard set: media `max-width: 100%`, `overflow-wrap`/`min-width:0` where the audit shows offenders, container clamp) — only as the audit demands, not speculatively.

### Phase 4 — Drive the loop to zero + surface residue

Iterate fix → rebuild (`build:public` / `npm run build` per the touched path) → re-audit until the violation list is empty across all five pages × all widths, or a bounded iteration budget is hit. Set `collapse_mobile` per each page's evident intent for the two pre-identified judgment cases — the **Demo 50/50 split** and the **Pricing à-la-carte 2×2** — then surface the `before`/`after` mobile captures and the residue set (those toggle decisions + any Swiper carve-out + any public-containment note) for the user's review.
