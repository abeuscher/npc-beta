# 046. Site Theme Admin

Build the admin UI for managing the public-facing theme. Two distinct audiences are served from a single CMS › Site Theme page: non-technical admins who set brand colors and fonts through simple form controls, and developers who need direct access to the SCSS theme file. Session 045 must be complete before this session begins — the SCSS file structure and Vite build pipeline it established are prerequisites.

## Part 2 — Site Theme page

### `app/Filament/Pages/SiteThemePage.php`

A standalone Filament `Page` in the CMS navigation group.

```
navigationGroup : 'CMS'
navigationLabel : 'Site Theme'
navigationSort  : 6  (adjust if needed to sit logically after Navigation)
title           : 'Site Theme'
```

Breadcrumbs follow the standard pattern — page links to itself, current view label is non-linked:

```php
public function getBreadcrumbs(): array
{
    return [
        static::getUrl() => 'Site Theme',
        'Edit Site Theme',
    ];
}
```

`canAccess()`: requires either `view_any_page` (i.e. any CMS user can see the Appearance tab) or `edit_theme_scss`.

The page uses Filament tabs. Two tabs are always rendered; the SCSS Editor tab is only visible to users with `edit_theme_scss`.

## Part 3 — Appearance tab

A standard Filament form backed by `GeneralSettings`. Fields:

### Brand color

A colour picker input for `primary_color`. Use Filament's `ColorPicker` component. On save, the value is stored in `GeneralSettings` and will be picked up by the inline `<style>` block in the public layout on the next page request — no build step required.

### Fonts

Two `Select` fields: `heading_font` and `body_font`. Offer the following options (system stacks first, then a small curated Google Fonts list):

```php
[
    ''                                          => '— Default (Pico) —',
    'system-ui, sans-serif'                     => 'System UI',
    'Georgia, serif'                            => 'Georgia (serif)',
    "'Inter', system-ui, sans-serif"            => 'Inter',
    "'Lato', system-ui, sans-serif"             => 'Lato',
    "'Merriweather', Georgia, serif"            => 'Merriweather',
    "'Montserrat', system-ui, sans-serif"       => 'Montserrat',
    "'Open Sans', system-ui, sans-serif"        => 'Open Sans',
    "'Playfair Display', Georgia, serif"        => 'Playfair Display',
    "'Raleway', system-ui, sans-serif"          => 'Raleway',
    "'Source Sans 3', system-ui, sans-serif"    => 'Source Sans 3',
]
```

**Important:** Google Fonts values require a `<link>` tag in the public layout head. When a font value contains a Google Font name (detectable by the font name being in a curated list), the public layout must render the appropriate Google Fonts `<link>` tag. Add a helper method to `GeneralSettings` or a Blade component that returns the Google Fonts URL for the currently selected fonts, if any.

System stack values do not require any extra markup.

On save, stored to `GeneralSettings`. Takes effect on next public page request — no build required.

### Save action

A standard Filament form save action. Show a success notification. No build is triggered.

## Part 4 — SCSS Editor tab

Only visible when `auth()->user()?->can('edit_theme_scss')` is true. If the viewing user lacks the permission, the tab is hidden entirely (not shown as disabled).

### Editor field

A `Textarea` component:
- Loads its content from `resources/scss/_theme.scss` at mount
- `->rows(30)` minimum
- `->extraAttributes(['style' => 'font-family: monospace; font-size: 0.85rem;'])`
- `->label('Theme SCSS')`
- No character limit

The textarea is not a Filament form field backed by a model — it is a raw Livewire public property (`$themeScss`) populated in `mount()` and submitted via a dedicated save action.

### Save and build action

A Filament page action (not a form action) labelled **"Save & Build"** with a spinner/loading state.

On execute:

1. **Validate SCSS** — use `scssphp/scssphp` (check if it is available as a Composer dependency; if not, require it: `composer require scssphp/scssphp`). Attempt to compile the submitted SCSS content. If compilation throws, catch the exception, display a danger notification with the error message, and abort — do not write to disk.

2. **Write to file** — write the validated SCSS content to `resources/scss/_theme.scss`.

3. **Run Vite build** — shell out to `npm run build` from the project root using `Process::run()` or `shell_exec()`. Capture stdout and stderr.

4. **Show result** — if the build exits 0, show a success notification: "Theme built successfully." If it exits non-zero, show a danger notification with the first 500 characters of stderr so the developer can diagnose the failure. In either case, display the full build output in a collapsible section below the editor.

Build output is stored in a Livewire property (`$buildOutput`, `$buildSuccess`) and rendered in the blade view after the editor. The user waits on the page while the build runs — this is intentional and acceptable for a developer-only tool.

### Stored build output

Keep the last build output visible on the page even after the action completes (not just in a notification). Render it in a `<pre>` block styled monospace beneath the editor with a clear heading ("Last build output").

`resources/views/filament/pages/site-theme.blade.php`

Renders the two-tab structure. The Appearance tab renders the Filament form component. The SCSS Editor tab renders the textarea and build output block. Only render the SCSS Editor tab markup if the user has `edit_theme_scss`.
