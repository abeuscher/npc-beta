# 064. Sample Data Generator Library — Log

## What was built

### `app/Data/SampleLibrary.php` (new)

A single static class with seven methods returning plain PHP arrays of themed sample data:

- `firstNames()` — 66 entries drawn from Looney Tunes, Hanna-Barbera, Rocky & Bullwinkle, Tiny Toons, Animaniacs, and Buckaroo Banzai characters. Includes quoted nicknames (`"New Jersey"`, `"Perfect Tommy"`, `"Rawhide"`, `"Pinky"`) to exercise unusual string handling.
- `lastNames()` — 70+ entries from the same universes plus Dr. Seuss-adjacent invented surnames. `Gump` removed per review.
- `emailDomains()` — 15 clearly fictional domains (e.g. `acme.com`, `jellystone.park`, `wossamotta.edu`).
- `streetAddresses()` — 36 cartoon/Seuss-themed addresses with low house numbers.
- `cities()` — 20 fictional place names.
- `states()` — 8 US state abbreviations.
- `eventTitles()` — 40 cartoon-themed event names structured like real nonprofit events.

### `database/factories/ContactFactory.php` (updated)

- `first_name` and `last_name` now draw from `SampleLibrary::firstNames()` and `lastNames()`.
- `email` constructed as `firstname.lastname.###@domain` (80% populated, 20% null). The unique numerify suffix prevents collisions on large batches.
- `address_line_1`, `city`, `state` draw from `SampleLibrary` with 60% probability; null otherwise.
- All other fields (phone, postal_code, date_of_birth, booleans) remain Faker-generated.
- Added `withGmailBase(string $base)` factory state: strips `@gmail.com` if present, then constructs each contact's email as `localpart+firstname_lastname@gmail.com`. Allows all generated emails to route to one real inbox for Mailchimp integration testing.

### `database/factories/EventFactory.php` (updated)

- Event title now draws from `SampleLibrary::eventTitles()`. Slug still derives from the title as before.

### `app/Filament/Widgets/DashboardDebugGeneratorWidget.php` (updated)

- Added `Membership` model import and `members` case to both `generate()` and `wipe()`.
- Added `gmailBase` public property (string, default empty).
- Extracted `generateContacts()` helper: applies `withGmailBase()` when `gmailBase` is non-empty, otherwise generates normally.

### `resources/views/filament/widgets/dashboard-debug-generator-widget.blade.php` (updated)

- Added `Members` option to the record type dropdown.
- Added Gmail base input field, labelled with triple ⚠️ to flag its dev-only nature. Constructs emails as `localpart+firstname_lastname@gmail.com` when populated.
- Fixed select dropdown width (`min-w-40 pr-8`) — caret was colliding with option text.
- Removed "Only visible when APP_DEBUG_TOOLS=true. Remove before go-live." subtitle — redundant given the widget's visibility gate.

### `app/Http/Controllers/EventController.php` (updated)

Fixed a pre-existing bug: the public event registration form had no duplicate-registration guard, allowing the same email to register multiple times for the same event. Added a silent-dedup check before `EventRegistration::create()` — if a registration already exists for this `event_id` + `email`, the controller redirects with `registration_success` without creating a duplicate. The neutral response prevents the check from acting as an oracle.
