# 064. Sample Data Generator Library

Replaces Faker's generic output in the debug data generator with a curated library of cartoon-universe names, addresses, and event titles. The library draws from Warner Bros / Looney Tunes, Hanna-Barbera, Rocky & Bullwinkle, Tiny Toons, Animaniacs, and Dr. Seuss-adjacent invented names — recognisable, clearly fictional, and safe for demo sessions. Adds Members as a fourth content type in the generator widget. No schema changes required.

---

## Architecture decisions

- **Single library class**: `app/Data/SampleLibrary.php` with static array methods — `firstNames()`, `lastNames()`, `emailDomains()`, `streetAddresses()`, `cities()`, `states()`, `eventTitles()`. No subclasses, no config files, no dependencies. Plain PHP arrays checked into the repo.
- **Factories call the library**: `ContactFactory` and `EventFactory` replace Faker calls with `fake()->randomElement(SampleLibrary::someMethod())`. The library does not replace Faker entirely — Faker still generates booleans, amounts, phone numbers, and other non-themed values.
- **Email construction**: `strtolower($firstName . '.' . $lastName) . '@' . randomElement(SampleLibrary::emailDomains())` — always produces a recognisably fake address.
- **Member generation**: `Membership::factory()->count($count)->create()`. The `MembershipFactory` already creates a linked `Contact` via `Contact::factory()`, so member records automatically inherit library-themed contact data once `ContactFactory` is updated. No separate member factory changes required.
- **Member wipe**: Wipes memberships only (not their associated contacts), consistent with how the `donations` wipe does not touch contacts.

## Phase 2 — Update ContactFactory

**File:** `database/factories/ContactFactory.php`

Replace Faker calls for the following fields with `fake()->randomElement(SampleLibrary::method())`:
- `first_name` → `SampleLibrary::firstNames()`
- `last_name` → `SampleLibrary::lastNames()`
- `email` → constructed as `strtolower($firstName . '.' . $lastName . '@' . fake()->randomElement(SampleLibrary::emailDomains()))` — make unique with a numeric suffix if needed
- `address_line_1` → `SampleLibrary::streetAddresses()`
- `city` → `SampleLibrary::cities()`
- `state` → `SampleLibrary::states()`
- `address_line_2`, `postal_code`, `phone`, `date_of_birth`, `do_not_contact`, `mailing_list_opt_in` — leave as Faker

## Phase 3 — Update EventFactory

**File:** `database/factories/EventFactory.php`

Replace the event title / name field with `fake()->randomElement(SampleLibrary::eventTitles())`. Slug derives from the title as it currently does. All date, status, capacity, and other fields remain unchanged.

## Out of scope

- Volunteer generation — deferred until the Volunteer module exists
- Replacing Faker for phone numbers, postal codes, amounts, or dates — Faker is fine for these
- Any UI changes to the generator widget beyond the new option
- Library entries for organisations — HouseholdResource and OrganizationResource are not generated by the widget
