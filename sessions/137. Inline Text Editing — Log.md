## What was built

### Phase 1 — Template annotations

Added `data-config-key` and `data-config-type` attributes to widget templates so the inline editing JS knows which rendered elements map to which config fields:

- `hero.blade.php` — `content` (richtext) on `.hero-content` div
- `text_block` — DB template wrapped in annotated div (seeder + live DB updated)
- `donation-form.blade.php` — `heading` (text) on `<h2>`
- `events-listing.blade.php` — `heading` (text) on `<h2>`
- `three-buckets.blade.php` — `heading_N` (text) on `<h3>`, `body_N` (richtext) on body div

### Phase 2 — Contenteditable activation

When a widget is selected and its preview renders, JS scans for `[data-config-key]` elements and makes them editable:

- **Richtext fields:** `contenteditable="true"` — edit HTML in place, saves `innerHTML` on blur and via 600ms debounced input
- **Text fields:** `contenteditable="plaintext-only"` — Enter key suppressed, saves `innerText` on blur/debounce
- Visual affordance: dashed outline on hover, solid outline on focus (CSS in AdminPanelProvider)
- Guard against double-wiring with `_inlineWired` flag on elements

### Phase 3 — Quill toolbar (explored and removed)

Explored three approaches for inline richtext formatting toolbar (floating, in-flow between header and preview, no toolbar). After testing, the toolbar introduced UX conflicts with multiple editable regions and didn't add to the feeling of control. Decision: ship without toolbar — plain contenteditable provides clean in-place editing. Formatting remains available via the inspector's Quill editor.

### Phase 4 — Two-way inspector sync

- `PageBuilderBlock::updateInlineConfig()` — persists config to DB without re-rendering preview (user is editing live in DOM), dispatches `inline-config-updated`
- `PageBuilderInspector::onInlineConfigUpdated()` — updates inspector state, dispatches `inspector-field-updated` for Quill sync
- Inspector richtext partial listens via `Livewire.on('inspector-field-updated')` to update its Quill content
- Inspector → inline direction works via existing `widget-config-updated` → Livewire re-render cycle
- Livewire `morph.updated` hook re-initializes inline editing after preview HTML is replaced
- Cleanup event (`inline-editing-cleanup`) fires when block is deselected
