## What was built

### Phase 1 — Read-through, audit, sign-off

- Cataloged the PII scanner's rule set (5 spoof targets across 4 distinct rejection reasons: SSN-hyphenated, valid-Luhn CC PAN, ABA-prefix bare-9, non-ABA-prefix bare-9; plus ZIP-allowlist suppression and the whole-fixture header-blocked path).

### Phase 2–7 — Generator, builders, transforms, manifest, encoding (one continuous unit)

Built as a single coherent unit because the parts are tightly coupled. New artifacts:

- **Artisan command** `import-fixtures:generate` — [app/Console/Commands/GenerateImportFixtures.php](../app/Console/Commands/GenerateImportFixtures.php). Flags: `--importer`, `--shape`, `--source-preset`, `--encoding`, `--rows`, `--seed`, `--output-dir`. Emits CSV + sibling `.expected.json` per (importer, shape, preset, encoding, seed) tuple.
- **Generator service** [app/Services/Import/Fixtures/FixtureGenerator.php](../app/Services/Import/Fixtures/FixtureGenerator.php) — dispatcher used by both the artisan command and the Pest runner (so test-side generation doesn't have to shell out).
- **Per-importer FixtureBuilders** for all seven importers under [app/Services/Import/Fixtures/Importers/](../app/Services/Import/Fixtures/Importers/). Each declares `headers($preset)`, `customFieldColumns($preset)`, `cleanRow($i, $preset, $faker)`, `columnMap($preset)`, `customFieldSentinel()`, `contactMatchKey()`. Custom-field type coverage across importers: text, number, date, boolean, select, rich_text (six total per session 250's E3 work).
- **Four shape transforms** under [app/Services/Import/Fixtures/Transforms/](../app/Services/Import/Fixtures/Transforms/):
  - `MessyTransform` — date-format shuffling + leading/trailing whitespace padding (case-flipping on enum-like headers was deliberately dropped during implementation — see Findings below).
  - `CorruptTransform` — per-importer required-field blanking strategy producing deterministic skip/error outcomes (table of seven importers' blank targets); ~30% of rows additionally carry permissive-corruption strategies (control chars, malformed email, oversized cell, custom-field type mismatch) recorded in `corruption_kinds_by_row`. The importer accepts these today; manifest documents the permissiveness.
  - `PiiTransform` — five-row catalog (`PiiTransform::CATALOG`), one row per scanner detection rule, including one row planting its violation in a custom-field column to verify scanner walks custom-field cells.
  - `StressTransform` — default 1k rows, opt-in to 100k via `--rows=N`. Wide-row variant adds 50 noise CF columns to test JSONB column size.
- **Manifest writer** [app/Services/Import/Fixtures/ManifestWriter.php](../app/Services/Import/Fixtures/ManifestWriter.php) — fields: `rows_total`, `rows_expected_imported`, `rows_expected_skipped`, `rows_expected_errored`, `rows_expected_pii_rejected`, `skip_reasons_by_row`, `error_reasons_by_row`, `pii_violations_by_row`, `corruption_kinds_by_row`, `custom_field_columns`, `fixture_sha256`.
- **Encoding-aware writer** [app/Services/Import/Fixtures/CsvWriter.php](../app/Services/Import/Fixtures/CsvWriter.php) — `utf8` / `utf8-bom` (LF) / `windows-1252` (CRLF, transcoded from UTF-8 source).
- **BuilderRegistry** [app/Services/Import/Fixtures/BuilderRegistry.php](../app/Services/Import/Fixtures/BuilderRegistry.php) — central importer-name → builder-class map.

### Phase 8 — FixtureRunner service + parametrized Pest runner

- **FixtureRunner service** [app/Services/Import/FixtureRunner.php](../app/Services/Import/FixtureRunner.php) — replays `processOneRow` off-Livewire via Reflection. For `pii`-shape fixtures, invokes `PiiScanner` directly (whole-fixture rejection semantics). Pre-seeds three things automatically: a `fixture-runner@example.test` user (events `author_id`, note authorship), one `Contact` per non-blank email cell (for importers whose `contactMatchKey` is non-null), `CustomFieldDef` rows from the manifest's `custom_field_columns` (events' dual `event` / `event_registration` targets handled via `target` field).

## Findings surfaced during implementation

These are recorded here (not patched) so B2 + future importer sessions inherit them without re-discovering. Track G's standing motivation — adversarial fixtures expand coverage where real data has plateaued — was validated by the count below.

1. **Events `status` not normalized before DB insert.** A case-flipped enum value like `"Draft"` violates `events_status_check`. Discovered when `MessyTransform`'s case-flipping landed on Event Status. Resolution this session: dropped the case-flip from MessyTransform (preserves the messy shape's "should fully import" contract). Underlying gap belongs in a future session — events importer needs `mapEventStatus` parallel to donations'/memberships' normalization helpers.
2. **Importer is permissive about most "corruption."** Control chars, malformed emails, oversized cells, custom-field type mismatches all import cleanly today. CorruptTransform documents these via `corruption_kinds_by_row` annotation rather than asserting they error. If the importer ever tightens, manifests get updated.
3. **`hide_pending_imports` global scope** on Contact filters auto-created import contacts from default queries until the import session is approved. Not a bug — intentional staging gate. Worth surfacing to operators in B2's playbook so they understand pending-import contacts don't appear in normal lists.
4. **PII rule "header-blocked" path not exercised by `pii` shape.** Value-shape rules are covered (5 rows, one per detection reason). The whole-fixture-rejection-via-blocked-header path is a separate flow; left out of scope.
5. **Two sources of truth for the "Imported from X" note body** — the contacts page's private `importNoteBody('imported')` and the trait's new `importedCreationNoteBody()` produce identical strings. Low-priority cleanup deferred.

## Security checklist — verified

- [x] PII fixtures contain only synthetic patterns. SSN spoofs use the SSA-reserved 9XX area code (never issued); CC PAN spoofs use Stripe / Visa / MC public test card numbers (already non-functional in production).
- [x] `storage/app/import-test-fixtures/` and `storage/app/test-fixtures-runtime/` gitignored.
- [x] FixtureRunner inherits Pest's `RefreshDatabase` transaction wrap — no leaked rows pollute downstream tests in the suite.
- [x] PiiScanner rule coverage matches PiiTransform 1:1 — every scanner-emitted reason has a fixture row that fires it; no fixture row fires a rule that doesn't exist.
- [x] Stress shape doesn't exhaust memory under default flags. 1k rows is the default; 100k is explicit opt-in.
- [x] Generator command is dev-only — not registered as a route, not exposed via the admin UI, not scheduled.
- [x] Custom-field PII coverage — `PiiTransform::CATALOG`'s `ssn_hyphenated_custom_field` entry plants its violation inside a custom-field-shaped column.
