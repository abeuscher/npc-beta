## What was built

### Phase 1 — Schema

- `database/migrations/2026_04_20_120100_add_external_id_to_notes.php` — adds `notes.import_session_id` (UUID, FK→import_sessions, nullOnDelete), `notes.external_id` (string, nullable), and the composite index `notes_import_external_idx` on `(import_source_id, external_id)`.
- `database/migrations/2026_04_20_120200_add_notes_scope_to_import_sources.php` — adds `notes_field_map` (jsonb, default `{}`), `notes_custom_field_map` (jsonb, default `{}`), and `notes_contact_match_key` (string, nullable) to `import_sources`.
- Split into two migrations per the "one migration per table" preference.
- `App\Models\ImportSource::$fillable` + `$casts` extended with the three new note columns.
- `App\Models\Note::$fillable` extended with `import_session_id` and `external_id`.
- `docs/schema/notes.md` — new rows for `import_session_id` and `external_id`; new index listed.
- `docs/schema/import_sources.md` — three new rows documenting the notes-scoped columns; the `notes_custom_field_map` row calls out that there is no CustomFieldDef surface for notes (values are written into `notes.meta` at import time).

**Deviation from the prompt:** `import_session_id` was not in the original Phase 1 spec — it was added mid-session so the review / approve / rollback flow and the Playwright `countNotesInSession(sessionId)` helper could scope to a specific session. The first migration was rolled back, amended, and re-applied before any downstream work landed.

### Phase 2 — Field registries

- `app/Importers/NoteFieldRegistry.php` — nine hand-curated fields: `type`, `subject`, `status`, `body`, `occurred_at`, `follow_up_at`, `outcome`, `duration_minutes`, `external_id`. `notable_type` / `notable_id` / `author_id` / `import_source_id` / `import_session_id` / `meta` are intentionally excluded; these are system-managed or flow through non-dropdown paths.
- `app/Importers/NoteImportFieldRegistry.php` — aggregates `Note` + contact-match subset (email / external_id / phone). `groupedOptions()` exposes `'Note fields'` (prefixed `note:*`), `'Contact match'` (prefixed `contact:*`), and `'Other'` with `__custom_note__` and `__tag_contact__`. `split()` restricts to the two valid namespaces.

No drift between `NoteFieldRegistry` labels and the `notes` table columns — every field key in the registry matches a real column name on the `notes` table.

### Phase 3 — Admin pages + Blade views

- `app/Filament/Pages/ImportNotesPage.php` — Wizard-based admin page, `ImporterPage` breadcrumb, `getBreadcrumbs()` wired, four-step wizard (Upload → Review → Map Columns → Preview & Confirm).
- `app/Filament/Pages/ImportNotesProgressPage.php` — progress page using `InteractsWithImportProgress`, with an inline duration parser (`parseDurationMinutes`) and a `buildNoteStageAttrs` helper for staged updates.

- `resources/views/filament/pages/import-notes-review-preview.blade.php` — modal preview content for the Importer review queue; lists first 20 notes with their attached contacts.
- `resources/docs/import-notes.md` — help doc stub with the full `routes:` frontmatter array.
- `app/Filament/Pages/ImporterPage.php` — `note` added to the model_type filter; approve / rollback / delete / preview paths now branch on `note`; `rollBackNoteSession()` helper added; `downloadNotesTemplate()` method added.
- `resources/views/filament/pages/importer.blade.php` — sixth card ("Import Notes") and sixth template-download button added.

**Deviations from the prompt:**

- **No `contact_missing_strategy` radio.** The spec said "contact not found = row errors out, no auto-create", so the radio that other importers surface was deliberately omitted. The progress page has no `$contactStrategy` property and no `contact_strategy` query param.
- **Custom duplicate-strategy radio.** The shared `duplicateStrategyRadio` trait helper only exposes `skip` / `update`. The prompt called for a third `duplicate` option (create a new note anyway). The Notes page defines the radio inline with all three options rather than modifying the shared trait.
- **Contact-not-found is an error, not a skipped row.** No `skipReasons.blank_contact_key` or `skipReasons.contact_not_found` entries on the dry-run report — both conditions produce an `outcome = 'error'` row with a specific message ("Contact not found for {field} = {value}. Run the contacts importer first.") directing the admin to the contacts importer.
- **No `auto_create_custom_fields` toggle.** `processUploadedFileNamespaced(..., supportsAutoCustom: false)` — unrecognised columns stay unmapped unless the admin explicitly maps them to `__custom_note__`.

### Phase 4 — CSV template + generator + seeder

- `App\Services\Import\CsvTemplateService::noteHeaders()` — emits each `NoteFieldRegistry` label prefixed with `'Note '`, followed by `Email` / `User ID` / `Phone`. `stream()` now accepts `'notes'`.
- `App\Services\Import\FakeCsvComposer::composeNotes(int $count, array $contactEmails)` — produces one row per note with randomised type (call / meeting / email / note / task / letter / sms), subject (from a 10-phrase pool), status (completed ~80%, scheduled / cancelled / no_show otherwise), body (from an 8-line pool), `occurred_at` between `-2 years` and now, `follow_up_at` set ~50% for non-completed notes, `outcome` + `duration` (5–90 min) only for call/meeting, `External ID` `N-######`, and an email from the biased 60% reference pool.
- `App\Console\Commands\GenerateFakeImportCsvsCommand` — emits `notes.csv` at 300–600 rows (range preserved from the prompt; actual generated counts during manual verification: 366 / 598 across two runs).
- `App\Console\Commands\SeedFakeImportSourcesCommand` — sixth demo source `Demo Fake Data — Notes` seeded with `notes_field_map`, `notes_custom_field_map = []`, `notes_contact_match_key = 'contact:email'`. Description updated from "five demo import sources" to "six".

## Deferred decisions

- **Organization or Event as `notable_type` targets.** Contact only in v1; deferred.
- **Custom Fields UI for notes.** `meta` remains a schemaless catchall; no dedicated admin page planned.
- **Update-strategy Playwright spec for notes.** Pest covers the service layer; the Playwright happy-path spec is the only UI coverage. Adding a `notes-update-strategy.spec.ts` mirror of the donations spec is a natural follow-up but not needed for Beta 1.
- **Additional competitor source presets for notes** (Raiser's Edge Actions, Salesforce Task/Event, CiviCRM Activities, Bloomerang Interactions). Session 203's `guessDestination` covers the most common free-text column names as a starting point; the competitor-coverage stub will formalise per-platform presets when sample exports are in hand.
- **Canonical-type consolidation UI.** Session 198 deferred this; still deferred. The importer writes whatever `type` string the CSV has.
- **Follow-up / task dashboard.** `follow_up_at` is now populated by the importer, but the dashboard surface that uses it is a separate post-Beta stub.
- **Notes edit-restricted-to-creator permission.** Flagged during manual testing as a likely tenant-level option. Scoped into the "Notes Permissions & Permissions Audit" stub rather than session 203.
