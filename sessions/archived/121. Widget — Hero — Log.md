## What was built

### Hero widget

- Hero widget type: full-width banner with richtext content, background image, configurable overlay opacity, 9-position text alignment, minimum height selector, nav overlap toggle
- Buttons config field type (`type: buttons`): Alpine-driven repeater with add/remove/reorder, text/url/style per button, persists via `$wire.updateConfig()`
- Button component classes (`btn-primary`, `btn-secondary`, `btn-text`) in `_custom.scss` using Tailwind `@apply`
- Frontend buttons rendering partial at `widgets/components/buttons.blade.php`
- Nav overlap: controller shares `__navOverlap` flag, layout wraps header in transparent bg when hero overlaps

### Inspector field refactor

- Extracted monolithic `inspector-field.blade.php` (`@if`/`@elseif` chain) into individual partials under `inspector-fields/`
- 8 built-in types extracted: text, richtext, textarea, toggle, number, url, select, image
- Dispatcher uses `view()->exists()` lookup — adding a new field type = adding a file
- `buttons` field type added as the first custom inspector field partial

### Full-width widget system

- New `full_width` boolean column on `widget_types` table (migration)
- Container `max-w-7xl mx-auto px-4` moved from layout `<main>` into `page-widgets.blade.php`
- Non-full-width widgets get the container wrapper; full-width widgets render edge-to-edge
- Per-instance override via `style_config.full_width` checkbox in the inspector's Advanced panel
- Hero, site_header, site_footer default to `full_width: true`

### Other changes

- H1 added to Quill editor toolbar in both richtext locations
- Heading sizes restored in `public.scss` base layer (Tailwind Preflight was stripping them)
