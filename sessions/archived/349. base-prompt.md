## Process rules for every session

Base-prompt process rules apply in full. Session-specific:

- **The embedded ProseMirror editor is isolated to this widget.** Do not touch Quill, `useInlineEdit`, the 347 inline-format toolbar, or any existing rich-text surface. If the work reaches there, scope has drifted toward the deferred migration track — stop and surface.
- **Store HTML, not ProseMirror-JSON.** The table persists as sanitized HTML in `config.table_html`; the editor parses it back on open. This keeps it non-destructive and bundle-clean — the same key decision the deferred track rests on.
- **The sanitizer extension is the one back-end change and it handles untrusted (pasted) input.** Extend `HtmlSanitizer`'s `ATTRS_PER_TAG` for `colspan`/`rowspan` (bounded-integer-validated) + a constrained alignment class; keep stripping style/script/handlers. Security-reviewed allow-list, with Pest cases.
- **Builds:** `npm run build` after the ProseMirror editor / field changes (the page-builder bundle), **and** `docker compose exec app php artisan build:public` after the widget `styles.scss` (the public widget asset bundle). The public render carries **no editor JS**.
- **Settle the open questions at session start** (editor surface placement; v1 toolbar/command set; storage key + paste scope) — log them.
