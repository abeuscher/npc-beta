### Flag W6/A (NEW) — `canAccess()` coverage gap → CLOSED in-session

Pre-fix: 18 of 21 admin Resources lacked explicit `canAccess()`; 3 had it (CollectionResource, MembershipTierResource, RoleResource). All resource permissions were already seeded via `PermissionSeeder.php`; per-action `->visible()` and `abort_unless()` gates compensated functionally — but the codebase-wide invariant the audit prompt names ("every Filament Resource has `canAccess()`") was unmet, and resources still appeared in the side nav for users who lacked any permission to act on them.

Resolution: closed in-session as commit `0bed43e`. 18 Resources gained `canAccess(): bool { return auth()->user()?->can('view_any_X') ?? false; }` with X mapped to the existing `PermissionSeeder` snake_case convention.

### Flag W6/B (NEW) — Observer coverage gap → CARRIED to 272 W3

Four CRM models lack observers: `Note`, `Organization`, `Affiliation`, `DonationCredit`. Other tracked models all have observers (Contact, Donation, Membership, Event, Page, Product, ProductPrice, Purchase, EventRegistration, CustomFieldDef). Note's gap is the standing 206 Flag C (deferred to C1 Notes Permissions session). Organization/Affiliation/DonationCredit gaps are NEW — B1a/B1b shipped without observers; org/affiliation/credit CUD does not appear in `activity_logs`.

**Decision points (272 W3 proposes a path):**
- (a) Add observers in 273 to match the established pattern.
- (b) Defer Org/Affiliation/DonationCredit to a dedicated audit-trail session.
- (c) Confirm the gap is intentional (post-Beta-1 audit-log redesign?).
