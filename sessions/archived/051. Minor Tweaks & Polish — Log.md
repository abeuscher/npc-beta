## Workstream 2 — Upload Field UX

**Investigation findings:** Livewire's file upload runs as a two-phase process. Phase 1 is the XHR upload to `/livewire/upload-file`; phase 2 is a component state sync round-trip that sets the field green. `livewire-upload-finish` fires after phase 1.

**Implemented:**
- **Site-wide loading bar** — A `wire:loading.delay` fixed top bar (1px, primary colour, `z-[200]`) injected via `PanelsRenderHook::BODY_START` in `AdminPanelProvider`. Covers all Livewire round-trips across the admin with a 200ms delay to avoid flashing on fast responses.
- **Importer upload blocker** — `import-contacts.blade.php` wrapped in an Alpine block that listens for `livewire-upload-start/finish/error` events, applies `pointer-events-none opacity-60` to the form during upload, and shows "Uploading file, please wait…" below it.
- **Helper text updated** — CSV field helper text updated to "Max 10 MB. CSV or plain text only. Wait for the field above to turn green before advancing to the next stage." The old "Wait for the progress bar to disappear" instruction was removed.

**Investigated and deferred:** Auto-advancing the wizard on upload complete. A `$wire.dispatchFormEvent('wizard::nextStep', 'data', 1)` approach was attempted but reverted — the timing between `livewire-upload-finish` and the component state being fully ready proved unreliable. Documented in log; the loading bar covers the remaining gap visually.

## Workstream 7 — Debug Data Generator Widget

`app/Filament/Widgets/DashboardDebugGeneratorWidget.php` and its blade view created.

- Gated on `filter_var(env('APP_DEBUG_TOOLS', false), FILTER_VALIDATE_BOOLEAN)` — handles both `.env` file boolean casting and raw Docker environment string values (`"true"`).
- Supports contacts, events, and donations (all three have self-contained factories).
- Quantity capped at 200 server-side.
- Wipe uses `forceDelete` with soft-delete awareness (`class_uses_recursive` check).
- `APP_DEBUG_TOOLS=false` added to `.env.example`.
- Sort: 99 (renders last on dashboard). Column span: full.

**Production debugging note:** The initial `=== true` comparison failed on production because Docker-injected env vars arrive as strings, not booleans. `filter_var` with `FILTER_VALIDATE_BOOLEAN` resolves this for all environments.
