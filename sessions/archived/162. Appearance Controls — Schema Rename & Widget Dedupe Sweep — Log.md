### Post-implementation — RTE typing race investigation

After manual testing, the user reported character drops / editor wipes when typing into the page builder Quill RTE at moderate speed. Investigation traced this to a structural Vue/timing race that pre-dates session 162 (originated in the session 149 Vue migration), made more visible by the freshly-seeded baseline state introduced by `migrate:fresh --seed`.

Mechanism: bulk default-write on first widget click queues a save before the user types → store debounce timer fires before Quill's field-level debounce flushes → server response arrives and `Object.assign` overwrites the local widget's `config` (including in-flight typing) → Quill watch detects the stale `innerHTML` and wipes the editor.

Surgical mitigation applied this session:

- Deleted the `onMounted` defaults block from [resources/js/page-builder-vue/components/WidgetAppearanceControls.vue](resources/js/page-builder-vue/components/WidgetAppearanceControls.vue). This block was auto-writing default `appearance_config.background.color` and `appearance_config.text.color` values on every widget click, which was queuing a save at exactly the moment the user would start typing. Render-time `?? '#ffffff'` / `?? '#000000'` fallbacks at lines 38–39 cover the visual default; nothing else changes. Reduces the frequency of the race but does not close it structurally.

The full structural fix (delete the parallel anti-pattern in `InspectorField.vue:29-37`, make `editor.ts updateWidget`'s `Object.assign` skip user-mutable fields when the widget is in `dirtyWidgets`, remove the redundant 300 ms field-level Quill debounce in `RichTextField.vue:57-63`) is scoped into session 163.

## Production deployment notes

- Running the rename migration.
- A full reseed (or a one-time data migration) is recommended because the seeder-supplied widget instances and the demo `Template.definition` rows still reference the old column shape until reseeded. The session prompt explicitly called for `migrate:fresh --seed`.
