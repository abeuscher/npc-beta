## What was built

### Phase 1 — Page builder: default selection, add-widget selection, sticky inspector tab

- Added `inspectorTopTab` and `inspectorBottomTab` refs to the Pinia editor store so the active tab survives widget switches within a session. `InspectorPanel.vue` now binds its `topTab` / `bottomTab` to store-backed computed refs with setters.
- Added `selectFirstRootItemIfNone()` store action, called from `App.vue` onMounted after `loadTree`. When a page has at least one root widget or layout, the first one is auto-selected — inspector populates immediately, no empty-state placeholder on first paint.
- `loadTree` also clears `selectedItemId` if the selection no longer resolves after a tree reload.
- Add Widget already selected the new widget via the `widget-created` event handler ([App.vue:24-32](resources/js/page-builder-vue/App.vue#L24-L32)); Add Columns already selected the new layout ([PreviewCanvas.vue:73-83](resources/js/page-builder-vue/components/PreviewCanvas.vue#L73-L83)). No changes needed to those paths.

### Phase 2 — Page builder: focus scroll behaviour

- Must-have shipped: new utility at [resources/js/page-builder-vue/utils/focusScroll.ts](resources/js/page-builder-vue/utils/focusScroll.ts) resolves the selected item by `[data-widget-id]` / `[data-layout-id]` and calls `scrollIntoView({ block: 'center', behavior: reduceMotion ? 'auto' : 'smooth' })`. `PreviewCanvas.vue` watches `[selectedItemId, selectedItemType]` and calls the util after `nextTick`.
- **Stretch clamp descoped.** The scroll-lock and tall-widget-exception pieces were skipped before implementation. Reason: per the session prompt's escape hatch, the clamp fight turns adversarial against wheel/touch inertial scrolling and the column-widget nested-scroll case — low ROI for session 204's scope. Successor stub added to `session-outlines.md`.

### Phase 3 — Admin left nav: expand-into-view

- New [resources/js/admin/sidebar-expand-scroll.js](resources/js/admin/sidebar-expand-scroll.js) imported from `resources/js/admin.js`. Click-delegated listener on `.fi-sidebar-group button`; waits 320ms for Filament's collapse transition to settle, then scrolls the nearest `overflow-y: auto`/`scroll` parent so the expanded group fits. If the group is taller than the scroll container, the scroll prioritises keeping the group header visible (per prompt's "header takes precedence").

### Phase 4 — Settings pages: per-section Save buttons

- New shared trait [app/Filament/Pages/Concerns/InteractsWithSectionedSettings.php](app/Filament/Pages/Concerns/InteractsWithSectionedSettings.php) providing `sectionSaveAction(id, label)` (Filament Actions builder) and `saveSection(id, label)` (public Livewire entry point). Each page implements `persistSection(string $id)` with a `match` on the section ID.
- Per-section Save buttons shipped on:

  - **MailSettingsPage** — Sending / MailChimp Configuration
- Global Save button retained on all four pages. Action-only sections (secret-key flows, QuickBooks environment toggle, QuickBooks connect/disconnect) deliberately excluded — they have their own modal actions.
- Routing section preserves the full slug-cascade logic (blog/events/system prefix rename walks `CmsPage`) and the demo-mode guard when invoked via per-section save.

### Incidental fixes made during the session

- **Hero content field redundant label.** User reported the "Content" label at the top of the Hero content tab was redundant with the tab title. Changed Hero's `content` field in the schema to `'label' => ''`; [InspectorField.vue](resources/js/page-builder-vue/components/InspectorField.vue) now skips rendering the `<label>` when `field.label` is falsy (no layout gap).
- **Events listing columns dropdown off-by-one.** User reported the Columns-per-row control saved N-1 instead of N (e.g. selecting "4" stored "3"). Root cause: Laravel's `JsonResource::removeMissingValues()` calls `array_values()` on any array whose keys are all numeric, which silently re-indexed `['1'=>'1', …, '6'=>'6']` to `['1','2',…,'6']`. Vue then iterated it as an array and emitted `<option value="0">1</option>`. Fix: `public $preserveKeys = true;` on [WidgetResource.php](app/Http/Resources/WidgetResource.php) (inherited by `WidgetPreviewResource`). Regression test: [WidgetResourceOptionsPreserveKeysTest](tests/Feature/WidgetResourceOptionsPreserveKeysTest.php).
- **`build-styles` npm script.** Added `"build-styles": "docker compose exec app php artisan build:public"` to [package.json](package.json) so the public widget asset bundle can be kicked off without remembering the artisan incantation.
