## What was built

### Phase 1 ‚Äî Reusable wizard action factory

- Created `app/Filament/Actions/EmailPreviewWizardAction.php` ‚Äî static factory class wrapping a 3-step Filament wizard action.
- Step 1 (Confirm): shows email type name and recipient summary (supports lazy closure).
- Step 2 (Preview): renders the first-recipient email in a sandboxed `<iframe srcdoc>` with a link to System Emails settings.
- Step 3 (Send): editable test-email address field (pre-populated with current admin's email), outlined "Send Test Email" button, and a confirmation message followed by the final submit button.
- Factory parameters: `name`, `emailTypeName`, `recipientSummary`, `previewHtmlResolver`, `sendCallable`, `step1ExtraSchema`, `submitLabel`, `testHtmlResolver` (optional ‚Äî falls back to `previewHtmlResolver` if null, used to pass scrubbed placeholder data instead of real recipient data).

### Phase 2 ‚Äî Retrofit to Donors page

- Replaced the previous `ActionGroup`-wrapped confirmation actions on `DonorsPage` with two standalone wizard actions (Send Pending, Force Re-send All).
- Added `receiptPreviewHtml(bool $forceAll)` method for live preview from first recipient.
- Added `receiptTestHtml()` method rendering the donation receipt template with bracket placeholders (`[Recipient Name]`, `[Fund Name]`, `[0.00]`, etc.) ‚Äî used as `testHtmlResolver` so no real financial data is sent in test emails.
- Both actions conditionally visible only when the donor table has entries.

### Phase 3 ‚Äî Retrofit to Admin Invitations

- Replaced `EditUser` invitation and resend actions with wizard actions.
- Added `invitationPreviewHtml()` method using placeholder `#` URL for preview.
- Role selector (`step1ExtraSchema`) retained in Step 1 for the invite action.

### Phase 4 ‚Äî Event Cancellation moved from observer to wizard

- Removed auto-send from `EventObserver::updated()` ‚Äî cancellation emails are no longer sent on model status change.
- Added Cancel Event wizard action to `EditEvent` with visibility guard (`status !== 'cancelled'`).
- Added `cancellationPreviewHtml()` method on `EditEvent`.
- Send callable loops registered attendees, sends `EventCancellation` mailable, updates event status to cancelled, shows notification.
- Rewrote `EventCancellationTest` to reflect the new behaviour (observer no longer auto-sends; subject test instantiates the mailable directly).

### Phase 5 ‚Äî Patches

- **patch-001**: Fixed silent failure caused by Filament v3 `ActionGroup` + wizard `->steps()` incompatibility (Livewire swallows the exception in production). Removed the `ActionGroup` wrapper; both donor actions became standalone header buttons.
- **patch-002**: Added conditional visibility on donor actions; added System Emails link in Step 2; added "Send Test Email" button in Step 3.
- **patch-003**: Made test email address editable (TextInput pre-populated with admin email); added optional `testHtmlResolver` parameter; added `receiptTestHtml()` to DonorsPage with scrubbed placeholder tokens.

### Phase 6 ‚Äî Documentation

- Updated `resources/docs/tax-receipts.md` to describe wizard flow.
- Updated `resources/docs/user-invitations.md` to describe wizard send/resend.
- Updated `resources/docs/events.md` to add Cancelling an Event section.
