# 220. Widget Primitive — Phase 4e (BoardMembers + LogoGarden Retrofit)

Retrofit the last two slot-declaring SWCT widgets onto data contracts. Both have **per-instance dynamic `fields` lists** (same shape as BarChart in 219): admins map Collection-field keys to widget concepts (`name_field`, `image_field`, etc.), and the contract's whitelist is the union of those non-empty mappings. After this session lands, the `WidgetRenderer::render()` line 46 legacy loop iterates 0 times for every widget in `app/Widgets/*`, and `WidgetDataResolver::resolveCustom` becomes unreferenced from the renderer — formal retirement of the slot-strip mechanism becomes possible (and is tracked as the Phase 4 follower).

---

## Design decisions (resolved before starting)

- **Two retrofits, one session.** Both widgets are SWCT, both have per-instance dynamic fields, both ship the same standard test-lock pair (Test A: fail-closed whitelist + sentinel; Test B: one-collections + one-collection_items + one-media query pin). Mechanically simpler per-widget than 219; the pair fits in one session because the second widget mirrors the first.
- **Contract shape: per-instance, dynamic mixed text+image fields.** Both widgets' `dataContract()` collects all non-empty user-configured field mappings and declares them on an inline `ContentType`. Image-typed mappings (`image_field`) get `'type' => 'image'` so the projector emits `_media[$imageField]`. Text-typed mappings get `'type' => 'text'`. The contract's `fields` array is the union of all mapped keys (no per-type splitting in `fields` — that's a `ContentType` shape concern, not a contract whitelist concern).
- **BoardMembers field set.** Up to 8 text mappings (`name_field`, `title_field`, `department_field`, `description_field`, `linkedin_field`, `github_field`, `extra_url_field`, `extra_url_label_field`) + 1 image mapping (`image_field`). `requiredConfig` only requires `collection_handle`; everything else is optional. ContentType handle: `'board_members.member'`.
- **LogoGarden field set.** 1 image mapping (`image_field`, required) + up to 1 text mapping (`name_field`, optional, gated on `show_name`). ContentType handle: `'logo_garden.logo'`.
- **Source: `SOURCE_WIDGET_CONTENT_TYPE`.** `resourceHandle` = `$config['collection_handle']`. Same source as Carousel and BarChart; the resolver already handles this case (`ContractResolver::resolveWidgetContentType`).
- **Template change.** `$collectionData['members'] ?? []` → `$widgetData['items'] ?? []` (BoardMembers). `$collectionData['logos'] ?? []` → `$widgetData['items'] ?? []` (LogoGarden). Downstream `$member[$nameField]` / `$logo[$nameField]` access patterns unchanged — the projected DTO row carries those user-mapped keys verbatim. The `$member['_media']` / `$logo['_media']` reads continue to work because both widgets' content types declare image fields.
- **`collections()` removal for both.** Per the system rule established in 218: contract-locked widgets do not declare slots. Remove `BoardMembersDefinition::collections()` and `LogoGardenDefinition::collections()`. Admin preview shows the unmounted state for empty/unconfigured widgets (correct UX). Demo data flows through each widget's existing `DemoSeeder`.
- **Run the seeder after removing `collections()`.** `docker compose exec app php artisan db:seed --class=WidgetTypeSeeder --force` syncs `widget_types.collections` for both widgets to `[]`. Skipped in 218 (left `carousel.collections` stale at `["slides"]`) and 219 (left `bar_chart.collections` stale at `["data"]`); user surfaced the bug in 219 manual testing. Carry-forward discipline.
- **No sibling-test slot-strip retargeting.** 219 retired Test 1 of `PageBuilderApiTest`'s slot-strip pair and rewrote Test 2 to use `'carousel'` (a contract-locked widget). No further swap needed in 220.
- **Test-lock fields.** Two new test files: `tests/Feature/BoardMembersContractRetrofitTest.php` and `tests/Feature/LogoGardenContractRetrofitTest.php`. Each follows the BarChart/Carousel shape. Test A asserts the projector emits exactly the configured field keys plus `_media` (since both content types declare image fields). Sentinels go in keys without a derived whitelisted counterpart — for these widgets, that means undeclared Collection-item `data` keys not mapped to any widget concept. Use sentinel keys `internal_notes` and `legacy_id` (same pattern as Carousel/BarChart). Test B asserts exactly one `select` against `"collections"`, one against `"collection_items"`, one against `"media"`.
- **Public-surface gate.** `ContractResolver::resolveWidgetContentType()` filters by `is_public = true` when `$context->publicSurface` is true (canvas-slot default). Test fixtures' Collections must be `is_public => true, is_active => true`.
- **Demo seeders unchanged.** Both `DemoSeeder.php` files already create real Collections — no fallback path involved. The dev demo routes render against live data via the contract. No seeder edits.
- **No new schema.** No migration, no `docs/schema/` updates.
- **Front-end builds.** `script.js` (none for BoardMembers; LogoGarden has no script either), SCSS, and `assets()` declarations are untouched. Default expectation: no `npm run build`, no `build:public`. Run only if cleanup unexpectedly touches widget asset files.
- **Out of scope: query settings on contracts.** Queued as the Phase 4 follower in `session-outlines.md`. Belongs in the next session, not this one.

## Phase 2 — Contract additions

`WidgetContentTypeProjector` already handles SWCT projection generically — no projector edits needed. The projection map is the contract's `fields` list (which the widget controls per-instance).

### BoardMembers

Add `dataContract(array $config): ?DataContract` to `BoardMembersDefinition.php`:

```php
public function dataContract(array $config): ?DataContract
{
    $imageField = (string) ($config['image_field'] ?? '');
    $textFieldKeys = ['name_field', 'title_field', 'department_field', 'description_field', 'linkedin_field', 'github_field', 'extra_url_field', 'extra_url_label_field'];
    $textFields = array_values(array_filter(
        array_map(fn ($k) => (string) ($config[$k] ?? ''), $textFieldKeys),
        fn ($f) => $f !== '',
    ));

    $imageFields = $imageField !== '' ? [$imageField] : [];
    $allFields = array_values(array_merge($imageFields, $textFields));

    $contentTypeFields = [];
    foreach ($imageFields as $f) {
        $contentTypeFields[] = ['key' => $f, 'type' => 'image'];
    }
    foreach ($textFields as $f) {
        $contentTypeFields[] = ['key' => $f, 'type' => 'text'];
    }

    return new DataContract(
        version: '1.0.0',
        source: DataContract::SOURCE_WIDGET_CONTENT_TYPE,
        fields: $allFields,
        resourceHandle: $config['collection_handle'] ?? null,
        contentType: new ContentType(handle: 'board_members.member', fields: $contentTypeFields),
    );
}
```

### LogoGarden

Add `dataContract(array $config): ?DataContract` to `LogoGardenDefinition.php`:

```php
public function dataContract(array $config): ?DataContract
{
    $imageField = (string) ($config['image_field'] ?? '');
    $nameField = (string) ($config['name_field'] ?? '');

    $imageFields = $imageField !== '' ? [$imageField] : [];
    $textFields = $nameField !== '' ? [$nameField] : [];
    $allFields = array_values(array_merge($imageFields, $textFields));

    $contentTypeFields = [];
    foreach ($imageFields as $f) {
        $contentTypeFields[] = ['key' => $f, 'type' => 'image'];
    }
    foreach ($textFields as $f) {
        $contentTypeFields[] = ['key' => $f, 'type' => 'text'];
    }

    return new DataContract(
        version: '1.0.0',
        source: DataContract::SOURCE_WIDGET_CONTENT_TYPE,
        fields: $allFields,
        resourceHandle: $config['collection_handle'] ?? null,
        contentType: new ContentType(handle: 'logo_garden.logo', fields: $contentTypeFields),
    );
}
```

Add `use App\WidgetPrimitive\ContentType;` and `use App\WidgetPrimitive\DataContract;` imports to both definition files.
