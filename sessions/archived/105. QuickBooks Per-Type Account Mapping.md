# 105. QuickBooks Per-Type Account Mapping

Replace the single global income account with a per-transaction-type mapping so that donations, product purchases, and refunds can post to separate QuickBooks income accounts. Optionally allow per-fund overrides for donations.

---

## Design decisions (resolved before starting)

- **Mapping granularity:** by transaction subject type. The system recognises three subject types that produce transactions: `Donation`, `Purchase`, and generic (no subject / manual). Refunds inherit their original transaction's subject type. Each type can be mapped to a different QB income account.
- **Per-fund override (stretch):** donations can optionally be mapped at the fund level — Fund A to one account, Fund B to another. If a fund has no override, the donation-level default applies. Decision: implement this if time permits, otherwise defer to a future session. Design the schema to support it either way.
- **Fallback:** if a transaction's type has no specific mapping, fall back to the global `qb_income_account_id` from session 103. If that is also empty, sync is skipped (existing behaviour).
- **Storage:** JSON site setting `qb_account_map` — a keyed object, e.g. `{"donation": "42", "purchase": "55", "default": "42"}`. Per-fund overrides stored on the `funds` table as `quickbooks_account_id` (nullable string column).
- **UI location:** Finance Settings page, in the existing "QuickBooks — Transaction Sync" section. Replace the single dropdown with a mapping table.
- **Account type correction:** session 103 queried Income-type accounts for the deposit dropdown, but QuickBooks requires `DepositToAccountRef` to be a Bank or Other Current Asset account (e.g. "Undeposited Funds"). The account query must be updated to fetch the correct types. The "income account" label should be corrected throughout the UI and help docs.
- **Account dropdown reuse:** all dropdowns in the mapping table use the same corrected cached account list.

## Phase 1 — Account Resolution Logic

### Migration

Add to the `funds` table:

- `quickbooks_account_id` — nullable string. Per-fund QB income account override.

Update `docs/schema.md` after writing the migration.

### QuickBooksClient — account resolution

Add a method to `QuickBooksClient`:

- `resolveIncomeAccount(Transaction $transaction): string` — determines which QB income account to use for a given transaction. Resolution order:
  1. If the transaction's subject is a `Donation` with a `fund_id`, and that fund has a `quickbooks_account_id` set, use it.
  2. If `qb_account_map` has an entry for the transaction's subject type key (`donation`, `purchase`), use it.
  3. Fall back to `qb_income_account_id` (the global default).
  4. If nothing is configured, throw the same "no income account" exception as today.

Update `createSalesReceipt` and `createRefundReceipt` to call `resolveIncomeAccount` instead of reading the global setting directly.

### Subject type key mapping

Define a simple mapping from model class to key:

- `App\Models\Donation` → `donation`
- `App\Models\Purchase` → `purchase`
- Everything else (null subject, manual transactions) → `default`

## Phase 2 — Settings UI

### Replace single dropdown with mapping table

In the "QuickBooks — Transaction Sync" section of Finance Settings:

- **Default account** — a select dropdown (replaces the existing single `qb_income_account_id` dropdown). This is the fallback for any unmapped type.
- **Donations** — a select dropdown. Label: "Donations". Helper text: "Income account for donation payments. Leave blank to use the default."
- **Product purchases** — a select dropdown. Label: "Product Purchases". Helper text: "Income account for product sales. Leave blank to use the default."
- **Refunds** — informational note: "Refunds post to the same account as their original transaction."

All dropdowns share the same cached QB income account options list.

The "Refresh Accounts" button remains and refreshes all dropdowns.

### Save logic

On save, construct the `qb_account_map` JSON from the form fields and store as a site setting. Continue storing `qb_income_account_id` as the default so session 103's fallback logic remains intact without changes to the sync job's skip conditions.

### Per-fund override on Fund resource

Add an optional "QuickBooks Account" dropdown to the Fund create/edit form. Only visible when QuickBooks is connected. Uses the same cached account list. Stores in `funds.quickbooks_account_id`.
