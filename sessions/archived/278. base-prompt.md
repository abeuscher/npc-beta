## Style rules

The style rules in `sessions/template-base-prompt.md` apply. Session-specific:

- **Iteration commit messages describe the deliverable** — e.g. "Iteration /1 — TicketTier model + migration + General-tier backfill" or "Iteration /2 — Admin tier repeater" or "Iteration /3 — Public tier picker + EventRegistration FK".
- **Tier shape is event-scoped, not global.** Tiers belong to one event (`events hasMany ticket_tiers`); no shared tier pool. Renaming "General" later is fine.
- **The data migration is part of the schema migration, not a separate seeder.** Backfill runs once on `migrate`; never re-runs. If an event was free at migrate time and later becomes paid, the operator adds the tier(s) manually — no auto-backfill on price change.
- **No docstrings, comments, or type annotations on code you did not write.**
- **Simple, correct, well-considered solutions over fast or clever ones.**
