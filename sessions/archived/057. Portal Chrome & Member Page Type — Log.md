# 057. Portal Chrome & Member Page Type — Log

**Date:** 2026-03-22
**Branch:** session-057 + patches 001–004

## What was built

### Phase 1 — Portal layout

Created `resources/views/layouts/portal.blade.php`. Shares the same Vite asset bundle as the public layout (`public.scss` / `public.js`). Includes a distinct portal header showing the site name + "Member Area" label, the authenticated member's first name, and a logout button. No public site nav or footer. Supports `$inlineStyles` and `$inlineScripts` so page builder widgets render correctly on member pages.

Added portal header CSS to `resources/scss/_layout.scss`: `#f1f1f1` background in light mode, `#1e1e1e` in dark mode, with explicit text colours. Both Pico's `[data-theme="dark"]` attribute and `prefers-color-scheme: dark` are handled.

### Phase 2 — Portal prefix SiteSetting

Added a **Routing** section to `GeneralSettingsPage` with a `portal_prefix` field (alpha-dash, required, default `members`). Helper text warns that changing the value will break bookmarked links. Mounted from and saved to `SiteSetting`.

### Phase 3 — Member page type

Migration `2026_03_22_000001_add_member_type_to_pages` adds a PostgreSQL check constraint enforcing `type IN ('default', 'post', 'event', 'member')`.

`PageResource` updated:
- Create form: a **Page Type** select (Web Page / Member Page) sits to the right of the title field. Post and Event are not offered at creation — those types are set programmatically.
- Edit form: type is displayed as a read-only `Placeholder` in the page heading (via `EditPage::getTitle()`) — "Edit Member Page", "Edit Web Page", etc. Type cannot be changed after creation.
- Slug field: when type is `member`, the field is disabled, shows the `portal_prefix/` as a decorative prefix, and uses `formatStateUsing` to display only the slug tail (e.g. `my-page` rather than `members/my-page`).

`PageObserver` updated:
- Added `created()` hook so member pages receive the `portal_prefix/` slug prefix immediately on first save.
- Added `member` handling to `updated()` for completeness.
- Extracted `applyMemberPrefix()` private helper.

`PageController::show()` updated: if the page type is `member`, unauthenticated visitors are redirected to `portal.login`; verified-but-unverified users are redirected to `portal.verification.notice`.

`resources/views/pages/show.blade.php` updated: member-type pages extend `layouts.portal` rather than `layouts.public`.

### Phase 4 — Account view

`resources/views/portal/account.blade.php` converted to extend `layouts.portal`. Logout button removed from the view body — it is now in the portal header.

### Phase 5 — Help doc stub

Created `resources/docs/portal-pages.md` with standard frontmatter. Route `portal.account` registered.
