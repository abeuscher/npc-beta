# 118. Help System — Search, Standalone Pages & Related Articles

Add searchable help, standalone article pages with related-article navigation, and a `category` taxonomy to the existing help system. Remove the placeholder dashboard widgets and put a help search box in the admin top bar.

---

## Design decisions (resolved before starting)

- Help search is a custom **Livewire component** in the Filament top bar — not Filament's built-in global search (which is designed for resource records, not help content).
- The existing **slide-over** contextual help (the `?` icon on each page) remains unchanged. Standalone help pages are a separate, deeper reading experience — think of them as the slide-over content surrounded by helpful chrome and context.
- **Category** is a new top-level grouping that maps to the admin left-nav groups: CRM, CMS, Finance, Tools, Settings, General. It is added to frontmatter and the data model but is *not* used for navigation in this session — it paves the way for a future navigable help index.
- Search ranking in this session is simple and deterministic: title match > tag match > content match. Weighted/semantic search using the `embedding` column is deferred.
- No tag management UI — tags are authored in markdown frontmatter and synced via `help:sync`.

## Phase 3 — Help search in the top bar

### 3a. Livewire search component

Create a Livewire component `HelpSearch` that:

- Renders as a search input in the Filament top bar (use `PanelsRenderHook::GLOBAL_SEARCH_BEFORE` or another appropriate hook — investigate which hook places the component to the left of the user menu)
- On input (debounced ~300ms), queries `help_articles` with simple ranking:
  1. **Title contains** the search term (highest priority)
  2. **Tags contain** the search term
  3. **Description contains** the search term
  4. **Content contains** the search term (lowest priority)
- Displays a dropdown of results (max 8) with title, description excerpt, and category badge
- Clicking a result navigates to the standalone help page (Phase 4)
- Escape or clicking outside closes the dropdown
- Empty input closes the dropdown

### 3b. Styling

Style the search input and dropdown to be consistent with the existing Filament admin theme. The search input should be compact — similar in size to Filament's built-in global search bar. Use the existing `public/css/admin.css` for any custom styles.

## Phase 4 — Standalone help page

### 4a. Filament page

Create a Filament page at `/admin/help/{slug}` that:

- Renders the article's markdown content using `Str::markdown()`
- Shows the article title, description, category badge, last-updated date, and version
- Does **not** appear in the left navigation
- Has breadcrumbs: `Dashboard > Help > {Article Title}`
- Is accessible to all authenticated admin users (no permission gate)

### 4b. Related articles sidebar

On the standalone help page, show a "Related Articles" sidebar (or section below content on mobile). Related articles are determined by tag overlap:

- Query `help_articles` where the article shares at least one tag with the current article
- Exclude the current article
- Order by number of overlapping tags (descending), then by title
- Show up to 6 related articles with title, description, and category badge
- Each links to its standalone help page

### 4c. "View full article" link from slide-over

In the existing slide-over component (`help-slide-over.blade.php`), add a link at the bottom that navigates to the standalone help page for the current article. Something like "Open in full page →".
