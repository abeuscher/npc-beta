# Session 098 — Additional Widget Types — Calendar, Chart & Video — Log

**Date:** 2026-03-30

## What was built

### Phase 1 — Install dependencies

- Added `jcalendar.js` (Calendar.js v2.12.6) and `chart.js` v4.4 to `package.json`.
- Copied dist files to `public/vendor/calendarjs/` and `public/vendor/chartjs/` for conditional loading.

### Phase 2 — Event calendar widget

- New public JSON endpoint `/api/events.json` returning published events (id, title, from, to, description, url). Scoped to published status, last 6 months onward.
- New `event_calendar` widget type with Calendar.js integration. Month/week views, fully read-only (all editing, export, import, search, config dialogs disabled). Events rendered as holidays with `onClick` handlers linking to event landing pages.
- Calendar.js toolbar stripped to essentials — only month/week navigation remains.

### Phase 3 — Bar chart widget

- New `bar_chart` widget type using Chart.js. Feeds off a collection via the field-selector pattern (same as carousel). Renders a `<canvas>` element with configurable axis labels and bar colour. Falls back to `--color-primary` for bar colour.
- Widget renders nothing when no collection data is connected.

### Phase 4 — Video embed widget

- New `video_embed` widget type. Parses YouTube and Vimeo URLs via regex, renders privacy-enhanced `<iframe>` using `youtube-nocookie.com`. Responsive 16:9 container via padding-bottom trick. Graceful "Unsupported video URL" fallback for non-matching URLs.
- Pure server-rendered HTML, no JavaScript required.

### Widget asset pipeline

- New `assets` JSONB column on `widget_types` table with keys: `css` (string[]), `js` (string[]), `scss` (string[]).
- PageController and ChromeRenderer collect and deduplicate asset paths from active widgets.
- Public layout renders `<link>` tags in `<head>` for CSS assets, `<script>` tags before `</body>` for JS assets.
- SCSS assets compiled at runtime via `scssphp/scssphp` with 1-hour cache.
- Calendar and chart widgets migrated to use the asset pipeline — no more inline `<link>` tags in template bodies or dynamic `document.createElement('script')` hacks.

### Calendar theme integration

- SCSS partial `resources/scss/widgets/_event-calendar.scss` maps site theme CSS variables (`--color-primary`, `--font-family-body`) to Calendar.js CSS variables for visual consistency.

### Chart Demo collection

- `ChartDemoSeeder` creates a "Chart Demo" collection with 10 monthly data points (label + value fields) for testing the bar chart widget.

### Add Block modal fix

- Fixed multi-click bug in both root and column child Add Block modals. Alpine `picked` flag prevents additional widget creation between the first click and modal close.

### Session 099 & 100 prompts

- Wrote session prompts and base prompts for the template system (data model + migration in 099, UI + page creation flow in 100).

## Production deployment notes

- Run `npm install` inside the app container to install `jcalendar.js` and `chart.js`.
- Copy dist files: `cp node_modules/jcalendar.js/dist/calendar.min.js public/vendor/calendarjs/`, `cp node_modules/jcalendar.js/dist/calendar.js.min.css public/vendor/calendarjs/`, `cp node_modules/chart.js/dist/chart.umd.js public/vendor/chartjs/`.
- Run `php artisan migrate` for the `assets` column.
- Run `php artisan db:seed --class=WidgetTypeSeeder` to register the three new widget types and their asset declarations.
- Run `php artisan db:seed --class=ChartDemoSeeder` to create the demo collection (optional — demo data only).
