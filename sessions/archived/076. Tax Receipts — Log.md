## What was built

### Phase 1 — Fund designation

- Migration: added `restriction_type` (string, default `unrestricted`) to `funds`; added `fund_id` (nullable UUID FK → funds, nullOnDelete) to `donations`
- `Fund` model: added `restriction_type` to `$fillable`
- `Donation` model: added `fund_id` to `$fillable`; added `fund()` BelongsTo
- `DonationCheckoutController`: accepts optional `fund_id`, validated `exists:funds,id`, stored on the Donation at creation
- `DonationResource`: fund name column added to table; fund field added to view (placeholder `—` when null)
- `FundResource`: `restriction_type` select added to form (read-only on edit); restriction column added to table
- `donation-form.blade.php` widget: queries active funds server-side; renders optional fund select below the amount section only when active funds exist; passes `fund_id` in the fetch payload

### Phase 2 — Receipt infrastructure

- Migration: created `donation_receipts` table (`id` bigint PK, `contact_id` UUID FK cascade, `tax_year` integer, `sent_at` timestamp, `total_amount` decimal 10,2, `breakdown` JSON, `created_at` timestamp; no `updated_at`)
- `DonationReceipt` model: `$fillable`, casts, `const UPDATED_AT = null`, `contact()` BelongsTo
- `donation_receipt` email template seeded in `DatabaseSeeder` and `EmailTemplate::defaults()`: subject, body with fund-grouped line items table and IRS no-goods-or-services acknowledgement line, footer reason
- `DonationReceipt` Mailable: renders breakdown as an HTML table inside the existing email wrapper via `resolveWrapper()`

### Phase 3 — Donors page (replaced GenerateTaxReceiptsPage)

- `DonorsPage` created under Finance nav group (replaces `GenerateTaxReceiptsPage`, which is retired with nav hidden)
- Table of donors: one row per contact with active donations; columns: name (sortable by last name), roles (same logic as ContactResource), total donated (sortable), last donation date (sortable); phone, email, address toggleable hidden
- Page-level filter bar: tax year dropdown (current year default, prior years from DB, "All time"), minimum total money field (default $250), "Include donors below threshold" checkbox — all reactive via Livewire `wire:model.live`
- Header actions: "Create Mailing List" stub (modal with pre-filled year/threshold form, sends info notification — wired up in session 077); ellipsis `ActionGroup` containing "Send System Emails to Pending Recipients" and "Force Re-send System Emails to All", both with confirmation modals
- Send logic: queries eligible contacts, builds fund-grouped breakdown per contact, creates `DonationReceipt` record, sends `DonationReceipt` mailable via existing Resend/mail system
- Help doc: `resources/docs/tax-receipts.md` created and synced (1 route: `filament.admin.pages.donors`)

## Production deployment notes

Run `php artisan migrate` then `php artisan db:seed --class=DatabaseSeeder` to seed the `donation_receipt` email template. Alternatively seed only the template: `php artisan tinker` → `(new \Database\Seeders\DatabaseSeeder)->run()` will use `firstOrCreate` and is safe to re-run.

Also run `php artisan help:sync` to register the new `tax-receipts` help doc route.
