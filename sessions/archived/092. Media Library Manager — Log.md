## What was built

### Phase 1 — Media Library page

- Built `app/Filament/Pages/MediaLibraryPage.php` under the CMS navigation group, using `HasTable` to query the Spatie `media` table directly.
- Columns: thumbnail preview (images only), file name, mime type (short badge label), size (human-readable KB/MB), owner (resolved from morph — shows model type and label/title/handle), collection name, conversion status (green/yellow/gray badge), upload date (relative).
- Searchable by file name. Filterable by owner type (PageWidget, CollectionItem, EmailTemplate) and mime group (Images, Documents, Other). Sortable by file name, size, and date. Default sort: newest first.
- Access gated behind `view_any_page` permission.
- Breadcrumbs follow the standard pattern.

### Phase 2 — Delete action

- Per-row delete action with confirmation modal showing the file name.
- Uses Spatie's `->delete()` to remove both the database record and files from disk.
- Delete visibility gated behind `view_any_page` permission.

### Additional

- Created help doc at `resources/docs/media-library.md` with documentation of browsing, deletion behaviour (including the orphan-image warning), and upload limitations. Synced via `help:sync`.
- Created blade view at `resources/views/filament/pages/media-library.blade.php`.
- Storage summary stats (file count, total size, incomplete conversions) were built but removed during review — user preferred a clean table without header stats.
