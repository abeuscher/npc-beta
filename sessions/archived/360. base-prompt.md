## Process rules for every session

Base-prompt process rules apply in full. Session-specific:

- **Surface the architectural choices at the Open Gate, build after.** The reset **scope** (2FA-only vs. full credential recovery incl. password/super-admin promotion), the **endpoint + envelope shape**, the **protected-admin handling**, and the **audit mechanism** all propagate — confirm the recommended leans with the owner before going deep, don't commit-and-unwind.
- **The boundary discipline is mandatory.** This is a contract change. Update `docs/fleet-manager-agent-contract.md` (body + CHANGELOG + `Contract Version` + `HealthController::CONTRACT_VERSION`) and the Cross-Repo block in `sessions/session-outlines.md` in the same pass as the endpoint. Follow `fleet-manager-planning-spec.md`'s two-repo protocol — the FM repo consumes this later.
- **No recovery secret in the app or FM.** Per the agreed design, the endpoint trusts the operator verified identity out-of-band against their external vault. Do **not** add a PIN column, a hash, or a verify endpoint. The conscious tradeoff (procedure-not-control; the audit logs "operator reset 2FA for user Y," not "…after PIN verification") is accepted for the solo-operator trust model.
- **The break-glass CLI hard-gates like `demo:restore`/`demo:reset`** — refuse to run in demo mode; never let a destructive auth reset reach a customer node by accident. The node endpoint is the primary path; the CLI is for a node cut off from FM.
- **Audit every reset** (who/what/when, by which path). Decide the sink at the gate (activitylog vs. a dedicated table vs. a log channel).
- **nginx config edits use `up -d --force-recreate nginx`**, never `restart` (bind-mount breakage). Regenerate local certs via `bin/dev-certs.sh` if absent.
- **Verify objectively; pull the owner in only for genuine judgment.** Close the loop yourself: the endpoint resets under mTLS and refuses in demo; the break-glass command clears the columns; the protected admin *can* be reset; the audit record lands; the contract-parity test is green; the full suite is green. There is little UX here (no CRM screen) — the FM UX is FM-side.
