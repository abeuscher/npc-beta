## What was built

### Phase 1 — DuplicateHeaderDetector service

- New `App\Services\Import\DuplicateHeaderDetector` with two rules (post-refinement): `exact_match_normalized` (lowercase + strip non-alphanumeric) and `trailing_digit_suffix` ("Address 2", "Name_1", "Email (alt)").
- Priority-based deduplication — a header is reported only in the highest-priority finding it qualifies for.
- Carve-out for the address-line family (`Address` / `Address 1` / `Address 2` / `Address Line N`, same for `Street`) so legitimately distinct line fields don't flag as duplicates.
- A third `word_subset` rule was originally implemented (prefix-subset on tokens) but removed during UAT after it produced compound-header false positives such as `Item` / `Item quantity` / `Item price` / `Item amount`. Synonym pairs like `Email` / `Email address` now surface during mapping-step collision detection instead of being pre-flagged.

### Phase 2 — Review Data wizard step

- Added `buildReviewStep()` to `InteractsWithImportWizard`, inserted as the second step across all five wizard pages (Contacts, Events, Donations, Memberships, Invoice Details) between Upload and Map Columns.
- Per-column Keep / Ignore decisions seeded with "keep first column, ignore the rest" within each finding group.
- Empty state and findings state both show `N rows detected` so users always get feedback from the step.
- Sample values rendered inline: up to 5 non-blank values per column, truncated at 80 chars.
- `topNav()` indices shifted across all five wizard pages (Upload=0, Review=1, Map=2, Preview=3).

### Phase 3 — Ignored-columns propagation into mapping

- `finalizeReviewDecisions()` writes `$data['ignored_columns']` and patches `column_map` entries for ignored columns to `null`, clearing any `cf_label_N` / `cf_handle_N` / `cf_type_N` for those columns.
- `processUploadedFileNamespaced()` (shared trait) and contacts' own `processUploadedFile()` both honour `$data['ignored_columns']`, so saved-mapping and `guessDestination()` defaults for ignored columns are suppressed on re-entry.
- Parse reset logic: review decisions / ignored columns are preserved when the same file is re-uploaded (unchanged headers) and cleared when headers change.

### Phase 4 — Cosmetic polish across the wizard

- Review Data step always shows `N rows detected in the uploaded file.`, regardless of whether findings exist.
- Importer landing page relabels the Events card to `Import Events and Event Registrations` (verbose but accurate; downstream templates follow).
- Preview-step submit action renamed from `Run Import` to `Stage Import` across all five wizards (correct vocabulary — commit is a second step on the progress page).

### Phase 5 — Notes and relational-naming unification (Batch 1)

- Contacts' notes sub-form upgraded from `delimiter` + `skip_blanks` to the richer `split_mode` dropdown (`none` / `date_prefix` / `regex`) + `split_regex` text input — matches the pattern used by the four namespaced importers.
- Contacts sentinels renamed: `__note__` → `__note_contact__`, `__tag__` → `__tag_contact__`, `__org__` → `__org_contact__`.
- Relational-map type strings renamed: `note` → `contact_note`, `tag` → `contact_tag`, `organization` → `contact_organization`.
- Progress-trait `applyPerRowNotes()` now returns `int` (fragment count) so contacts' dry-run report can display per-row note counts without re-counting.
- Contacts now uses the progress trait's shared note-split logic, picking up date-prefix parsing of `occurred_at` as a side benefit.
- Behavioural loss: `skip_blanks=false` retention of empty fragments is gone. `preg_split(PREG_SPLIT_NO_EMPTY)` always drops empties; the corresponding test was removed and a new `date_prefix` test added.

### Phase 6 — Contact resolution consolidation (Batch 2)

- Extracted three helpers on `InteractsWithImportProgress`: `resolveContactByIdMap()`, `resolveContactByField()`, `resolveContactByMatchKey()`.
- Namespaced `resolveContactByNamespacedKey()` and contacts' own `findExistingMatch()` collapsed onto those helpers; contacts' inline version deleted.
- Side benefit: contacts now matches email case-insensitively (was a latent inconsistency with the namespaced importers).
- Error message on ambiguous match unified to `"Ambiguous contact match on {field} = {value}"` everywhere.

### Phase 7 — ImportSource column rename + skip-reason vocabulary (Batch 3)

- Migration `2026_04_17_130000_rename_contact_columns_on_import_sources.php` renames the four bare `ImportSource` columns to the `contacts_*` prefix for symmetry with events / donations / memberships / invoices scopes: `field_map` → `contacts_field_map`, `custom_field_map` → `contacts_custom_field_map`, `match_key` → `contacts_match_key`, `match_key_column` → `contacts_match_key_column`.
- Model, seeder, wizard, progress page, schema doc, and four test files updated to the new names.
- Contacts' skip-reason keys renamed for self-documenting phrasing: `no_identifier` → `missing_identifier`, `match_skip` → `duplicate_skipped`. Namespaced importers' keys left alone (`blank_event_id` / `blank_contact_key` / `contact_not_found` / `blank_invoice_number` describe genuinely different conditions).

### Phase 8 — Tests

- New `tests/Feature/ImportSession192Test.php` with 22 tests covering detector rules, rule priority, address carve-out, the Item-family false-positive regression, and wizard-trait state flow (seed defaults, finalize decisions, sample truncation, mapping-step suppression).
- `ImportRelationalDestinationsTest.php` updated to the new sentinels and relational-map types; `skip_blanks=false` test replaced with a `date_prefix` occurred_at test.
- `ImportSourcePresetTest.php`, `ImportContactsDryRunTest.php`, `ImportEventsTest.php` updated for the renamed `ImportSource` columns.

## Deferred decisions

- Dedicated "delimiter" split mode in the notes sub-form. When contacts' notes UI was unified onto `split_mode` (none / date_prefix / regex), users who want a literal character-delimiter split must now write a regex (e.g. `\|` for pipe). A dedicated `delimiter` option on the dropdown is a natural follow-up if users find regex too abstract — applies to all five importers uniformly.
- Custom-field sentinel rename (`__custom__` → `__custom_contact__`). Flagged in the audit as low-value, optional; left alone to minimize churn.
- Saved-per-source ignore-column preferences. Session prompt allowed for deferral; not implemented. Current behaviour: review decisions clear when headers change; persist only within a single wizard session.
- Commit-phase UX polish (loading indicators on `runCommit` / `saveMapping` / navigation links, plus the ~1k-row Chrome JS error at the committing → done transition) deferred to session 193. Prompt is in place.
