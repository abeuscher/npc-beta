**Date:** 2026-03-23
**Branch:** session-066

## What was built

### Phase 1 — Membership tier system (DB + models)

- **Migration:** `2026_03_23_000001_create_membership_tiers_table` — uuid PK, name, slug (unique, Spatie sluggable), billing_interval enum (`monthly`, `annual`, `one_time`, `lifetime`), default_price (decimal, nullable), renewal_notice_days (integer, default 30), description (text, nullable), is_active (boolean, default true), sort_order (integer, default 0).
- **Migration:** `2026_03_23_000002_update_memberships_add_tier_id` — drops `tier` string column, adds `tier_id` uuid FK → `membership_tiers.id` (`onDelete('set null')`).
- **`MembershipTier` model** — `HasSlug`, `HasUuids`, `HasFactory`; slug auto-generated from name; `memberships()` hasMany.
- **`Membership` model** — `tier` removed from `$fillable`, replaced by `tier_id`; `tier()` belongsTo `MembershipTier` added.
- **`MembershipTierSeeder`** — creates one default "Standard / Annual" tier (null price, renewal_notice_days 30) on fresh install if no tiers exist. Registered in `DatabaseSeeder`.
- `docs/schema.md` updated — new `membership_tiers` section added, `memberships` section updated.

### Phase 2 — MembershipTierResource

- New Filament resource at `app/Filament/Resources/MembershipTierResource.php`.
- Nav group: **Membership**, sort 1, icon `heroicon-o-tag`. Gated to `isSuperAdmin()`.
- Form: two sections — main (Name, Billing Interval, Default Price) and collapsed Advanced (Description, Renewal Notice Days, Is Active, Sort Order). Defaults: billing_interval `lifetime`, renewal_notice_days 30, sort_order 0, is_active true.
- Table: name, interval badge, default price (formatted, "Complimentary" if null), active icon, sort order. Filter by active. Default sort by sort_order.
- Pages with `getBreadcrumbs()` on all three.
- Help doc stub at `resources/docs/membership-tiers.md`.

### Phase 3 — MembershipResource updated

- `tier` TextInput replaced with `tier_id` Select loaded from active tiers ordered by sort_order.
- Table column updated to `tier.name` badge.
- `MembershipTierFactory` created with sensible defaults.
- `MembershipFactory` updated — `tier` string replaced by `MembershipTier::factory()`.
- **Nav group changed to Finance** (moved from CRM — treasurer is the primary audience).

### Phase 4 — Promote to Member action

- `EditContact.php` extended with `getHeaderActions()` containing a `DeleteAction` + an `ActionGroup` (ellipsis ⋯ menu).
- Ellipsis contains **Promote to Member**: hidden when contact has `status = 'active'` membership; modal form with tier select (name + interval hint), starts_on DatePicker (default today), expires_on DatePicker (hidden for lifetime tiers, forced null on save), amount_paid TextInput (default 0).
- On submit: creates `Membership` with `status = 'active'`, fires success notification.

### Phase 5 — MembershipsRelationManager

- New relation manager at `ContactResource/RelationManagers/MembershipsRelationManager.php`.
- Shows **all memberships** (no status filter) — full history visible.
- Columns: tier name, status badge (colour-coded), starts_on, expires_on, amount_paid.
- Registered in `ContactResource::getRelations()` alongside `NotesRelationManager`.

### Post-session tweaks (same session, in-conversation fixes)

- **MembershipTierResource:** added `->default(30)` and `->default(0)` to `renewal_notice_days` and `sort_order` form fields to prevent NOT NULL violations when Advanced section is left collapsed.
- **MembershipTierResource:** `billing_interval` Select defaults to `lifetime`.
- **ContactResource Roles column:** converted from plain text to colour-coded badges — `Member` (green), `Donor` (yellow), `Contact` (gray, shown as default when neither applies). Previously returned a comma-separated string; now returns an array for multi-badge rendering.

## Deferred decisions

- **Portal access on promotion:** "Promote to Member" intentionally does not create a portal account. Portal access is a separate concern — some members never need it. A dedicated "Grant Portal Access" action will be added in session 067 alongside consolidation of the portal actions into the ellipsis menu.
- **Members list view:** A dedicated Members nav entry (filtered contacts list scoped to active members, analogous to PostResource/Page) was discussed but deferred. When built, it replaces the MembershipResource as the primary member-facing view. MembershipResource stays under Finance for treasurer use.
- **Notes sub-page:** Notes relation manager currently loads with the contact record and causes layout jank (data hops while loading). Session 067 will move notes to a dedicated sub-page (`/contacts/{id}/notes`) with breadcrumbs and a header action button, removing `NotesRelationManager` from the contact edit page.
- **Renewal emails, self-serve portal renewal, Stripe Billing for memberships, tier colours/public descriptions, grace period logic, auto-calculation of expires_on** — all out of scope, unchanged from original deferral.
