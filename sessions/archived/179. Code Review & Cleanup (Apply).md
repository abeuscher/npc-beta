## Open flags from 178 (need a decision before implementation)

Walk through these first. Each one is a fork in the road — the session's scope depends on which way the user goes.

### Flag A — `PagePreviewController` and `/admin/preview/{page}`

Orphaned in the live UI (no button or link exercises it in the Vue builder). Originally flagged in session 141 and deferred. Has its own test file (`tests/Feature/PagePreviewTest.php`) and a registered route (`app/Providers/Filament/AdminPanelProvider.php:71`). Decision: **retire** (delete controller, view `resources/views/admin/page-preview.blade.php`, route, and test file) or **repurpose** (wire a preview button into the Vue builder inspector/header).

If retired, items #8 in the W6 table shrinks by half automatically.

### Flag B — `docs/widget-system-spec.md` "Migration from current system" section

Lines 277-309 describe the migration plan executed in session 172; the paths it references (`resources/views/widgets/...`, `resources/views/livewire/partials/inspector-fields/...`) no longer exist. Decision: **annotate** the section as "migration complete (session 172), kept as historical record" or **retire** the entire section.

### Flag C — Arbitrary-key writes in `PageBuilderApiController::update()`

`update()` validates `config` / `appearance_config` / `query_config` only as `array`, not against the widget's schema keys. A user with `update_page` can POST arbitrary keys; templates ignore them, but the JSONB storage is unbounded. Known since session 141; tolerated. Decision: **enforce** (filter incoming keys against `widget_type.config_schema`, reject or silently drop unknown keys) or **document** the tolerance in `docs/widget-system.md` and move on.

Security checklist in 178 flagged this as "architectural"; the call belongs to the user.
