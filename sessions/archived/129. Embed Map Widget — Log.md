## What was built

### Phase 1 ‚Äî Map URL parser

- Created `App\Services\MapEmbedParser` with a static `extractEmbedUrl()` method.
- Accepts iframe snippets (extracts `src`), Google Maps place/@ URLs, short links (`goo.gl/maps`, `maps.app.goo.gl`), and existing embed URLs.
- Converts place/@ URLs to the keyless embed format (`google.com/maps?q=...&output=embed`) ‚Äî no API key required.
- Sanitises output: only `*.google.com` hosts are allowed. Returns `null` for anything else.

### Phase 2 ‚Äî Widget type registration + notice field type

- Added `map_embed` to `WidgetTypeSeeder` with config fields: privacy notice, heading, map_input (textarea), aspect_ratio (select: 16:9, 4:3, 1:1, 21:9), min_height/max_height (number, advanced), full_width (toggle).
- Created `notice` inspector field type (`resources/views/livewire/partials/inspector-fields/notice.blade.php`) ‚Äî a display-only banner with `info` and `warning` variants, styled with left border accent and icon. No `key`, stores nothing in config.
- Fixed `WidgetType::getDefaultConfig()` to skip fields without a `key` (the `notice` field was causing an `Undefined array key` error).

### Phase 3 ‚Äî Blade template and SCSS

- Created `resources/views/widgets/map-embed.blade.php` with responsive iframe, Alpine.js scroll-lock overlay ("Click to interact with map"), Escape/click-outside to deactivate.
- Created `resources/scss/widgets/_map-embed.scss` with BEM-style classes for container, iframe, overlay, hint pill, and responsive heading.
- Map fills full container width with `aspect-ratio` driving proportional height.
- iframe uses `referrerpolicy="no-referrer-when-downgrade"` and `loading="lazy"`.

### Bug fix ‚Äî keyless embed format

- Initial implementation used `/maps/embed/v1/place?q=` which requires a Google API key.
- Switched to the keyless format: `google.com/maps?q=...&output=embed`.
- Iframe snippets from Google's "Share > Embed" pass through directly (they already use the keyless `pb=` format).
