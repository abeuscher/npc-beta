# 120. Filament Custom Theme — Admin Panel Tailwind Build

Set up a Filament custom theme so the admin panel compiles its own Tailwind CSS from project blade files, giving access to the full Tailwind utility class set in custom admin views like the page builder.

---

## Design decisions (resolved before starting)

- The custom theme uses Filament's `->viteTheme()` panel method, which adds a Vite entry point compiled with a Filament-aware Tailwind config.
- The theme CSS file lives at `resources/css/filament/admin/theme.css` (Filament's conventional path).
- A dedicated `tailwind.config.js` (or section in the existing one) scans `resources/views/**/*.blade.php`, `app/Livewire/**/*.php`, and `app/Filament/**/*.php` so all custom admin classes are included in the build.
- The Filament preset (`@filament/support/tailwind.config.preset`) is used to ensure all built-in Filament classes remain available.
- No visual changes to existing UI — this session only adds the build infrastructure. The widget picker's inline-style workarounds (if any) are then replaced with proper Tailwind classes.

## Phase 1 — Filament custom theme infrastructure

### 1a. Create theme CSS entry point

Create `resources/css/filament/admin/theme.css` with:

```css
@import '/vendor/filament/filament/resources/css/theme.css';

@config 'tailwind.config.filament.js';

@tailwind base;
@tailwind components;
@tailwind utilities;
```

### 1b. Create Filament Tailwind config

Create `tailwind.config.filament.js` using the Filament preset:

```js
import preset from './vendor/filament/support/tailwind.config.preset'

export default {
    presets: [preset],
    content: [
        './app/Filament/**/*.php',
        './app/Livewire/**/*.php',
        './resources/views/filament/**/*.blade.php',
        './resources/views/livewire/**/*.blade.php',
        './vendor/filament/**/*.blade.php',
    ],
}
```

### 1c. Register Vite entry point

Add the theme CSS to `vite.config.js` as an additional entry point.

### 1d. Register theme in AdminPanelProvider

Add `->viteTheme('resources/css/filament/admin/theme.css')` to the panel configuration.

### 1e. Build and verify

Run `npm run build` (or `npm run dev`) and confirm the admin panel loads with the custom theme. All existing UI should look identical.

## Phase 2 — Replace inline style workarounds

### 2a. Widget picker modals

In both `page-builder.blade.php` and `page-builder-block.blade.php`, replace any inline `style="padding-top: ..."` workarounds with proper Tailwind classes (e.g. `pt-5`, `pb-3`, `gap-1.5`) that are now available in the compiled CSS.

### 2b. Verify spacing

Confirm the widget picker modals render with correct spacing — header, search input, category toolbar, and tile grid should all have comfortable padding.
