## Process rules for every session

Base-prompt process rules apply in full. Session-specific notes:

- **The hash is computed once, at upload, and persisted.** Do not read files to hash them at request time on the hot path — populate `content_hash` when media is added (a model event / Spatie hook) and backfill existing rows in the migration or a one-shot command. The 318 duplicate scan should then prefer the column over reading bytes.
- **Dedup is a warn-and-offer, never silent.** When an upload matches an existing hash, the operator chooses to reuse or upload anyway. Never auto-substitute a file — two visually identical files may be intentionally separate, and silent substitution is a surprise.
- **Reuse semantics under the current (non-CAS) path generator copy bytes.** Be explicit in the log: reusing an existing asset for a new owner still writes a physical copy until CAS lands. The win this session is behavioral (fewer fresh uploads, a findable library), not yet disk reclamation. Do not pretend otherwise.
- **Every upload surface or none.** A dedup check on the page-builder upload but not on the Filament `FileUpload` fields leaves the leak open. Cover them uniformly per `feedback_uniform_ux_over_per_type_plumbing`, or explicitly document a surface as out-of-scope with rationale.
- **PM-level tone for status summaries** per `feedback_pm_level_tone`. **Questions are not instructions** per `feedback_questions_are_not_instructions`.
