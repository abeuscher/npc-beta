# 166. Appearance Controls — Docs Finalization & Widget Spacing Harmonization

Final phase of the Appearance Controls project (sessions 161–165). Closes out developer documentation for the universal Appearance layer and harmonizes hard-coded Swiper spacing values across three widgets into user-controllable config fields.

## Design decisions (resolved before starting)

### Documentation targets

Two existing help docs need updating:

- `resources/docs/widget-types.md` — the admin-facing widget type reference. Add a section documenting the universal Appearance layer: what it controls (background color, gradient, image with alignment/fit, text color, full width, padding, margin), the `appearance_config` storage shape, and the column-child width override.
- `resources/docs/widget-development.md` — the developer-facing widget authoring guide. Add sections covering:
  - The three shared primitives (`NinePointAlignment`, `ColorPicker`, `GradientPicker`) and the `compact` prop on the latter two.
  - The `AppearanceStyleComposer` service and how it maps `appearance_config` keys to inline styles.
  - The layered rendering rule: gradient over image over background color via a single CSS `background-image` shorthand.
  - A concise per-widget "what changed" table covering the removes and renames from session 162.
  - The `BackgroundPanel`, `TextPanel`, and `SectionLayoutPanel` components and how they write to the store via `updateLocalAppearanceConfig`.

### Widget spacing harmonization — `gap` config field

Three widgets hard-code a Swiper `spaceBetween` value that should be user-controllable:

| Widget | Current hard-coded value | Template file |
|--------|------------------------|---------------|
| `logo_garden` | `spaceBetween: 16` | `resources/views/widgets/logo-garden.blade.php` |
| `blog_listing` | `spaceBetween: 24` | `resources/views/widgets/blog-listing.blade.php` |
| `events_listing` | `spaceBetween: 24` | `resources/views/widgets/events-listing.blade.php` |

For each widget:

1. Add a `gap` field to its `config_schema` in `database/seeders/WidgetTypeSeeder.php`. Type: `number`, label: "Slide spacing (px)", group: `appearance`, default: the current hard-coded value. Place it after any existing appearance-group fields.
2. In the Blade template, replace the hard-coded `spaceBetween` value with `{{ $config['gap'] ?? {default} }}` where `{default}` is the current hard-coded value.

No migration needed — this is a seeder-only change. Existing widget instances will render identically because the Blade `??` fallback matches the current hard-coded value.
