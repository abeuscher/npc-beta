### Flag A — LayoutInspectorPanel architectural divergence *(carried from 205, re-framed at 272)*

**272 finding:** the 205-time architectural divergence is **substantially closed** by 207's `page_layouts.appearance_config jsonb` migration plus `LayoutInspectorPanel.vue:540` reusing `BackgroundPanel` and `:531` reusing `SectionLayoutPanel` via data-driven props (`config`, `show-image: false`, `show-full-width: false`). **Pre-resolution:** re-frame as W7 #5 mechanical sub-component split. Extract the Column Settings tab to `LayoutColumnSettingsTab.vue` (~400 LOC), leaving `LayoutInspectorPanel.vue` as a ~380 LOC orchestrator. No architectural choice required. Lands in iteration `/5`.

### Flag B — ImportContactsPage convergence with the namespaced pattern *(carried from 206)*

**272 finding:** divergence is real and load-bearing — bare `__custom__` sentinel (vs `__custom_contact__`), bare field grammar (`email` vs `contact:email`), bespoke 3-strategy collision-resolution UI (prefer / split / drop), inline ~50-LOC `runImport()` serialization that mirrors but doesn't use `serializeColumnMaps()`. 11 sessions of namespaced-pattern maturity haven't shifted the cost calculus — the 6 namespaced wizards are 230-356 LOC each, ImportContactsPage is 1014 LOC, and each is doing the work it needs to.

**Pre-resolution:** re-affirm 206 disposition — leave as is. Carry forward as a standing won't-fix; revisit only if a forcing function emerges (e.g. a feature that needs the namespaced grammar uniformly across all 7 importers). Drops off the 273 docket.

### Flag W6/B — Observer coverage gap *(NEW at 271)*

**272 finding:** `Note`, `Organization`, `Affiliation`, `DonationCredit` have no observers. CUD bypasses `App\Services\ActivityLogger::log` entirely. Pattern comparison: every other tracked CRM model has an observer (Contact, Donation, Membership, Event, Page, Product, ProductPrice, Purchase, EventRegistration, CustomFieldDef).

**Pre-resolution:** path (a) — add 4 new observers in 273 to match the established pattern. `NoteObserver`, `OrganizationObserver`, `AffiliationObserver`, `DonationCreditObserver`, each mirroring `ContactObserver`'s shape (`created` / `updated` / `deleted` / `restored` events calling `ActivityLogger::log`). Register via `#[ObservedBy(...)]` PHP attribute (the modern pattern — see W8 #1). Tests: 4 observer tests with `RefreshDatabase` + activity-log assertions. Lands in iteration `/3`.

**Note coordination caveat:** `NoteObserver` lands here as plain CUD activity logging. The C1 Notes Permissions stub (release-plan position 27) covers the *finer-grained permission gates* around structured-interactions, separate from observability. Capture the boundary in `/3`'s commit message so C1 doesn't relitigate the observer question.
