## What was built

### Phase 1 — New factories

- `ProductFactory` — generates a name by combining a random franchise and character from `SampleLibrary`, derives a unique slug, sets status `active`, random capacity 10–200.
- `ProductPriceFactory` — realistic label (General Admission, Early Bird, VIP, etc.), random amount $10–$500, sort_order provided by caller.
- `PurchaseFactory` — status randomised between `active` and `completed`, `occurred_at` within the last year, `amount_paid` overridden by caller. `product_id`, `product_price_id`, and `contact_id` must be supplied by the caller.
- `DonationFactory` improved — added `started_at` defaulting to a random date within the current calendar year.

### Phase 2 — Widget expansion

- Added `taxYear` public property (int, default `now()->year`) initialised in `mount()`.
- Added `products` and `purchases` options to the type dropdown in the Blade view.
- Added Tax Year number input to the Blade view (shown always, meaningful only for donations).
- `generateDonations(int $count)` — picks randomly from existing contacts (aborts with feedback if none); picks randomly from existing active funds or sets `fund_id` null; spreads `started_at` across the selected tax year; creates one Transaction row per donation (`subject_type = Donation`, `stripe_id = null`).
- `generatePurchases(int $count)` — picks randomly from existing contacts and existing active products with prices (aborts with feedback if either is missing); sets `amount_paid` to the actual price amount; creates one Transaction row per purchase (`subject_type = Purchase`, `stripe_id = null`).
- `generateProducts(int $count)` — creates N products, each with 1–3 randomly labelled prices.
- `wipe()` refactored: `wipeDonations()` deletes linked Transaction rows then donations; `wipePurchases()` deletes linked Transaction rows then purchases; `wipeProducts()` deletes prices then products; `wipeGeneric()` handles contacts, events, and members as before.
- `generate()` match updated for all six types; feedback message uses the new per-method early-return pattern so abort messages surface correctly.

### Phase 3 — Sample library

- Added `SampleLibrary::productFranchises()` — ~35 1980s toy franchise names (Transformers, G.I. Joe, Care Bears, Madballs, etc.).
- Added `SampleLibrary::productCharacters()` — ~60 1980s toy character/item names (Optimus Prime, Cobra Commander, Tenderheart Bear, Slimer, etc.).
- `ProductFactory` recombines a random franchise with a random character to produce absurd but recognisable product names (e.g. "Care Bears Starscream", "Ghostbusters Tenderheart Bear").
