# 066. Promote Contact to Member

Adds a configurable membership tier system and a one-click "Promote to Member" admin action on the contact record. Membership history is preserved — a contact may have multiple membership records over time (lapsed, cancelled, active). The action is tucked into the ellipsis menu on the contact edit page header, not surfaced as a primary action.

---

## Architectural decisions

- **Tiers are admin-configurable**, stored in a `membership_tiers` table. The `tier` string column on `memberships` is replaced by a `tier_id` FK. Pre-launch — the migration drops the old column.
- **Billing interval** drives behaviour: `monthly`, `annual`, `one_time`, `lifetime`. Lifetime tiers set `expires_on` to null and hide the expires field in the modal. All other intervals leave `expires_on` editable by the admin.
- **Membership history:** multiple records per contact are normal and expected. The relation manager shows all memberships, not just the active one.
- **"Promote to Member" visibility:** hidden when the contact already has a membership with `status = 'active'`. Lapsed or cancelled contacts can be re-enrolled.
- **Amount paid defaults to zero.** Admin-created memberships are often complimentary — the field exists for treasurer accountability ("12 complimentary memberships granted this year") but is never required.
- **Simple path first:** a fresh install ships with one seeded default tier ("Standard", annual, free/null price). An org that wants only one tier never needs to touch tier management.
- **Renewal machinery is out of scope.** The `renewal_notice_days` field is stored on the tier record for future use but no scheduled jobs, emails, or renewal flows are built in this session.

## Phase 1 — membership_tiers table and model

**Migration:** create `membership_tiers`

| Column | Type | Notes |
|---|---|---|
| id | uuid | PK |
| name | string | e.g. "Standard", "Patron", "Lifetime Friend" |
| slug | string | unique, auto-generated from name via Spatie sluggable |
| billing_interval | enum | `monthly`, `annual`, `one_time`, `lifetime` |
| default_price | decimal(8,2) | nullable — null means complimentary / price not set |
| renewal_notice_days | integer | default 30, for future renewal flow use |
| description | text | nullable |
| is_active | boolean | default true |
| sort_order | integer | default 0 |
| timestamps | | |

**Migration:** update `memberships`

- Add `tier_id` (uuid, nullable, FK → `membership_tiers.id`, `onDelete('set null')`)
- Drop `tier` string column

**Model:** `app/Models/MembershipTier.php`

- `HasUuids`, `HasFactory`
- `$fillable`: all columns except id and timestamps
- `memberships()` hasMany relationship
- Spatie sluggable — auto-generates slug from name

**Update `Membership` model:**

- Replace `'tier'` in `$fillable` with `'tier_id'`
- Add `tier()` belongsTo `MembershipTier` relationship
- Remove or update any reference to the old `tier` string

**Seeder:** `MembershipTierSeeder`

Create one default tier if no tiers exist:
- name: `Standard`
- billing_interval: `annual`
- default_price: `null`
- renewal_notice_days: `30`
- is_active: `true`
- sort_order: `0`

Register in `DatabaseSeeder` so it runs on `migrate:fresh --seed`.

## Phase 2 — MembershipTierResource

**File:** `app/Filament/Resources/MembershipTierResource.php`

Navigation:
- Group: `Membership`
- Label: `Membership Tiers`
- Icon: appropriate icon (e.g. `heroicon-o-tag`)
- Sort: 1 within the group
- Permission: `super_admin` only for now — gate with `canAccess()` returning `auth()->user()?->isSuperAdmin()`

Form layout — two sections:

**Main section (always visible):**
- Name (TextInput, required)
- Billing Interval (Select, required) — options: Monthly / Annual / One-time / Lifetime
- Default Price (TextInput, numeric, prefix `$`, placeholder `Leave blank for complimentary`)

**Advanced section (collapsed by default, label "Advanced"):**
- Description (Textarea, nullable)
- Renewal Notice Days (TextInput, numeric, suffix `days`, helperText `How many days before expiry to send a renewal reminder`)
- Is Active (Toggle)
- Sort Order (TextInput, numeric)

Table columns: Name, Billing Interval, Default Price (formatted, "Complimentary" if null), Is Active badge, Sort Order.

Sortable by Sort Order. Filterable by Is Active.

Create a help doc stub at `resources/docs/membership-tiers.md` and register the resource routes in its `routes:` array. Include `getBreadcrumbs()` following the established pattern.

## Phase 4 — Promote to Member action on ContactResource

**File:** `app/Filament/Resources/ContactResource/Pages/EditContact.php`

Add a `getHeaderActions()` override (or extend the existing one) that appends an `ActionGroup` (ellipsis menu) to the right of the existing Delete action. The group contains one action: **Promote to Member**.

The action:
- Label: `Promote to Member`
- Hidden when `$record->memberships()->where('status', 'active')->exists()`
- Opens a modal form with:
  - **Tier** (Select, required) — options from `MembershipTier::where('is_active', true)->orderBy('sort_order')->get()`, labelled with name + billing interval hint e.g. "Standard (Annual)"
  - **Member since** (DatePicker, required, default today) — maps to `starts_on`
  - **Expires on** (DatePicker, nullable) — hidden via `hidden(fn ($get) => ...)` when the selected tier has `billing_interval = 'lifetime'`; when hidden, value is forced to null
  - **Amount paid** (TextInput, numeric, prefix `$`, default `0`, label `Amount paid`, helperText `Leave as 0 for complimentary`)
- On submit: create a `Membership` record with `contact_id = $record->id`, `status = 'active'`, `registered_at = now()`, and the modal field values
- Success notification: `Membership created.`

**Resolving expires_on for non-lifetime tiers:**

When the tier is not lifetime, leave `expires_on` fully editable — the admin sets it manually. Auto-calculation from billing interval is deferred (requires the renewal flow to be in place to be meaningful).

If it is currently filtered, remove the filter so the full history is visible. The table should show: Tier name, Status badge, Member since (`starts_on`), Expires on, Amount paid.

## Out of scope

- Renewal emails or scheduled renewal jobs
- Self-serve renewal via the member portal
- Stripe Billing integration for membership payments
- Tier colour, badge, or public-facing tier descriptions
- Grace period logic for lapsed memberships
- Auto-calculation of `expires_on` from billing interval
