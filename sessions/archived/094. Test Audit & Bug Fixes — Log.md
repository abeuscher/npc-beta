### Phase 3 — Bug fix: inspector stays open after widget deletion

Added check in `PageBuilder::deleteBlock()` — if the deleted block is the currently selected one, clears `selectedBlockId` and dispatches `block-selected` with empty ID to close the inspector panel and remove the highlight.

### Phase 4 — Testing policy

Agreed policy codified in updated templates:

- **Fast suite** runs at the end of every session (`--exclude-group=slow`).
- **Slow group** (tests >5s individually) runs only when the session prompt says to.
- **New tests expected** for every session that changes application behaviour. Pure CSS/template/copy sessions may skip.
- **5-minute ceiling** on the fast suite; revisit the split if exceeded.
