## What was built

### Phase 1 ‚Äî `ImportSource` as preset

- Migration adds `field_map` (jsonb `{}`), `custom_field_map` (jsonb `{}`), `match_key` (string, nullable), `match_key_column` (string, nullable) to `import_sources`.
- `ImportSource` model: fillable + array casts for the jsonb columns.
- New `ImportSourceSeeder` seeds three built-in rows (Generic CSV / Wild Apricot / Bloomerang) with their preset field maps and `match_key='email'`. Wired into `DatabaseSeeder`.
- `ImportContactsPage` pre-populates the mapping step from a saved source's `field_map` + `custom_field_map`, including restoring `__custom__` handles/labels/types. Shows "Using saved mapping from <Source>" banner instead of the detection banner.

### Phase 2 ‚Äî Match-key configuration

- Migration adds `match_key` string column (default `'email'`) to `import_logs`.
- New "Match contacts by" select on the mapping step. Options are dynamically derived from mapped columns (standard fields + custom-field handles). Default: `external_id` if mapped, else `email`, else first mapped field.
- Saved-source's `match_key` wins over the runtime default.
- `validateAtLeastOneIdentifier` now also enforces that the chosen match key is actually mapped.
- `ImportProgressPage::processRow()` generalised. New `findExistingMatch()` routes `external_id` through `ImportIdMap`, standard fields through `where($matchKey, $value)`, custom fields through `custom_fields->>handle`. Rows with two-or-more matches throw `"Ambiguous match on {key} = {value}"`.

### Phase 3 ‚Äî Dry-run with halt-on-error

- `ImportProgressPage` refactored into a 4-phase state machine: `awaitingDecision` ‚Üí `committing` ‚Üí `done` (plus `rejected` for PII).
- `mount()` auto-runs the dry-run inside `DB::transaction()` + a `DryRunRollback` exception to force rollback. Counts and errors accumulate on `$dryRunReport`.
- `runCommit()` (renamed from `commit()` to dodge a Livewire 3 built-in collision ‚Äî see below) resets counters and enters the chunked tick-polling loop. Tick skips rows whose numbers are in `$skipRowNumbers` from the dry-run.
- `cancel()` deletes the session + log and redirects to the importer.
- `saveMapping()` writes `field_map` / `custom_field_map` / `match_key` / `match_key_column` back to the source. Headers are normalized to lowercase on save.
- `downloadErrors()` streams errored rows as CSV.
- New blade view drives the phase machine: dry-run summary, commit/cancel buttons, errors table with CSV download, then the existing progress bar for commit, then completion + "Save mapping" CTA.

### Phase 4 ‚Äî Collision-option relabel

- Radio relabelled to **Skip / Update / Create duplicate anyway**, each with inline descriptions.
- `processRow` short-circuits the match lookup when strategy is `duplicate` ‚Äî every row becomes a new contact, including when two existing rows would have tripped ambiguous-match.

### Phase 5 ‚Äî Timeline notes carry source reference

- Migration adds `import_source_id` uuid FK (nullOnDelete) to `notes` and to `import_logs`.
- `Note` and `ImportLog` models gain `importSource()` relations + fillable.
- `ImportProgressPage` and `ImporterPage` (approve + rollback) note bodies all reference the source name by name and carry `import_source_id`.
- `ImportHistoryPage` accepts a `?source=UUID` query param to filter by source and shows a Source column.
- Contact timeline (`ContactNotes`) renders a clickable source chip on import notes that deep-links to `ImportHistoryPage` filtered by that source.

### Phase 6 ‚Äî UAT polish (landed across 6a‚Äì6g)

- **6a ‚Äî UX flow polish.** Merged Source + Upload wizard steps into one "Upload" step. Added "Create custom fields for unrecognized columns" toggle. New `FieldTypeDetector` service infers boolean / number / date / text from up to 10 sample rows. Mirrored Back/Next buttons at the top of every wizard step. Password / pwd / passwd / password_hash headers added to `FieldMapper::SKIPPED_HEADERS` ‚Äî always ignored regardless of preset or toggle.
- **6b ‚Äî PII scanner rewrite.** Batch up to 50 violations instead of stopping at the first. ZIP-like columns (`zip`, `zipcode`, `postal`, `postalcode`, `postcode`) skip the bare 9-digit ABA/SSN heuristic since US ZIP+4 is 9 digits and trips that rule falsely. Credit-card Luhn and hyphenated SSN still check everywhere. Rejection card shows a table of violations with the original row data underneath each; Download Report button streams a readable CSV.
- **6c ‚Äî Skip-reason aggregation.** `dryRunReport.skipReasons` breaks skipped rows into `no_identifier` (no email/first name in row) and `match_skip` (matched existing + strategy=skip). View shows an amber explainer card under the dry-run summary.
- **6d ‚Äî Duplicate-column resolution.** When two+ headers map to the same destination field, they're pulled out of the main mapping list and grouped into a "Duplicate column mappings" section. Per-group radio: **Prefer + fallback** / **Split as custom field** / **Drop the rest**, plus a primary column picker. Runtime gains `column_preferences` (new jsonb column on `import_logs`) ‚Äî `processOneRow` applies a second pass so the preferred column wins when populated and sibling columns serve as blank fallbacks. Essential for Wild Apricot exports where "First name"/"FirstName" and "Email"/"Email address" are common duplicates.
- **6e ‚Äî Wizard nav bug fixes.** Top Back/Next were using Filament Actions with `wire:click="mountAction(...)"` racing against our custom `x-on:click`, which silently swallowed the `previousStep` event. Replaced with raw HTML buttons dispatching the wizard events directly. Layout changed to Back-on-left (text link) / Next-on-right (filled button) to mirror the native footer.
- **6f ‚Äî Relational destinations (Organization / Note / Tag).** New mapping destinations beyond contact fields + custom. **Organization** ‚Äî subform radio with `auto_create` / `match_only` / `as_custom` strategies; dry-run preview lists the org names that would be created, matched, or left unmatched, with row counts per name. **Note** ‚Äî optional delimiter + `skip_blanks` toggle; splits a cell into multiple Note records, each carrying `import_source_id`. **Tag** ‚Äî optional delimiter, auto-creates missing tags by name (namespaced `type='contact'`), attaches via `syncWithoutDetaching`. All three backed by a new `relational_map` jsonb column on `import_logs`. `buildRowContext` snapshots existing org/tag names once per run so the dry-run preview classifies "would create" vs "would match" without per-row queries.
- **6g ‚Äî `commit()` ‚Üí `runCommit()` rename.** Classic Livewire 3 gotcha: `$wire.commit` resolves to the built-in reactive-state sync method, not your public `commit()`. The fallback in `Wrapped::__call` silently returns null when the target doesn't have a method ‚Äî which is why the POST returned 200 with no exception and `commit()` never ran. Renamed to `runCommit()` throughout (Page + blade + 7 test files). Documented in the method's docblock so future devs don't repeat it.

## Deferred decisions

- **Event / donation / membership importers** ‚Äî session 189 (events + registrations) and session 190 (donations + memberships + invoice details) carry the same pattern forward with multi-entity resolution and the new `Transaction ID` universal external key.
- **Shared `ImportWizardBase` class** ‚Äî deliberately not extracted yet. Session 191 polish will abstract the commonality once there are two-plus concrete importers (contacts + events at minimum) to inform the shape.
- **Source-specific noise detection + downloadable CSV templates** ‚Äî deferred to session 191. Today `FieldMapper::SKIPPED_HEADERS` is a single global list; per-source skip lists and value-pattern heuristics (for cells like `"Field&&Visibility Field&&Visibility..."` from Wild Apricot's `Details to show` column) need a broader design.
- **Performance tuning for very large imports** ‚Äî explicitly out of scope for this session. WCG's 1,051 contacts import ran in acceptable time. Tuning (queue-based background processing, chunked transaction boundaries, bulk insert) will be addressed if real-world imports in the 100k+ range show up.
- **Ticket tiers as a first-class Event feature** ‚Äî added as a pre-Beta 1 stub in `session-outlines.md`. Session 189 stores `ticket_type`/`ticket_fee` as flat fields on registrations in the meantime.
- **LLM-assisted data-prep tutorial** ‚Äî added as a post-Beta 1 stub. Second path to success for messy exports the importer's presets don't cover: point users at Claude Code with a canonical CSV template and a copy-pasteable reshape prompt.
- **Concrete-values rule vs `match_key`/`match_key_column` nullability.** The session prompt specified `nullable` for both fields on `ImportSource`; memory's "concrete values" rule pushed toward a default. Went with the prompt as written since seeded presets populate both fields with concrete values. Worth revisiting if we want to enforce the concrete rule uniformly.
