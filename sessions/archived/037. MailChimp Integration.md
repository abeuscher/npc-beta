# 037. MailChimp Integration

Connect the Mailing List Manager to MailChimp. Administrators manually sync a mailing list to a configured MailChimp audience. Each list's contacts are pushed as audience members with merge fields and tagged with the list name. Unsubscribe events posted by MailChimp are received by a webhook endpoint and written back to the contact record.

---

## Background & decisions

- The app maintains one MailChimp audience (configured via `.env`). Lists map to MailChimp **tags** on members within that audience — not separate audiences.
- Sync is **manual and one-directional** (CRM → MailChimp) except for the unsubscribe webhook (MailChimp → CRM).
- The sync always respects contactability: any contact with `do_not_contact = true` OR `mailing_list_opt_in = false` is excluded from every mailing list query. This is a base filter applied universally in `MailingListQueryBuilder`, not a per-list option.
- The MailChimp batch API is used for all pushes. Batch calls are asynchronous — the UI confirms submission, not completion.
- The PHP package is `mailchimp/marketing` (official Intuit SDK). Add via Composer.
- Credentials live in `.env` — not in the admin UI for v1.

---

## Contactability base filter

Update `MailingListQueryBuilder::buildSimple()` and `buildAdvanced()` to apply this before any user-defined filters:

```php
$query->where('do_not_contact', false)
      ->where('mailing_list_opt_in', true);
```

This affects all list queries: contact counts shown in the form, CSV export, and MailChimp sync. Document the behaviour clearly in the help article (update `resources/docs/mailing-lists.md`).

## Environment variables

Add to `.env.example` (under the existing MailChimp block):

```
MAILCHIMP_API_KEY=
MAILCHIMP_SERVER_PREFIX=
MAILCHIMP_AUDIENCE_ID=
MAILCHIMP_WEBHOOK_SECRET=
```

- `MAILCHIMP_SERVER_PREFIX` — the data centre prefix from the API key (e.g. `us14`). Required by the SDK.
- `MAILCHIMP_AUDIENCE_ID` — the audience (list) ID from the MailChimp dashboard.
- `MAILCHIMP_WEBHOOK_SECRET` — a random string you set in `.env` and register in MailChimp's webhook settings. Used to verify incoming webhook requests.

## `MailChimpService`

`app/Services/MailChimpService.php`

Wraps the `mailchimp/marketing` SDK. Methods:

### `syncList(MailingList $list): void`

1. Evaluate `$list->contacts()->with('tags')->lazy()` (the base contactability filter is already applied by the query builder).
2. Chunk into groups of 500.
3. For each chunk, build a batch operations array. Each operation is an `PUT /lists/{audience_id}/members/{hash}` upsert:
   - `status_if_new`: `'subscribed'`
   - `email_address`: contact email
   - `merge_fields`: `FNAME`, `LNAME`
   - `tags`: array containing the list's name (MailChimp tags are applied via a separate `POST /lists/{id}/members/{hash}/tags` call — however the batch API doesn't support tag operations directly, so apply tags in a separate pass after the member upsert batch completes — see note below)
4. Submit each chunk via `POST /batches`.
5. Store nothing — fire and forget. Return void. The caller shows a success notification.

**Tag pass**: After submitting the member upsert batch, submit a second batch applying the list's tag to each member. Use `POST /lists/{id}/members/{hash}/tags` with `{ tags: [{ name: "List Name", status: "active" }] }`. This can be a separate batch per chunk.

**Contacts without an email address** — skip silently (MailChimp requires an email).

### `removeTag(string $email, string $tag): void`

Used if a future session needs to remove a tag. Stub with `// TODO` for now — not needed in this session.

### `isConfigured(): bool`

Returns `true` if `api_key`, `server_prefix`, and `audience_id` are all non-empty. Used to guard the sync action.

## Merge fields

MailChimp audiences include `FNAME` and `LNAME` by default. Push only those two — no other merge fields for v1. Address and phone data serve no purpose in the email sending context and MailChimp requires manual audience setup for any non-default field.

## Sync action on `EditMailingList`

Add a second header action to `EditMailingList::getHeaderActions()`:

```
Actions\Action::make('syncMailchimp')
    ->label('Sync to MailChimp')
    ->icon('heroicon-o-arrow-path')
    ->color('gray')
    ->requiresConfirmation()
    ->modalHeading('Sync to MailChimp')
    ->modalDescription('This will push all contactable members of this list to your MailChimp audience and apply the list tag. The sync runs in the background — large lists may take a minute to appear in MailChimp.')
    ->visible(fn () => app(MailChimpService::class)->isConfigured())
    ->action(function (MailingList $record): void {
        app(MailChimpService::class)->syncList($record);
        Filament::notify('success', 'Sync submitted to MailChimp.');
    })
```

Place it between the Export and Delete actions.

## Unsubscribe webhook

### Route

`routes/web.php` — outside the Filament admin group, no auth middleware:

```php
Route::post('/webhooks/mailchimp', [\App\Http\Controllers\MailChimpWebhookController::class, 'handle'])
    ->name('webhooks.mailchimp');

Route::get('/webhooks/mailchimp', fn () => response('OK', 200))
    ->name('webhooks.mailchimp.verify');
```

The `GET` route is required because MailChimp sends a verification GET when you register the webhook URL.

### `MailChimpWebhookController`

`app/Http/Controllers/MailChimpWebhookController.php`

```php
public function handle(Request $request): Response
{
    // Verify secret
    if ($request->query('secret') !== config('services.mailchimp.webhook_secret')) {
        abort(403);
    }

    $type  = $request->input('type');
    $email = $request->input('data.email');

    if ($type === 'unsubscribe' && $email) {
        Contact::where('email', $email)->update(['mailing_list_opt_in' => false]);
    }

    return response('OK', 200);
}
```

The secret is passed as a query parameter in the webhook URL registered in MailChimp (`/webhooks/mailchimp?secret=your-secret`). This is MailChimp's recommended verification pattern.

Exclude this route from CSRF protection in `bootstrap/app.php` (or wherever middleware exceptions are configured).

### Registering the webhook

Document in the help article — not automated. The admin registers the URL in MailChimp under **Audience → Manage Audience → Settings → Webhooks**. Subscribe to: `Unsubscribes` only for v1.

## Out of scope

- Syncing address fields to MailChimp
- Pulling MailChimp subscriber data back into the CRM
- Handling MailChimp `subscribe` events (someone subscribes via MailChimp form) — would need to create or update a contact record
- MailChimp `profile update` webhook events
- Storing batch job IDs or polling for completion
- Multiple MailChimp audiences
- Admin UI for MailChimp credentials (`.env` only for v1)
- Removing a contact from MailChimp when they are deleted or set to do_not_contact in the CRM
