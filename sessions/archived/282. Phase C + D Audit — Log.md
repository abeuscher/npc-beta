## Audit findings

### Finding 1 — Naming overload

"Track" was overloaded in the planning docs:
- **Tracks A–G** inside `release-plan.md` = release-bounded scope categories for Beta 1.
- **Tracks** inside `sessions/tracks/` = cross-release long-running architectural arcs (Widget Primitive, Fleet Manager Agent, Code Review & Cleanup, Widget Autonomy).

Same word, different concept. Resolved: A–G in `release-plan.md` renamed to **Phases A–G**. The `sessions/tracks/` docs keep the "tracks" name because they genuinely span releases.

### Finding 2 — Session 281 was never executed

The 282 base prompt assumed session 281 had closed and shipped the concurrent-editing lock system (record_locks table, RecordLock model, all the rest). Verification: no `record_locks` migration, no RecordLock model, no "Close session 281" commit. The planning files exist (`sessions/281. Concurrent Admin Editing.md`, `sessions/281. base-prompt.md`) but were never executed.

This means the Phase C accounting was off — #32b was treated as "closed" when in fact it was "planned but never started." Significant because the entry's (a)-scope plan (full pessimistic locking + polymorphic lock table + takeover rules + beforeunload hook) was over-scoped for a small-team admin context. With nothing shipped yet, the scope could be redesigned before any code lands.

### Finding 3 — C3 #32c scope drift

The original release-plan C3 success-criterion bullet for #32c was tight: protect endemic non-public fields from public flip + per-field protection-mechanism docs + ubiquitous public-content indicator (Path A).

The 282 session prompt as written drifted into a documentation-walk shape (data-classification section + carry-forward absorptions + indicator UX choice).

What the user actually wanted (surfaced at audit open): protection (Path A) **plus** three new features that aren't in the original bullet — accountability (actor stamped on publish/unpublish), notification (actor notified of their action), audit trail (queryable page-action log).

Three competing scopes, none aligned. Resolved: #32c stays Path-A-only; accountability/notification/audit-trail lifts to a new prereq stub **C3a** that must ship before the #32c drill.

### Finding 5 — Phase D drift

- **D2 prereqs language stale.** Said "C1–C7 must close"; after the Phase C refit, C6 lifted and C7 dropped.
- **D2 flaky-connection criterion** named the day-of event check-in flow as the test target; that flow was lifted post-Beta. Replaced with mobile donation submit.
