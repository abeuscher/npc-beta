## Workstream 1 — Role Deletion Safety

### What changed

**`app/Filament/Resources/RoleResource.php`**

Replaced the default `DeleteAction` with a custom `Tables\Actions\Action::make('delete')` that gates deletion behind a user-count check:

- **Zero users assigned** — opens a simple modal confirming the role is safe to delete.
- **One or more users assigned** — modal presents a `Radio` choice:
  - *Reassign* — a `Select` of all other non-super_admin roles; on confirm, all affected users are moved to the chosen role before the old role is deleted.
  - *Delete with acknowledgment* — a `TextInput` requiring the literal string `DELETE` before proceeding; validated via an inline Closure rule so the confirmation cannot be bypassed. Confirmed working with live validation.
- `super_admin` and `cms_editor` roles remain hidden from the delete action (unchanged).

Replaced `DeleteBulkAction` with an overridden version that:
- Checks all selected roles for assigned users before doing anything.
- If any have assigned users, blocks the entire bulk delete and sends a persistent danger notification listing the affected role names.
- If none have assigned users, deletes all selected roles normally.

Added imports: `Closure`, `Filament\Forms\Get`, `Filament\Notifications\Notification`.

### Tested manually
Delete with zero users, delete with reassignment, delete with `DELETE` acknowledgment, and bulk delete block all confirmed working.

## Workstream 2 — Birthday & Age Fields

### What changed

**Migration** `2026_03_21_000001_add_date_of_birth_to_contacts_table.php` — adds `date_of_birth` (date, nullable, after `postal_code`) to `contacts`. Ran successfully.

**`app/Models/Contact.php`** — added `date_of_birth` to `$fillable` and cast as `'date'`.

**`app/Filament/Resources/ContactResource.php`** — added `DatePicker` (columnSpan 5) and a read-only `Placeholder` showing calculated Age (columnSpan 7), placed after the postal code row in the Contact Information section. Age uses `$record->date_of_birth->age` (Carbon's own age property — positive integer, no decimal, called from DOB toward now). A `->maxDate(now()->subYears(13)->toDateString())` guard prevents entering a date of birth that would make the contact under 13, with helper text referencing the help article.

**`app/Filament/Resources/ContactResource/Pages/ListContacts.php`** — added `date_of_birth` to both the CSV header row and the data row (formatted as `toDateString()`).

**`docs/schema.md`** — added `date_of_birth | date | yes` row to the contacts table.

### Age field bug fix (during session)

Initial implementation used `now()->diffInYears($record->date_of_birth)` which returned a negative float in this Carbon build. Fixed to `$record->date_of_birth->age` (Carbon's built-in age property — always a non-negative integer).

### Date picker fix — global native pickers

The Flatpickr-based `DatePicker` (Filament's default) does not commit year changes unless a day is re-clicked — a known Flatpickr UX limitation. Fixed globally in `AppServiceProvider::boot()` using `configureUsing()` to set `->native()` on both `DatePicker` and `DateTimePicker` across the entire application. This covers all date fields: campaign dates, membership dates, donation date, event dates, page/post publish date, note timestamps, transaction timestamps, and any dynamic date fields in custom fields and collections — without touching individual resource files.
