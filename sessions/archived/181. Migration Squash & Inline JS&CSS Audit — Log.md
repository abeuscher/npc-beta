## Production deployment notes

Squash deploy requires a one-time clean replay of the schema dump rather than a normal `migrate`. Two options depending on whether production data is worth keeping:

- **Disposable data:** `docker exec nonprofitcrm_app php artisan migrate:fresh --seed`.
- **Keep data:** manual `pg_restore` or `psql < database/schema/pgsql-schema.sql` replay, followed by verifying `migrations` table state.

After the squash replay, future `migrate` runs behave normally.
