## What was built

### Phase 1 — Secure password generator on user create/edit form

- Added a "Generate" hint action button on the password field in `UserResource` using `alpineClickHandler()`.
- Uses `crypto.getRandomValues()` to generate a 20-character random password (mixed case, digits, symbols) entirely client-side — no server round-trip.
- Fills both Password and Confirm Password fields via `$wire.$set()`.
- Copies the generated password to the clipboard using `navigator.clipboard.writeText()`.
- Shows a Filament toast notification ("Password copied to clipboard") via `FilamentNotification`.

### Phase 2 — Data generator audit

- **2a — Removed financial record types:** Removed `generateDonations()`, `wipeDonations()`, `generatePurchases()`, `wipePurchases()` from `DashboardDebugGeneratorWidget`. Removed the donations and purchases options from the Blade type dropdown. Removed the Tax Year input. Removed unused model imports (`Donation`, `Transaction`, `Purchase`, `Fund`, `Carbon`). Products remain.
- **2b — Improved event generation:** Events now vary in price (~50% free, ~50% $5–$100) and capacity (~60% unlimited, ~40% 20–200). Free events receive 0–N random registrations using existing contacts where available (with RFC email validation fallback) or Faker-generated names/emails. Paid events receive no registrations.
- **2c — Improved blog post generation:** Each generated post now gets two `page_widgets` rows — a `text_block` widget with 2–4 paragraphs of lorem ipsum HTML, and a `blog_pager` widget with empty config. Widget type IDs are resolved by handle from the `widget_types` table at generation time.
- **2d — Wipe cleanup:** Removed `wipeDonations()` and `wipePurchases()`. Added a dedicated `wipeEvents()` method that deletes `event_registrations` before deleting events. Blog post wipe relies on FK cascade for `page_widgets` (unchanged).
