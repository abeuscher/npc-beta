# Session 072 — Activity Log — Log

**Date:** 2026-03-24
**Branch:** session-072

---

## What was built

### Phase 1 — Schema and infrastructure

- Migration creating `activity_logs` table: `id` (bigint PK), `subject_type`, `subject_id`, `actor_type`, `actor_id` (nullable), `event`, `description` (nullable), `created_at` (DB default). Indexes on `(subject_type, subject_id)` and `(actor_type, actor_id)`.
- `App\Models\ActivityLog` — fillable, `$timestamps = false`, `CREATED_AT` constant set, `created_at` cast as datetime.
- `App\Services\ActivityLogger::log()` — resolves current actor from `web` then `portal` guard, writes entry, wraps in try/catch for silent failure.
- `docs/schema.md` updated.

### Phase 2 — Observers

- Created `ContactObserver`, `MembershipObserver`, `DonationObserver` — each implements `created`, `updated`, `deleted`, `restored`. Registered in `AppServiceProvider::boot()`.
- Updated `PageObserver` — added activity logging to existing `created` and `updated` methods (preserving slug-prefix logic), added `deleted` and `restored`. Description set to "Published" or "Unpublished" when `is_published` changes.
- Updated `EventObserver` — added `created` and `deleted` hooks, added activity logging to existing `updated` method (preserving cancellation email logic). Description set to "Status changed to {status}" when `status` changes.

### Phase 3 — Contact Timeline

- `ContactNotes` renamed to Timeline (page title, breadcrumbs, button label on `EditContact`).
- Converted from `InteractsWithTable` to `InteractsWithActions`.
- `getTimeline()` fetches notes and activity log entries, pre-loads admin users in a single query, merges, sorts by `created_at` descending, caps at 200.
- Notes display with inline edit/delete actions (scoped to the contact's own notes for security). Activity log entries are read-only showing event, actor attribution, optional description, and timestamp.
- Filter added: ellipsis menu to the right of "Create note" with Show all / Contact notes / Activity log options. Active filter indicated with checkmark. Skips unnecessary DB queries when filter is set.
- Breadcrumbs: `Contacts › Edit › Timeline`.
- Note attribution displays "by [Author Name]".

### Phase 4 — Admin user activity

- Collapsible "Activity" section (collapsed by default) added to User edit form via `Placeholder` with rendered Blade partial.
- Shows 50 most recent entries where `actor_type = admin` and `actor_id = user.id`, sorted newest first.
- Section is `visibleOn('edit')` only.

## Security checklist — verified

- [x] `ActivityLogger::log()` is wrapped in try/catch — logging failures never surface to the user or break the primary operation.
- [x] No route, action, or UI allows editing or deleting activity log entries.
- [x] Portal actor attribution uses the authenticated portal guard user only — no cross-contact attribution possible.
- [x] Admin activity section on the User edit page is gated by existing user-view permissions.
- [x] Note edit/delete actions in Timeline scope queries to the current contact's notes — cannot edit another contact's notes by tampering with arguments.
