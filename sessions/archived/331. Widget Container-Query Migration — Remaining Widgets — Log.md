### Container-query migration (the planned core)

Each establishes `container-type: inline-size; container-name: np-widget` only on widgets whose SCSS actually declares an interior `@container` query, per the 330 blast-radius scoping rule.
- **BoardMembers** ‚Äî two `@media (max-width: $bp-md / $bp-sm)` ‚Üí `@container np-widget`. Verified by real-wrapper Playwright injection: cards collapse 4-up ‚Üí 2-up off the widget's *own* width (195.5px cards at a 918px container ‚Üí 290px at 668px) on a fixed 1280px viewport, proving container-driven, not viewport-driven.
- **EventsListing** ‚Äî one `@media` ‚Üí `@container np-widget`, and the hardcoded `48rem` literal normalized to `#{$bp-md}` (the published 768px constant). Featured hero collapses side-by-side ‚Üí stacked.
- **LogoGarden** ‚Äî four `@media` ‚Üí `@container np-widget` initially (cell + flip-container wrap thresholds); the widget was then rebuilt further (below).
- **Consumption gate** (`WidgetStyleBoundaryConsumptionTest`) ‚Äî each migrated widget's baseline entries deleted in the same pass; the `WIDGET_SCSS_VIEWPORT_MEDIA_BASELINE` is now **empty**, so the gate asserts **zero** viewport `@media` in widget SCSS.

### MapEmbed ‚Äî heading field removed entirely (owner call)

The migration surfaced that MapEmbed's only responsive rule (a heading font-size step) was *inert* ‚Äî out-specified by the host `.np-site h2` typography (0,1,3 vs the widget's 0,1,0). Owner chose to remove the heading outright (a rich-text field may return later; a sibling Text block stacks above just as well):

- Removed `heading` from `MapEmbedDefinition::schema()` + `defaults()`, from the Blade template (and the `$heading`-derived iframe title), and the `.map-embed__heading` rules (base + the just-added `@container`) from the SCSS ‚Äî leaving MapEmbed with no interior query, so it was dropped from the `np-widget` rule in `_layout.scss`.
- Demoted `inlineEditable()` ‚Üí `false` (the heading was its only inline-editable surface ‚Äî the same demotion BarChart/ProductCarousel took at session 308); moved `map_embed` from the true‚Üífalse roster in `InlineEditingFoundationSession304Test` (eligibility roster 10 ‚Üí 9).
- Fixed the line-number coupling: the `WidgetColorTokenConsumptionTest` baseline for the `color: #fff` scrim moved `MapEmbed/styles.scss:39` ‚Üí `:34`.
- Removed the now-vacuous `MapEmbed ‚Äî heading parity` case from `BuilderPublicRenderParityTest` (it asserted a heading-annotation surface that no longer exists).

### LogoGarden ‚Äî overlap fix + separation of concerns (owner-directed)

Investigating a reported "margins collapse 1024‚Üí768" turned up the real defect: real logos were **overlapping** each other (a partner/sponsor-logo contract concern). Root cause was three things fighting over sizing (the flex `gap`, a per-cell `width: calc()` that re-encoded that same gap, and the image's `max-width/max-height` doubling as a square cap that crushed wide logos). Rebuilt with separated concerns:

- **Bounding-box container** ‚Äî a new `logo-garden__media` wrapper per logo (`width: 100%`, fixed height cap); the image fits inside via `object-fit: contain`, so any aspect ratio renders centered, faithful, never overflowing or distorted. `<picture>` made `display: contents` so the class-bearing `<img>` (x-picture emits the class on the img) contains either way. Verified across ten varied intrinsic aspect ratios (incl. a 30√ó120 sliver and a 600√ó100 wordmark): **0/10 overflowing, 0/10 distorted**.
- **Layout** ‚Äî first tried CSS grid (clean single-source gap + column count) but **walked it back to flexbox + `gap` + `justify-content: center`** on the owner's call: grid left-aligns a partial last row, and logos must stay centered. The container fix was kept.
- **Single-source gap** ‚Äî one `--logo-gap` token now drives both the flex `gap` and the per-cell width share (the operator's "Slide spacing" config feeds it; default 16px = the old `1rem`), so the duplicate-spacing problem is gone.
- Template: each of the five `<x-picture>` instances (carousel, smooth, flipper front/back, static) wrapped in `logo-garden__media`; `--logo-gap` added to the static + flipper inline styles.
