## What was built

### Phase 1 — Manifest methods on the base class

- Added six optional methods to `App\Widgets\Contracts\WidgetDefinition` with safe defaults: `version()` → `'1.0.0'`, `author()` → `'Nonprofit CRM'`, `license()` → `'MIT'`, `screenshots()` → `[]`, `keywords()` → `[]`, `presets()` → `[]`. None abstract — every existing widget inherits defaults unchanged.
- Deliberately not added to `toRow()`. Metadata is code-only; no `widget_types` column, no migration.

### Phase 2 — `manifest()` aggregator

- Added `manifest(): array` on `WidgetDefinition` returning `handle`, `label`, `description`, `category`, `version`, `author`, `license`, `screenshots`, `keywords`, `presets` in one shape. This is the stable surface Stage 5's widget browser will consume.

### Phase 3 — `WidgetRegistry::manifests()`

- Added `manifests(): array` returning handle → `$def->manifest()` for every registered definition. One call populates the Stage 5 browser grid.

### Phase 4 — CI validation test

- New `tests/Feature/WidgetManifestTest.php` with six rules, each iterating every registered definition and failing with the offending widget handle:
  1. `version()` matches `/^\d+\.\d+\.\d+$/`.
  2. `license()` is in the allow-list (`MIT`, `Apache-2.0`, `GPL-3.0`, `BSD-3-Clause`, `proprietary`).
  3. Every path in `screenshots()` exists on disk (resolved against `app/Widgets/{Folder}/`).
  4. Every keyword matches `/^[a-z0-9-]+$/`.
  5. Every preset has the correct shape (`handle`, `label`, `description` nullable, `config`, `appearance_config`) and every key in `preset.config` exists in `schema()`.
  6. `manifest()` returns exactly the expected top-level key set.
- Each test asserts `$registry->all()` is non-empty before the inner loop so the vacuously-passing case (every widget returns `[]`) still produces real assertions.

### Phase 5 — Documentation

- `docs/widget-system.md`: added a "Manifest & Metadata" section documenting the six methods, the aggregator, the preset shape, the CI validation rules, and a worked preset example. `WidgetRegistry` bullet list now includes `manifests()`. Base-class optional-methods list extended with the six new methods. "Current status" bumped to Stage 4.
- `resources/docs/widget-development.md`: added a "Declaring manifest metadata" section with the method table, preset shape example, and the CI-enforced constraint that `preset.config` keys must appear in `schema()`. Quick-Start Checklist step 2 extended to mention the manifest overrides.
- `sessions/session-outlines.md`: Stage 4 of the Sovereign Widget arc marked *(Completed session 173)* with a summary of what landed and what was deferred (`requiredCapabilities()`, `minAppVersion()`).
