## What was built

### Phase 1 — Audit and report

- Scanned all 29 widget types across Blade templates, SCSS partials, and the WidgetTypeSeeder
- Produced a findings table covering: Tailwind residue, misplaced inline CSS/JS, color field types, orphaned config fields, unused variables, missing SCSS partials, BEM naming, and accessibility gaps
- Key findings: 4 color fields using `text` instead of `color`, inline margin/padding styles across 8 widget templates, 2 accessibility gaps (carousel nav buttons, hero video)
- Confirmed no Tailwind residue — `text-muted`, `text-sm`, `text-muted-light`, `icon-sm` are all project-defined utility classes

### Phase 2 — Fix

- Changed 4 color config fields from `type: 'text'` to `type: 'color'` in WidgetTypeSeeder: `bar_chart.bar_color`, `carousel.link_color`, `carousel.text_color`, `logo_garden.background_color`
- Added `aria-label` attributes to carousel prev/next navigation buttons
- Added `aria-hidden="true"` to hero widget's decorative background video
- Replaced inline styles in `donation-form.blade.php` with BEM classes: `.donation-section`, `.donation-section__heading`, `.donation-section__sub`
- Replaced inline styles in `product-display.blade.php` with BEM classes: `.widget-product-display__description`, `__status`, `__subtitle`
- Replaced inline styles in 5 portal templates with shared classes: `.portal-alt-link`, `.portal-description`, `.portal-description--tight`
- All new CSS classes added to `resources/scss/_custom.scss`
- Ran `build:public` to recompile the public CSS bundle
