- **CRM contract at v2.3.0** — unchanged. 332 was non-boundary (front-end CSS/JS + page-builder preview only).
- **Schema baseline:** no change since 328 (`pages.locked` + `edit_locked_pages`). **This session expects no schema change.**
- **Fast Pest suite green at 2756 / 0** (332 close baseline; verify as-found at 333 start). +4 vs 331's 2752 = the new `WidgetStyleLayerDisciplineTest`.
- **332 landed two things on `session-332/1`:** (1) `@layer` cascade isolation — all host CSS in `reset / host / widgets` layers, widget interiors win by layer order (commit `3fdf754`); (2) **container-query rollback** — the 330/331 `@container` width-overrides were removed because they keyed widget collapse to mobile *viewport* breakpoints applied to *container* width, so a column-placed widget rendered its phone layout on desktop **and overrode the operator's configured count**. Widgets now render their configured count (logos-per-row, items-per-row, column_count, slidesPerView) at **every** width; the `np-widget` container context and LogoGarden's Swiper viewport `breakpoints` ramp were removed (commit `33278b6`). `widget-development.md` updated to "no width-based collapse; mobile is future widget-system work."
- **Consequence for this session:** because widgets now render flat config at all widths, **editor↔public count parity is largely width-independent** — both surfaces should render the same configured count regardless of the editor's simulated preview width. That simplifies the harness; the residual divergences are the three bugs below.
- **Three confirmed carry-forward bugs (this session's targets), all observed live, none introduced by 330/331:**
  - **C — Editor Swiper load race (priority; the WYSIWYG-killer).** The page-builder preview races loading the Swiper library against the widget's Alpine init; when it loses, `window.Swiper`/`window.SwiperModules` are undefined, the widget throws on `SwiperModules.Navigation`, the carousel never initializes, slides stack, and the pager has zero bullets. **Confirmed intermittent** via real-browser runs (Swiper absent on one editor load, present on the next). Public is unaffected (library loads normally there). Loader code last changed at sessions 325/148 — predates 330/331.
  - **B — Uneven grid tracks.** `.content-card` grid items lack `min-width: 0`, so `grid-template-columns: repeat(N, 1fr)` distorts to content min-width at narrow widths (measured `172/207/159` for three "equal" tracks). One-line fix; affects BlogListing + EventsListing.
  - **E — Possible layout-column drop.** A layout column was lost at some point (the Welcome-page layout is labeled "3 Column Layout" but `page_layouts.columns = 2`; a first measurement saw 3 content columns, a later one saw 2). Current state is **consistent and renders fine** (no stranded widget). Owner hypothesis: changing a widget's `columns` config knocked out a layout column. **Owner guidance: one bounded repro attempt, then report — do not wild-goose-chase. If real it will recur.**
- **Planning state:** 332 grew beyond its planned `@layer` scope to include the container-query rollback (documented in the 332 log). This session is **promoted ahead of Mobile Nav**; Mobile Nav and the rest of the Mobile Public-Site Readiness push shift to 334+. The Widget Styling Contract arc concluded at 332.

## Process rules for every session

Base-prompt process rules apply in full (`sessions/template-base-prompt.md`). Session-specific notes:

- **Reproduce before you fix; observe, don't assert.** Every bug here was found by driving the real browser (Playwright against the live local site, and the admin editor with owner-provided local creds), not by reading code. Each fix is proven by re-driving the same surface and seeing the divergence gone — not by reasoning that the code now looks right. This is the explicit lesson from the 332 walk-back.
- **Playwright against the editor is read-mostly and must not reset the owner's data.** The project's standard specs run `resetDatabase()`; do **not** point them at the live local site. For ad-hoc diagnosis use a standalone script (login + navigate + measure) like the 332 investigation. The standing harness runs against the isolated e2e stack (its own DB), per 308.
- **WYSIWYG is the bar.** A fix is not done until the **editor preview and the public render match in appearance and behavior** for the affected widget. "Matches in HTML" (the 307 guard) is necessary but not sufficient — the carousel must actually initialize and the pager must render in the editor.
- **Found means fixed — no identify-then-defer.** Surface every divergence the sweep finds inclusively; the "won't-fix / intentional-divergence" set is dispositioned with the owner during the apply walkthrough and locked into the harness exclusion list, per 308 and the standing audit-findings rule.
- **Front-end builds:** page-builder Vue / `resources/js/**` / `resources/scss/**` → `npm run build`; widget SCSS/JS or `widget_types.assets` → `docker compose exec app php artisan build:public`. The editor Swiper-load fix is in `resources/js/page-builder-vue/**` (Vite). Rebuild the touched surface before re-driving the browser.
