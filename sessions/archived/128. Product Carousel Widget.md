# 128. Product Carousel Widget

Swiper-based coverflow carousel showcasing published products with images, names, descriptions, and buy buttons. Includes adding an image field to the Product model and a demo product seeder.

---

## Design decisions (resolved before starting)

- Carousel uses Swiper's `coverflow` effect for the Apple-style spotlight presentation: centred active slide, partially visible adjacent slides receding to each side.
- A CSS reflection effect is rendered beneath each slide using a pseudo-element (flipped, gradient-masked copy of the image).
- Product images use Spatie media library, same pattern as carousel/collection image fields.
- Buy buttons post to the existing `ProductCheckoutController` — one button per price tier.
- A `success_page` config field lets the admin specify a thank-you page slug. The checkout controller is updated to accept this value as a hidden form field, following the same pattern as `DonationCheckoutController`.
- The widget uses the existing `products` data source in `WidgetDataResolver` with `query_config` support (limit, order).
- **The user will supply sample product images before the session starts.** If no images are present at `resources/sample-images/products/`, ask the user before proceeding.

## Phase 1 — Add image field to Product model

- Register Spatie media library on the `Product` model (`HasMedia` interface, `InteractsWithMedia` trait) with a `product_image` media collection (single file).
- Register media conversions matching the existing pattern (`webp`, responsive breakpoints via `ImageSizeProfile`).
- Add a `product_image` file upload field to `ProductResource` (admin form) following the existing image upload conventions.
- Update `docs/schema/products.md` to note the media collection.

---

## Phase 2 — Update `WidgetDataResolver::resolveProducts()`

Extend `resolveProducts()` to include product image and price tiers in the resolved data:

- Load `prices` and `media` relationships.
- Include `image_url` in the returned array (from the `product_image` media collection, preferring webp conversion).
- Include `prices` as a nested array with `id`, `label`, `amount`, and `stripe_price_id` for each tier.

---

## Phase 3 — Update `ProductCheckoutController` for `success_page`

- Accept an optional `success_page` hidden field in the checkout form.
- When present and valid (exists as a page slug), use it as the Stripe success redirect URL instead of the Referer.
- Follow the exact pattern from `DonationCheckoutController`.

## Phase 4 — Product Carousel widget type registration

Add a `product_carousel` entry to `WidgetTypeSeeder` with:

- **handle:** `product_carousel`
- **label:** Product Carousel
- **description:** Coverflow carousel of published products with images, pricing, and buy buttons.
- **category:** `['giving_and_sales']`
- **collections:** `[]` (uses `resolveProducts()` directly, not collection-backed)
- **assets:** SCSS at `resources/scss/widgets/_product-carousel.scss`
- **config_schema:**
  - `heading` — text
  - `limit` — number, advanced. Max products to display (blank for all).
  - `autoplay` — toggle, default: true
  - `interval` — number, default: 5000, advanced. Autoplay interval in ms.
  - `background_color` — color, default: `#000000`
  - `text_color` — color, default: `#ffffff`
  - `success_page` — select, `options_from: pages`. Thank-you page after purchase.
  - `full_width` — toggle, default: true. Extend edge-to-edge without content container.

## Phase 5 — Product Carousel Blade template and SCSS

Create `resources/views/widgets/product-carousel.blade.php`:

- Swiper with `coverflow` effect: `rotate: 30`, `stretch: 0`, `depth: 100`, `slideShadows: false`.
- `centeredSlides: true`, `slidesPerView: 'auto'`.
- Navigation arrows and pagination dots.
- Autoplay controlled by config (`autoplay` toggle and `interval`).
- Each slide contains:
  - Product image (if present) as an `<img>` with `object-fit: cover`.
  - Reflection pseudo-element beneath the image (CSS `transform: scaleY(-1)` with a gradient mask fading to transparent).
  - Product name in `<h3>`.
  - Product description in `<p>`.
  - Buy buttons: one per price tier, each in its own `<form>` posting to `route('products.checkout')` with `product_price_id` and `success_page` hidden fields.
- `full_width` toggle controls whether the widget renders with or without the page content container, following the hero widget pattern.

### SCSS (`resources/scss/widgets/_product-carousel.scss`)
- `.widget-product-carousel`: background-color, text-color, padding
- `.product-slide`: centered card layout
- `.product-slide__image`: image container with overflow hidden
- `.product-slide__reflection`: flipped gradient-masked reflection
- `.product-slide__name`, `__description`, `__prices`: semantic classes
- `.product-slide__buy-btn`: buy button styling
- Swiper overrides for coverflow presentation (slide width, spacing)
