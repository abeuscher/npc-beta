## What was built

### Phase 1 — `requiredPermission` field on `DataContract` + centralized resolver gate

- [app/WidgetPrimitive/DataContract.php](app/WidgetPrimitive/DataContract.php) — added `public readonly ?string $requiredPermission = null` constructor parameter, declared between `$cardinality` and `$formatHints` per Decision 1. Docblock entry written in the existing style — names the Spatie permission semantics, the null-means-public-by-construction convention, and the cardinality-appropriate empty-shape contract that callers receive on deny.
- [app/WidgetPrimitive/ContractResolver.php](app/WidgetPrimitive/ContractResolver.php) — restructured `resolve()`'s per-contract loop body so the centralized gate fires before the source `match`. When `$contract->requiredPermission !== null && ! auth()->user()?->can(...)`, the loop assigns `['item' => null]` (CARDINALITY_ONE) or `['items' => []]` (CARDINALITY_MANY) into `$results[$i]` and `continue`s, skipping source dispatch entirely. One enforcement point per contract per resolve call, regardless of source — the Decision 6 "all four sources covered for free" property falls out naturally.

### Phase 2 — Lifted the three inline gates from the resolver arms

- [app/WidgetPrimitive/ContractResolver.php](app/WidgetPrimitive/ContractResolver.php) — deleted the three-line `if (! auth()->user()?->can('view_X')) { return ...; }` block from each of `resolveNote` / `resolveMembershipOne` / `resolveDonationList`. Updated each method's docblock to point to the centralized enforcement (replaced "Permission gate: fail-closed when the authenticated user lacks `view_X`" with "Permission gate is centralized in `resolve()` via the contract's `$requiredPermission`"). Ambient-type gates (`! $record instanceof Contact` for membership/donation, `$record === null` for note) stay inline per Decision 5 — they're a separate axis from permissions and out of scope here.

### Phase 3 — Three widget definitions declare `requiredPermission`

- [app/Widgets/RecentNotes/RecentNotesDefinition.php](app/Widgets/RecentNotes/RecentNotesDefinition.php) — `requiredPermission: 'view_note'` on the returned `DataContract`.
- [app/Widgets/MembershipStatus/MembershipStatusDefinition.php](app/Widgets/MembershipStatus/MembershipStatusDefinition.php) — `requiredPermission: 'view_membership'`.
- [app/Widgets/RecentDonations/RecentDonationsDefinition.php](app/Widgets/RecentDonations/RecentDonationsDefinition.php) — `requiredPermission: 'view_donation'`.
