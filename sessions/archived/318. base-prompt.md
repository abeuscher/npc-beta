## Process rules for every session

Base-prompt process rules apply in full. Session-specific notes:

- **Read-only first; destructive operations confirmed.** The scans produce reports; deletions / merges require operator confirmation per-row or per-batch. Never auto-delete.
- **Embedded-media-reference inventory is load-bearing.** The unused-media scan can produce false positives (real references the scan didn't recognize). Build the surface inventory deliberately — every place media gets referenced needs to be in the scan or marked explicitly as out-of-scope.
- **Rich-text fields are the hardest case.** Quill content stored in JSON columns may carry `<img>` tags whose `src` references media via various forms (URL, attachment-id, hash). The scan needs to parse rich-text content properly, not just regex-match URLs.
- **Tool surface mirrors existing admin tools.** Tools-group page, `manage_cms_settings` gate (or whichever existing permission fits), header actions, narrative sections explaining what each scan does.
- **PM-level tone for status summaries** per `feedback_pm_level_tone`.
- **Questions are not instructions** per `feedback_questions_are_not_instructions`.
