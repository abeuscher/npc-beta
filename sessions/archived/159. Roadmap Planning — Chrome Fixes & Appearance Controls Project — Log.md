### Phase 5 — Base prompt template improvements (`sessions/template-base-prompt.md`)

Three additions plus one consistency fix that bring the template into alignment with current project reality:

- **Archive the previous session** — new step in the close gate, between "Update session-outlines.md" and "Commit." When closing session N, all files matching `sessions/(N-1). *.md` are moved into `sessions/archived/`. Skips silently if those files don't exist or have already been archived. The current session's files stay in `sessions/` until the *next* session's close.
- **JS build verification** — new sub-bullet under "When implementation is complete." If the session touched any code under `resources/js/`, run `npm run build` and confirm it succeeds. Catches Vue/JS build failures that the Pest suite cannot see.
- **Widget asset build verification** — same place. If the session touched widget SCSS/CSS/JS or any `WidgetType.css`/`js` columns, run `docker compose exec app php artisan build:public` to recompile the public widget asset bundle.
- **Docker prefix consistency fix** — the test command in process rules previously read `php artisan test --exclude-group=slow` while the migration command three bullets above read `docker compose exec app php artisan migrate`. Both now use the explicit `docker compose exec app` prefix.

The same changes were applied directly to [sessions/160. base-prompt.md](sessions/160.%20base-prompt.md) since that file was created earlier in this session from the pre-update template.
