### Phase 2 — User-revision pass (5 visual changes)

After the first review the user requested five small visual changes; all applied JSON-only (with one DB-side change for the site footer):

1. **Hero top padding halved.** Band 1 layout padding 200/200 → 100/200. Reduces the gap between the page header and the hero text.
2. **Hero CTAs left-aligned.** Band 1 Hero `text_position: center-center` → `top-left`; `fullscreen: true` → `false`. The G11 substring-match bug means any text_position containing "center" forces center-aligned CTAs; `top-left` is the cleanest way out under the existing schema and pairs naturally with content-natural hero height once `fullscreen` is dropped.
3. **"What it Does" composite extends through the grid.** Band 4a grid layout background `#ffffff` → `#d4d4f2`; Band 4b CTA-footer layout background `#ffffff` → `#d4d4f2`. Bands 3 + 4a + 4b now read as one tinted slab.
4. **Final CTA band switches to solid black.** Band 8 layout background gradient → `#000000`. White text + `secondary-dark` CTAs read cleaner with the harder contrast than against the gradient.
5. **Site footer.** Default Template `footer_bg_color` → `#333333` via direct model update. Footer chrome-page text_block widget `appearance_config.text.color` → `#ffffff`. White text on `#333` background. (Templates and chrome pages live outside the page JSON, so this was a tinker-style DB update, not a JSON import.)

### Phase 3 — Gap-report appends

Four new rows surfaced during the rebuild and appended to `sessions/public website/gap-report.md` under a new "Session 286 findings" section:

- **G15 — Hero widget renders its own `.site-container` inside a layout cell, causing nested containers.** Visual impact is inert (both containers cap to the same max-width) but architecturally noisy. Recommended: either suppress the inner `.site-container` when the Hero renders inside a layout slot, or default to Text widget for hero-band roles (the user-revision pass implicitly chose the second path on this page).
- **G16 — Hero `fullscreen: true` inside a layout cell stretches the cell to 100vh.** The viewport-relative height propagates through `align-items: center` and `grid-auto-rows` defaults so the whole Band 1 row becomes 100vh. Recommended lift: add `fullscreen` to `layout_config` as a layout-container option (the home will use the layout-level knob; the Hero widget can keep its own as a back-compat path or drop it).
- **G17 — TextBlock requires placeholder content (`<p><br></p>`) to render cleanly as a CTA-only band.** TextBlock renders `.widget-text-block__content` even when content is empty. Low priority; either skip the wrapper conditionally or accept the placeholder as the convention.
- **G18 — Image widget has no built-in cell-placement controls.** Images fill their column cells edge-to-edge. Deferred until real photography lands; full-cell fill is the predictable default for now.

Also appended a Session 286 visible-change validation note: the layout-spec approach delivered what it was authored to deliver (zero importer warnings, eight bands in spec order on first pass, cross-section rhythm matched the spec's prediction). Equivalent specs should follow for About / Pricing / Contact / Demo.
