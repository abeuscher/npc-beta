**Date:** 2026-03-28

## What was built

### Phase 1 — Infrastructure & production fixes

- Fixed `SiteThemePage.php` Vite build command: `npm run build` → `./node_modules/.bin/vite build` to resolve `vite: not found` in the Docker container.
- Fixed `Dockerfile` permissions: added `resources/scss` to the `chown` line so the PHP-FPM process can write `_theme.scss`.
- Refactored the Site Theme logo field to follow the Admin Settings pattern: `Placeholder` for current logo preview, `FileUpload` for new upload, conditional save only when a new file is provided.

### Phase 2 — Event widget slug fix

- Changed all event widget config keys from `event_id` (UUID) to `event_slug` (string) across `WidgetTypeSeeder`, `PageController`, and `EventResource::createLandingPageForEvent()`.
- Updated all affected feature tests (`EventTest`, `EventRegistrationModeTest`, `MailingListOptInTest`, `EventLandingPageTest`) to use `event_slug`.

### Phase 3 — Blog pager widget

- Added `blog_pager` widget type to `WidgetTypeSeeder` with server render mode.
- Created `resources/views/widgets/blog-pager.blade.php` using `COALESCE(published_at, created_at)` for robust ordering of posts with or without an explicit publish date.
- Added `'currentPage' => $page` to the `Blade::render()` data array in `PageController`.
- Diagnosed a `Blade::render()` anonymous-component scoping bug that prevents `$currentPage` from reaching `@include`'d widget templates — full fix deferred to Session 084 (`PageContext` architecture). Blog pager debug output left in place pending that fix.

### Phase 4 — Blog post authors & event authors

- Migration: added `author_id` (bigint unsigned, not null, FK → `users.id` with `restrictOnDelete`) to the `pages` table via two migrations (nullable first, then backfilled and set not null).
- Migration: added `author_id` (same pattern) to the `events` table.
- Added `author()` relationship and `author_id` to `$fillable` on both `Page` and `Event` models.
- Added Author select field (required, defaults to authenticated user, positioned at top of Settings column) to `PageResource`, `PostResource`, and `EventResource`.
- Renamed the right-hand section in `PostResource` from "Publication" to "Settings" for consistency.
- `EventResource::createLandingPageForEvent()` now passes `author_id` from the event to the created landing page.
- `CreatePage` and `CreateEvent` page classes set `author_id` from `auth()->id()` in `mutateFormDataBeforeCreate`.
- Updated `EventFactory` and `PageFactory` to include `author_id => User::factory()`.
- Updated `BasePageSeeder`, `PortalPageSeeder`, and `SystemPageSeeder` to pass `author_id` (first user in DB) when seeding pages.
- Updated all feature tests that called `Page::create()` directly to use `Page::factory()->create()` so mass-assignment protection doesn't silently drop `author_id`.
- Fixed `DashboardDebugGeneratorWidget::generateBlogPosts()` to assign `author_id` from a random existing user.

### Phase 5 — Remaining polish items

- Added `auto_publish_pages` and `auto_publish_posts` toggles to `CmsSettingsPage`. `CreatePage` reads these settings to set `is_published` and `published_at` on new pages/posts.
- Changed the default event publish state to published (`event_auto_publish` default → `'true'`).
- Added `->default('#1a56db')` to the `header_color` ColorPicker in `EmailTemplateResource` to match the wrapper fallback colour.
- Added `scroll-behavior: smooth` to `.fi-sidebar-nav` in `public/css/admin.css`.
- Fixed `RoleResource::canDelete()` to allow deletion of all non-`super_admin` roles (removed unexplained restriction on `cms_editor`).
- Added `blog_posts` generate/wipe case to `DashboardDebugGeneratorWidget`.

### Phase 6 — Session 084 preparation

- Authored `sessions/084. Widget Render Context.md` (full session spec) and `sessions/084. base-prompt.md`.
- Updated `session-outlines.md`: blog pager scoping bug documented, Sessions 084 and 085 stubbed as the two-part widget context migration.

## Production deployment notes

- Run `php artisan migrate` — two migrations add and constrain `author_id` on `pages` and `events`. Both backfill nulls with the first user in the DB before applying the not-null constraint, so they are safe to run against existing data as long as at least one user exists.
- Run `php artisan db:seed --class=WidgetTypeSeeder` — updates the `blog_pager` widget type template and adds the `blog_posts` generator entry.
