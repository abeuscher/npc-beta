### Notable finding

The session prompt's original Test A sketch used `{{item.meta_description}}` with a sentinel in the Page's `meta_description` column and asserted the sentinel does not appear in the rendered HTML. First attempt failed: the sentinel **did** appear — not through the `{{item.meta_description}}` token (that substitution correctly failed-closed), but through the `{{item.excerpt}}` token, because `SystemModelProjector::projectPost()` legitimately derives `excerpt` from `meta_description` (via `Str::limit(strip_tags($post->meta_description ?? ''), 160)`).

The fail-closed invariant is about **contract keys**, not about **source-column content**. The projector is allowed to derive a whitelisted field from any source column; what the contract prevents is undeclared keys appearing in the DTO. Test A was restructured to use genuinely non-derived sentinels (`meta_title`, `head_snippet`) for the content-leak assertions, plus a direct key-level assertion decoded from the `listingData` JSON — `items[0]`'s keys canonically equal the seven contract fields.

This clarification belongs carried forward into Phase 4b and subsequent retrofit sessions: choose leak-test sentinels in columns that have no derived whitelisted counterpart. BlogListing's derived fields (`url` from `slug`, `date`/`date_iso` from `published_at`, `excerpt` from `meta_description`, `image` from media) are all intentional.

## Deferred decisions

- **Contract-level `limit` and `order_by` filters for BlogListing.** Explicitly out of scope — pagination and sort are Alpine client-side today. If server-side pagination becomes a feature, it is a separate session.
- **Widening `SystemModelProjector::projectPost()`'s field map.** Each added field is a fail-closed boundary loosening; do it deliberately, not as part of a retrofit audit.
- **`blog_prefix` in the `url` derivation.** `SystemModelProjector::projectPost()` accepts a `$blogPrefix` parameter (passed from `config('site.blog_prefix', 'news')`) but currently composes `url` as `url('/' . $post->slug)` without applying the prefix. This is a pre-existing behavior, unrelated to the retrofit. Not touched; flagged here for whenever public blog routing is revisited.
- **Default `vendor/bin/pest` memory limit.** The `-d memory_limit=2G` override worked single-pass. Whether to bake this into the project's test tooling (PHP ini overrides, composer script, `tests/Pest.php` setup) is a separate question and not widget-primitive-track work.
- **Remaining ~19 page-builder widgets.** Phase 4b (EventsListing) drafted as session 217; the rest follow. The 216 retrofit was near-trivial because the contract was already substantially landed in sessions 209–211; EventsListing is a true retrofit (no contract yet, template walks relationships directly) and is the better stress test for the pattern.
