# 151. Widget Delete & Livewire Inspector Removal

Add widget deletion to the Vue inspector, then remove the Livewire inspector and its supporting code. After this session, the editor runs entirely on Vue (except the widget picker and save-as-template modals, which stay on Livewire permanently).

**Reference document:** `docs/editor-migration-plan.md` ‚Äî read this file in full before writing any code. The teardown plan is in Part 4 (session 151 section). The component tree (Part 3, section 3.2) and store design (Part 3, section 3.1) are relevant.

**Prior sessions:** Sessions 147‚Äì150 built the API endpoints, Pinia store, preview canvas, toolbar, inspector panel with all field types, appearance controls, spacing, query settings, and color swatches. All of those are in place and working. The Livewire inspector is still rendered but is now redundant.

---

## Design decisions (resolved before starting)

- **Delete confirmation modal:** Vue component (`ConfirmDeleteModal.vue`), not browser `confirm()`. Displays "Are you sure you want to delete this widget: [Widget Label]?" with Cancel and Delete buttons. Delete button is red/destructive styled.
- **Delete button placement:** Small delete button (trash icon or similar) to the right of the widget title in the `InspectorHeader.vue` component. Disabled for required widgets (`is_required`).
- **Livewire inspector hiding:** The Livewire inspector blade (`page-builder-inspector.blade.php`) and block blade (`page-builder-block.blade.php`) are removed entirely. The Livewire components (`PageBuilderInspector.php`, `PageBuilderBlock.php`) are deleted.
- **Alpine modules:** The four Alpine.js modules (`preview-manager.js`, `spacing-controls.js`, `richtext-editor.js`, `button-list-manager.js`) and their registration entry point (`index.js`) are deleted. Their functionality has been ported to Vue.
- **PageBuilder.php becomes thin:** Strip all block manipulation methods except `createBlock` (called by the widget picker modal) and `saveAsTemplate` (called by the save-as-template modal). Keep `mount()`, `getBootstrapData()`, and `render()`. Remove preview rendering methods ‚Äî the Vue app handles previews via the API.
- **Vite rebuild required:** After removing Alpine module imports, run `npm run build` to update the JS bundle.

## Phase 1 ‚Äî Widget deletion in Vue

### 1a. `ConfirmDeleteModal.vue`

Create `resources/js/page-builder-vue/components/ConfirmDeleteModal.vue`:

- Props: `visible` (boolean), `widgetLabel` (string)
- Emits: `confirm`, `cancel`
- Renders a centered overlay modal with:
  - Message: "Are you sure you want to delete this widget: **[Widget Label]**?"
  - Cancel button (neutral)
  - Delete button (red/destructive)
- Closes on Cancel, Escape key, or backdrop click
- Transitions for open/close

### 1b. Update `InspectorHeader.vue`

- Add a small delete button (trash icon) to the right of the widget title
- Button is hidden when `widget.is_required` is true
- Clicking opens the `ConfirmDeleteModal`
- On confirm, calls `store.deleteWidget(widget.id)`

### 1c. Verify deletion works

- Delete removes the widget from the preview and store
- If the deleted widget was selected, selection clears
- Required widgets cannot be deleted

## Phase 2 ‚Äî Livewire teardown

### 2a. Strip `PageBuilder.php`

Remove all methods except:
- `mount()` ‚Äî keep for page setup and bootstrap data
- `getBootstrapData()` ‚Äî keep, provides initial data to Vue
- `render()` ‚Äî keep, returns the view
- `createBlock()` ‚Äî keep, called by the widget picker modal
- `openAddModal()` / `closeAddModal()` / modal properties ‚Äî keep, drives the widget picker
- `saveAsTemplate()` / template modal properties ‚Äî keep, drives save-as-template
- `loadBlocks()` ‚Äî keep if `createBlock` depends on it for refreshing the Livewire block list after creation

Remove:
- `deleteBlock()`, `copyBlock()`, `moveUp()`, `moveDown()`
- `updateOrder()`, `onMoveToMain()`, `onMoveToColumn()`
- `selectBlock()`, `switchToEdit()`, `switchToHandles()`
- `refreshAllPreviews()`, `renderWidgetForPreview()`, `renderColumnChildren()`
- `buildInlineStyles()`, `buildDemoCollectionData()`, `collectLibs()`
- `resolveWidgetImageUrls()`
- Any properties only used by removed methods (`$mode`, `$previewBlocks`, etc.)

### 2b. Simplify `page-builder.blade.php`

Replace with a thin shell:
- Keep the widget picker modal (`<x-widget-picker-modal>`)
- Keep the save-as-template modal
- Main content area is just `<div id="page-builder-app" data-bootstrap='@json($bootstrapData)'></div>`
- Remove the Livewire toolbar, block list, inspector panel, and edit/handles mode toggle
- Remove any Alpine.js `x-data` registrations for the old editor modules

### 2c. Delete Livewire components

Delete these files:
- `app/Livewire/PageBuilderBlock.php`
- `app/Livewire/PageBuilderInspector.php`

### 2d. Delete blade partials

Delete these files:
- `resources/views/livewire/page-builder-block.blade.php`
- `resources/views/livewire/page-builder-inspector.blade.php`
- `resources/views/livewire/partials/inspector-field.blade.php`
- `resources/views/livewire/partials/inspector-field-group.blade.php`
- `resources/views/livewire/partials/inspector-fields/` (entire directory ‚Äî all 13 field partials)

### 2e. Delete Alpine.js modules

Delete these files:
- `resources/js/page-builder/index.js`
- `resources/js/page-builder/preview-manager.js`
- `resources/js/page-builder/spacing-controls.js`
- `resources/js/page-builder/richtext-editor.js`
- `resources/js/page-builder/button-list-manager.js`

Remove any import of `resources/js/page-builder/index.js` from the Vite entry points (check `vite.config.js` and `resources/js/main.js`).

### 2f. Rebuild Vite

Run `npm run build` after removing Alpine module imports.

### 2g. Verify event bridge

Confirm that the Vue ‚Üî Livewire event bridge still works:
- Vue dispatches `open-widget-picker` ‚Üí Livewire opens the modal
- Livewire dispatches `widget-created` ‚Üí Vue replaces the tree
- Vue dispatches `open-save-template-modal` ‚Üí Livewire opens the modal
