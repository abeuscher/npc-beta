# 183. Page Builder Typography Inheritance & Theme Editor Polish

Make the page builder's preview canvas (and the Quill rich-text editor inside it) render with the same typography the public site uses, so designers can tweak theme styles and see them in-editor without publishing. Also apply a round of UI polish to the Theme → Text Styles tab based on notes the user will hand over at session start.

---

## Design decisions (resolved before starting)

- **Source of truth stays the same.** Typography is read from `SiteSetting('typography')` via `App\Services\TypographyResolver::load()` and compiled by `App\Services\TypographyCompiler::compile()`. No new data model. The page builder consumes the same compiled CSS the public layout already injects.
- **Scope is the Vue page builder's preview region only**, not the whole Filament admin panel. Typography CSS should not bleed into the Filament chrome (sidebars, tabs, modals, form fields).
- **Quill editor inherits too.** When a user edits a rich-text field in the page builder, the content area inside the Quill editor should render headings/paragraphs/lists with the theme's fonts, sizes, and spacing — so "write an H2" looks the way it'll look on the published page.
- **The `:not(nav ...)` exclusion stays.** Current selectors in `TypographyCompiler` already dodge `<nav>`. Confirm the preview region does not include `<nav>` markup that would be incorrectly excluded; if it does, decide whether to scope differently for preview.
- **Google Fonts.** The Theme editor page already preloads the full curated set (`resources/views/filament/pages/design-system.blade.php`). The page builder needs the same preload so previewed fonts actually render — not just system fallbacks.
- **No new storage, no migration.** This is rendering/integration work.

## Phase 1 — Inject theme typography into the page builder preview

1. Extend `PageBuilder::getBootstrapData()` (or add a sibling Livewire-rendered blade partial) to emit the compiled typography CSS and the Google Fonts `<link>` once per page-builder mount.
2. The injection must be scoped so it only affects the preview region — not the Filament shell, the inspector panel, or the rest of the admin chrome. Options to consider (pick one, flag the choice in the log):
   - Wrap the preview region in a scoping class (e.g. `.np-preview`) and recompile selectors under that scope for this mount, **or**
   - Render the preview region inside an iframe that loads its own `<head>` with typography CSS + fonts, **or**
   - Use a shadow-DOM root for the preview region.
3. Whichever approach is chosen, ensure the Google Fonts are preloaded (reuse the preload pattern from the Design System page blade).
4. Verify: with a theme where H1 = Georgia and a page containing a Text Block widget with an H1, the H1 in the preview renders Georgia. Changing the theme and returning to the page builder shows the updated font.

---

## Phase 2 — Quill editor inherits typography

1. The Quill content area is `.ql-editor` inside each rich-text field. Apply the theme typography to elements inside that editor so headings, paragraphs, and lists render with the theme's fonts/sizes/spacing while the user is typing.
2. Do not leak typography rules onto the Quill toolbar itself (`.ql-toolbar` should stay in its admin-chrome styling).
3. Verify: open a rich-text field, choose H2 from the Quill heading menu, type a line. The line renders in the theme's H2 style.
