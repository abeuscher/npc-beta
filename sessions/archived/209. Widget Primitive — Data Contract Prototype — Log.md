## What was built

### Phase 1 ‚Äî Scaffold the contract primitives

New namespace `app/WidgetPrimitive/` with four files and two test files.

- `DataContract.php` ‚Äî typed value object. Constructor takes `version`, `source` (one of `SOURCE_PAGE_CONTEXT` / `SOURCE_SYSTEM_MODEL` / `SOURCE_WIDGET_CONTENT_TYPE`), `fields` (declared field list), `filters`, `model`, `resourceHandle`, `contentType`. All properties `readonly`.
- `ContentType.php` ‚Äî widget-declared content shape. Carries `handle` and `fields`. Exposes `fieldKeys()` and `imageFieldKeys()` derivations.
- `SlotContext.php` ‚Äî wraps `PageContext` with an optional `currentPageOverride` for callers that need to bind the slot to a specific Page without mutating the shared `PageContext`.
- `ContractResolver.php` ‚Äî single service, `resolve(array $contracts, SlotContext $context): array`. Per-call fetch-dedup cache keyed by source fingerprint (`sha1(serialize($filters))` for system models, `collection:{handle}` for widget content types). Fail-closed field projection: only declared fields appear in the returned DTO.
- `tests/Unit/WidgetPrimitive/DataContractTest.php` + `ContractResolverTest.php` ‚Äî 10 unit tests, 37 assertions.

### Phase 2 ‚Äî Migrate Blog Listing

- `WidgetDefinition::dataContract(array $config): ?DataContract` ‚Äî new base method, default returns `null`.
- `BlogListingDefinition::dataContract()` ‚Äî `SOURCE_SYSTEM_MODEL` contract, `model: 'post'`, fields `['title', 'slug', 'url', 'date', 'date_iso', 'excerpt', 'image']`.
- `WidgetRenderer` ‚Äî looks up the widget's definition via `WidgetRegistry`. If a contract is declared, resolves via `ContractResolver` and injects the DTO into template vars as `$widgetData`.
- `BlogListing/template.blade.php` ‚Äî the `$posts = $pageContext->posts()` + 30-line `->map(...)` block collapsed to `$items = $widgetData['items'] ?? [];`. Rest of the template (token substitution, Swiper config, card pre-render) unchanged.
- `tests/Feature/ContentListingWidgetsTest.php` ‚Äî 7 existing Blog Listing template-isolation tests updated to inject `widgetData` instead of `pageContext`. Shared `blogListingWidgetData()` helper added at the top of the file.
- `tests/Feature/WidgetPrimitiveIntegrationTest.php` (new) ‚Äî end-to-end render through `WidgetRenderer` asserting the full contract ‚Üí resolver ‚Üí template path produces expected HTML.

### Phase 3 ‚Äî Migrate Text Block

- `SlotContext` ‚Äî grew `currentPageOverride` constructor parameter and a `currentPage()` method that honors it, so the owner-page fallback used by the legacy path (admin preview, standalone render) threads through the contract path cleanly.
- `TextBlockDefinition::dataContract()` ‚Äî `SOURCE_PAGE_CONTEXT` contract declaring the full `PageContextTokens::TOKENS` union.
- `WidgetRenderer` ‚Äî contract resolution moved before the token-substitution block. For widgets with a `SOURCE_PAGE_CONTEXT` contract, substitution uses DTO values (fail-closed to declared fields). For non-contract widgets, the legacy `PageContextTokens` path runs unchanged.
- Added one integration test ‚Äî renders a real Text Block through `WidgetRenderer`, asserts `{{title}}` and `{{date}}` tokens substitute correctly.

### Phase 4 ‚Äî Migrate Carousel

Image field handling lifts `ContentType::imageFieldKeys()` to populate `_media` on each row.

### Follow-on refinement ‚Äî query-count regression fix

Initial full-suite run surfaced the `PageBuilderApiTest > buildTree eager-loads owner` test at 32 queries (test ceiling `< 32`). Diagnosed as the `SOURCE_PAGE_CONTEXT` contract firing `PageContextTokens::values()` on every widget render, which lazy-loads `$page->event` and `$page->author`. The legacy path short-circuited on `Str::contains($text, '{{')`; the contract path had no such check. Added a token-presence short-circuit in `WidgetRenderer::configHasTokens()` ‚Äî the resolver is skipped for `SOURCE_PAGE_CONTEXT` contracts when no config value contains `{{`. Query count dropped back to 30. Short-circuit is a stop-gap; structural batching is one of the four refinements carried into session 210.

## Deferred decisions

All four named in the findings doc; each becomes a phase in session 210:

- Fallback-data semantics should move from `WidgetRenderer` into `ContractResolver` so no DTO ever bypasses field projection.
- `SOURCE_PAGE_CONTEXT` batching ‚Äî singleton-binding `PageContextTokens` with per-page-id memoization, replacing the `{{` short-circuit as the load-bearing defence.
- Projection logic extraction ‚Äî per-source-type projector classes under `app/WidgetPrimitive/Projectors/`, so adding more widgets in Phase 4 of the migration plan doesn't turn `ContractResolver` into a switchboard.
- Richtext-consumer case ‚Äî name the "declare the union" pattern explicitly in the `WidgetDefinition::dataContract()` docstring and in the migration doc, so Phase 2 widgets follow it without re-opening the question.

Forward hook added to `widget-primitive-migration.md`: appearance as a shared contract. Not committed to any phase; reachable if slot taxonomy surfaces a concrete appearance-constraint need.
