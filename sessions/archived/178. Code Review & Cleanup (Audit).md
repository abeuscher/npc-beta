## Workstream 1 — Stale reference audit

Grep for references that no longer resolve after the Livewire→Vue migration and sovereign widget reorganisation.

- **Deleted Livewire components.** Sessions 148–151 removed the Livewire page builder (`PageBuilder`, `PageBuilderBlock`, `PageBuilderInspector`). Grep for any remaining `App\Livewire\PageBuilder*` references in blade, PHP, or JS code.
- **Retired blade partials.** `resources/views/livewire/partials/inspector-fields/` and related Livewire blade templates should all be gone. Grep for `@include('livewire.partials.*')` and for Livewire components that are still referenced but whose class file no longer exists.
- **Deleted widget view paths.** Session 172 retired `resources/views/widgets/` and `resources/scss/widgets/`. Grep for any absolute path references to these directories in PHP, blade, SCSS, or JS.
- **Renamed appearance keys.** Session 162 renamed appearance config schema keys and ran a dedupe sweep. Grep for any references to the old key names.
- **Retired Alpine state.** Sessions 148+ replaced Alpine state with Vue/Pinia for the builder. Grep for `x-data`, `x-on`, or `$wire` references in page-builder blade files that no longer back real Livewire methods.
- **Dead `use` statements.** Imports referencing non-existent classes after the reorganisation.

Fix confirmed stale references in place.

## Workstream 2 — Widget sovereignty invariants

Sessions 170–172 moved every widget to `app/Widgets/{PascalName}/`. For every widget:

- Has a `{PascalName}Definition.php` extending `WidgetDefinition`.
- Has a `template.blade.php`.
- Is discovered by `WidgetRegistry::all()` (or equivalent).
- `WidgetTypeSeeder` is a thin wrapper over `WidgetRegistry::sync()` — no widget is hardcoded into the seeder outside the sync call.
- `toRow()` values match the DB row after seeding.
- No widget missing a `demoSeeder()` where the dev demo page would otherwise render empty (use session 174 and session 177 observations as a guide).
- No widget with `requiredConfig()` keys that don't exist in `schema()`.
- No widget whose `presets()` entries reference `config` keys that don't exist in `schema()` or that live under a non-`appearance` group (CI validates this — confirm the assertion still runs).

Flag mismatches. Fix obvious ones (missing `demoConfig()` keys, orphaned folder references); flag anything that changes runtime behaviour for discussion.

## Workstream 3 — Preset and thumbnail pipeline consistency

Sessions 175–177 shipped the preset system and the preset/static thumbnail pipeline.

- Verify `WidgetType::forPicker()` and `PageBuilderApiController::widgetTypes()` produce identical preset and thumbnail shapes — both were touched in 177; confirm they haven't drifted.
- Verify `WidgetType::resolvePresetThumbnails()` and `WidgetType::resolveStaticThumbnail()` use the same folder-resolution logic, and that neither synthesises a filesystem path from an untrusted handle.
- Verify `WidgetThumbnailController::show()`'s filename regex still matches the filename conventions actually produced by `scripts/generate-thumbnails.js` (`static.png`, `preset-{slug}.png`).
- Verify the cache-buster (`?v={filemtime}`) is applied consistently across both static and preset thumbnail URLs.
- Verify the preset apply semantics are implemented the same way in the Vue store (`applyPreset`) and in the dev preset route (`WidgetDemoController::showPreset`) — overlay `config`, replace `appearance_config`.

Fix mismatches in place; flag uncertain findings.

## Workstream 4 — Dead code and orphaned files

- **Dead imports.** `use` statements that import classes not referenced in the file. Focus on controllers, services, and Vue components touched since 141.
- **Unreachable methods.** Public methods on controllers, services, or models that are no longer called from any route, view, or test.
- **Orphaned blade partials.** Any partial in `resources/views/` that is no longer `@include`d from anywhere — especially under `resources/views/livewire/` which should be mostly gone post-session-151.
- **Orphaned JS/TS/Vue files.** Any file under `resources/js/` that was created during the Vue migration or sovereign-widget arc but is no longer imported from the entry point or a component.
- **Orphaned SCSS.** Any partial under `resources/scss/` that is no longer `@use`'d or `@import`'d.
- **Retired Livewire controllers, views, and routes.** The Livewire page builder is fully gone; sweep for any residue including `PagePreviewController` and `page-preview.blade.php` (flagged as orphaned in session 141 and deferred).

Fix confirmed dead code in place. Flag uncertain cases for discussion.

## Workstream 6 — Duplicated logic

Scan PHP and Vue/TS files for blocks of 10+ lines that appear in substantially similar form in multiple files. For each:

- Describe what the repeated code does.
- List where it appears.
- Propose an extraction target (model method, trait, service, Blade/Vue component).

Known candidates to investigate:

- Thumbnail URL construction in `WidgetType::forPicker()` vs `PageBuilderApiController::widgetTypes()`.
- Preset-apply logic in the Vue store vs. the dev preset route (both overlay `config` and replace `appearance_config` — confirm the semantics are extracted to a shared helper, or propose one).
- Appearance style composition paths (`AppearanceStyleComposer` vs. any duplicate inline style building in templates or services).
- Any widget bootstrap-payload shape that appears both in a Filament page and in the Vue builder API.

Do not refactor. Present findings for session 179.

## Workstream 7 — Framework alignment

Same diagnostic question as sessions 116 and 141:

> "Is this code doing something the framework already does? If so, are we working with the framework or around it?"

Focus areas:

- **Laravel / Eloquent.** Manual queries or array manipulations that should use scopes, relations, or accessors. Collection operations done with `foreach` that would read better as `->map()` / `->filter()` / `->groupBy()`.
- **Vue / Pinia.** Ad-hoc reactivity that could use Pinia's built-in `computed` / `watch` / `storeToRefs` patterns. Vue component patterns that reinvent built-in directives.
- **Filament.** Admin pages that duplicate Filament form or table component behaviour that could be leveraged.
- **Blade components.** Repeated HTML patterns that should be anonymous Blade components.

Record findings in a table: File, Line(s), What the code does, What the framework provides, Risk if left as-is, Suggested approach.

Do not fix. Present the table for session 179.
