### Phase 1 — Toolchain installation

- `composer.json` / `composer.lock` — added `infection/infection ^0.32` to `require-dev`. The `infection/extension-installer` plugin landed in `config.allow-plugins` so `composer install` on a fresh checkout doesn't prompt.
- `Dockerfile` — added `pecl install pcov` after the existing `redis` PECL install. Deliberately **not** followed by `docker-php-ext-enable pcov` — PCOV is installed but not auto-loaded, so the default Pest profile runs at the same speed it did pre-241. Loading PCOV is opt-in per Infection invocation via `php -d extension=pcov -d pcov.enabled=1`. App image rebuilt to pick up the new extension.
- `infection.json5` — new file at repo root. `source.directories: ["app/WidgetPrimitive"]` (the session-241 slice — swap for future audits), `@default` mutator set, JSON + HTML + text + summary log emission, `--exclude-group=slow` on the inner Pest run, `phpUnit.customPath: "bin/pest-for-infection.php"` (the adapter shim — see below).
- `.gitignore` — appended exclusions for Infection's report artifacts at repo root: `infection.html`, `infection.json`, `infection-log.txt`, `infection-summary.txt`, `.infection.cache/`.
- `bin/pest-for-infection.php` — Pest 2 + Infection 0.32 adapter shim. PHP rather than bash because Infection's PHPUnit adapter spawns `phpUnit.customPath` as `php <path>`. Two compatibility fixes inside:
  1. **Per-mutant config `<file>` path rewrite.** Infection generates a PHPUnit config under `/tmp/infection/` for each mutant with relative test paths. PHPUnit/Pest resolve those relative to the config-file directory, producing `/tmp/infection/tests/Feature/X.php` — which doesn't exist — and Pest exits non-zero with "Test file not found." Without the fix, Infection records every mutant as `killed` from the non-zero exit alone (no test ever runs); the first Phase 2 attempt produced a fake 100% MSI / 159 killed where every `processOutput` was the "Test file not found" line. The shim rewrites every `<file>` in any `/tmp/infection/` config to absolute project paths before passing argv on to Pest.
  2. **Junit log namespace rewrite.** PCOV's coverage-xml emits Pest's mangled namespace (`P\Tests\Feature\…`) while PHPUnit's junit log emits the user-facing namespace (`Tests\Feature\…`). Infection cross-references the two by class name; the prefix mismatch hits `TestNotFound`. The shim runs Pest, then `str_replace`-rewrites the junit log so both sides agree on the `P\` form.
  - Optional env-gated mode: `PEST_FOR_INFECTION_NO_STOP_ON_FAILURE=1` rewrites the per-mutant configs' `stopOnFailure="true"` to `"false"`. Required for the redundancy/dead-test analysis (default Infection runs only see the *first* failing test per mutant; the env var produces the complete catcher set per mutant). 4–5x slower per mutant — used for Phase 2 Pass 2 only.
- Smoke test: `vendor/bin/infection --filter=ContractResolver.php` produced 169 mutations / 155 killed / 14 timed out / 100% MSI in 32s — toolchain end-to-end working.
