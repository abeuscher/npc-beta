**Cluster 1 — Media hygiene**
- `[0.336.01 · 2026-06-02]` — **Orphan media-file sweep — close the documented "wholesale-wipe gap."** Physical files in `storage/app/public/cas/<content_hash>/` are only removed when a media row is deleted *through the model* (`ContentAddressedFileRemover` on `->delete()`). Bulk row clears (`migrate:fresh --seed`, `demo:reset`, scrub regeneration, Playwright `resetDatabase()`) truncate the `media` table without firing that event, so the files orphan. Measured at session 336: **228 of 280 `cas/` directories (81%) were orphans**; the tree was 416 MB (swept to 308 MB as a one-time cleanup). `media-library:clean` does **not** cover this (it removes orphan *rows* + deprecated *conversions*, not tree dirs with no row). Fix: a `media:prune-orphans` command keyed on `content_hash` — delete `cas/<hash>/` dirs whose hash is in no live media row. Local-only growth (the demo droplet only restores, never seeds), so the demo baseline blob should be authored from a swept tree to ship lean.

**Cluster 2 — Widget + admin-tool polish**
- `[0.329.02 · 2026-05-31]` — **EventMiniCalendar internal margins collapse** as the widget narrows. After session 330 removed the widget's `display:none`, it renders at narrow column widths (capped at `max-width: 21rem`, centred) where its fixed internal `margin`/`padding`/`gap` values crowd. Self-verifiable CSS spacing fix; **not** a `@media`→`@container` boundary item (internal-spacing only). Keep the 330 anti-morph guarantee (square cells, centred) intact.
- `[post-milestone]` — **Random Data Generator: add Organizations** to the entity-generation list. The generator (`app/Services/RandomDataGenerator.php`) currently spans contacts / events / registrations / donations / memberships / products; Organizations is absent. Mirror the existing arms (generate / count / clear) with `source => Source::SCRUB_DATA` so `demo:reset` and scrub-clear round-trip. Small admin-tool addition.
