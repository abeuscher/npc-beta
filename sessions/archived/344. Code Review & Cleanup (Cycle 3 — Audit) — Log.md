### Boolean-settings / config-shape sweep (new)

**Canonical convention codified:** a boolean `SiteSetting` is a row of `type='string'` storing the literal `'true'`/`'false'`; reads compare `=== 'true'`; writes use `? 'true' : 'false'`. Real model boolean *columns* legitimately use Eloquent `'boolean'` casts (all 31 verified backed by real DB boolean columns — no mismatches). Two drifts + one dead branch:
1. **`use_pico`** — the only key seeded with `type='boolean'` (`DatabaseSeeder.php:96`), routing through the dead `filter_var` cast branch; also **never read anywhere** (orphan). Fix: change to `type='string'`/`'false'` or drop entirely.
2. **`event_auto_publish`** — read-default mismatch: `CmsSettingsPage.php:47` defaults `'false'`, `EventResource.php:116` defaults `'true'`. On a fresh install the toggle reads OFF while events auto-publish ON. Align the defaults.
3. **`SiteSetting::castValue()` `'boolean'` branch** (`:65`) — the only `FILTER_VALIDATE_BOOLEAN` in the app; unreachable once `use_pico` is fixed. Delete or document. This is the anchor for the convention-drift Pest test.

Exhaustive boolean-key list for the apply test: `horizon_enabled`, `notes_edit_only_by_creator`, `auto_publish_pages`, `auto_publish_posts`, `noindex_global`, `stripe_tos_url_configured`, `stripe_dashboard_branding_confirmed`, `event_auto_publish`.

The session-308 typography seam is **resolved** (importer design merge gated behind `merge_design` default-OFF + shared resolver defaults + manifest pre-flight). Live findings:
1. **EventRegistration import-email seam (APPLY PICK).** `EventRegistrationObserver::created()` sends a **synchronous** `RegistrationConfirmation` email and auto-creates a Contact for every registration with an email. Fix: `withoutEvents()` at the two seams, or a `source`-guard early-return in the observer. **Verified by hand** (`EventRegistrationObserver.php:12-49`).

## Open Flags block

### Flag 344-A — SVG upload sanitizer wired to nothing *(security; downgraded by owner — no real PII in any system)*

SVGs accepted at 5+ upload chokepoints (`PageBuilderApiController:364/440`, `DashboardBuilderApiController:221`, `RecordDetailViewBuilderApiController:377`, `InlineImageUploadController:15`, `Collection.acceptedFileTypes:134`); `registerMediaConversions()` **explicitly skips** SVG (stored raw); `App\Services\Media\SvgSanitizer` exists, complete, with **zero callers**. Execution vector is direct-URL navigation to the stored `.svg` (standalone-document context runs `<script>`), gated only by upload permission. **Session 329's log + prompt documented "SVGs run through SvgSanitizer" as a live protection that does not exist.** Owner call at audit: not worried (no real PII) — treat as ordinary cleanup, no pull-forward. **Apply (345/1):** wire `SvgSanitizer::sanitize()` into the media-upload pipeline (store-time observer/handler) OR drop SVG from accepted types OR serve uploaded SVG as attachment; **and correct the 329 record** either way. Non-boundary.

### Flag 344-B — `CollectionResource::getRelationManagers()` inert dead code

See static-reflection scan above. **Apply (345/2):** rename to `getRelations()` (activates `CollectionItemsRelationManager`) or delete both — decide whether the items relation manager duplicates `ContentCollectionResource`'s `ManageContentCollectionItems` surface before activating.

### Flag 344-C — EventRegistration import-email seam

See integration-seam sweep #1. **Apply (345/2):** suppress events at the two importer create-loops (`withoutEvents()`) or `source`-guard the observer.

### Flag 344-D — `demo:restore` does not lock pages

`demo:reset` runs `Page::query()->update(['locked'=>true])` (`DemoResetCommand.php:38`); `demo:restore` (the path the node cron uses) has no lock step (`DemoRestoreCommand.php`) — relies entirely on the baseline blob carrying `locked=true`. **Apply (345/1):** add the lock step + a test.

### Flag 344-E — `BlockDemoUploads` defense-in-depth gap

Two builder routes (`admin-dashboard-api.php:27`, `admin-record-detail-view-api.php:28` → `uploadAppearanceImage`) lack the middleware. Not exploitable today (the `demo` role lacks the gating permissions) but breaks the single-chokepoint invariant. **Apply (345/1):** add `BlockDemoUploads` to both.

### Flag 344-F — `canViewAny`-only Resources (style)

6 Resources gate via `canViewAny()` instead of explicit `canAccess()`. Functionally protected. **Apply (345/5):** add explicit `canAccess()` for convention uniformity, OR formally bless the delegation pattern (then assert it in the convention test). No live gap either way.

### Flag 344-G — Transaction/Campaign/Fund have no audit-trail observer *(judgment call)*

Not a regression (never in the tracked-CUD set). **Apply:** decide whether financial Transaction CUD deserves an `ActivityLogger` observer; defer Campaign/Fund unless wanted.
