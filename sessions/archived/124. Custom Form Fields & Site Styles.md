# 124. Custom Form Fields & Site Styles

Build custom-styled form controls (select dropdown, toggle switch, radio buttons) that replace browser defaults on all public-facing forms. Work with the user to polish overall site visual appearance.

---

## Design decisions (resolved before starting)

- Custom controls are built with HTML + CSS + Alpine.js. Each control hides the native element visually and renders a styled replacement that syncs its value to the hidden native input — forms submit normally.
- The three controls to build: styled select (custom dropdown), toggle switch (replaces checkbox where appropriate), styled radio buttons.
- Controls live in Blade components under `resources/views/components/` so any widget or template can use them via `<x-custom-select>`, `<x-toggle>`, `<x-radio-group>`.
- Styling uses the SCSS partial system established in session 123 (`_forms.scss`, `_variables.scss`). No Tailwind utilities.
- The second half of this session is collaborative: walk through the site with the user and adjust typography, spacing, colours, and other visual details based on their feedback. Changes go into the SCSS partials.

## Phase 1 — Custom select dropdown

Build `<x-custom-select>` Blade component. Alpine.js opens/closes a dropdown panel. Supports:
- `name`, `id`, `required` attributes passed through to the hidden native `<select>`
- `options` array (value/label pairs)
- `placeholder` text
- Keyboard navigation (arrow keys, enter, escape)
- Accessible: `role="listbox"`, `aria-expanded`, `aria-activedescendant`

SCSS in a new `_controls.scss` partial.

---

## Phase 2 — Toggle switch

Build `<x-toggle>` Blade component. A styled on/off switch that replaces a checkbox. Supports:
- `name`, `id`, `checked` attributes
- `label` text
- Accessible: `role="switch"`, `aria-checked`

Use for the theme dark/light toggle in the footer and for boolean form fields where appropriate.

---

## Phase 3 — Styled radio buttons

Build `<x-radio-group>` Blade component. Replaces native radio buttons with styled alternatives (button-style or card-style selectors). Supports:
- `name`, `required` attributes
- `options` array (value/label pairs)
- `layout` — inline or stacked

---

## Phase 4 — Site style polish

Collaborative phase. Walk through pages with the user and adjust:
- Typography (sizes, weights, line heights)
- Spacing (section gaps, widget margins)
- Colour refinements (muted text, borders, backgrounds)
- Any other visual issues the user flags

All changes go into the existing SCSS partials.
