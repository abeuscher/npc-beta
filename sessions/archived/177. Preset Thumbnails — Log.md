## What was built

### Phase 1 — Dev route for preset variants

- Added `WidgetDemoController::showPreset()` rendering a widget at 800×500 with a named preset applied — overlays `preset.config` onto the demo config, replaces `appearance_config` wholesale (mirrors inspector apply semantics).
- Refactored shared rendering into `renderWidget()` and `resolveBaseline()` helpers so `show()` and `showPreset()` don't duplicate the seeder + registry + renderer dance.
- Registered `GET /dev/widgets/{handle}/presets/{presetHandle}` in `routes/dev.php`, named `dev.widgets.preset`. Same `DevRoutesMiddleware` gate — 404 in production.

### Phase 2 — Thumbnail capture script extension

- Extended `scripts/generate-thumbnails.js` to iterate each widget's manifest `presets[]` after (or instead of) the static capture, writing each preset PNG to `app/Widgets/{PascalFolder}/thumbnails/preset-{handle}.png`.
- Added `--preset=<handle>` filter (composable with `--widget=<handle>`) for targeted regeneration.
- Added `--no-static` and `--no-presets` flags so the script can be used in either mode independently.

### Phase 3 — Thumbnail serving controller

- New `App\Http\Controllers\WidgetThumbnailController::show()`. Public route `GET /widget-thumbnails/{handle}/{file}` registered in `routes/web.php` as `widget-thumbnails.show`.
- Filename validated against `^(static|preset-[a-z0-9-]+)\.png$`. PascalFolder derived from `WidgetRegistry::find()` + `class_basename()` (never synthesised from the URL handle). Response is streamed with `Content-Type: image/png` and `Cache-Control: public, max-age=3600`.

### Phase 4 — Bootstrap payload plumbing

- Added `WidgetType::resolvePresetThumbnails()` static helper that decorates code-authored presets with a `thumbnail` key — the public `widget-thumbnails.show` URL when the PNG exists on disk, `null` otherwise. URL includes a `?v={filemtime}` cache-buster.
- Added `WidgetType::resolveStaticThumbnail()` for per-widget static thumbnails, same resolution + cache-buster pattern.
- Wired both helpers into `WidgetType::forPicker()` and `PageBuilderApiController::widgetTypes()`. The widget-picker `thumbnail` falls back to the file-based `static.png` only when the media-library upload is absent (admin upload wins when present).
- Extended the Vue `WidgetPreset` type with `thumbnail?: string | null`.

### Phase 5 — Inspector card rendering

- `InspectorPresetGallery.vue` code-authored preset cards now render `<img :src="preset.thumbnail" class="preset-card__thumb" alt="" loading="lazy" />` when the thumbnail is present, and fall back to the existing empty placeholder div when it isn't. Blank card and draft cards are unchanged.
- Added `img.preset-card__thumb { object-fit: cover }` so the PNG doesn't squish.

### Phase 6 — Capture Hero presets and commit PNGs

- Ran `npm run generate-presets` (introduced mid-session) to capture all Hero presets. Produced `preset-bold-statement.png`, `preset-draft-2.png`, `preset-minimal-left.png`, `preset-campaign-cta.png`, `preset-compact-announcement.png`, and refreshed `static.png`.
- Discovered and fixed a text-colour bug in `HeroDefinition::demoConfig()` / `demoAppearanceConfig()` — the demo content had an inline `style="color:#ffffff"` on the `<h1>` that overrode the Minimal Left preset's dark text. Removed the inline color; moved the default white to `demoAppearanceConfig.text.color` so each preset's own `text.color` drives the final render. Recaptured all PNGs.

## Security checklist — verified

- [x] Dev preset route is gated by `DevRoutesMiddleware` and returns 404 in production — same behaviour as the existing widget demo route
- [x] `WidgetThumbnailController` validates the filename against a strict regex and rejects traversal attempts with 404
- [x] The PascalFolder is derived from `WidgetRegistry::find($handle)` rather than re-implementing the handle-to-folder transform
- [x] Thumbnail URLs returned in the bootstrap payload route through the named controller — not raw `file://` paths or direct `public_path()` URLs
- [x] Cache header on the controller is `public, max-age=3600` — no aggressive caching that would prevent a designer from seeing an updated thumbnail
