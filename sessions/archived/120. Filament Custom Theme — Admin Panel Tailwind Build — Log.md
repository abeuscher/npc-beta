### Docker build restructure

The Filament theme needs `vendor/filament/` for the Tailwind preset and blade content scanning, but `vendor/` is gitignored and only exists after `composer install`. The Docker node-builder stage (Stage 1) has no Composer — it only builds JS/CSS.

Resolution: the full `npm run build` (public + admin theme) now runs in the PHP stage (Stage 2) after `composer install`, where `vendor/` is available. A new `vite.config.public.js` gives the node-builder a public-only build for Docker layer caching.

### Widget picker fix (unrelated to theme)

- Member Login and Member Signup widgets changed from `['member']` page-type restriction to `null` (all page types) — these are public-facing forms that belong on regular pages, not portal pages
- Updated tests in `WidgetPickerSession119Test` to match
