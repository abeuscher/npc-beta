# 033. Admin Branding & Dashboard

Add three admin-specific settings to the existing General Settings page: a logo and title displayed in the Filament panel header, and a plain-text welcome message shown on the dashboard. All three values are stored in `site_settings` and managed from a new "Admin Panel" section in `GeneralSettingsPage`.

---

## Background & decisions

The Filament panel provider (`AdminPanelProvider`) has a hard-coded identity — no custom logo, no custom title. The dashboard shows only the default Filament account and info widgets with no admin-facing welcome message.

**Why General Settings, not a new page:** The admin branding fields are few and naturally belong alongside the site URL that already lives in General Settings. A dedicated Admin Settings page would be disproportionate to three fields.

**File upload storage strategy:** Filament's `FileUpload` component expects a disk-relative path (e.g. `site/admin-logo.png`) in its state, not a `storage/`-prefixed URL fragment. Feeding it a URL fragment prevents pre-fill from working. For the admin logo we therefore store the disk-relative path in `site_settings` under the key `admin_logo_path`. The panel provider derives the public URL at render time via `Storage::disk('public')->url($path)`. This differs from how `CmsSettingsPage` stores `logo_path` — do not change that page's behaviour.

**Dashboard welcome as plain text:** A rich-text field in a panel header welcome risks broken formatting in the admin UI. A plain `TextInput` is safer and sufficient.

---

## What to build

### 1. Extend `GeneralSettingsPage`

File: `app/Filament/Pages/Settings/GeneralSettingsPage.php`

**Add a new Section** to the form schema after the existing "Site" section:

**Section "Admin Panel"** (columns: 2):

- `TextInput::make('admin_brand_name')` — label `'Panel title'`, nullable, helper text `'Replaces the default "Filament" text in the admin header. Leave blank to use the default.'`
- `TextInput::make('dashboard_welcome')` — label `'Dashboard welcome message'`, nullable, `->columnSpanFull()`, helper text `'Displayed at the top of the admin dashboard. Leave blank to hide.'`
- `FileUpload::make('admin_logo_upload')` — label `'Panel logo'`, nullable, `->disk('public')->directory('site')->visibility('public')`, `->acceptedFileTypes(['image/png', 'image/jpeg', 'image/svg+xml'])`, `->columnSpanFull()`, helper text `'Displayed in the top-left of the admin panel. Leave blank to use the default.'`

**Update `mount()`** — add these keys to the existing `$this->form->fill([...])` call:

```php
'admin_brand_name'  => SiteSetting::get('admin_brand_name', ''),
'admin_logo_upload' => SiteSetting::get('admin_logo_path', ''),
'dashboard_welcome' => SiteSetting::get('dashboard_welcome', ''),
```

The `admin_logo_upload` field is intentionally pre-filled with the stored disk-relative path so that the component can display the current logo and the user does not have to re-upload it on every save.

**Update `save()`** — add after the existing `SiteSetting::set('base_url', ...)` call:

```php
SiteSetting::set('admin_brand_name', trim($data['admin_brand_name'] ?? ''));
SiteSetting::set('admin_logo_path', $data['admin_logo_upload'] ?? '');
SiteSetting::set('dashboard_welcome', trim($data['dashboard_welcome'] ?? ''));
```

Setting `admin_logo_path` to an empty string when the field is cleared is intentional — it disables the logo in the panel provider.

---

### 2. Panel provider — dynamic brand name and logo

File: `app/Providers/Filament/AdminPanelProvider.php`

Add `use Illuminate\Support\Facades\Storage;` to the imports if not already present.

Read both settings at the top of the `panel()` method, before the `return $panel` chain:

```php
$brandName = SiteSetting::get('admin_brand_name', '');
$logoPath  = SiteSetting::get('admin_logo_path', '');
```

Add `use App\Models\SiteSetting;` to the imports.

Chain onto the panel builder, placed **after** `->authMiddleware([...])` and **before** the first `->renderHook(...)`:

```php
->when($brandName !== '', fn ($p) => $p->brandName($brandName))
->when($logoPath  !== '', fn ($p) => $p->brandLogo(Storage::disk('public')->url($logoPath)))
```

Do not change anything else in the panel provider.

---

### 3. Dashboard welcome widget

Create `app/Filament/Widgets/DashboardWelcomeWidget.php`:

```php
namespace App\Filament\Widgets;

use App\Models\SiteSetting;
use Filament\Widgets\Widget;

class DashboardWelcomeWidget extends Widget
{
    protected static string $view = 'filament.widgets.dashboard-welcome-widget';

    protected static ?int $sort = -10;

    public static function canView(): bool
    {
        return SiteSetting::get('dashboard_welcome', '') !== '';
    }
}
```

Create `resources/views/filament/widgets/dashboard-welcome-widget.blade.php`:

```blade
<x-filament-widgets::widget>
    <x-filament::section>
        {{ \App\Models\SiteSetting::get('dashboard_welcome', '') }}
    </x-filament::section>
</x-filament-widgets::widget>
```

`DashboardWelcomeWidget` will be auto-discovered via the panel provider's `->discoverWidgets(...)` call — no manual registration needed.

---

## Out of scope

- Removing or toggling the default Filament account or info widgets
- Dark-mode logo variant
- Admin favicon
- Any change to the public-facing logo stored under `logo_path` in CMS Settings
