The demo LP builds against today's registration widget and does not wait on any of these.

### Phase 2 — architecture decisions settled

- **Per-page permission:** a `restricted_roles` jsonb column on `pages` (default `[]`), deny-list of role names barred from editing that page. Enforced in `PagePolicy::update()` **and** wired into the page-builder API route (which today does not check the page policy per-page). super_admin bypasses via the existing `Gate::before`. demo:reset seeds the demo arrival page with `restricted_roles:['demo']`. Chosen over a `page_permissions` join table (per-action granularity is YAGNI) and over an `is_locked_from_demo` bool (the exception shape the owner rejected). Generalizable, not an exception.
- **Calendar:** server-rendered Blade month grid via Carbon (all date math server-side → no "date-exception hell"), small vanilla `script.js` for scroll-to-day + month-nav, as its own widget in the index right-rail column. **vue-cal retired from the plan** — the public side has zero Vue runtime, the old `EventCalendar` already used a vanilla lib, and a density-dot mini-month-grid doesn't justify standing up Vue on the public side.
- **Driver.js tour:** reserve a `data-tour` attribute scheme (dot-namespaced by surface) now; build sessions add anchors as they touch widgets; the tour itself is a downstream session. Recorded why this doesn't contradict session 249 (that was the Filament/Livewire admin DOM; this is our own stable public CMS DOM).

## Security considerations

None beyond standard convention — planning-only, no code. The one security-adjacent design
decision (the `restricted_roles` per-page permission) was deliberately shaped as a deny-list
with a concrete default and super_admin bypass via the existing `Gate::before`, and the plan
flags that enforcement must cover **both** the Filament resource and the page-builder API
route (the real edit surface). The portal-login deferral keeps a second public auth surface
off the demo node until it can get its own lockdown audit (mirror of `DemoRoleLockdownTest`).
