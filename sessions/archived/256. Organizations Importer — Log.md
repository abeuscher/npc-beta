## Security checklist — verified

- [x] Match-key SQL uses parameter binding (`whereRaw('LOWER(TRIM({$matchField})) = ?', [$value])` with column name `$matchField` whitelisted to `name|email` only — external_id matching uses parameterized `where()` directly, scoped by `import_source_id`).
- [x] The importer respects the `import_data` permission gate via `canAccess()` matching the other five importers.
- [x] No widening of `Organization` permissions — the standard `OrganizationPolicy` still gates create / update / delete elsewhere.
- [x] PII-scan check applies to the Organizations CSV upload via `InteractsWithImportProgress::mount()` — same path the other namespaced importers use.
- [x] The new `__tag_organization__` sentinel does not inadvertently expose tag-create on Contact tags or vice versa — `Tag::create(['name' => $name, 'type' => 'organization'])` is type-segregated, and the morph relations in `Tag::contacts()` / `Tag::organizations()` are independent.
- [x] Custom-field handle validation matches the other importers — `alpha_dash` rule on the trait's `cf_handle_{$n}` TextInput.
