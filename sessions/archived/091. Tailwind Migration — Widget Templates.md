# 091. Tailwind Migration — Widget Templates

Convert all public-facing widget templates from Pico's classless HTML to Tailwind utility classes. Session 090 installed Tailwind and migrated layouts — this session completes the migration by restyling every widget template.

---

## Design decisions (resolved before starting)

- **Like-for-like conversion.** Each widget should look and behave the same as it did under Pico. No new features, no redesigns.
- **Dark mode on every widget.** Every widget template must include `dark:` variants for backgrounds, text, and borders where the Pico version relied on automatic dark mode.
- **Form widgets use `@tailwindcss/forms`.** The forms plugin provides a clean unstyled baseline. Widget-specific form styling is applied via utility classes.
- **The form grid system** (`.form-grid`, `.col-N`) is replaced by Tailwind's `grid grid-cols-12` and `col-span-N` utilities. Update the `public-form` component and any widget that uses the grid.
- **Carousel widget styles** (`_custom.scss` or equivalent from session 090) may need minor adjustments but should largely work since they were already written as custom CSS scoped to `.widget-carousel`.

## Phase 1 — Content widgets

Convert the following widget templates to Tailwind:

- `blog-listing.blade.php` — post list with headings and dates.
- `blog-pager.blade.php` — previous/next post navigation.
- `events-listing.blade.php` — upcoming events list.
- `event-description.blade.php` — single event detail display.
- `event-registration.blade.php` — registration form with fields, capacity messaging, honeypot.
- `column-widget.blade.php` — CSS grid column layout (already uses inline grid styles — verify it works with Tailwind's reset).

## Phase 2 — Commerce widgets

- `donation-form.blade.php` — amount presets, frequency toggle, Stripe Elements container, custom amount input. This is the most complex widget — take care with form element styling and Alpine.js state.
- `product-display.blade.php` — product details and checkout form.

## Phase 3 — Form & utility widgets

- `web-form.blade.php` — delegates to `<x-public-form>`. Update the `public-form` component to use Tailwind form classes and replace the `.form-grid` / `.col-N` system with `grid grid-cols-12` / `col-span-N`.
- `image.blade.php` — simple image wrapper, likely minimal changes.
- `carousel.blade.php` — verify Swiper integration works correctly with Tailwind's reset. Adjust `.widget-carousel` custom CSS if needed.
