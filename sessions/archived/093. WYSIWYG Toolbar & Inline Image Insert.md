# 093. WYSIWYG Toolbar & Inline Image Insert

Standardise the Quill rich text editor toolbar across all usage sites and add inline image upload/insert via a custom handler that stores images through Spatie Media Library.

---

## Design decisions (resolved before starting)

- **Quill v2** is already loaded globally via CDN in `AdminPanelProvider`. All toolbar changes are configuration, not library upgrades.
- **Two toolbar rows.** The editor lives in a sidebar inspector, so vertical space is acceptable. Row 1: formatting. Row 2: structure, media, cleanup.
- **Font family removed.** The site's design system dictates font. Users should not override it.
- **Heading levels H2–H5 + Paragraph.** H1 is reserved for the page title. H6 is rarely needed.
- **Image upload uses a custom Quill handler**, not the built-in base64 embed. The handler uploads via Livewire/endpoint, stores through Spatie Media Library on the parent model, and inserts an `<img>` tag with the returned URL.
- **Editor minimum height: 16rem.** Both the Filament form component and the inspector field Quill instances.

---

## Phase 1 — Toolbar standardisation

Update the Quill toolbar configuration in both usage sites:

- `resources/views/forms/components/quill-editor.blade.php` (Filament form component)
- `resources/views/livewire/partials/inspector-field.blade.php` (page builder inspector)

### Toolbar layout

**Row 1:** Heading dropdown (P, H2, H3, H4, H5) | Bold | Italic | Underline | Strikethrough | Text Color | Background Color

**Row 2:** Bullet List | Numbered List | Blockquote | Text Align (left, center, right, justify) | Link | Image | Clean

### Other changes

- Remove the `font: []` option from the toolbar.
- Remove the `size: []` option — replaced by the heading dropdown.
- Remove the `FontAttributor` whitelist code from `AdminPanelProvider` (no longer needed with font removed).
- Set editor container minimum height to `16rem` in both templates.

---

## Phase 2 — Inline image upload handler

Build a custom Quill image handler that:

1. Opens a file picker when the toolbar image button is clicked.
2. Accepts image files (PNG, JPEG, GIF, WebP, SVG).
3. Uploads the file via Livewire to the server.
4. Stores the file through Spatie Media Library on the parent model (the PageWidget, Event, or EmailTemplate that owns the editor content).
5. Inserts an `<img>` tag with the public URL into the editor at the cursor position.

The handler must work in both the Filament form component context (EventResource, EmailTemplateResource) and the page builder inspector context. In the inspector, the parent model is the `PageWidget`; in Filament forms, it is the model being edited.

### Upload endpoint

Add a Livewire method (or dedicated controller endpoint if cleaner) that:

- Accepts the uploaded file.
- Validates it (image types only, reasonable size limit — 10 MB).
- Stores it via `$model->addMedia()->toMediaCollection('inline-images')` using a dedicated `inline-images` collection.
- Returns the public URL.

### Rendering

Inline images are stored as standard `<img src="...">` tags in the HTML content. No `<x-picture>` conversion — these are inline editorial images, not widget-level optimised images. Responsive/WebP conversions on the `inline-images` collection are out of scope for this session.

---

## Security checklist

Before closing:

- [ ] Image upload validates file type server-side (not just client-side accept attribute).
- [ ] Image upload enforces a size limit (10 MB max).
- [ ] Upload endpoint is auth-gated — only accessible to authenticated admin users.
- [ ] No XSS vectors introduced through the image URL insertion.
- [ ] File picker does not expose filesystem paths or internal storage structure.

---

## Out of scope

- Responsive/WebP conversions for inline images (future session)
- `<x-picture>` rendering for inline images
- Media library "pick from existing" browser in the editor
- Tracking inline image references for deletion protection
- Drag-and-drop image insertion
- Image resize handles in the editor
