### Iteration `/1` — Flag W10/A FilePond polling + W8 #1 ObservedBy + W7 #1 password-mismatch (`d249ae1`)

- **Flag W10/A — FilePond upload-completion polling.** Three-layer hardening at `tests/e2e/helpers/wizard.ts:fillUploadStep`:
  1. Wait for `.filepond--root` to be visible before `setInputFiles` — the field is mounted via Filament's Alpine `x-load` lazy import; setting files before initialization completes drops the change event on the floor.
  2. Replace the precise-state `expect(...).toBeVisible({ timeout: 30s })` with `expect.poll(() => count > 0, { timeout: 90s, intervals: [200, 500, 1000] })`. Re-queries the DOM each interval, tolerating the brief window where Filament's `->live()` FileUpload triggers a Livewire roundtrip that detaches the item element.
  3. `selectExistingSource()` now waits for `networkidle` after `selectOption()` — choosing an existing source pre-loads prior mappings via Livewire, and `setInputFiles` against the about-to-be-replaced FilePond DOM element was the second-pass flake shape observed in the trace.
- **W8 #1 — `#[ObservedBy]` attribute unification.** Contact / Membership / Donation models now declare their observers via the modern `#[ObservedBy(...)]` PHP attribute. Three imports + three `Model::observe()` calls deleted from `AppServiceProvider::boot()`.
- **W7 #1 + W12 rows 4-6 — password-mismatch JS factory.** Three sites inlining the same ~15-LOC IIFE now share `resources/js/portal/password-mismatch.js`'s `attachPasswordMismatch()` exported function, registered globally as `window.NPPasswordMismatch`. Vite gains the file as a top-level input so the standalone admin invitation page can `@vite` it independently. Widget gains an `assets()` declaration on its Definition + `app/Widgets/PortalChangePassword/script.js` calling the global factory.

### Iteration `/6` — editor.ts composable extraction (W7 #4) (`7e24bcb`)

Final W7 row from the 271+272 audit. Four cohesive composables under `resources/js/page-builder-vue/composables/`:

- **`useDebouncedSave({ updateWidget, afterSave })`** — owns `pendingConfigChanges` + the `debounceSaveTimer` closure variable. Returns `{ flushPendingSaves, flushDebouncedSave }`. The post-save preview-refresh decision (the existing `payloadAffectsPreview` check) is now a callback the store provides at construction time rather than an inline branch. (52 LOC.)
- **`useRefreshPreview({ widgets, dirtyWidgets, requireApi, flushPendingSaves })`** — owns the per-widget refresh-counter, abort-controller Map, blur/spinner timer Maps, indicator-stage ref, and preview-error ref. Returns `{ refreshPreview, widgetRefreshing, widgetIndicatorStage, widgetPreviewError }`. The cascading indicator timers (250ms blur → 500ms spinner) and AbortController cancellation semantics from the original `refreshPreview()` are preserved exactly. (133 LOC.)
- **`useUploadActions({ widgets, dirtyWidgets, saving, requireApi, refreshPreview })`** — four image upload/remove fns (uploadImage, removeImage, uploadAppearanceImage, removeAppearanceImage). Each flips `saving`, hits the API client, mutates the local widget config / image_urls / appearance_image_url, marks dirty, fires a preview refresh. (82 LOC.)
- **`usePresetActions({ widgets, dirtyWidgets, widgetTypes, requireApi, flushPendingSaves, flushDebouncedSave })`** — saveDraftPreset, renameDraftPreset, deleteDraftPreset, applyPreset. (80 LOC.)

**Circular wiring.** `useDebouncedSave`'s `afterSave` callback fires `refreshPreview`, and `useRefreshPreview` awaits `flushPendingSaves`. The store wires both via two forward-declared `let` variables (`refreshPreviewRef` / `flushPendingSavesRef`) that are assigned the moment each composable returns; the closures don't fire until user interaction, well after both sides are wired.

`editor.ts` shrinks **930 → 700 LOC**. New composables total 347 LOC. Net surface across 5 files vs prior single-file shape is +117 LOC due to per-file boilerplate (imports, dep interface declarations) — accepted cost for the per-composable cohesion gain.

### Bonus commit — `.btn` anchor specificity bugfix (`07bd6d4`)

Three-line addition in `resources/scss/_base.scss`: `&.btn { color: var(--btn-color, inherit); }` scoped inside the anchor selector. Anchor tags styled as buttons (`<a class="btn">`) were inheriting the parent anchor's `color` declaration from the section above, overriding `--btn-color` on every link-shaped button across the public site (visible symptom: blue-on-blue link blue text on the button's intended background). Pre-existing bug surfaced during /1-/5 deploy testing; user direction was to commit it separately at /6 close.
