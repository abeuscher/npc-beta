# 144. Column Widget — Drag-and-Drop Reorder

Complete the column widget by solving its core UX gap: drag-and-drop in the handles view. The column widget is fully functional — data model (`parent_widget_id`, `column_index` on `page_widgets`), widget type, child rendering, inspector controls, and public rendering all exist from session 086. The missing piece is the ability to drag widgets into and out of column slots in the handles view — currently, child widgets only support move-up/move-down buttons, which makes column layout management clunky.

---

## Design decisions (resolved before starting)

- The column widget is a special widget type that renders as a horizontal row of connected drop zones in the handles view, not a single card.
- Drag-and-drop uses Livewire 3's `x-sort` directive with the `group` option so the main list and each column slot are connected sortable containers.
- The column widget inspector controls: number of columns (2–4) and column width preset (e.g., 6/6, 4/8, 8/4, 4/4/4, 3/6/3, 3/3/3/3).
- Width presets use the existing 12-column CSS grid. Public rendering wraps children in a CSS grid row with appropriate `grid-column: span N` values.
- Column widgets cannot be nested inside other column widgets.
- On mobile viewports, columns stack vertically (single-column fallback).

## Phase 1 — Connected sortable groups in handles view

**Goal:** Drag widgets into and out of column widget slots in the handles view.

1. **Column widget card in handles view** — when a widget is a column type, render it differently: a horizontal row showing each column slot as a separate drop zone, each with its own `x-sort` container sharing the same `group` as the main list.
2. **Update `x-sort` on the main list** — add `group="blocks"` so items can move between the main list and column slots.
3. **Update the sort callback** — the current callback collects IDs in DOM order and calls `$wire.updateOrder(ids)`. The new callback needs to also detect which container each widget ended up in (main list vs column slot) and pass that context: `{ id, parentId, columnIndex }` for each widget.
4. **Extend `updateOrder()` on PageBuilder** — accept the richer payload and persist `parent_widget_id`, `column_index`, and `sort_order` for each widget. Widgets dragged out of a column get `parent_widget_id = null, column_index = null`. Widgets dragged into a column get the column widget's ID and the slot index.
5. **Prevent nesting** — column widgets themselves should not be draggable into column slots. Add a guard in the `x-sort` accept logic or in the Livewire handler.
