## What was built

### Phase 1 — Config requirements system

- Added `required_config` JSONB column to `widget_types` table (nullable, default null)
- Updated `WidgetType` model: added to `$fillable` and `$casts`
- Seeded `required_config` for 8 widget types: carousel, logo_garden, board_members, video_embed, map_embed, web_form, event_registration, event_description
- Corrected config key names from session prompt starting point to match actual template checks: `form_handle` (not `form_id`), `event_slug` (not `event_id`), `map_input` (not `embed_code`)
- Exposed `widget_type_required_config` in: Livewire bootstrap data (root + children), API `formatWidget`, `formatWidgetWithPreview`, `formatChildren`, and `widgetTypes` endpoint
- Added `required_config` to `WidgetType::forPicker()` output
- Updated TypeScript `Widget` interface (`widget_type_required_config`) and `WidgetType` interface (`required_config`)
- `PreviewRegion.vue` shows a styled notice (widget type label + message) when required config keys are missing
- Auto-refreshes preview when `needsConfig` transitions from true to false (user fills in required config)
- Added `.widget-preview-notice__label` and `.widget-preview-notice__message` styles to `admin.css`
- Removed all 8 `@include('widgets.components.widget-placeholder', ...)` calls from widget Blade templates
- Placeholder partial file retained but no longer referenced

### Phase 2 — Drag-and-drop reordering

- Installed `vuedraggable@next` (SortableJS wrapper for Vue 3)
- Wrapped `PreviewRegion` list in `PreviewCanvas.vue` with `<draggable>` component
- Computed setter updates `store.rootOrder` immediately on drag so drops stick visually
- `onDragEnd` builds reorder items and calls `store.reorderWidgets()` to persist
- Ghost class provides 40% opacity drag feedback (unscoped style for SortableJS compatibility)
