## Design decisions (resolved before starting)

- **Squash recipe lineage: 062 → 082 → 108 → 142 → 181 → 208 → 274.** This is the seventh squash. The procedure follows the session-181 / session-208 shape verbatim: `schema:dump` → delete all current migration files → verify `migrate:fresh --seed` identity → update `docs/schema/README.md` squash-note block. No schema changes during the squash itself — schema changes this session happen as **new** migrations written during the optimization phase, committed before the squash runs, and absorbed into the dump alongside everything else.
- **Squash cut-off is today's `database/schema/pgsql-schema.sql` state plus whatever new migrations this session commits.** No edits to historical migrations, no reordering — reordering has no effect on a squash (the dump erases sequence) and editing historical files is wasted effort when the files are about to be deleted.
- **Pre-beta license applies throughout.** No backward-compat shims, no deprecation periods. When the optimization phase identifies a column, method, route, or config file that isn't pulling its weight, cut it. The squash absorbs the cut-migration into the baseline; the old shape disappears from history. Same posture as 208.
- **Optimization phase is inventory-first.** Produce a categorised table of candidates, present to user, user picks what to apply. Mirrors the session-181 / session-208 Phase 2 pattern. Cap: if more than ~10 candidates surface, present the list and let the user narrow before applying. The rest become deferred stubs.
- **Help-docs coverage is a delta audit, not a fresh sweep.** Session 271's W9 already closed the 208-cutoff coverage gaps (B1c importer routes, Notes Resource routes, retired tax-receipts doc deletion). 274's Phase 1c walks every admin view added since 271 — primarily anything 273's apply-half work surfaced — and confirms no drift in the existing coverage.
- **Playwright teardown audit is also a delta audit.** Session 208 added `afterAll` hooks to all 14 importer specs via the `cleanupAllImportSessionsOfType` artisan-command pattern. 274's Phase 1d audits the page-builder spec set that has grown since 208 (`tests/e2e/page-builder/`) plus any other non-importer specs that may persist out-of-band rows. Use the existing pattern as the reference.
- **No agent contract change.** No `docs/fleet-manager-agent-contract.md` body edit, no version bump. The 273 W8 #3 parity test continues to gate code-vs-spec drift.
- **Iteration strategy.** All phases sequence within a single branch `session-274/1`. Inventory + user picks happen first (no code changes yet). Then each pick is its own logical commit. The squash itself is the final commit before test + manual-test announcement. If scope balloons mid-session the user may bless a `/2` carve-out — but the squash must still land in this session, not be deferred.

## Testing

- **Slow test groups to run this session:** none by default.
- **New tests expected:** yes, but scoped. Any optimization pick that changes observable behaviour needs coverage. Pure deletions of already-unused code don't need new tests. The query-count regression-guard pattern from 208 is the reference shape for any perf pick.
- **Playwright:** required — `npm run test:e2e` must pass before pausing for manual test. The `migrate:fresh --seed` in each spec's `beforeAll` validates the squash end-to-end by construction; any seed breakage will surface here.
- **Widget asset rebuild:** not expected unless a pick touches widget SCSS/CSS/JS.
- **Pest + Playwright must NOT run in parallel.** They collide on DB / PHP-FPM (Pest's `RefreshDatabase` transactions vs Playwright's globalSetup `migrate:fresh`) and can take the local site down. Run sequentially.
- **Mass-ECONNRESET in Playwright is a stack-state symptom, not a test-logic failure.** Observed at 273/5 close: a parallel-run mistake left stale PHP-FPM child / nginx upstream / Postgres connection state that survived a single `docker compose restart nginx`. Recovery ladder, in order:
  1. **`docker compose restart nginx`** — clears stale fastcgi upstreams from a brief contention burst. Fixes the simple case.
  2. **`docker compose restart`** (full stack) — clears PHP-FPM child memory and Postgres connection pool. Fixes most cases nginx alone can't.
  3. **Host reboot or Docker Desktop restart** — clears WSL2 port-forwarding state and any kernel-level socket leftovers. The case-of-last-resort that resolved 273's cascade.
  When mass-ECONNRESET appears across many specs (vs a single test's assertion failure), don't chase test logic — climb the ladder.
