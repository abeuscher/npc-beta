## What was built

### Phase 1 — Secondary colour setting

- Added `admin_secondary_color` site setting with default `#73bbbb`. Wired through:
  - `GeneralSettingsPage` — new `ColorPicker` directly below the primary picker, with seed in `mount()` and write in `save()`.
  - `AdminPanelProvider` — loaded inside the same `try/catch` as the primary, registered as the `secondary` palette key via `Color::hex()` alongside `primary`.
  - `AppServiceProvider` — shared as `site.admin_secondary_color` for any Blade that needs the raw hex.
- No migration — site settings are JSON-keyed.

### Phase 2 — Header-action audit and swaps

- Relocated `view_transactions` into the ellipsis overflow group on `EditContact` (both `getReadOnlyHeaderActions()` and `getHeaderActions()`) and on `EditProduct`. Permission gate, label, and URL preserved.
- Audited every `->color('gray')` under `app/Filament/`. Swapped to `->color('secondary')` on header-action affordances; left as `gray` on table rows, ellipsis filter groups, form-internal actions, and badges.

  **Swapped → `secondary`:**
  - `EditContact.php` — `notes` (Timeline) in both header methods.
  - `ContactNotes.php` — `back_to_contact`.
  - `UserResource/EditUser.php` — `Resend Invitation`.
  - `FormResource/Pages/EditForm.php` — `download_json`.
  - `FormResource/Pages/ListForms.php` — `import_json`.
  - `EventResource/Pages/EditEvent.php` — both `editLandingPage` actions (read-only View, editable Edit).
  - `MailingListResource/Pages/EditMailingList.php` — `Export CSV` and `Sync to MailChimp`.
  - `DonorsPage.php` — `Send System Emails to Pending Recipients`.

  **Kept as `gray`:**
  - `ContactNotes.php` — Source / Type filter ActionGroup labels.
  - `ListContacts.php`, `ListEvents.php`, `EditEvent.php` — ellipsis-vertical groups.
  - All table row Actions (DonationResource, TransactionResource, FormResource, ProductResource, MembershipTierResource, FundResource, ViewRegistrations, ImporterPage, TransactionsRelationManager).
  - Form-internal actions (EmailPreviewWizardAction, CmsSettingsPage, FinanceSettingsPage).
  - `MediaLibraryPage` — badge color, not a button.

### Phase 3 — Admin form style tweaks (added during the session)

Phase 3 was deliberately discretionary in the prompt; the user added items inline during the session driven by designer feedback. All work is in `public/css/admin.css`:

- Section header bar tinted with `--np-control-bar-bg`. Targets `.fi-section-header` (the full bar, not just the heading text), with rounded top corners matching the section's `rounded-xl`.
- Form field input wrapper, fieldset, and repeater item ring color now reads from the existing `--np-control-border` token instead of a hardcoded gray-400. User adjusted the token's value during the session.
- New `--np-form-label` variable controls the colour of all `.fi-fo-field-wrp-label` text. Light/dark values defined in `:root` and `.dark` blocks.
- New `--np-form-input-size` variable controls the font-size of input/textarea/select values inside `.fi-input-wrp`. Single value in `:root` (no dark variant needed).
- Quill rich-editor explicitly excluded from the input-size rule by selector (the `input/textarea/select` element-type filter and the absence of `.fi-input-wrp` ancestor on QuillEditor's wrapper).

Playwright was used in-session to confirm the section-header DOM structure before settling on the correct selector. Inspection script was created at the project root, used, and removed.

- **Form-input border tightening from gray-400 to gray-300.** The Filament `fi-section` / `fi-input-wrp` ring overrides shifted to `--np-control-border` (gray-300). Section card outlines (`.fi-section`) and table outlines (`.fi-ta-*`) still use their own gray-400 — unifying those onto the same token is a one-line follow-up if desired, deferred to session 200.

Whether these collect under a new `--np-form-*` namespace or stay mingled is a naming question that pairs naturally with the session 200 tightening pass.
