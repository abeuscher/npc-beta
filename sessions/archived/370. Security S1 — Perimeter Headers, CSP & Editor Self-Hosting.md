## Session-specific deltas

- **CSP is the hard part; the other headers are near-mechanical.** HSTS / `X-Content-Type-Options` / `Referrer-Policy` / frame protection have obvious correct values for this app (single-origin, no legitimate framing, HTTPS-only in prod — mind HSTS on the plain-HTTP dev stack). Budget the session's judgment for the CSP.
- **CSP reality constraints known going in:** the renderer emits inline `style` attributes pervasively (widget appearance composition — `AppearanceStyleComposer` inline styles, `<img style>` allowed by the sanitizer), so `style-src` almost certainly needs `'unsafe-inline'` at v1 — that is an accepted, documented trade-off, not a failure; the win to fight for is **`script-src` without `'unsafe-inline'`** (nonce or hash any legitimate inline scripts; the public bundle and admin bundles are same-origin files). Alpine/Livewire/Filament CSP compatibility on the admin side is the risk pocket — if the admin panel can't reach enforcement inside this session, ship **public-surface enforcement + admin report-only** and record the residual as a tracked item for the Gate-3 findings register, rather than weakening the public policy.
- **Two delivery layers, deliberately redundant:** app middleware (so headers exist regardless of proxy config) + nginx (so they exist even for responses that short-circuit PHP). Dedupe rule decided in-session; parity across `prod.conf` and `default.conf` where the dev stack allows.
- **Editor self-hosting:** every admin page loads Quill from `cdn.jsdelivr.net` via `AdminPanelProvider` render hooks. Vendor the exact pinned version locally (npm-managed if clean, static-vendored if not) and point the render hooks at the local copy. This is both the supply-chain fix and the CSP unblock — after it, no third-party origin should remain in the admin `script-src`.
- **Enforced defaults, not conventions:** `SESSION_SECURE_COOKIE` gets a production-safe default that the local HTTP dev stack and the e2e stack can still override; `.env.example`'s `APP_DEBUG=true` posture gets a guard so a copied example file can't ship debug to prod (shape decided in-session — config default, deploy-time check, or both).
- **Folded guard 1 — Stripe test-mode on the Random Data Generator:** key self-identifies by prefix (`sk_live_` / `sk_test_`). Default lean per the stub: **hard refuse-to-render** when the install is configured against a live key. Confirm at the Open Gate.
- **Folded guard 2 — demo-role seeding:** wrap `PermissionSeeder`'s `demo`-role block in `isDemoMode()`. Mind the demo server's own seeder re-run path and the tests referencing the role (`DemoRoleLockdownTest`, the demo-dashboard seeder item) — the role must keep existing where demo mode is on.
- **Do not touch:** the five FM `/api/*` mTLS location blocks (S2's territory — verify byte-untouched at close), `HtmlSanitizer` and write paths (S3), portal/permission surfaces (S4/S5).
