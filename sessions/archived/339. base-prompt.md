## Process rules for every session

Base-prompt process rules apply in full (`sessions/template-base-prompt.md`). Session-specific notes:

- **Work local; the owner pushes the export.** The deliverable is the local CMS/theme state, exported by the owner. **No seeder changes** (edit live/imported state, never `DesignSettingsSeeder`). **No code changes if avoidable** — if a change needs code, flag it at the gate; it breaks the export-only round-trip.
- **Replace, don't add.** The new colours replace the token values they correspond to in the comp (brand, background, the dark surface), not new tokens beside them. Sample the hexes off the comp and **surface them for sign-off before applying.**
- **Site-wide vs page-level.** The brand-accent + background colours are **theme tokens** (site-wide, ripple everywhere). The home page's per-section backgrounds, 80px spacing, copy, and layout are **page-builder appearance** edits on the imported page.
- **Build for colours.** A theme-colour change requires `docker compose exec app php artisan build:public` to recompile `--np-color-*` into the public bundle. Verify the rendered site, not just the setting.
- **Demo is not this.** Nothing here touches the demo node or demo pages.
