## What was built

### Multi-stage Dockerfile
Converted the single-stage `Dockerfile` to a two-stage build:
- **Stage 1 (`app`)**: PHP 8.4-FPM with all system deps, PHP extensions, Composer install. Code baked in — no bind mounts in production.
- **Stage 2 (`web`)**: `nginx:alpine` with the nginx config and `public/` directory copied from the app stage. Static assets are baked into the image.

A node-builder stage was attempted but removed: `vite.config.js` had `input: []` so Vite refused to build. `vite.config.js`, `package.json`, and `package-lock.json` were removed from the repo as they serve no purpose.

### docker-compose.prod.yml
New production compose file:
- References GHCR images (`ghcr.io/${GHCR_OWNER}/nonprofitcrm-app` and `nonprofitcrm-web`) at `${IMAGE_TAG}`
- No bind volume mounts for code
- Named volumes for `storage/`, `bootstrap/cache/`, `postgres_data/`, `redis_data/`
- PostgreSQL and Redis ports not exposed to the host
- `env_file: .env` on all services

### GitHub Actions deploy workflow
`.github/workflows/deploy.yml`:
- Triggers on push to `main`
- Builds and pushes both images to GHCR (tagged with 7-char git SHA and `latest`)
- SSHes into the droplet using `appleboy/ssh-action`
- On the server: pulls images, `docker compose up -d`, `migrate --force`, `storage:link`, `docker image prune`

### Droplet setup documentation
`docs/deployment/droplet-setup.md` — full one-time setup guide covering droplet creation, deploy user, SSH keypair, GitHub Secrets, file placement, and `.env` configuration.

### First successful deploy
App is live and accessible at the DigitalOcean droplet IP. Database seeded, admin user created.

## Issues encountered and resolved

| Issue | Resolution |
|-------|-----------|
| `npm ci` failed in CI | Node stage removed entirely — no Vite entrypoints in project |
| `ufw` rate-limiting blocked GitHub Actions SSH | Changed port 22 rule from `LIMIT` to `ALLOW` |
| Wrong droplet IP in GitHub Secrets | Corrected by user |
| `.env` permission denied on server | `chown deploy:deploy` on `.env` and `docker-compose.prod.yml` |
| `GHCR_OWNER` placeholder not replaced | Set `GHCR_OWNER=abeuscher` in server `.env` |
| Migrations not run before seed | Ran `php artisan migrate --force` first |
