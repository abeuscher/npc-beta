# 108. Beta-One Bug Fixes & Migration Squash

A focused bug-fix and polish session addressing QA issues collected during beta-one testing, followed by a migration squash to clean up the schema history before the beta milestone.

## Phase 1 — Bug fixes

Work through all eight bugs, then run the test suite and announce ready for testing.

### Bug 1 — Header/logo not showing in theme

The site header is not rendering in the default theme. There is no way to add a logo to the front-page header. The header and footer must be present and controllable in the default theme. If new widget types are needed to support this, create them.

### Bug 2 — Blog post publish dates

Blog posts with a future publish date that are in `published` status should not be visible on the public site until the publish date has passed. Update the relevant query scopes and/or controller logic.

### Bug 3 — Unpublished page links from admin

When an admin clicks the "View page" link on an edit record screen and the page is unpublished (draft), the app should show an error or notice and keep the user on the edit screen rather than navigating to a 404 or blank page.

### Bug 4 — Convert forms to Tailwind

Public-facing form widgets are still styled with Pico CSS. Convert them to Tailwind, matching the styling conventions used in the event registration and donation form widgets.

### Bug 5 — Event date UX

When creating a new event, selecting a start date should auto-set the end date to one hour later. This is a convenience default — the user can still change it.

### Bug 6 — CMS Settings breakpoints layout

The breakpoints section in CMS Settings takes up too much vertical space. Tile the breakpoint fields horizontally, up to 5 columns across.

### Bug 7 — Horizon endpoint

The Horizon dashboard endpoint should be closed off by default. Make it toggleable via an admin setting, defaulting to not available. When disabled, the route should return 403.

### Bug 8 — Widget inspector auto-open

When adding a new widget to a page via the page builder, the widget should automatically open in the inspector (settings panel) so the user doesn't have to click it after adding.
