## Design decisions (resolved before starting)

- **Color picker:** Each color field renders a freeform colour wheel (HTML `<input type="color">`) alongside the hex text input (matching the existing Blade partial). Below the picker, show a row of **theme palette presets** — the template's `primary_color` (from `--color-primary` CSS variable) and any future design tokens. Users can click a preset to apply it. Additionally, support **saved swatches**: when a user picks a colour, they can click a "+" button to save it as a swatch. Swatches are stored as a JSON array on the `site_settings` table (key: `editor_color_swatches`). The swatch list is loaded once from bootstrap data and shared across all color fields. Swatches can be deleted (×) from the list.
- **Universal appearance controls:** Every widget gets three controls at the top of the Appearance tab (before any widget-specific appearance fields): full-width toggle, background colour, and text colour. These are stored on `style_config` (keys: `full_width`, `background_color`, `text_color`) — NOT on `config`. The inspector reads/writes these via the same debounced `updateWidget` API, using `style_config` instead of `config`. Widgets that already define `background_color` or `text_color` in their `config_schema` as `advanced` fields (events_listing, blog_listing) should have those per-widget fields removed from the seeder — the universal controls replace them.
- **Image/video upload:** Uses the existing `POST /admin/api/page-builder/widgets/{id}/image` and `DELETE /admin/api/page-builder/widgets/{id}/image/{key}` endpoints already implemented in session 147. The component shows a thumbnail preview, file input, and remove button.
- **Button list manager:** Port the Alpine `buttonListManager` to a Vue component. Array of `{text, url, style}` objects. Add, remove, move up/down. Style select uses the button variant options from `field.style_options` (default: primary, secondary, text). Changes persist through the same debounced config save.
- **Spacing controls:** Port the Alpine `spacingControls` to a Vue component. Padding and margin each have an "All" field plus four individual fields (left, top, right, bottom). "All" is a computed getter/setter: shows the value when all four are equal, shows "mixed" placeholder when they differ, and sets all four when typed into. Spacing changes persist to `style_config` via debounced API save.
- **Query settings:** Port the Livewire query settings panel to Vue. Renders for widgets whose `widget_type_collections` array is non-empty. Each collection handle gets: limit (number), direction (asc/desc), order by (sort_order, created_at, updated_at, published_at), include tags (checkbox list), exclude tags (checkbox list). Tags come from the store's `tags` array (loaded from bootstrap data). Changes persist to `query_config` via debounced API save.
- **Store changes for style_config and query_config:** The `updateLocalConfig` action in the store currently only handles `config` and `label`. Extend it (or add parallel actions) to support `style_config` and `query_config` updates with the same debounced save pattern.

## Phase 1 — Color picker with swatches

### 1a. Bootstrap data for swatches

Add `editor_color_swatches` to the bootstrap data in `app/Livewire/PageBuilder.php`:
- Read from `SiteSetting::get('editor_color_swatches', '[]')` and JSON-decode
- Add to the `$bootstrapData` array
- Add `color_swatches: string[]` to the `BootstrapData` TypeScript interface
- Store in Pinia: `colorSwatches` ref, loaded from bootstrap data

### 1b. API endpoint for saving swatches

Add a `PUT /admin/api/page-builder/color-swatches` endpoint:
- Request: `{ swatches: string[] }` — the full swatch array
- Saves to `SiteSetting::set('editor_color_swatches', json_encode($swatches))`
- Response: `{ swatches: string[] }`
- Requires `update_page` permission

### 1c. `ColorPickerField.vue`

Create `resources/js/page-builder-vue/components/fields/ColorPickerField.vue`:

- Props: `field`, `modelValue`
- Renders `<input type="color">` + hex text input (matching existing Blade layout)
- Below the inputs: a row of preset swatches (theme primary colour from CSS variable, plus saved swatches from the store)
- Click a swatch to apply it
- "+" button to save the current colour as a new swatch (calls store action, which calls API)
- "×" on each saved swatch to remove it (calls store action, which calls API)
- Emits `update:modelValue` on change

### 1d. Update `InspectorField.vue`

Replace the color fallback message with the `ColorPickerField` component.

## Phase 2 — Image/video upload and button list

### 2a. `ImageUploadField.vue`

Create `resources/js/page-builder-vue/components/fields/ImageUploadField.vue`:

- Props: `field`, `widget`, `modelValue`
- For `image` type: accepts image/png, image/jpeg, image/gif, image/webp, image/svg+xml
- For `video` type: accepts video/mp4, video/webm
- Shows thumbnail preview (image or video element) when a file exists
- File input for uploading
- Remove button that calls `store.removeImage(widgetId, key)`
- Upload calls `store.uploadImage(widgetId, key, file)`
- Loading indicator during upload
- Emits `update:modelValue` is NOT used — uploads go directly through the store's image actions

### 2b. `ButtonListField.vue`

Create `resources/js/page-builder-vue/components/fields/ButtonListField.vue`:

- Props: `field`, `widget`, `modelValue` (array of `{text, url, style}`)
- Emits: `update:modelValue`
- Renders each button as a row with: text input, URL input, style select, move up/down buttons, remove button
- "Add Button" button appends `{text: '', url: '', style: 'primary'}`
- Style options from `field.style_options` (default: `{primary: 'Primary', secondary: 'Secondary', text: 'Text Only'}`)
- Move up/down reorders the array
- All changes emit `update:modelValue` with the full updated array

### 2c. Update `InspectorField.vue`

Replace the image, video, and buttons fallback messages with the real components.

## Phase 3 — Spacing controls and query settings

### 3a. `SpacingControl.vue`

Create `resources/js/page-builder-vue/components/SpacingControl.vue`:

- Props: `styleConfig` (the widget's current `style_config` object), `widgetId`
- Collapsible panel (matching existing Blade accordion pattern)
- Full-width checkbox bound to `style_config.full_width`
- Padding section: "All" field + left/top/right/bottom. "All" getter shows the value when all four match, empty when they differ (placeholder: "mixed"). "All" setter sets all four.
- Margin section: same pattern as padding
- All changes call a store action that updates `style_config` via debounced save

### 3b. `QuerySettings.vue`

Create `resources/js/page-builder-vue/components/QuerySettings.vue`:

- Props: `widget` (the selected widget)
- Only renders if `widget.widget_type_collections` is non-empty
- Collapsible panel (matching existing Blade accordion pattern)
- For each collection handle in `widget.widget_type_collections`:
  - Limit (number input)
  - Direction (asc/desc select)
  - Order By (sort_order, created_at, updated_at, published_at select)
  - Include Tags (checkbox list from `store.tags`)
  - Exclude Tags (checkbox list from `store.tags`)
- All changes call a store action that updates `query_config` via debounced save

### 3c. Update `InspectorPanel.vue`

Add `SpacingControl` and `QuerySettings` to the Appearance tab, below the field group (matching their position in the Livewire inspector).

## Phase 4 — Universal appearance controls

### 4a. `WidgetAppearanceControls.vue`

- Props: `widget`
- Renders at the top of the Appearance tab, before widget-specific appearance fields

### 4c. Remove redundant per-widget color fields

- Add `colorSwatches` ref and `saveColorSwatches(swatches)` action
