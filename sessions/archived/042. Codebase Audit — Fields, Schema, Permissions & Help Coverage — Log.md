## Workstream 1 — Edit Page Field Audit

### Columns dropped via migration

**Migration `2026_03_20_100001`** — dropped from `contacts`:
- `notes` — superseded by the morphMany `Note` relationship (NotesRelationManager)
- `email_secondary` — removed from form and not reinstated
- `phone_secondary` — removed from form and not reinstated

**Migration `2026_03_20_100002`** — dropped:
- `contacts.preferred_name` — not exposed in any UI; no current use case
- `contacts.is_deceased` — not exposed in any UI; removed filter from ContactResource
- `donations.organization_id` — not in edit form; donation attribution is contact-only for now
- `tags.color` — not exposed in any UI

### Model `$fillable` updates
- `Contact` — removed `email_secondary`, `phone_secondary`, `notes`, `preferred_name`, `is_deceased`; added system-managed comments for `household_id`, `custom_data`, `source`, `import_session_id`
- `Donation` — removed `organization_id`; removed `organization()` relationship
- `Tag` — removed `color`
- `EmailTemplate` — added system-managed comment for `handle`
- `Event` — added system-managed comment for `landing_page_id`
- `Role` — expanded `$fillable` inline with system-managed comment for `guard_name`

### Resource form updates
- `ContactResource` — removed `preferred_name` field, `is_deceased` toggle, `is_deceased` table filter; removed `notes` column from CSV export
- `DonationResource` — no change (organization_id removed from both form and model)
- `TagResource` — removed `color` ColorPicker

### Deliberate omissions documented
- `Contact.household_id` — managed from HouseholdResource MembersRelationManager
- `Contact.custom_data` — system-managed SchemalessAttributes written by importer
- `Contact.import_session_id` — set by importer
- `Contact.source` — set at creation; shown read-only via Placeholder in form
- `EmailTemplate.handle` — template identifier set by seeder
- `Event.landing_page_id` — set by EventObserver
- `Role.guard_name` — always 'web'; set by Spatie base class
- `Organization.notes` — morphMany Notes relationship is the correct editing surface; column retained pending a separate decision
