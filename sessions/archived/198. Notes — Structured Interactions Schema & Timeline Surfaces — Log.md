## What was built

### Phase 1 ‚Äî Migration, model, factory, schema doc

- New migration `2026_04_19_120000_add_structured_fields_to_notes.php` adds six columns to `notes`:
  - `type` string, not-null, default `'note'`
  - `subject` string, nullable
  - `status` string, not-null, default `'completed'`
  - `follow_up_at` timestamp, nullable
  - `outcome` text, nullable
  - `duration_minutes` integer, nullable
  - `meta` jsonb, nullable
- Three new indexes: `notes_type_index` on `(type)`; `notes_notable_occurred_at_index` on `(notable_type, notable_id, occurred_at DESC)` for Timeline reads; `notes_follow_up_at_index` partial on `(follow_up_at) WHERE follow_up_at IS NOT NULL` for the future follow-up dashboard.
- `app/Models/Note.php` ‚Äî six new keys added to `$fillable`; casts: `follow_up_at ‚Üí datetime`, `duration_minutes ‚Üí int`, `meta ‚Üí array`.
- `database/factories/NoteFactory.php` ‚Äî regenerated with structured defaults. Type randomised over the canonical set; status defaults to `completed`; `follow_up_at` populated ~20% of the time; `outcome` + `duration_minutes` only for `call` / `meeting` types; `meta` always null.
- `docs/schema/notes.md` ‚Äî updated to document all columns and every index, and to note that `body` now stores rich HTML.

### Phase 2 ‚Äî Form surfaces

- `App\Filament\Resources\NoteResource` gained two public constants (`TYPE_OPTIONS`, `STATUS_OPTIONS` ‚Äî the canonical vocabularies) and a `coreFormSchema()` static helper returning the reusable field stack.
- Four surfaces now consume `coreFormSchema()`:
  - `NoteResource::form()` (prepends an Attachment section with `notable_type` / `notable_id` / `author_id`).
  - `ContactResource/RelationManagers/NotesRelationManager.php` (adds hidden `author_id`).
  - `OrganizationResource/RelationManagers/NotesRelationManager.php` (adds hidden `author_id`).
  - `ContactNotes::getHeaderActions()` `create_note` modal and `editNoteAction` modal.
- Final form layout after user feedback (replaced the initial two-section design):
  - Single `Section` with `->columns(12)`.
  - Row 1: `Type` (`Select` with the 7 canonical options, default `note`, `native(false)`, `selectablePlaceholder(false)`) spanning 6 ¬∑ `Occurred At` (DateTimePicker) spanning 6.
  - Row 2: `Title` (TextInput, formerly labelled "Subject") full-width 12.
  - Row 3: `Note` (QuillEditor ‚Äî rich HTML) full-width 12.
  - Row 4: `Status` (TextInput, default `completed`) 4 ¬∑ `Next action` (DateTimePicker) 4 ¬∑ `Duration` (TextInput numeric, `min` suffix) 4.
  - Row 5: `Result / outcome` (Textarea, 3 rows) full-width 12.
- Modals bumped to `modalWidth('2xl')` to accommodate the 12-col grid.
- `meta` intentionally omitted from the form ‚Äî it is write-only from the importer path.

### Phase 3 ‚Äî Timeline rendering

- `ContactNotes::getTimeline()` projection extended to include `type`, `subject`, `status`, `body` (HTML), `outcome`, `duration_minutes`, `follow_up_at`, and `meta`.
- `ContactNotes` added `public string $typeFilter = 'all'`, threaded it through the notes query, and added `getNonCanonicalTypes()` to expose imported non-canonical type strings to the filter menu.
- `getHeaderActions()` now renders two ActionGroups: the existing `Source` filter (labelled, with a funnel icon) and a new `Type` filter (tag icon) seeded from `TYPE_OPTIONS` plus any non-canonical types present. Filament 3 `ActionGroup` has no `disabled()` method, so the Type filter is hidden entirely when the Source filter is `activity` ‚Äî equivalent intent (the filter has no target), cleaner plumbing than faking a disabled state.
- `resources/views/filament/pages/contact-notes.blade.php` rebuilt from scratch:
  - Two treatments in one stream: full note cards vs compact activity one-liners.
  - Card header: subject (or stripped-body preview, truncated to 60 chars) as semibold title, right-aligned type-icon badge (with `¬∑ N min` suffix for call/meeting when `duration_minutes` is set), inline status badge when status is not `completed`, relative timestamp, inline Edit / Delete buttons.
  - Body rendered as rich HTML (`{!! $item->body !!}`) with added paragraph/list spacing in the embedded stylesheet. Title-fallback strips tags before truncation.
  - Outcome rendered italic-muted below the body.
  - Follow-up pill: future + status‚âÝ`completed` ‚Üí neutral `Follow up: {relative}`; past + status‚âÝ`completed` ‚Üí amber `Overdue: {relative}` (with dark-mode variant); past + `completed` ‚Üí suppressed.
  - Meta `<details>` disclosure titled `Source fields`, rendered as a `<dl>` key/value list. Scalar values rendered escaped; non-scalars JSON-encoded then escaped.
  - Activity rows: one-liner with a cog icon, event label, actor label, optional description, and relative timestamp on the right.
  - All chrome tokenised against session-197's `--np-control-*` custom properties.
- Relative timestamp helper inlined in the Blade (`today`, `Yesterday`, weekday name, `M j`, `M j, Y` tiers).

### Phase 4 ‚Äî Organization parity

- Organization's `NotesRelationManager` now consumes `coreFormSchema()` ‚Äî the form is at parity with Contact.
- **Gap flagged:** Organization has no equivalent to `ContactNotes` (no Timeline page on the Organization edit record). Not in scope for session 198 per the prompt. Natural follow-up if Organization becomes a first-class interaction parent. The `organizations.notes` legacy text column remains as a Notes section in the main Organization form ‚Äî its naming collision with the polymorphic `notes()` relationship was observed but left alone.

### Phase 5 ‚Äî Tests

New Pest suite at `tests/Feature/NotesStructuredFieldsTest.php`, 12 tests, 74 assertions:

- Schema ‚Äî all six new columns present on the `notes` table.
- Defaults ‚Äî bare insert lands `type='note'`, `status='completed'`.
- Casts ‚Äî `meta` round-trips as array, `duration_minutes` coerces string to int.
- Create persistence ‚Äî ContactNotes `create_note` path persists `type`, `subject`, `status`, `outcome`, `duration_minutes` for a representative call note.
- Task persistence ‚Äî `type='task'` + `status='scheduled'` + `follow_up_at` round-trips.
- Meta persistence ‚Äî factory-created note with `meta = ['civicrm_activity_id' => ..., 'participants' => [...]]` round-trips.
- Factory variety ‚Äî 10 factory records produced, all have a type in the canonical set and a string body.
- Timeline projection ‚Äî a call note with `subject` / `outcome` / `duration_minutes` / `meta` surfaces all fields on the projected timeline item.
- Timeline type filter ‚Äî setting `typeFilter='call'` scopes results to call notes only.
- Timeline filter interaction ‚Äî `filter='activity'` + `typeFilter='call'` returns only activity items (type filter is a no-op in activity-only mode).
- Non-canonical type surfacing ‚Äî `getNonCanonicalTypes()` returns strings present in the DB but not in the canonical set.
- Backwards-compat rendering ‚Äî a bare pre-structured note renders with the expected default `type`, `status`, and nullable field values.
