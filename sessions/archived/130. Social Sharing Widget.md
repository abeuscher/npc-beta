# 130. Social Sharing Widget

Thin, inline row of share icons using pure anchor tags and platform share URLs — no third-party scripts, no tracking. Fediverse and non-corporate channels are ordered first.

---

## Design decisions (resolved before starting)

- **Platform order (left to right):** Bluesky, Mastodon, Email, Copy Link, LinkedIn, Facebook. Fediverse and open channels lead; corporate platforms trail.
- **Mastodon sharing** uses the web intent pattern (`https://mastodon.social/share?text=...`). The admin can set a preferred instance domain (default: `mastodon.social`). The share link opens on that instance — the user can still redirect to their own.
- **Bluesky** uses `https://bsky.app/intent/compose?text=...`.
- **Email** uses a `mailto:?subject=...&body=...` link.
- **Copy Link** is a small Alpine.js click handler that copies `window.location.href` and shows a brief "Copied!" tooltip. No external JS.
- **LinkedIn** uses `https://www.linkedin.com/sharing/share-offsite/?url=...`.
- **Facebook** uses `https://www.facebook.com/sharer/sharer.php?u=...`.
- **Icons:** inline SVGs provided by the user at session start, placed in `resources/views/widgets/partials/share-icons/`. Monochrome, inheriting text color via `currentColor`. Each wrapped in an `<a>` with a `title` attribute for tooltip.
- **Full width toggle:** controls only the background — the background extends edge-to-edge when enabled, but the icon row stays within the `site-container` content bounds. This is the same pattern as the hero widget's full-width mode: the widget wrapper goes full-bleed while inner content is constrained.
- **No collections, no heavy JS** — just SCSS + Alpine for copy-link.

## Phase 1 — Widget type registration

Add a `social_sharing` entry to `WidgetTypeSeeder` with:

- **handle:** `social_sharing`
- **label:** Social Sharing
- **description:** Row of share buttons for social platforms, email, and link copying. No third-party scripts.
- **category:** `['content']`
- **collections:** `[]`
- **config_schema:**
  - `heading` — text (optional heading above the row)
  - `platforms` — checkboxes displayed 3 per row, options: `bluesky`, `mastodon`, `email`, `copy_link`, `linkedin`, `facebook`. All selected by default. Store as a JSON array of enabled platform keys.
  - `alignment` — select: `left`, `center`, `right` (default: `center`)
  - `icon_size` — select: `small` (20px), `medium` (28px) (default: `small`)
  - `background_color` — color, default empty (transparent)
  - `text_color` — color, default empty (inherit)
  - `full_width` — toggle, default: false (background only — content stays in `site-container`)
  - `mastodon_instance` — text, label: "Mastodon instance domain", default: `mastodon.social`, advanced
- **assets:** SCSS at `resources/scss/widgets/_social-sharing.scss`

### Phase 1a — Add `checkboxes` config field type to the inspector

Add a new inspector field type `checkboxes` that renders a grid of checkboxes. Stores an array of selected values in config.

**Field definition example:**

```php
['key' => 'platforms', 'type' => 'checkboxes', 'label' => 'Platforms', 'columns' => 3, 'default' => ['bluesky', 'mastodon', 'email', 'copy_link', 'linkedin', 'facebook'], 'options' => [
    'bluesky'   => 'Bluesky',
    'mastodon'  => 'Mastodon',
    'email'     => 'Email',
    'copy_link' => 'Copy Link',
    'linkedin'  => 'LinkedIn',
    'facebook'  => 'Facebook',
]]
```

**Implementation:**

- Create `resources/views/livewire/partials/inspector-fields/checkboxes.blade.php`.
- Render a grid of checkboxes using CSS `grid-template-columns` driven by the `columns` attribute (default: 2).
- Each checkbox toggles its value in/out of the config array. Use `wire:model.live` bound to `block.config.{key}`.
- Verify that `getDefaultConfig()` in `WidgetType` returns the `default` array for `checkboxes` type fields.

## Phase 2 — Blade template and SCSS

### Blade template (`resources/views/widgets/social-sharing.blade.php`)

- Extract config values at top.
- If `platforms` is empty, render nothing.
- Build share URLs using the current page URL (`request()->url()`) and page title (`$config['heading']` or a generic fallback).
- Render each enabled platform as an `<a>` (or `<button>` for copy-link) containing the inline SVG partial.
- Each icon link gets `title="Share on {Platform}"` (or "Copy link" / "Share via email").
- Copy Link uses Alpine.js `x-data` with a `copied` state — clicking copies `window.location.href` via `navigator.clipboard.writeText()` and shows "Copied!" for 2 seconds.
- All external share links open in a new tab (`target="_blank" rel="noopener"`).
- Full-width background: when `full_width` is true, the widget wrapper gets the full-bleed class but the icon row is wrapped in a `<div class="site-container">` to constrain content.

### SCSS (`resources/scss/widgets/_social-sharing.scss`)

- `.widget-social-sharing`: full container, supports background-color via inline style
- `.social-sharing__row`: flexbox row, gap, alignment via modifier classes (`--left`, `--center`, `--right`)
- `.social-sharing__link`: inline-flex, icon sizing via CSS custom property `--icon-size`, color inherits from text_color or parent, hover opacity transition
- `.social-sharing__copied`: small "Copied!" indicator, absolutely positioned, fades in/out
