## What was built

### Phase 1 — Stripe Payment Method Configuration

- Added "Payment Methods" section to Finance Settings with a checkbox list of Stripe-supported types: card, ACH Direct Debit, Link, Cash App Pay, Amazon Pay.
- Card is always enabled (disabled checkbox, cannot be unchecked).
- Stored as JSON site setting `stripe_payment_method_types` (type: `json`).
- DonationCheckoutController and ProductCheckoutController now read the setting and pass `payment_method_types` to Stripe Checkout Sessions.
- For recurring donations (subscription mode), non-subscription-compatible methods (Cash App, Amazon Pay) are automatically filtered out.
- Updated help doc with payment methods section explaining available methods and recurring donation behaviour.

### Phase 2 — QuickBooks OAuth Connection

- Installed `quickbooks/v3-php-sdk` (Intuit's official PHP SDK).
- Created `QuickBooksAuth` service with: authorization URL generation (with CSRF state), code exchange, token refresh, disconnect, connection status check.
- All QB tokens stored as encrypted site settings (`qb_access_token`, `qb_refresh_token`, `qb_realm_id`, `qb_token_expires_at`).
- QB Client ID and Client Secret stored via the same encrypted Set Key / Change Key UI pattern as Stripe keys (not in `.env`).
- Created `QuickBooksCallbackController` with connect, callback, and disconnect routes — all gated behind auth + `manage_financial_settings` / `super_admin`.
- OAuth callback validates the `state` parameter to prevent CSRF.
- Connect/disconnect actions logged in the activity log.
- Finance Settings UI shows: credential entry sections, connect button (when credentials are set), connected state with Realm ID and token expiry (when connected), disconnect button with confirmation.
- Redirect URI derived from `url()` — no `.env` variable needed.
- Updated help doc with QuickBooks credential setup, connect/disconnect flow, and status explanations.
- Wrote session 103 base prompt and session prompt for QuickBooks Transaction Sync.

## Schema changes

No migrations. New site setting keys created on-the-fly:

- `stripe_payment_method_types` (type: json) — array of enabled payment method type strings
- `qb_client_id` (type: encrypted) — QuickBooks OAuth Client ID
- `qb_client_secret` (type: encrypted) — QuickBooks OAuth Client Secret
- `qb_access_token` (type: encrypted) — QuickBooks access token
- `qb_refresh_token` (type: encrypted) — QuickBooks refresh token
- `qb_realm_id` (type: encrypted) — QuickBooks company ID
- `qb_token_expires_at` (type: encrypted) — token expiry timestamp

Updated `docs/schema.md` to document all new keys.
