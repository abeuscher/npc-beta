## Phases

### Phase 1 — D1 walk: Playwright spec discipline

No deletions or rewrites in this phase. The deliverable is a single verdict table the user reviews before any spec changes land.

- Inventory every spec file under `tests/e2e/`.
- For each, characterize what its assertions actually test: visible appearance, JS-driven UI, user interaction flow, CSS rendering, browser-mediated side effects (file downloads, FilePond uploads, drag-drop, Choices.js init).
- Bucket per spec: **Keep** (browser-required), **Redundant** (data/business logic covered by Pest), **Mixed** (some browser, some data — rewrite to UI-only or split).
- For Redundant verdicts in the importer suite: verify the same data path is covered by a `tests/Feature/Generated/ImportFixtureRunnerTest.php` case (importer × shape pairing). If FixtureRunner doesn't cover it, the verdict re-evaluates.
- For the page-builder cluster: separately tag specs as **Browser-required-but-flaky** if the cold-load timing is the issue rather than the assertion shape. Surface these as candidates for either selector hardening or deletion based on user policy.
- Output: verdict table, per spec, with rationale and recommended action. Surface to the user before Phase 2 starts.

### Phase 2 — D1 apply: deletions, rewrites, workflow doc

After the user approves the Phase 1 verdict table, apply the dispositions:

- **Delete** specs marked Redundant. Each deletion lands a `[test-integrity]` housekeeping line per the A003 pattern (what was removed, coverage tradeoff, Pest coverage that absorbs the loss).
- **Rewrite** specs marked Mixed — keep the UI-required assertions; remove the data assertions. The post-rewrite spec is materially smaller and reads as "exercises the browser-only contract."
- **Keep** specs marked Browser-required as-is. For Browser-required-but-flaky entries, apply the user-chosen path (selector hardening with concrete fixes, or delete).
- **Write `docs/testing/playwright-discipline.md`** — the durable workflow doc. Mirror `docs/testing/mutation-audits.md`'s shape:
  - Why Playwright (browser-only criterion in three sentences max).
  - The decision rule (Keep / Redundant / Mixed, with examples from this cycle).
  - The local-first authoring discipline (run the spec locally before commit; `npm run test:e2e -- --grep "<spec name>"` for the inner loop; never commit a spec that hasn't been seen green at least once).
  - Anti-patterns this discipline catches (URL mismatch against the Filament route registry; ambiguous `getByRole` selectors; `.first()` on `[role="dialog"]` placeholders; `getHeaderActions` `requiresConfirmation()` on custom Pages — the A003 patterns named explicitly).
  - The FixtureRunner-is-the-safety-net rule for importer spec dispositions.
  - The forward queue of audit dimensions (link to the track doc).

### Phase 3 — D2 drain: spec-claim integrity

Walk every `[test-integrity]` entry in `sessions/housekeeping-inbox.md`. For each:

- **Rename** — the test's title overclaims; rename to match what the body actually asserts. Default lean per the track doc: rename to match the actual assertion rather than expand the test.
- **Extend** — the test's title implies a stronger claim; add the missing assertion to honor the title. Only when the missing assertion is small and obviously right; otherwise rename.
- **Delete** — the test is a factory-echo / fixture-mirror / magic-count change-detector that asserts nothing meaningful. Per the s296 Tier-3 entries that flagged this class.
- **Reaffirm-and-mark** — the user reviews and decides keep-as-is with a `// guards:` comment per the s241 override-keep convention.

Output: per-item disposition table; surface to user; apply.

### Phase 4 — Forward scan (time-permitting)

If Phase 1–3 leave session capacity, run a forward scan for the spec-claim anti-pattern: grep `it(` / `test(` titles for promise verbs ("renders", "persists", "creates", "every", "only", "all", "ensures") across the full test suite. Output a delta table of any new mismatches the s296 walk missed. Apply dispositions per Phase 3's rules.

If the session is full at this point, queue Phase 4 as next-cycle work and close without it.

Verify the new `docs/testing/playwright-discipline.md` reads cleanly end-to-end. The doc is the most durable artifact this session ships.
