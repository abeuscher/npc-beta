## Process rules for every session

Base-prompt process rules apply in full (`sessions/template-base-prompt.md`). Session-specific notes:

- **Idempotent by construction.** `widgets:sync` must be safe to run on every deploy — it wraps `WidgetRegistry::sync()` (`updateOrCreate` by handle), which already is. The command adds nothing stateful; it must not duplicate rows or fail on an already-synced DB.
- **Run after `migrate --force`, before `build:public`** in each path, mirroring the existing ordering. The command depends on the schema being current; the public bundle build is downstream.
- **Touch every deploy path for parity** — `deploy.yml`, `deploy-demo.yml`, and the documented droplet/FM upgrade procedure in `docs/deployment/*`. A fix in one path but not the others reproduces the bug on the missed path.
- **Deploy-workflow changes are CI-affecting** — note this; the user may want to push for a real deploy run to prove the path. Do not push or deploy without explicit instruction.
- **Record the manual unstick** in the log / docs: `docker exec nonprofitcrm_app php artisan db:seed --class=WidgetTypeSeeder --force` (the immediate fix for an already-affected node) alongside the new command.
