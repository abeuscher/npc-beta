## What was built

First session of the **Widget Styling Contract** arc (source spec `sessions/styles-rework-spec.md`, Widget Autonomy track), taken as a deliberate switch off the paused demo-site track. Audit-style, two-phase with a hard pause between.

### Phase 1 — Audit (no application code)

- Surveyed **all** `app/Widgets/*/*.scss` (14 files) + the host seam (`AppearanceStyleComposer`, `page-widgets.blade.php`, `PageBlockRenderer`, `_variables`/`_base`/`_layout`, bundle concatenation order) by five matrices: spacing / type / breakpoints / specificity-cascade / general host-var leaks. Findings doc: `sessions/330. Widget Styling Boundary — Audit & Cleanup — Audit.md`.
- **Headline result:** the leak surface is far smaller than the source spec's completionist framing implied. Only two host-var crossings exist — `$gutter` (one widget, `ThreeBuckets`) and viewport `@media` (`$bp-*`). **No type leak, no other host-var leak** (grep-confirmed). The audit also surfaced two `@media` users the spec's snapshot missed (`EventsListing`, `EventMiniCalendar` — hardcoded literals) and the `Nav` runtime-breakpoint inline `<style>` special case.
- **Key scoping finding:** converting viewport `@media` → `@container` is behavior-preserving only for always-full-width widgets; for column-capable widgets it deliberately changes when collapse fires (the mobile-appearance improvement, deferred). `@layer` carries the most blast radius (coordinated all-host rollout) → recommended its own session.

### Phase 2 — Action (the owner-agreed fix list)

- **`ThreeBuckets` deleted entirely** (owner call mid-session — also removed the sole `$gutter` leak): the widget directory; the `WidgetServiceProvider` registration (registry-synced seeder drops it automatically on fresh DBs; `sync()` does not prune existing rows); `BoardMembersThreeBucketsTest` → renamed `BoardMembersSeederTest` minus its two `three_buckets` blocks; the `ThreeBuckets` block in `BuilderPublicRenderParityTest`; the `three_buckets` entries in `InlineEditingFoundationSession304Test` (roster 11→10, Tier-B map); the `WidgetType` count assertion in `WidgetPickerSession119Test` (40→39); the `_custom.scss` comment and the `widget-system.md` folder example. Confirmed no seeded/demo page referenced `three_buckets` before deleting.
- **`@container` migration — the two always-full-width widgets** (`pricing_chart`, `product_carousel`): added a per-handle `np-widget` named-container rule to `resources/scss/_layout.scss` (scoped to those handles, **not** every `.widget` — `container-type: inline-size` carries a containing-block/stacking blast radius); converted each widget's `@media (max-width: $bp-x)` to `@container np-widget (max-width: #{$bp-x})`. Verified objectively via real-wrapper Playwright injection: `pricing_chart` renders 3 columns at a 900px container, collapses to 1 at ≤768px container width.
- **`EventMiniCalendar` — option 3** (owner call, prompted by a reported `events-demo` bug): removed its `@media (max-width: 1023px) { display: none }`. It now stacks below the listing when the host column layout collapses, instead of self-hiding. This fixes a **pre-existing** glitch — its 1023px hide threshold sat above the layout's 768px collapse threshold, leaving a dead empty right column (and a squished left column) in the 769–1023px window. Confirmed pre-existing (not a 330 regression) via disposable read-only Playwright; verified the fix visually at 1000/768/375px.
- **Enforcement gate** — new `tests/Feature/WidgetStyleBoundaryConsumptionTest.php` (in the `design` group, mirroring `WidgetColorTokenConsumptionTest`): bans bare `$gutter` in widget SCSS (baseline empty — ThreeBuckets was the only reader) and bans new viewport `@media` (reviewed baseline = the four deferred column-capable widgets). `DesignGroupIntegrityTest` updated (20→21).
- **Doc** — `resources/docs/widget-development.md` SCSS section rewritten: the host-container/widget-interior boundary, `@container`-not-`@media`, the `np-widget` per-handle opt-in, the colour/spacing/type token rules.
