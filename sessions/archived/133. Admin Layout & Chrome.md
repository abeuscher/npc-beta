# 133. Admin Layout & Chrome

Interactive exploration of the admin panel's structural UI — left navigation menu, page body area, and header bar. This is a design-driven session: we review the current state, discuss alternatives, and implement agreed changes.

---

## Design decisions (resolved before starting)

- **This is an interactive, exploratory session.** The user will direct which areas to examine and which changes to make. Do not pre-plan a full implementation — wait for direction at each step.
- **Filament 3 is the admin framework.** All layout changes must work within Filament's panel builder, service provider configuration, and Blade component override system.
- **The custom Filament theme is already built** (session 120). CSS changes go through the existing Vite-based admin theme, not the public build server.

---

## Before writing any code

Read the following to understand the current admin layout:

- `app/Providers/Filament/AdminPanelProvider.php` — panel configuration, navigation groups, branding
- `resources/views/filament/` — any existing Filament view overrides
- `resources/css/filament/` — custom admin theme CSS
- `app/Filament/Pages/` — custom pages (dashboard, settings, tools)
- The Filament 3 panel builder documentation patterns for navigation customisation, render hooks, and Blade component overrides

---

## Phase 1 — Review current state

Present the current admin layout structure:

- Left navigation: groups, ordering, icons, collapsed/expanded state
- Header bar: what's currently shown, branding, user menu
- Page body: content width, spacing, sidebar behaviour

Describe what each area looks like today and wait for the user to identify what they want to change.

---

## Phase 2 — Implement agreed changes

Apply changes as directed by the user. Possible areas include but are not limited to:

- Navigation group reordering, renaming, or icon changes
- Header bar customisation (logo, breadcrumbs, quick actions)
- Page body width and spacing adjustments
- Sidebar collapse behaviour and responsive breakpoints
- Color or typography tweaks to the admin theme

Each change should be implemented, verified, and shown to the user before moving to the next.
