## What was built

### Phase 1 — Zoom/focus CSS proof of concept

- Focused block gets `zoom: 1.1` with primary ring highlight; sibling blocks blur (4px) + reduced opacity + `pointer-events: none`
- Column parent blocks exempt from blur when a child is focused and auto-open
- Clicking a blurred block transfers focus (outer wrapper stays clickable, inner content gets `pointer-events: none`)
- Escape key deselects; smooth transitions on all states
- Fixed column widget ellipsis menu clipping (removed `overflow-x-auto` from column slot container)

### Phase 2 — Edit mode: live widget rendering

- `PageBuilderBlock::getRenderedWidgetHtml()` renders the actual widget via `WidgetRenderer::render()` with recursive column child rendering
- `WidgetRenderer::render()` accepts optional `$fallbackCollectionData` param for demo data injection when real collection data is empty
- `DemoDataService` fallback for collection-bound widgets with no bound data
- Preview rendering is lazy — only runs when a block is actually selected (via `block-selected` Livewire event), not on initial page load
- Inspector dispatches `widget-config-updated` event on every save; block listens and re-renders
- Full-viewport zoom-down preview container: inner div set to viewport pixel width, CSS `zoom` scales it down to fit the editor panel; Alpine computes zoom factor dynamically on mount and window resize

### CSS isolation — `.np-site` namespace

- All bare element selectors (`body`, `h1`, `a`, `button`, `input`, `select`, etc.) in `_base.scss` and `_forms.scss` wrapped under `.np-site`
- Public layouts (`public.blade.php`, `portal.blade.php`) get `.np-site` on `<body>`
- Preview container gets `.np-site` class alongside `widget-preview-scope`
- Build server bundle (which includes all public styles) loaded in admin via `AdminPanelProvider` render hook — no `@layer` hacks or Vite CSS needed
- Body-level layout rules (`min-height: 100vh`, `display: flex`) overridden on `.widget-preview-scope.np-site`

### Phase 3 — Preview mode: full-page iframe

- `PagePreviewController`: admin-authenticated route at `/admin/preview/{page}` renders full page with public layout, no Filament CSS
- Each widget region gets a transparent click-target overlay with label tooltip on hover
- Clicking a widget handle sends `postMessage` to parent frame with widget ID
- Edit/Preview toggle in the page builder toolbar
- Default mode is preview; clicking a widget in the iframe switches to edit mode with that widget focused
- Deselecting in edit mode (Escape or click-away) auto-returns to preview mode

### Phase 4 — LoremFlickr placeholder images

- `DemoDataService` image fields return LoremFlickr URLs with context-derived keywords and dimensions (background → 800x600/nature, portrait → 400x400/portrait, thumbnail → 400x300/city, etc.)
- Collection generators (events, blog posts, products) return LoremFlickr URLs for thumbnail/image fields
- Video fields remain `null`

### Infrastructure fixes

- Docker `USER www-data` in Dockerfile for production security
- `docker-compose.yml`: `user: "${UID:-1000}:${GID:-1000}"` so artisan commands match host user UID locally
- Vite `emptyOutDir: false` prevents Vite from deleting the build server's `widgets/` subdirectory
- "Return to Preview" button added to toolbar (later replaced by automatic deselect → preview flow)
