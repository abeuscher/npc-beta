## Workstream 1 — Duplicate Detection Service

Created `app/Services/DuplicateContactService.php` with three public methods:

- `check(Contact|array $subject, ?string $excludeId = null): array` — accepts a Contact model or a plain data array and returns `['hard' => ?Contact, 'probable' => Collection]`. Hard match is exact email (case-insensitive); probable match requires both `last_name` (case-insensitive) and `postal_code` (normalised — spaces stripped, uppercased) to match. Dismissed pairs are excluded from probable results when `$excludeId` is supplied.
- `findAllProbablePairs(): Collection` — loads all non-deleted contacts with `last_name` and `postal_code` populated, groups them in PHP by normalised key, generates all within-group pairs, and filters out dismissed pairs. Returns a Collection of `['a_id' => uuid, 'b_id' => uuid]` with `a_id < b_id` (string comparison) for consistent dismissal lookup.
- `mergeContacts(string $survivorId, string $discardId): void` — runs inside a DB transaction. Copies non-blank scalar fields from discard to survivor, merges `custom_fields` arrays (survivor value wins on collision), reassigns notes, tags, donations, memberships, and event_registrations to the survivor, then soft-deletes the discarded contact.

Also created `app/Models/ContactDuplicateDismissal.php` and migration `2026_03_21_200001_create_contact_duplicate_dismissals_table.php`.

**Migration note:** An initial migration attempt used `foreignUuid` for `dismissed_by`, but `users.id` is a bigint — corrected to `foreignId` before the migration ran successfully.

## Workstream 2 — Import-Time Duplicate Flagging

Modified `app/Filament/Pages/ImportContactsPage.php`:

- Added `use App\Services\DuplicateContactService` import.
- Extended `getPreviewSchema()` to build a `$fieldToColIndex` lookup mapping `email`, `last_name`, and `postal_code` to their source column indices.
- If at least one of those three fields is mapped, runs `DuplicateContactService::check()` for each preview row and appends a **Duplicate check** row to the existing preview table.
- Hard duplicate: red badge "Exact match". Probable duplicate: amber badge "Probable: [contact name]". No match: `—`.
- Rows with none of the three fields mapped are skipped silently.

## Workstream 3 — Contact List "Review Duplicates" Action

Modified `app/Filament/Resources/ContactResource/Pages/ListContacts.php`:

- Added public properties: `$duplicatePairs` (array of `[a_id, b_id]`), `$currentPairIndex` (int), `$survivorId` (?string).
- Added public methods: `loadDuplicatePairs()`, `skipPair()`, `dismissPair()`, `mergePair()`.
- Added a `reviewDuplicates` header action using `mountUsing` to load pairs when the modal opens and `modalContent` to render the pair view. `modalSubmitAction(false)` — all navigation is via `wire:click` buttons in the modal content.
- Survivor defaults to `a_id` (lexicographically lower UUID, consistent with the dismissals table ordering). Admin can change selection via radio button.
- After merge/dismiss/skip the component advances to the next pair. When all pairs are exhausted the modal is closed via `$this->unmountAction()`.

Created `resources/views/filament/modals/review-duplicates.blade.php` — renders two contact cards side by side with a radio selector, a pair counter, and Merge / Not a Duplicate / Skip buttons.

## Scale Note

`findAllProbablePairs()` loads all contacts with `last_name` and `postal_code` into PHP memory for grouping. For datasets significantly above ~50,000 contacts this should be replaced with a raw self-join SQL query or moved to a queued job.
