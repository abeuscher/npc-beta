## What was built

### Phase 1 — Custom select dropdown

- Built `<x-custom-select>` Blade component with Alpine.js: keyboard navigation (arrow keys, enter, escape, tab), ARIA attributes (`role="combobox"`, `aria-expanded`, `aria-activedescendant`, `role="listbox"`, `role="option"`), hidden native `<select>` for standard form submission.
- Added `:searchable="true"` variant with a filter input at the top of the dropdown — filters options client-side as you type, shows "No results" when empty.
- Created `_controls.scss` partial and registered it in `public.scss` and `AssetBuildService` partial order.
- Replaced native `<select>` elements in public-form.blade.php (select, state, country field types), donation-form.blade.php (fund select), and portal-signup.blade.php (tier select).
- Added `$dispatch('custom-select-change')` for parent Alpine scope integration.

### Phase 2 — Toggle switch

- Built `<x-toggle>` Blade component: styled on/off switch with hidden checkbox, `role="switch"`, `aria-checked`, focus ring, smooth slider animation. Supports `name`/`id`/`checked` for forms, `alpine-checked`/`alpine-change` for reactive Alpine contexts.
- Replaced the theme toggle checkbox in both site-footer widget and site-footer component.

### Phase 3 — Styled radio buttons

- Built `<x-radio-group>` Blade component: button-style radio selector with Alpine.js, hidden input for form submission, `role="radiogroup"` and `role="radio"`. Supports `layout="inline"` (default) or `layout="stacked"`.
- Replaced native radio inputs in public-form.blade.php.

### Additional — State select, searchable select, country defaults

- Built `<x-state-select>` convenience wrapper: all 50 US states, DC, and 5 territories (PR, GU, VI, AS, MP). Searchable by default.
- Updated country select: United States moved to top of list and pre-selected as default value. Made searchable.
- Replaced the plain text state input in event-registration.blade.php with `<x-state-select>`.

### Additional — Form field width config & grid layout

- Created `config/form-fields.json` with handle-to-width mapping (e.g. `first_name: 6`, `city: 5`, `state: 4`, `zip: 3`), empty `rules` array for future label-matching, and `default_width: 12` fallback.
- Created `App\Support\FormFieldConfig` resolver class: priority is explicit width > handle match > rule match > default.
- Converted widget forms from `form-stack` to `form-grid` layout: portal-signup, event-registration (including nested address section), portal-contact-edit, portal-change-password.
- Updated public-form.blade.php to use `FormFieldConfig::width()` as fallback when no explicit width is set in the field JSON.
