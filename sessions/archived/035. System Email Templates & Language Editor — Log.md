## What was built

### Priority 0 — Help system trigger restored

The `?` help button was missing from all admin pages. Root cause: the `help_articles` database table was empty (wiped by a prior `migrate:fresh`) — the button conditionally renders only when a matching help article exists. Fix: re-ran `php artisan help:sync` to repopulate from the markdown docs in `resources/docs/`. No code changes required.

### Email templates system

Replaced hardcoded subjects and Blade views in the three transactional mailables with a database-driven template system.

**Migration:** `2026_03_17_100001_create_email_templates_table`
- `email_templates` table: handle (unique), subject, body, header_color, header_image_path, header_text, footer_sender_name, footer_reply_to, footer_address, footer_reason, custom_template_path

**Model:** `app/Models/EmailTemplate.php`
- `forHandle(string $handle)` — returns or creates the record with safe defaults
- `replaceTokens(string $text, array $tokens)` — token substitution on any string
- `renderSubject(array $tokens)` — renders the subject line
- `render(array $tokens)` — renders the body HTML
- `resolveWrapper(string $renderedBody)` — assembles the full email HTML (custom template if uploaded, default Blade wrapper otherwise)

**Seeder:** `DatabaseSeeder` seeds default rows for all three handles via `firstOrCreate`.

**Filament resource:** `EmailTemplateResource` under Settings > System Emails
- No create/delete (seeded records only)
- Edit form: Content section (subject + Quill body + token hint), Branding section (header colour, image, headline, footer fields), Custom Template section (collapsed, HTML upload)
- File uploads follow the same pattern as the logo uploader (disk: public, directory: email-templates)

**Blade views:**
- `resources/views/mail/default-wrapper.blade.php` — table-based HTML email layout with header (colour/image/headline) and footer (sender, reply-to, address, reason)
- `resources/views/mail/system-email.blade.php` — single `{!! $html !!}` output view

**Updated mailables:** `RegistrationConfirmation`, `EventCancellation`, `EventReminder` — all now load their `EmailTemplate`, build a token array from model data, render subject and body, and pass the assembled HTML to `mail.system-email`.

**Test send:** `EditEmailTemplate` page has a "Send test email" header action — asks for a recipient address, substitutes Fred Flintstone dummy tokens, prefixes subject with `[TEST]`, and sends via `Mail::html()`. Same driver/API key guard as the Mail Settings page test.

**Help article:** `resources/docs/system-emails.md` — covers all three emails, token reference, branding section, custom template upload, and a CAN-SPAM compliance section with FTC link. Synced via `help:sync`.

## Infrastructure fix (patch commits)

### Storage volume not mounted in Nginx container

Uploaded files (logos, email template images) returned 404 in production. Root cause: the Nginx image had no `public/storage` symlink and no access to the `storage_data` volume.

**Fix:**
- `Dockerfile` (web stage): added `RUN ln -s /var/www/html/storage/app/public /var/www/html/public/storage`
- `docker-compose.prod.yml`: added `storage_data:/var/www/html/storage` volume mount to the nginx service

### Deploy workflow not copying compose file

`docker-compose.prod.yml` changes were never transferred to the production server, silently ignoring service configuration updates.

**Fix:** Added `appleboy/scp-action` step to `.github/workflows/deploy.yml` to copy `docker-compose.prod.yml` to `/opt/nonprofitcrm/` before the SSH deploy step.
