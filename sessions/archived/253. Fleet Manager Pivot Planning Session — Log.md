### Phase 2 — Land the three CRM-side artifacts

- **`bin/rotate-fm-cert.sh`** — operator-facing rotation script. Validates input PEM via `openssl x509 -noout -text` (bails with exit 1 on parse failure), stages new cert via temp-file + atomic `mv` into the bind-mount source directory `/opt/nonprofitcrm/nginx-certs/`, reloads nginx via `docker exec nonprofitcrm_nginx nginx -s reload` (graceful, near-zero polling blip), appends a host-side audit record (`YYYY-MM-DDTHH:MM:SSZ  fingerprint=...  subject=...`) to `/opt/nonprofitcrm/logs/fm-cert-rotations.log`. EXIT trap cleans up the temp file on any failure path. Style mirrors `bin/dev-certs.sh` (shebang, `set -euo pipefail`, brief header, explicit chmod, final operator-confirmation echo). 67 lines.

- **`docs/runbooks/fm-compromise-recovery.md`** — fleet-wide compromise-recovery runbook. Mirrors `docs/runbooks/fleet-manager-cert-paste.md` style (frontmatter, intro, sections, troubleshooting bullets). Covers: threat model and scope; detection signals (FM admin credential compromise / FM database or filesystem exfiltration / unauthorized SSH or console access / unexpected polling source); pre-installation break-glass cert generation (offline keypair, cold-storage discipline, fingerprint-recording for verification); at-recovery-time per-CRM cert swap procedure (triage → retrieve break-glass cert → SSH per-CRM → run rotation script → verify mTLS gate); post-recovery cold-storage invariant restore (regenerate break-glass cert offline, store in cold storage, optionally rotate active cert away from break-glass to a fresh-FM-issued cert); audit log review and fleet inventory check; troubleshooting bullets matching cert-paste runbook style.

- **`docs/fleet-manager-agent-contract.md` Security Posture revision** — new sub-section "Recovery posture and FM-side trust assumptions" inside the existing Security Posture section, with the FM-tightened section-header preface labeling each item shipped vs FM-side intended-posture, plus three numbered items: (1) break-glass recovery path — *shipped* at this revision (CRM-side artifacts shipping alongside); (2) FM's off-filesystem-key posture — *FM-side intended posture, Beta-1 scope*; (3) audit-sink discipline — *FM-side intended posture, Beta-1 scope*. New `2.1.0 revision — 2026-05-01 (session 253)` CHANGELOG entry takes the FM-tightened text. **No version bump.** `Contract Version:` field stays at `2.1.0`.

## Production deployment notes

- **`bin/rotate-fm-cert.sh` ships in the CRM repo at `bin/rotate-fm-cert.sh`** but the CRM deploy workflow does not currently sync `bin/` to `/opt/nonprofitcrm/bin/` on production droplets (manual placement pattern, same as `docker/nginx/prod.conf` and `nginx-certs/fm-client.crt`). Future deploy-workflow extension to sync `bin/` would retire this manual step.
