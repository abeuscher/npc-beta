# 046. Site Theme Admin — Log

## What was built

### Admin page — `SiteThemePage`

- New standalone Filament page at `app/Filament/Pages/SiteThemePage.php`
- Navigation: CMS group, label "Site Theme", icon `heroicon-o-paint-brush`, sort 6
- `canAccess()` requires `view_any_page` or `edit_theme_scss`
- Two-column layout (2/3 main, 1/3 sidebar) following the Contact resource pattern
- **Main column:** Header & Footer section (nav handles, logo upload, header content rich editor, custom override status placeholders) — **Colours section moved to right column during session**
- **Right column:** Colours section (Brand, Header bg, Nav link, Nav hover, Nav active, Footer bg — 6 pickers stacked) + Fonts section (heading/body selects)
- Mount reads all keys from `SiteSetting::get()`; save writes all via `SiteSetting::set()`
- `footer_bg_color` added for parity with `header_bg_color`

### SCSS Editor tab

- Gated by `edit_theme_scss` permission (not assigned to any role by default)
- Raw Livewire property `$themeScss` — not a Filament form field
- `Textarea` with `rows(30)` and monospace styling
- "Save & Build" action: validates via `scssphp/scssphp`, writes `resources/scss/_theme.scss`, runs `npm run build` via `exec()`, shows full output in a `<pre>` block with success/failure colouring

### Blade view — `site-theme.blade.php`

- Alpine-based tab switcher (Appearance / SCSS Editor)
- SCSS Editor tab only rendered in markup when user has `edit_theme_scss`

### Permission

- Added `edit_theme_scss` standalone permission to `PermissionSeeder`

### Public layout changes — `public.blade.php`

- Google Fonts `<link>` tag auto-generated when a Google Font is selected — includes `preconnect` hints
- Scoped CSS block for header/nav/footer colours injected after the `:root` block
- **Bug fixed:** `{{ }}` was HTML-encoding CSS values inside `<style>` tags (`'Lato'` → `&#039;Lato&#039;`). Changed all CSS output inside `<style>` to `{!! !!}`
- `nav_link_color` extended to also cover `footer nav a` and `footer .theme-toggle` for colour consistency

### Site header — `site-header.blade.php`

- Logo rendering added (reads `logo_path` from `SiteSetting`)
- `aria-current="page"` added to active nav links (current URL comparison)
- Nav dropdown simplified: removed Alpine `x-data`/`@mouseenter`/`@mouseleave`/`x-show`/`x-cloak` and the separate `<button>` for the caret entirely. Replaced with pure CSS `:hover` on `.nav-dropdown`. Caret moved inside `<a>` as a CSS `::after` triangle that rotates on hover.
- Gear icon dropdown (light/dark toggle) removed from header per user feedback

### Site footer — `site-footer.blade.php`

- Light/dark mode toggle added: moon icon / Pico `role="switch"` checkbox / sun icon
- Uses global Alpine `$store.theme` — `x-data` on the wrapper div

### New components

- `resources/views/components/svg-icon.blade.php` — reads Heroicons SVGs from vendor directory and renders inline. Named `svg-icon` (not `icon`) to avoid collision with `blade-ui-kit/blade-icons`' registered `x-icon` component.

### JS — `public.js`

- Alpine `theme` store added: persists light/dark preference in `localStorage`, applies `data-theme` on `<html>` at init

### SCSS — `_layout.scss`

- Header grid reverted to 2-column (`1fr 2fr`) after gear icon removal
- Nav dropdown: removed `button` rule, added `display: none` / `&:hover > ul { display: block }` for CSS-only behaviour
- `.has-dropdown` anchor: `::after` triangle with `transform: rotate(180deg)` on parent hover
- Footer layout: `flex`, `space-between` for nav + toggle
- `.theme-toggle`: inline-flex with icon sizing

### `vite.config.js`

- Added `css.preprocessorOptions.scss` with `api: 'modern-compiler'` and `silenceDeprecations: ['if-function']` to eliminate Dart Sass deprecation warnings from Pico CSS

### Dockerfile

- Node.js 22 installed in the `app` stage via NodeSource (required for `npm run build` from inside the container)
- `ln -s` changed to `ln -sf` in the `web` stage to make symlink creation idempotent across rebuilds

### `docker-compose.yml`

- `target: app` added to `app` and `worker` service build configs — previously the last Dockerfile stage (`web`/nginx) was being tagged as the app image
- `app_node_modules` named volume added to shadow the host's Windows `node_modules` with a Linux-native set inside the container (resolves `@rollup/rollup-linux-x64-gnu` missing module error)

### Help doc — `resources/docs/public-theme.md`

- Full rewrite from session 045 stub covering Appearance tab, SCSS Editor tab, Google Fonts behaviour, light/dark toggle, and brand colour precedence note
- Route `filament.admin.pages.site-theme-page` registered in frontmatter
