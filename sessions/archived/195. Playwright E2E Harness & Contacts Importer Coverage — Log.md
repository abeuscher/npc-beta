### Phase 2 — Global setup

- `tests/e2e/global-setup.ts` runs `docker compose exec -T app php artisan migrate:fresh --seed`, then delegates to `captureAdminStorageStateStandalone()` from the auth helper.
- `tests/e2e/helpers/auth.ts` exposes `loginAsAdmin(page)`, `captureFreshStorageState(browser)`, `captureAdminStorageStateStandalone()`, and `resetAndLogin(browser)` — the last is used by spec `beforeAll` hooks because `migrate:fresh --seed` wipes server-side sessions and invalidates the storageState cookies.
- Login drives the real `/admin/login` form, waits for navigation off `/admin/login`, and sanity-checks by navigating to `/admin/import-contacts-page` and confirming no redirect back to login.

### Phase 3 — DB helper

- `tests/e2e/helpers/db.ts` uses the `pg` client against `localhost:5432` (Postgres is host-exposed by docker-compose). Reads `DB_*` from `.env` via `dotenv`.
- Exports: `countContacts()`, `countContactsInSession(sessionId)`, `countStagedUpdatesForSession(sessionId)`, `findContactByEmail(email)`, `findLatestImportSessionId()`, `resetDatabase()`.

### Phase 4 — `data-testid` retrofit (contacts surfaces only)

- Shared trait [InteractsWithImportWizard.php](app/Filament/Pages/Concerns/InteractsWithImportWizard.php): `topNav()` emits `import-step-back-{n}` / `import-step-next-{n}`; `buildSourceSection()` tags the new-source input (`import-source-name`) and the existing-source select (`import-source-select`); `buildFileUpload()` tags the upload wrapper (`import-file-upload`). All five wizards inherit these.
- Contacts wizard [ImportContactsPage.php](app/Filament/Pages/ImportContactsPage.php): tag select (`import-tag-select`), new-tag input + action (`import-new-tag-input`, `import-new-tag-add`), auto-custom toggle (`import-auto-custom-toggle`), per-column mapping select (`map-column-{n}`), match-key select (`import-match-key`), duplicate-strategy radio (`import-duplicate-strategy`), Stage Import submit action (`import-commit-button`). Wizard root testid on the outer blade div (`import-contacts-wizard`).
- Progress page blade [import-progress.blade.php](resources/views/filament/pages/import-progress.blade.php): phase wrappers (`import-progress-phase-{awaiting|committing|done|rejected}`), commit/cancel buttons, progress bar, stat cells (`import-stat-imported`, `import-stat-updated`, `import-stat-skipped`, `import-stat-errors`, used in both dry-run and committing and done phases), save-mapping button, go-to-review link.
- Importer landing [ImporterPage.php](app/Filament/Pages/ImporterPage.php): `recordClasses` emits `importer-row-{id}` on each table row; per-row action `extraAttributes` closures emit `importer-action-{preview|approve|rollback}-{id}`; the approve/rollback modal submit buttons receive `importer-modal-approve-submit` / `importer-modal-rollback-submit` via `modalSubmitAction()` closures.
