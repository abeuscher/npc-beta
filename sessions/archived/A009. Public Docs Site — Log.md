## What was built

### Phase 0 — Stack bring-up (environment drift — see Deferred/notes)

- The cloud container's network policy blocks the Docker Hub blob CDN, GHCR blob storage, dl-cdn.alpinelinux.org, and deb.debian.org, so neither `docker compose up` (image build needs apk/apt) nor a GHCR image pull could bring the stack up. The host itself has PHP 8.4.19 with every required extension except bcmath (pdo_pgsql, gd, intl, redis, zip all present), Node 22, PostgreSQL 16, and Redis 7 — so the session ran the stack **natively on the host**: `service postgresql start`, `service redis-server start`, `composer install --ignore-platform-req=ext-bcmath` (Launchpad PPA blocked; bcmath has pure-PHP fallbacks via brick/math — full fast suite green confirms nothing fatal calls `bc*`), `npm install`, `migrate:fresh --seed`, `help:sync`. phpunit.xml's forced `DB_HOST=postgres` was satisfied with an `/etc/hosts` alias to 127.0.0.1 and the test role/database created to match.
- Corpus drift vs the prompt: **63** files in `resources/docs/`, not 62 (`widget-table.md` landed with session 349 after the A009 prompt was drafted). All counts below use 63.

### Phase 1 — Flag + helper

- `config/site.php`: `'public_website' => env('PUBLIC_WEBSITE', false)` — plain env read, deliberately not a SiteSetting; survives `config:cache`.
- `app/helpers.php`: `isPublicWebsite(): bool` returning `(bool) config('site.public_website')`, sibling to `isDemoMode()`.
- `.env.example`: `PUBLIC_WEBSITE=false` with a one-line marketing-instance comment.

### Phase 2 — Public docs routes + controller + views

- Routes registered with the SEO/utility block in `routes/web.php`, before the page-slug catchall: `GET /docs` (index), `GET /docs/{slug}.md` (raw — registered first), `GET /docs/{slug}` (show). Both slug routes constrain `{slug}` to `[^/.]+`, so the `.md` sibling and the HTML route match unambiguously. Every action opens with `abort_unless(isPublicWebsite(), 404)`.
- `app/Http/Controllers/DocsController.php`: `index()` groups all articles by category in the admin help surface's category order (crm/cms/finance/tools/settings/general, labels reused from `HelpCategoryPage::categoryLabel`); `show()` builds the breadcrumb ancestor chain with the same guarded `parent()` walk `HelpArticlePage` uses and pulls related articles from the service (see extraction note); `raw()` serves the `content` column behind a minimal header block (H1 title, description, `Last updated:`, `Canonical: {base_url}/docs/{slug}` — `base_url` from SiteSetting with the `config('app.url')` fallback, the `LlmsTxtController` pattern) as `text/markdown; charset=utf-8`.
- **Sanctioned extraction:** the tag-intersection related-articles query (raw JSONB SQL, non-trivial) moved from `HelpArticlePage::loadRelatedArticles()` into `HelpArticleService::relatedTo()`; the Filament page now delegates. No behavior change (admin help tests green).
- **Adaptation (noted per the drift rule):** every one of the 63 corpus bodies opens with its own `# Title` H1. Prefixing the spec'd header block verbatim would have produced two H1s per document on every public surface. `HelpArticleService::bodyWithoutLeadingH1()` strips the body's leading H1 on the public surfaces only (HTML article body, `.md` sibling, `llms-full.txt`), so each rendered document carries exactly one H1 — the extraction-structured shape the spec is after. Admin help rendering untouched.
- Views `resources/views/docs/index.blade.php` + `show.blade.php` extend `layouts.public` directly (the `PostController`-family shape — title/description/bodyClass via `@extends` params; chrome, SEO meta, global-noindex all come from the layout, which is `$page`-independent throughout). Article page: breadcrumbs (Docs → ancestors → title), `<h1>` = title, description lede, visible "Last updated", rendered body (`Str::markdown`, the admin call), related-articles links, "View as Markdown" link to the `.md` sibling, cross-link back to `/docs`. `<title>` = `{Title} — {Site Name} Docs`.
- Styling: new `resources/scss/_docs.scss` (readable long-form typography only, `@layer host`, `.np-site`-independent class namespace `docs-*`), wired into `public.scss` via `@use "docs"`. `npm run build` run (Vite surface only). The build-server bundle's explicit partial list (`AssetBuildService::$partialOrder`) deliberately does **not** include `_docs.scss` — docs styles ride the Vite public bundle, `build:public` untouched per the hard-no. The reviewed list in `WidgetStyleLayerDisciplineTest::LAYERED_HOST_PARTIALS` updated in the same pass (the guard's intended workflow).

### Phase 3 — AEO wiring (all flag-conditional)

- `LlmsTxtController::build()`: when flagged, a `## Documentation` section after `## Pages` (before `## Contact`) — one bullet per article linking the **`.md` sibling** URL, title as link text, description after an em-dash. Existing 1-hour cache and key kept (content is flag-stable per instance).
- `LlmsTxtController::full()` (new, + `GET /llms-full.txt` route, 404 off-flag): whole corpus — site-identity header (same name/description block `llms.txt` opens with), then each article as `---` separator + `# {title}` + canonical line + body (leading-H1-stripped). `Cache::remember` 1 hour, `text/plain; charset=utf-8`.
- `SitemapController::buildSitemap()`: when flagged, appends `/docs` (lastmod = max article lastmod) and every `/docs/{slug}` with `<lastmod>` from `last_updated`, falling back to `updated_at`, in the page entries' W3C format.
- robots.txt verified-only: still `Allow: /` for all agents + sitemap reference; a test asserts the docs-enabled instance stays permissive.

## Security considerations

The new surface is read-only over repo-authored content (no user input beyond the slug, which is pattern-constrained and used only in an Eloquent `where`). The gate was the review focus: all four new routes and both AEO extensions check `isPublicWebsite()`, which reads through config so `config:cache` cannot strand a stale env read; the flag-off state is proven byte-equivalent in behavior by the gating tests (404s + unchanged llms/sitemap output). Rendered markdown uses the same `Str::markdown()` call the admin help system already trusts for this same repo-authored corpus. No auth surface, no portal data, no contract surface touched.
