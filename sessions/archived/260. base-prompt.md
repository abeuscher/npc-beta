# Base Prompt — Session 260

We are about to begin a new session: **260. First-Deploy Migrate Race Fix**.

This session lands a small operational fix unblocking FM-side's upcoming install-automation work: the documented race between `docker compose up -d` returning and the postgres container finishing its first-time data-directory init. The deploy workflow's `migrate --force` step exits 0 on first install but no migrations actually run, and the documented workaround is the operator manually running `migrate:fresh --seed --force` after every fresh deploy. This session removes the workaround at root.

This session is **not boundary-touching** — Fleet Manager contract stays at v2.1.0.
