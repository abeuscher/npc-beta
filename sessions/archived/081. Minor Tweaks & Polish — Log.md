## What was built

### Phase 1 — UI Bug Fixes

- Block editor delete confirm button: changed from `bg-danger-600 text-white` (white-on-white) to `outlined` variant (`border border-danger-600 text-danger-600`)
- Portal email verification and email-change templates: moved `.btn` styles from `<head>` CSS class to inline styles on the `<a>` tag — fixes Gmail class-mangling that rendered buttons as blue blocks with invisible text

### Phase 2 — Admin Navigation & Quick Links

- "View public site" link added to the admin topbar via `TOPBAR_START` render hook — small outlined button with external-link icon, left-aligned
- "Donors" renamed to "Giving Summary" in navigation label, page title, and breadcrumbs (`DonorsPage`)
- Navigation icon updates: Contacts → `user-circle`, Members → `identification`, Mailing Lists → `inbox-arrow-down`, Giving Summary → `trophy`, Membership Tiers → `identification`, Finance Settings → `currency-dollar`

### Phase 3 — Contact & Donations Table Enhancements

- Portal Access section on contact edit form: removed `->collapsed()`, now expands by default
- Donations table: "View Contact" row action added, navigates to contact edit page; hidden when no `contact_id`

### Phase 4 — User Management: Password Confirmation

- Added `->same('password_confirmation')` validation with clear error message to the password field
- Added "Confirm New Password" field (always visible, `->dehydrated(false)`); required on create, optional on edit

### Phase 5 — API Key Rotation: Resend & Mailchimp

- `MailSettingsPage` restructured: Resend API key, Mailchimp API key, and Mailchimp webhook secret now use the high-friction `secretKeySection()` / `setKeyAction()` / `changeKeyAction()` pattern from `FinanceSettingsPage`
- Non-sensitive Mailchimp fields (server prefix, audience ID, webhook path) remain as regular editable inputs
- `preserveMailSettings()` ensures non-secret settings are not lost when a key action redirects
- Activity logging added to all key set/rotate actions

### Additional — Transaction Immutability

- `TransactionResource`: added `canEdit()` and `canDelete()` resource-level gates scoped to `!$record->stripe_id` — Stripe-originated transactions are now fully immutable at every entry point, not just the table action buttons

- **B. Mailing list field policy** — full policy agreed and stubbed as a future session (allowed filters, donor threshold boolean, age cutoffs at 13/18/21, household deduplication, `do_not_contact` as hard system exclusion)
