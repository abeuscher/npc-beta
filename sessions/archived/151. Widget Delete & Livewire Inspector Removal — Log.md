### Bug fixes discovered during testing

- **Script tag stripping broke Alpine data widgets:** The preview renderer's `preg_replace` stripped all `<script>` tags including `<script type="application/json">` data carriers used by blog listing and events listing widgets. Fixed with a negative lookahead regex that preserves `type="application/json"` scripts. Applied to both `PageBuilder.php` and `PageBuilderApiController.php`.
- **Swiper CSS overriding pager button positioning:** Listing widget pager buttons and pagination container used Swiper classes (`swiper-button-prev`, `swiper-button-next`, `swiper-pagination`) which Swiper's dynamically-loaded CSS overrode with `position: absolute` and `width: 100%`. Fixed by removing Swiper classes from the buttons and pagination container in blog and events listing templates — Swiper wires to elements via `x-ref`, not classes. Rewrote `_pager.scss` cleanly with our own classes. For the pagination container, Swiper adds classes at runtime, so used a 3-class specificity selector to override `width: 100%`.
- **Widget creation didn't auto-select in inspector:** `createBlock()` dispatched `widget-tree-updated` without the new widget's ID. Changed to dispatch `widget-created` with the ID; Vue listener reloads tree and selects the widget.
- **"No preview available" and placeholder notices:** Moved inline styles to `admin.css` classes (`widget-preview-notice`, `widget-placeholder`). Placeholder styles scoped to `.widget-preview-scope` for admin-only sizing (5rem title, 3rem message). Removed inline styles from `widget-placeholder.blade.php`.
