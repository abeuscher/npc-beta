### Phase 4 ‚Äî Value-pattern noise detection

- New service `app/Services/Import/NoiseDetector.php` with three heuristic rules: `Field&&Visibility` concatenations (Wild Apricot pattern), very-long-metadata strings (avg >200 chars), key-value dumps.
- Integrated into `processUploadedFileNamespaced()`; noise columns default to unmapped with per-column advisory helper text on the Map Columns step. Non-blocking ‚Äî user can override.
- Detected column indices stored in optional `$noiseColumns` property on each wizard page.

### Phase 5 ‚Äî Per-source skip-header lists

- Extended `FieldMapper` with `sourceSkippedHeaders()` method merging the global `SKIPPED_HEADERS` baseline with per-source lists.

- `isSkipped()` signature extended with optional `$sourcePreset` parameter.
- `presetFromSourceName()` maps seeded source names (`Wild Apricot`, `Bloomerang`, `Generic CSV`) to preset keys.

### In-session additions beyond the original prompt

- **Custom field support for Donation / Membership / Transaction.** Added `custom_fields` jsonb column to all three models via migration `2026_04_17_121000_add_custom_fields_to_financial_models.php`. The donation, membership, and invoice progress pages now resolve `CustomFieldDef` entries under `model_type = donation | membership | transaction` and persist values on the created records. The invoice importer merges custom fields across multi-row invoice groups (first-non-blank wins).
- **Update preview on the contacts dry-run report.** The `updatePreview` bucket collects up to 50 matched contacts (display name, match field, match value, row number). Rendered as a table on the import-progress blade above the "Why rows would skip" section so reviewers can see which existing contacts would be modified before committing.
- **Rich Text Custom Fields stub** added to `session-outlines.md` under pre-Beta 1.
- **Random Data Generator ‚Äî CSV Export for Import Testing stub** added to `session-outlines.md` under pre-Beta 1. Motivated by the UAT constraint that real client data contains PII and cannot be moved off the author's local machine for deploy-server testing.
- **Bug discovered and diagnosed in UAT.** Real WCG import surfaced a data-corruption bug: the Wild Apricot CSV has two email-bearing columns (`Email` at col 5 and `Email address` at col 21, the latter being the bundle-administrator's email for secondary members). The default WA preset mapped both to `email`, causing cascading overwrites ‚Äî bundle members' primary emails were replaced with the admin's email, which then shadow-matched the admin's own row on import and silently skipped it. Consequence: most of the org's admin/staff contacts (the people the tester spot-checks first) were missing from the imported data. Root cause documented; fix scoped out as session 192.

## Deferred decisions

- **Cascading duplicate-column corruption on re-import** ‚Äî deferred to session 192 (Data Review Step). The WA "Email address" bug is one instance of a broader pattern: source systems often export multiple columns whose headers suggest the same semantic field but whose values are not the contact's own data. Session 192 adds a dedicated wizard step between Upload and Map Columns to flag duplicate/similar headers and let the user resolve before mapping.
- **Update-existing strategy for events / registrations / donations / memberships / transactions** ‚Äî documented in session-outlines.md under pre-Beta 1. Architecturally distinct, benefits from the trait structure delivered this session.
- **Saving per-source "ignore" preferences** to `ImportSource` ‚Äî flagged in the session 192 prompt as possibly-deferrable scope.
