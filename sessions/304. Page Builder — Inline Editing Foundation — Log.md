## What was built

### Phase 1 — Interaction-model rework (shipped)

- Top-left **hover-in drag grip** (`.preview-region__handle`); the existing `vuedraggable` `handle=` repointed to it on both the root canvas and column slots (reorder payload/persistence unchanged).
- Top-right **hover-in "Edit" affordance** → selects the widget and opens the Inspector on its Content tab.
- **Body-click selects**; whole-panel selection (and its hover outline) removed. Overlay is no longer a drag handle.
- **Drag proxy:** a fixed labelled chip follows the cursor (`forceFallback` + zoom approach was tried and rejected as distorted; the chip is the shipped solution — solves a real "can't drag a tall full-width widget back into a column slot" usability bug the owner hit).
- Playwright: `tests/e2e/page-builder/inline-editing-foundation.spec.ts` (4 tests). **Now quarantined — see Test result.**

### Phase 2 — In-page text editing behind the code-declared gate (shipped)

- **Capability gate:** `WidgetDefinition::inlineEditable()` (fail-closed `false`; `true` only on TextBlock, Hero, ThreeBuckets, PricingChart). Plumbed to the Vue layer **live via `WidgetResource`** (`app(WidgetRegistry)->find(handle)?->inlineEditable()`) — **no `widget_types` column, no migration** (settled decision honoured).
- **Editors seed from / write the RAW `config[path]` value, never serialized rendered DOM** (the token / inline-image baking hazard). Plaintext nodes → `contenteditable`; richtext nodes → Quill (no toolbar; `theme:'snow'` for a usable box, then its chrome stripped — see WYSIWYG below). Path-addressed `updateLocalConfigPath` (dot path, arrays preserved) → existing debounced save.
- **Refresh-suppression + Quill lifecycle:** `inlineActiveWidgetId` suppresses the post-save preview echo for the actively-edited widget; one reconciling refresh on blur (`endInlineEdit`). Editor **mounts on activation, deferred via `requestAnimationFrame`** so the triggering click's gesture doesn't kill caret placement (this was a real bug found in manual UX testing); disposes on blur/deselect.
- **Templates:** shared `widget-shared/inline-prose.blade.php` partial + `WidgetRenderer` `$inlineEditing` (= `$isCanvas`) so annotated wrappers + builder-only empty placeholders render in the builder while **public output stays byte-safe** and PricingChart subgrid tracks always render. PricingChart annotated incl. nested repeater leaves (`columns.{i}.attribute_rows.{j}.value`, etc.).
- **Discoverability:** dormant `.inline-editable` CSS (admin.css) activated, schema-`label` ghost on empty slots, selection-scoped. Quill's snow chrome (border/bg/padding/`height:100%`/`overflow`/font/align) stripped so the in-place editor is **WYSIWYG and shrink-wraps** like the original element (fixes the "hop"; verified on Hero — generic CSS, applies to all richtext contexts; plaintext path never had the issue).
- **Security:** `PageWidget::saving` richtext sanitization now **recurses into repeater rows** so every inline path (incl. nested PricingChart richtext) is sanitized on save.
- **Tier-B exempt-set guard:** durable Pest assertion that no Tier-B field key carries a `data-config-key` annotation in any widget template.
- **Owner decision — token gate reversed:** the per-node `{{token}}` exclusion was **removed**. Token-bearing prose in the eligible widgets is inline-editable; the editor seeds/writes RAW so a `{{token}}` shows and round-trips literally (same Quill/raw path the Inspector already uses → no new risk). The genuinely-unsafe data-driven `{{item.*}}` templates remain excluded **at the widget level** (`inlineEditable()`=false). This **revises the prompt's "mandatory per-node token gate" framing** — recorded here per the owner's call.
- Playwright: `tests/e2e/page-builder/inline-editing-phase2.spec.ts` (5 tests; the token test was flipped to assert editable). **Now quarantined — see Test result.**

## Security considerations

Reviewed: the inline-edit write path **never serializes rendered DOM** — it reads/writes the raw stored `config[path]`, so resolved tokens / inline-image URLs cannot bake back. Nested richtext written via inline editing is now sanitized (`PageWidget::saving` recurses repeater rows). The token-gate reversal does not widen the attack surface: it's the same Quill + raw-write path the Inspector already exposes; data-driven `{{item.*}}` templates remain widget-level-excluded. Upload-limit fix raises ceilings only; Filament's 512 MB cap and the zip-slip/zip-bomb guards from s303 are unchanged. CRM contract unchanged at **v2.3.0**; no boundary surface touched.
