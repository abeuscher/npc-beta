## Discussion Points Resolved

**Dashboard widget taxonomy:** Aligned with Filament's existing vocabulary. Admin dashboard components are "dashboard widgets"; page builder components are "page widgets". No custom taxonomy — keeps the codebase readable for open-source contributors.

**CSRF on web forms:** Not a gap. Livewire handles CSRF automatically on all public form components.

**Site Theme color system:** Agreed approach — two-row layout (light / dark), mirror buttons in each direction using HSL inversion as a starting point (labelled as suggestion), live WCAG AA contrast checker with nearest-passing-color hint on failure, preset palettes with preview swatches. Auto-deriving dark from light without user review is not safe; the mirror is a convenience, not automation.

**MINIMUM DATA mode:** Agreed as a real feature. Event-level registrant cleanup is independently valuable and ships in the same session. Broader data minimisation flag encourages hygiene as normal practice, not an edge case. Aligns with existing privacy philosophy in memory.

**Communication Log / Notes lazy loading:** Notes panel should lazy-load regardless of the communication log work. The log itself is an extension of the notes model — typed, filterable entries. Depends on Member Portal so member self-service activity is captured from day one. All logging is transparency/accountability-facing, not behaviour tracking.
