**Date:** 2026-04-07

## What was built

### Phase 1 — API endpoints

- Created `PageBuilderApiController` with 16 endpoints under `/admin/api/page-builder`:
  - Widget CRUD: list tree, create, update, delete, copy, batch reorder
  - Preview: render single widget HTML
  - Lookups: widget types, collections, collection fields, tags, pages, events, data sources
  - Image upload/remove
- Created `routes/admin-api.php` with route definitions
- Registered routes via Filament's panel `->routes()` in `AdminPanelProvider.php` with `Authenticate` middleware
- Logic extracted from existing Livewire components (PageBuilder, PageBuilderBlock, PageBuilderInspector) — not rewritten from scratch
- Permission checks: read endpoints require `view_page`, write endpoints require `update_page`
- Widget ownership validation on all CRUD operations

### Phase 2 — Vue scaffold

- Installed `vue` (^3.5), `pinia` (^3.0) as dependencies; `@vitejs/plugin-vue` (^6.0) as devDependency
- Updated `vite.config.js` with Vue plugin and new `page-builder-vue/main.ts` entry point
- Created `resources/js/page-builder-vue/` with:
  - `types.ts` — TypeScript interfaces (Widget, WidgetType, BootstrapData, API payloads)
  - `api.ts` — fetch wrapper with CSRF token handling, methods for all 16 endpoints
  - `stores/editor.ts` — Pinia store with flat widget map, rootOrder, selection state, editor mode, all CRUD actions, preview refresh, image upload/remove
  - `App.vue` — root component, reads bootstrap JSON from mount element
  - `components/BlockListPoc.vue` — proof-of-concept: block list with select-to-highlight, JSON config dump, inline label editing via PUT API
  - `main.ts` — createApp + createPinia entry point
- Added `getBootstrapData()` to `PageBuilder.php` — assembles full bootstrap JSON (page, widgets with preview HTML, widget types, lookups, CSRF token, API URLs)
- Added `#page-builder-app` div with `wire:ignore` and `@vite` tag to `page-builder.blade.php`

## Security checklist — verified

- [x] All API endpoints require Filament session auth — no unauthenticated access
- [x] Write endpoints check `update_page` permission; read endpoints check `view_page`
- [x] Widget CRUD validates `page_id` ownership — cannot manipulate widgets on other pages
- [x] No sensitive data (API keys, user credentials) in API responses or bootstrap JSON
- [x] Image upload validates file type and size
- [x] CSRF token required on all mutating requests
- [x] Lookup endpoints return only data the authenticated user may view
