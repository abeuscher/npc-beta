### Phase 2 — User-revision passes (4 visual changes)

After the first review the user requested a series of changes; each applied JSON-only, re-imported clean, screenshot refreshed:

1. **Band 2 widened to full container width.** First attempt: kept the spec's 3-col `1fr 2fr 1fr` spacer-cell pattern, switched to `1fr` single-column. Result: the four stacked text widgets broke visually — each rendered at 643px tall, overflowing the parent column 4× and making the closing paragraph fall behind Band 3. Root cause: the global SCSS rule `.widget--text_block { height: 100% }` at `resources/scss/_custom.scss:144` collides with multi-text-widget single-column layouts; in a grid context the rule resolves `100%` to the largest sibling's content height and applies it to every sibling. **Fix:** split Band 2 into four sibling page-layout blocks (2a / 2b / 2c / 2d), each 1-col single-column white-background, each carrying one text_block. With one widget per grid cell the rule resolves to natural-content height. Inter-layout padding calibrated so the four read as one composite band (same pattern home Band 4's grid + CTA-footer composite uses).
2. **Pull-quote narrowed to ~80% content width.** Symmetric horizontal padding of 128px on the Band 2b text_block — at 1264px content area this gives ~1008px content (~80%). The pull-quote now reads as a distinct centered inset against the full-width prose above and below. Side-stepped the missing `appearance_config.max_width` knob; surfaced as part of the G19 surface (no border knob).
3. **Pull-quote real text in.** Replaced the placeholder with the verbatim Boris Cherny quote from AI Ascent 2026 (`https://youtu.be/SlGRN8jh2RI?t=747`), corrected spelling/casing per the user's typed-in-caps version. Attribution links "AI Ascent 2026" to the YouTube URL via Quill `<a href>` (HTML sanitizer allows `href / title / class` only on `<a>` — no `target / rel`, link opens in same tab; minor surface to revisit if outbound-link new-tab behavior is wanted later).
4. **Bold removed from pull-quote.** `<strong><em>` → `<em>` for the quote text. Italic-only treatment reads cleaner with real content than the bold + italic placeholder treatment did.

### Phase 3 — Gap-report appends

Two new rows surfaced; appended to `sessions/public website/gap-report.md` under a new "Session 287 findings" section:

- **G19 — `appearance_config` has no border knob for pull-quote / panel treatment.** Surfaced when Band 2's pull-quote spec called for a "left border (1-4px solid in a neutral accent color)" and the appearance composer had no field to express it. Workaround per spec: bold + italic content with vertical padding (later refined to italic-only after the user-revision pass). Recommended action: extend `appearance_config` with a border group (color / width / style / sides) wired through `AppearanceStyleComposer`, OR add a typography `blockquote` element row. The first is per-instance flexible; the second is typography-as-source-of-truth. Either keeps the eventual real-pull-quote treatment from depending on inline styles. Low priority — the workaround reads acceptably.
- **G20 — Global `.widget--text_block { height: 100% }` rule breaks stacked text widgets in a single layout cell.** Surfaced when the first widening attempt placed four text_block widgets in one column. The rule was added for the common one-text-widget-per-cell case (vertical balance across multi-column rows); collides with stacked-siblings-in-one-cell. Recommended: scope the rule with `:only-child` (`.layout-column > .widget--text_block:only-child { height: 100%; }`), or drop it and rely on per-widget `vertical_align` for cross-cell balancing. Either path lets single-column stacked text bands work without the four-sibling-page-layout workaround.
