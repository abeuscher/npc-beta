## What was built

### Phase 1 — paratest wired; isolation-defect set surfaced

- **Drift:** the release-plan assumed paratest was a transitive/suggested dep needing `composer require`. It is in fact a **hard requirement of `pestphp/pest` v2.36.1** — `php artisan test --parallel` already worked. Still pinned `brianium/paratest: ^7.4` explicitly in `require-dev` (we invoke it directly in CI; don't ride a transitive) and refreshed `composer.lock`'s content-hash (no package changes).
- First `--parallel --exclude-group=slow` run @ 8 procs: **2463 passed / 1 failed**, count 2464 (matches the serial baseline exactly), wall **105s** vs **675s** serial.
- The single failure was the **only** genuine isolation defect: `firstSampleLogo()` defined in `Session156Test.php` and called cross-file from `ContentImportExportTest.php` via an `if (! function_exists())` guard — works only because serial runs load every test file; under paratest a worker may not load the defining file. A codebase-wide scan confirmed it was the *only* genuinely cross-file test helper (`makeUser()` was a false positive — three files each define their own).

### Phase 2 — Root-caused the isolation defect at source

- Moved `firstSampleLogo()` into `tests/Support/helpers.php`, registered via `composer.json` `autoload-dev.files` (loaded in every process, incl. each paratest worker). Removed the in-file definition. `composer dump-autoload` regenerated.
- The release-plan-named hazards did **not** materialise as defects: the `storage/app/private/` tests already use `Storage::fake('local')` (per-test isolated); the `seedWidgetCollections` test is in `->group('slow')` and ran green under `--parallel --group=slow` (18/0, no row leak). Build-pipeline real-artifact writers were already temp-dir-isolated by session 296 — confirmed holds under `--parallel` (Open Q2: not re-solved).

### Phase 3 — Order-independence proven; `--parallel` made default

- Identical **2464 passed / 0 failed** across: 8-proc default (~105s), 8-proc `--order-by=random` (102s), 4-proc redistribution (180s); `--group=slow` 18/0 (115s); `--group=design` 213/0 (22.9s). Order-independent across within-worker order and cross-worker file distribution. (Assertion-count wobble 14735↔14736 is data-dependent random/canonical assertions, not a coverage delta — test count is the invariant and is identical every run.)
- `--parallel` wired as the default fast invocation: `./dev test`, `./dev test:design`, the CI `fast` job, and `CLAUDE.md`. `--group=design` + the integrity guard preserved.

### Phase 4 — Isolated e2e stack + parameterised harness

- **Open Q3 resolved with the user: faithful duplicate.** `docker-compose.e2e.yml` — a self-contained, port-shifted stack (project `nonprofitcrm_e2e`; nginx 8090/8443, postgres 5442 — 5433 was taken by `fleetmanager_postgres`), own throwaway `postgres_data_e2e`/`redis_data_e2e`/`app_node_modules_e2e` volumes, reusing the dev-built `nonprofitcrm-app:latest` image + repo bind-mount (no second checkout, no second build). `APP_URL` overridden in-container; `config:clear` on bring-up (shared bind-mounted config cache). `./dev e2e:up|e2e:down`, `npm run e2e:up|e2e:down|test:e2e:isolated`.
- **Parameterised the three hardcoded targets** via `tests/e2e/helpers/stack.ts` (`composeExecArgs`, `pgConnectionConfig`); `db.ts` + `fake-csv.ts` refactored to it. Dev-stack defaults are byte-identical, so existing local usage is unaffected; the isolated stack is opt-in purely via env. `APP_URL` was already env-driven.
- **Proven the parameterised reset targets the isolated DB and never the dev DB:** dev `users` count 1 → 1 (unchanged) across a destructive `migrate:fresh --seed` aimed at the isolated stack; e2e DB `relation "users" does not exist` → seeded. Fully separate Postgres instances/volumes/networks. A representative spec then passed end-to-end on the isolated stack.

### Phase 5 — Playwright CI job + e2e suite stabilisation

- New `playwright` CI job: brings up the same isolated compose stack in CI (Docker available on `ubuntu-latest`), reuses the parameterised harness with **zero CI-specific harness code**, runs chromium after the Pest jobs (`needs: slow` — Pest and Playwright never concurrent within a run, the never-together rule honoured in CI too), uploads the report on failure, tears down.
- **Deliberately NOT in the `tests` merge-gate `needs`.** The e2e suite is environment-sensitive; a flaky required check would block every merge to `main`. The job runs/reports on every push; `tests` still gates on `[fast, slow]` exactly as session 298 established. The workflow comments document the one-line promotion path once the suite is reliably green.
- **Full chromium e2e (49 specs) root-caused and fixed** — detail in commit `3ee789d`. Summary: (1) `donations-error-report` — a genuine Phase-4 miss (an inline `docker compose exec -T postgres` bypassed the harness → setup data hit the dev DB); fixed with a parameterised `insertDuplicateContactsByEmail()` in `db.ts`. (2/3) `events-happy-path` / `events-update-strategy` — **time-rotted fixtures** (hard-coded 2026 dates; `2026-05-15` is now past, and the admin list shows only upcoming events); fixed with `futureDatedEventsCsv()` (deterministic per-row future dating, rot-proof, update pass still title-only). (4) `donations-mapping-indicator` — clicked a lazily-initialised Choices.js dropdown before init; gated on the wrapper. (5) `ticket-tier-picker` — public landing page's first **cold render** (runtime SCSS compile) exceeds the window on a CPU-contended box (dev + isolated stacks share the host); networkidle + generous first-assertion timeouts + a **scoped `retries: 2`** on that serial describe.

## Serial-only / retry carve-out + rationale

`ticket-tier-picker` carries `test.describe.configure({ mode: 'serial', retries: 2 })`. This is **not** a skip or a coverage reduction — the test must still pass green; the retry only absorbs a *proven* environment transient: the public event page's first cold render (runtime SCSS compilation) on a host running the dev stack alongside the isolated e2e stack. Every warm run passes deterministically (stress: cold attempt 1 fails, attempts 2–5 pass, 4/4 each). Surfaced to and accepted by the user. Two full order-independent runs both exit 0: 48 passed + ticket-tier passing on retry, honestly reported by Playwright as `1 flaky` rather than hidden.
