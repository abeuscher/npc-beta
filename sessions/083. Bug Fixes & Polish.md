## Design decisions (resolved before starting)

- npm/Vite: switch from `npm run build` to `./node_modules/.bin/vite build` in `SiteThemePage.php` and verify end-to-end on the server.
- SCSS permission error: fix file ownership/permissions so the PHP-FPM process can write `_theme.scss`, or move the write target to a path that is already writable. Do not change the write path without confirming it will survive a container rebuild.
- Site Theme logo: follow the Admin Settings logo field exactly as the reference implementation.
- Event widget slug vs. ID: all event widgets must reference events by slug. Audit every event widget type and fix any that use a record ID.
- Blog/page authors: add a nullable `author_id` FK to the `pages` table (references `users`). Default to the authenticated user on creation. Expose as a searchable dropdown on the page edit form.
- Auto-publish: CMS Settings gets two new toggles (`auto_publish_pages`, `auto_publish_posts`), both defaulting to on. New events also default to published.
- CMS Editor role: investigate why it is not deletable. If no documented reason exists, make it deletable. Only `super_admin` must be indestructible.
- Nav scroll smoothing: attempt `scroll-behavior: smooth` on the Filament nav scroll container first. Do not fight the framework further if that alone is not sufficient.
