# 191. CRM Importer — Polish, Shared Code & Source Templates

Extract shared code from the five duplicate-then-diverge importer page pairs, add downloadable CSV templates per content type, add value-pattern noise detection for Wild Apricot exports, and extend per-source skip-header lists. This is the consolidation pass after three entity-focused sessions (188 contacts, 189 events, 190 financials).

---

## Design decisions (resolved before starting)

- **Traits, not an abstract base class.** The five wizard pages and five progress pages share large amounts of code but diverge in entity-specific logic. Extract shared behaviour into traits (`InteractsWithImportWizard` for wizard pages, `InteractsWithImportProgress` for progress pages) rather than an abstract base class. Traits are more composable and avoid the diamond-inheritance problems that would arise if a future importer needs to mix behaviours from two different bases.
- **Contacts importer stays divergent where it must.** The contacts importer has staged updates, duplicate strategies (skip/update/duplicate), and column-preference resolution that the other four importers don't. The trait extracts the shared subset; contacts-specific logic stays in `ImportContactsPage` / `ImportProgressPage`. Don't force-fit contacts into a shape it doesn't match.
- **CSV templates are generated from field registries.** Each content type's canonical CSV template is produced by reading its field registry (e.g. `ContactFieldRegistry`, `DonationFieldRegistry` + `TransactionFieldRegistry`). No separate template definition to maintain — the registry is the source of truth. Templates include a header row only (no sample data rows).
- **Templates are downloadable from the wizard page.** Each wizard's Upload step gets a "Download CSV template" link. Also available from a new section on the Importer landing page. No artisan command needed — the download is a Livewire action that streams a CSV.
- **Noise detection is heuristic, not blocking.** Value-pattern detection identifies columns whose cell values look like procedural cruft (Wild Apricot `Field&&Visibility` concatenations, structured key-value dumps, very long metadata strings). These columns default to unmapped with a banner. The user can still opt in. Detection runs during the mapping step, not during upload.
- **Per-source skip-header lists extend the global list.** `FieldMapper::SKIPPED_HEADERS` remains the global baseline. Each seeded source (Generic / Wild Apricot / Bloomerang) gains an additional array of known-noise headers. When a source is selected, its skip list is merged with the global list. Admin overrides are out of scope — they can always manually map a skipped column.
- **Update-existing strategy is deferred to a follow-up.** The session outlines stub lists this as a requirement, but it is architecturally distinct from the polish/refactor work in this session and large enough to warrant its own prompt. This session documents the gap clearly in the outlines and defers it. The refactored trait structure will make it easier to add later.

- `deriveContactMatchKey()` — identical.
- Upload step schema builder — the Source section (session label, source name/select, file upload) is identical. Extract as a method that each page calls, passing only the default label prefix and source helper text.

## Phase 3 — Downloadable CSV templates

Add a CSV template download for each content type. The template is a single header row with the canonical column names from the relevant field registries.

Templates:

- **Contacts** — all fields from `ContactFieldRegistry::fields()` keys, humanised. Plus `External ID`, `Organization`, `Tags`, `Notes`.
- **Events & Registrations** — event fields + registration fields + contact match fields + transaction fields, with namespace prefixes removed and human-readable names (e.g. `Event Title`, `Registration Status`, `Contact Email`, `Transaction Amount`).
- **Donations** — donation fields + transaction fields + contact match fields.
- **Memberships** — membership fields + contact match fields.
- **Invoice Details** — invoice detail fields + contact match fields.

Implementation:

- Each wizard page gets a `downloadTemplate()` Livewire action on the Upload step — a small link below the file upload field: "Download a blank CSV template for this content type."
- The Importer landing page gets a "CSV Templates" section below the import cards with download links for all five types.
- Each download streams a CSV with a single header row. No sample data — just the column names.
- Column names in the template should match the `guessDestination()` heuristics so that re-importing a filled template auto-maps perfectly.

## Phase 4 — Value-pattern noise detection

Add heuristic detection for columns whose values are procedural metadata rather than real data. Runs during the mapping step after CSV parsing.

Patterns to detect:

- **Wild Apricot `Field&&Visibility` concatenations.** Cells contain newline-separated `FieldName&&VisibilityLevel` pairs (e.g. `Email&&Nobody\nFirst name&&Anybody`). Regex: multiple lines matching `\w+&&\w+`.
- **Very long metadata strings.** Cells averaging >200 characters across the sample rows, with no clear delimiter pattern. Likely system dumps.
- **Structured key-value strings.** Cells containing repeated `key: value` or `key=value` patterns across multiple lines.

When detected:

- The column's mapping select defaults to `null` (unmapped) instead of auto-custom-field.
- A per-column helper text appears: "This column appears to contain system metadata. It has been left unmapped — you can still map it if the data is meaningful."
- No blocking — the user can always override.

Implementation: a `NoiseDetector` service in `App\Services\Import\` with a static `detect(array $sampleValues): bool` method. Called during `processUploadedFile()` for columns that would otherwise fall through to auto-custom-field or remain unmapped.

## Phase 5 — Per-source skip-header lists

Extend `FieldMapper` with per-source skip-header arrays. When a source preset is selected, its known-noise headers are merged with the global `SKIPPED_HEADERS` list.

Wild Apricot noise headers (from WCG UAT):
- `Password`
- `Archived`
- `Subscribed to emails`
- `Subscription source`
- `Opted in`
- `Administration access`
- `Profile last updated`
- `Last login`
- `Updated by`
- `Access to profile by others`
- `Details to show`
- `Photo albums enabled`
- `Member bundle ID or email`
- `Member role`
- `Group Participation`

These are already ignored by most users during mapping, but having them auto-skip saves time and reduces mapping noise on wide CSVs.

Implementation:

- Add `FieldMapper::sourceSkippedHeaders(string $sourcePreset): array` returning the merged list.
- Update `processUploadedFile()` in the wizard trait to call `FieldMapper::isSkipped()` with the source-aware list.
- Seeded source presets (`Generic`, `Wild Apricot`, `Bloomerang`) each carry their skip list. Generic has none beyond the global baseline.
