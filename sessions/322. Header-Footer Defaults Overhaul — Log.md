### Phase 1 — Nav widget "Columns" layout preset

- New **Layout** field on the Nav widget (`NavDefinition`): `horizontal` (default) / `columns`. Concrete default added to `defaults()`.
- `template.blade.php` gained a dedicated **columns render path** (branch on `orientation === 'columns'`): each top-level menu item renders as a heading column (`widget-nav__column-heading`, a link when it resolves to a real destination, else a plain label) with its children listed beneath as a visible vertical link list (`widget-nav__column-links` / `widget-nav__column-link`). **No dropdowns, no Alpine/JS** in this path — every link visible ("kitchen-sink" footer exposure). Guarded the `is-active` class against `#` placeholder hrefs (the existing `isActive` helper treats `#` as `/`).
- `styles.scss`: `.widget-nav--columns` — flex row of columns, `flex-wrap` so columns wrap on narrow widths, row placement following the existing `--nav-justify` alignment property; heading/link colors follow the nav link/hover color vars.
- Registry synced (`WidgetTypeSeeder`) so the new field surfaces in the config editor; `build:public` rebuilt the widget bundle (the columns CSS lives there).

This replaced an earlier intra-session `stacked` (single vertical column, dropdown children) take after the owner clarified the target with a reference image — the real ask was Stripe-style heading columns with all links visible.

### Phase 2 — Seeded default footer ships the columns nav

- `SystemPageSeeder::seedFooterWidgets()` now seeds a **columns** Nav widget (centered) above the copyright, pointed at a seeded `footer` NavigationMenu of headed groups — **Organization** (About, Contact) / **Activities** (Events, News) / **Legal** (Privacy Policy, Terms of Use as `#` placeholders). A genuinely fresh `migrate:fresh --seed` now renders a three-column footer instead of a bare copyright line. The early-return-if-widgets-exist guard means existing installs (incl. the owner's 284-edited footer) are untouched.
- **Copyright** centered (already), with **25px top / 150px bottom** padding via the widget's `appearance_config.layout.padding` (partial appearance config, matching the header-layout convention already in this seeder; avoids forcing a white background box if the band is ever tinted).
