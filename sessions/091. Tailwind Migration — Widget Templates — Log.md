## What was built

### Phase 1 — Content widgets

- `blog-listing.blade.php` — styled headings, post list with `space-y-3`, date in muted text, dark mode variants.
- `blog-pager.blade.php` — flexbox nav with border-top separator, text alignment for prev/next, dark variants.
- `events-listing.blade.php` — card-style articles with bottom borders, "Free" badge using green bg/text, dark variants.
- `event-description.blade.php` — heading, date paragraph, and description div all converted from Pico class names to Tailwind utilities.
- `event-registration.blade.php` — full form conversion: alert boxes (success/error/cancelled), labeled inputs with `@tailwindcss/forms` baseline, fieldset for address, honeypot preserved with original inline styles, buttons styled as primary.
- `column-widget.blade.php` — added `gap-6` Tailwind class alongside existing inline grid styles (dynamic column config requires inline styles).

### Phase 2 — Commerce widgets

- `donation-form.blade.php` — toggle-button groups for amount presets and frequency with Alpine `:class` for active state (bg-primary + white text), custom amount input, fund select, error alert, processing state on submit button.
- `product-display.blade.php` — price cards in bordered rows with flex layout, waitlist form, status alerts (success/cancelled/waitlist), delete old classless markup.

### Phase 3 — Form & utility widgets

- `public-form.blade.php` (component) — replaced `.form-grid` / `.col-N` CSS system with Tailwind `grid grid-cols-12` / `col-span-{width}` + `max-md:col-span-12` for mobile full-width. All field types (text, email, tel, number, textarea, select, state, country, radio, checkbox) styled with consistent label/input/hint/error pattern.
- `image.blade.php` — replaced inline `style="object-fit:..."` with Tailwind `w-full block object-cover`/`object-contain` classes via the `class` prop on `<x-picture>`.
- `carousel.blade.php` — no changes needed; custom CSS in `_custom.scss` works correctly with Tailwind's reset.

### Phase 4 — Portal widgets

- `portal-login.blade.php` — form with labeled inputs, remember-me checkbox, navigation links.
- `portal-signup.blade.php` — form with honeypot, labeled inputs, password hint, error spans, navigation link.
- `portal-forgot-password.blade.php` — single-field form, navigation link.
- `portal-change-password.blade.php` — three password fields with inline JS match-check preserved, success/error alerts.
- `portal-contact-edit.blade.php` — two sections (mailing address, email change) with `mb-8` separator, household address choice buttons.
- `portal-event-registrations.blade.php` — responsive table with `overflow-x-auto`, header borders, row dividers.
- `portal-account-dashboard.blade.php` — welcome text, definition list with flex layout for email/household.

### Phase 5 — Cleanup & visual audit

- Removed `.form-grid` / `.col-N` / mobile override CSS from `_custom.scss` (replaced by Tailwind grid in public-form component).
- Carousel CSS retained in `_custom.scss` — still needed for aspect-ratio container and Swiper integration.
- Build output: 54.37 kB CSS (9.82 kB gzipped), 135.41 kB JS.
- Fixed one test (`EventTest::event_description widget renders event date`) that asserted the old `event-date` class — updated to assert `<time datetime=`.

### Additional fixes (not in original scope)

- **Page builder inspector — spacing panel**: fixed the Padding/Margin section in Advanced Settings. Replaced cramped 6-column layout with 5 evenly-spaced columns using Filament's CSS variable pattern (`grid-cols-[--cols-default]`).
