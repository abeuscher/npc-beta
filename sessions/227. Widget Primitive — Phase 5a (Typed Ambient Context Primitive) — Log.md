### Notable findings

- **Laravel container auto-resolves `?Page` to a non-null unsaved Page.** During verification, my initial `PageBuilderCanvasSlotTest` assertion `expect($ctx->ambient->currentPage)->toBeNull()` for the no-override case failed because `app(PageContext::class)` resolves the `?Page $currentPage = null` parameter into a fresh, non-existent Page object rather than null — the container treats nullable type hints on concrete classes as resolvable. The public API `$ctx->currentPage()` still returned null because the existence check survived in the delegate (`$page && $page->exists`), so byte-equivalence is preserved. Dropped the inner-field assertion, kept the public-API one. Carry-forward: any future test that introspects `$ctx->ambient->currentPage` directly should construct the ambient explicitly with `new PageContext(null)` (or `new PageAmbientContext(null)` directly) rather than relying on `app(PageContext::class)`.
- **The "a+" View framing was locked in during close-gate scoping for 5b.** While drafting the 5b session prompt, the question of how `RecordDetailView` relates to `Page` and `DashboardConfig` surfaced. The decision: **a `View` is the shared widget-composition primitive expressed as the `App\WidgetPrimitive\IsView` PHP interface**, with concrete implementations free to be plain PHP classes or Eloquent models. `DashboardView` and `RecordDetailView` adopt View vocabulary natively from their introduction; CMS `Page` keeps its own table and full public-facing surface (slug, SEO, publishing, type, hierarchy, blog/post discrimination) and implements `IsView` as a small adapter only if/when "walk every widget-mounting surface" becomes load-bearing — could be never. **No unified `views` table, no Page refactor.** The marketplace's extensibility surface is widgets+slots+sources+ambients, not Views — third-party widget developers contract against those four primitives and the marketplace works whether Views are unified or sibling. Documented in `widget-primitive-migration.md`'s "Design decisions locked in" section (new entry) and "Forward hooks" section (new "Page-as-View elective" entry). The Templates/Themes record sub-navigation refactor (previously a separate stub at `session-outlines.md` line 457 area) folds into Phase 5c as a consumer of the same sub-nav primitive — annotated in the outlines.
