# 037. MailChimp Integration — Log

## What was built

### Contactability base filter
`MailingListQueryBuilder` updated to apply `do_not_contact = false AND mailing_list_opt_in = true` as a base filter on all list queries. This affects contact counts, CSV export, and MailChimp sync universally.

### MailChimp SDK
`mailchimp/marketing` installed via Composer.

### MailChimpService
`app/Services/MailChimpService.php` — handles all MailChimp API interaction:
- `isConfigured()` — checks that API key, server prefix, and audience ID are present
- `syncList(MailingList $list)` — chunks contacts in groups of 500, submits two batch API jobs per chunk: one to upsert members (with FNAME/LNAME merge fields), one to apply the list name as a MailChimp tag
- `removeTag()` — stub, not yet implemented

### Admin UI — Mail Settings
MailChimp credentials moved from `.env` to the admin Mail Settings page (DB-backed via `site_settings`, same pattern as Resend). Fields: API key, server prefix, audience ID, webhook path (the URL segment after `/webhooks/`), webhook secret (a user-chosen random string).

`AppServiceProvider` updated to load these values from `site_settings` into `config('services.mailchimp.*')` at boot.

### Sync action
"Sync to MailChimp" button added to the mailing list edit page header. Only visible when `MailChimpService::isConfigured()` returns true. Shows a confirmation modal, then calls `syncList()` and displays a success notification.

### Webhook endpoint
`app/Http/Controllers/MailChimpWebhookController.php` — handles `POST /webhooks/{path}?secret={secret}`:
- Validates secret against `config('services.mailchimp.webhook_secret')`; returns 403 if mismatch
- Parses `data` field as JSON string (MailChimp's format)
- On `type=unsubscribe`, sets `mailing_list_opt_in = false` on the matching contact by email

`GET /webhooks/{path}` returns 200 OK (MailChimp validation ping).

Both routes are excluded from CSRF via `bootstrap/app.php` (`/webhooks/*`).

The webhook path segment is configurable — set a random hash for security in production.

### Mailing list members widget
`MailingListMembersWidget` table widget added as a footer widget on the mailing list edit page, showing name, email, city, state of all members passing the contactability filter. Full width, paginated at 25/50/100.

### SSL & domain
- `beuscher.net` A record pointed at the production server (`167.172.141.225`)
- Let's Encrypt cert issued for `beuscher.net` via certbot standalone
- `docker/nginx/prod.conf` created with HTTP→HTTPS redirect and HTTPS server block with SSL cert paths
- `docker-compose.prod.yml` updated to mount `/etc/letsencrypt` into the nginx container and expose port 443
- UFW rules added for ports 80 and 443 on the host

### README
Deployment section updated with GitHub secrets table (`DEPLOY_HOST`, `DEPLOY_USER`, `DEPLOY_SSH_KEY`) and note that all other config lives in `.env` on the server.

## What was tested and confirmed working

- **Sync**: contacts pushed to MailChimp audience with correct merge fields and tags. Verified via MailChimp batch API response and MailChimp audience UI.
- **SSL**: `https://beuscher.net` serves the app correctly.
- **Webhook GET** (MailChimp validation ping): returns 200 OK.
- **Webhook POST via curl**: returns 200 OK and correctly updates `mailing_list_opt_in` on the matching contact.

## Outstanding issue — webhook 500 from MailChimp servers

Every real POST from MailChimp's servers returns HTTP 500 (consistent 6628-byte response). The curl simulation from the production server with MailChimp's exact payload format returns 200 OK. The 500 is not logged by Laravel or PHP-FPM.

A `Log::error()` call was added at the very top of `MailChimpWebhookController::handle()` (before the secret check) and deployed at the end of this session. If this log entry does not appear when MailChimp fires, the 500 is happening before Laravel boots.

Full debugging brief written at `sessions/038. MailChimp Webhook Debugging.md`.
