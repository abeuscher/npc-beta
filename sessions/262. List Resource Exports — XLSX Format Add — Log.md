### Phase 2 — Extend the writer service

- [`app/Services/ListExportService.php`](app/Services/ListExportService.php) gains a third format branch:
  - `stream()`'s content-type dispatch and stream-callback dispatch swap from if/else to `match` to accommodate three formats cleanly.
  - New private `streamXlsx(Builder $query, array $columnSpec, Collection $customDefs): void` parallel to `streamCsv` / `streamJson`. Uses `tempnam(sys_get_temp_dir(), 'xlsx-')` for the writer's target file; writes via `OpenSpout\Writer\XLSX\Writer::openToFile()`; iterates rows via `$query->each()`; emits via `addRow(new Row($cells))` after constructing cells manually for Option B styling; `close()` finalises; `readfile($tempPath)` streams to client.
  - **`try { ... } finally { @unlink($tempPath); }` wraps the entire write-and-stream sequence** (load-bearing per the prompt — disk-leak guard in case OpenSpout, the Eloquent iterator, or the Carbon parse throws mid-write).
  - New private `coerceForXlsx(mixed $value, ?string $type): bool|float|int|string|\DateTimeInterface|null`. Null/empty-string passes through; otherwise `match` on `'date'`/`'datetime'` → `Carbon::parse()`, `'number'` → `(float)`, `'boolean'` → `(bool)`, default → string.
- **Date/datetime cell styling.** OpenSpout writes `DateTimeCell` as a raw OOXML serial number; without a date-format style, Excel renders the cell as a number (e.g., `46036` instead of `2026-03-01`). Two cached `Style` instances (`(new Style())->setFormat('yyyy-mm-dd')` for `date`, `'yyyy-mm-dd hh:mm:ss'` for `datetime`) get attached per-cell via `Cell::fromValue($value, $style)` for the relevant typed columns. Costs ~5 lines, prevents the user-visible "looks like a number in Excel" failure mode.

### Phase 3 — Per-resource `'type'` hints

Touched 10 column specs to add the inventoried hints — all additive, no value-callable changes. No CSV/JSON behavior change since those paths ignore the `'type'` key.

|---|---|
| `ContactResource::exportColumnSpec` | `date_of_birth` (date), `created_at` (datetime) |

### In-session decisions

- **Cell-type semantics: Option B.** Confirmed at session start; landed verbatim per the inventory above.
- **Date-format styling.** Discovered during Phase 5 test-writing: OpenSpout's default DateTimeCell write produces a raw serial number, not a date-formatted cell. Added cached `Style` instances with `setFormat('yyyy-mm-dd')` / `'yyyy-mm-dd hh:mm:ss'` to Phase 2's `streamXlsx`. Costs ~5 lines, addresses the actual user-visible Excel render, distinct from the cell-type-detection question Option B resolved.
- **Test fractional value for number coercion.** A `goal_amount` of 5000.00 reads back as PHP int from OpenSpout's Reader (whole-number floats normalize). Switched the test fixture to 5432.10 to keep the `toBeFloat()` assertion meaningful. Doesn't affect production behavior — the cell is still a NumericCell either way; just a test-fixture choice.
- **`captureStream()` output-buffer leak under exception.** The existing `captureStream()` test helper does `ob_start(); $response->sendContent(); return ob_get_clean();` — if `sendContent()` throws, the buffer is never closed and PHPUnit flags the test as risky. The cleanup-on-exception test inlines its own `ob_start()` / `ob_end_clean()` around the throwing call rather than going through `captureStream()`. No change to the helper itself (other tests don't have throwing paths).
