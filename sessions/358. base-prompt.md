## Process rules for every session

Base-prompt process rules apply in full. Session-specific:

- **The two architectural choices were settled at planning — confirm, don't re-open.** (1) `demoContext()` returns a generic descriptor `['guard' => 'portal', 'seeder' => DemoPortalMemberSeeder::class, 'login' => DemoPortalMemberSeeder::ACCOUNT_EMAIL]`; the controller runs the seeder, resolves the guard's user provider, logs the member in by the `login` identifier, renders, and logs out in a `finally`. (2) **No per-widget framing/viewport feature** — strip widgets use the manual-thumbnail opt-out + a committed framed PNG. Both decisions are recorded in the session prompt's "Settled at planning" section; confirm they still hold at the gate, then build.
- **`demoContext()` is the sanctioned renderer change — keep it generic.** Session 357 deliberately avoided special-casing the renderer for portal widgets; this session adds the *general* lever (a widget declares it needs an auth context; the controller honors the declaration). It is **not** a per-widget `if (handle === …)` branch. The stand-in portal member is one shared fixture (the "portal is a package" framing): a Contact + a PortalAccount + one EventRegistration against the existing 357 demo event — no membership needed.
- **Manual-thumbnail opt-out before the strip-widget thumbnails.** The opt-out flag is the foundation the committed framed PNGs lean on; land it first so a framed capture is durable against `--all`.
- **Coverage guard is the capstone.** A Pest test asserting every registered widget's `static.png` is non-blank **or** on an explicit, justified allowlist (the blank-by-design set: RandomDataGenerator, SetupChecklist — and only those, with a one-line reason each). Mirror the pinned-list discipline of `WidgetManifestTest`'s `demoSeeder` list: editing the allowlist is a reviewed edit in the same pass.
- **Build / regenerate after changes.** Any `resources/js/**` change → `npm run build`. Regenerate affected widget thumbnails via `node scripts/generate-thumbnails.js` per its documented flow (host-side, global chromium); commit the regenerated `static.png` artifacts. Revert out-of-scope thumbnail churn — touch only the widgets this session addresses.
