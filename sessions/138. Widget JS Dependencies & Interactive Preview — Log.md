## What was built

### Phase 1 — Library dependency declarations

- Added `libs` key to the `assets` JSONB for 7 widget types: `carousel`, `product_carousel`, `logo_garden`, `events_listing`, `blog_listing` (all `["swiper"]`), `bar_chart` (`["chart.js"]`), `event_calendar` (`["jcalendar"]`).
- Updated `WidgetTypeSeeder` with `libs` entries for all affected widget types.
- Migration `2026_04_06_074630_add_libs_to_widget_type_assets` updates the live DB.
- Updated `docs/schema/widget_types.md` to document the `libs` key.

### Phase 2 — Build server per-library bundles

- Extended `AssetBuildService` with a `LIBRARY_SOURCES` constant defining JS source and CSS/SCSS paths for each library (swiper, chart.js, jcalendar).
- `buildLibraryBundles()` sends one `POST /build` per library to the existing build server, producing self-contained JS and CSS bundles at `public/build/libs/{name}.js` and `public/build/libs/{name}.css`.
- Manifest (`public/build/widgets/manifest.json`) gains a `libs` key mapping each library identifier to `{js, css}` paths.
- Updated `BuildPublicAssets` artisan command to report library bundles.
- No changes required to the bundleserver repo.

### Phase 3 — Admin JS loader

- `PageBuilderBlock::loadBlock()` now includes `widget_type_assets` in the `$block` array.
- `AdminPanelProvider` injects `window.__widgetLibs` from the manifest via a render hook.
- Preview blade template includes a dynamic loader (`loadLibs()`) that injects `<script>` and `<link>` tags for missing libraries before widget Alpine components initialize.
- Added `x-ignore` wrapper and `reinitWidgetAlpine()` to control Alpine init timing (will be superseded by session 139's page-level loading approach).

### Phase 4 — Responsive preview toggle

- Added Desktop (1920px), Tablet (1024px), Mobile (375px) segmented icon toggle above the preview.
- `viewportW` controlled by preset selection instead of `window.innerWidth`.
- Zoom factor recalculates on preset change.

### Session 139 prompt & outlines reorganization

- Wrote session 139 "Unified Preview & Edit Layout" prompt — pivots the page builder from per-block preview rendering to a persistent unified preview pane with page-level library loading, eliminating Swiper init timing issues.
- Reorganized session outlines: inserted session 139, removed numbers from remaining future stubs, reordered.
