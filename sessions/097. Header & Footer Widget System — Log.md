# Session 097 — Header & Footer Widget System — Log

**Date:** 2026-03-30
**Branch:** session-097

## What was built

### Phase 1 — System pages for header and footer

- Seeded `_header` and `_footer` Page records (type `system`, status `published`) via `SystemPageSeeder`.
- Added routing guard in `PageController::show()` — any slug starting with `_` returns 404.
- Registered `edit_site_chrome` permission in `PermissionSeeder` (not assigned to any role by default).

### Phase 2 — Header & Footer tab on Site Theme

- Split the combined "Header & Footer" concept into two separate tabs — **Header** and **Footer** — on the Site Theme page, each containing its own `PageBuilder` Livewire component instance. Gated by `edit_site_chrome` permission.
- Each tab includes an auto-save notice and a Save button that forces blur (triggering Livewire persistence).
- Removed the existing "Header & Footer" section from the Appearance tab form (logo upload, nav handles, header content, custom override status placeholders). Site settings remain in the database for fallback rendering.
- Reorganized Appearance tab layout — Colors and Fonts now side by side.

### Phase 3 — Purpose-built widget types

- Seeded `site_header` widget type: config fields for logo (image), nav_handle (text), header_content (richtext). Template replicates existing `components/site-header.blade.php` HTML structure.
- Seeded `site_footer` widget type: config fields for nav_handle (text), show_theme_toggle (toggle), copyright_text (text). Template replicates existing `components/site-footer.blade.php` HTML structure.
- Created widget Blade templates at `resources/views/widgets/site-header.blade.php` and `resources/views/widgets/site-footer.blade.php`.

### Phase 4 — Public layout rendering

- Created `ChromeRenderer` service (`app/Services/ChromeRenderer.php`) that renders a system page's widgets to HTML + styles + scripts, reusing the existing widget rendering logic.
- Updated `resources/views/layouts/public.blade.php` with three-tier precedence for both header and footer: custom Blade override > widget-driven > legacy fallback partial.
- Chrome widget styles/scripts are computed once and injected alongside page widget assets (no duplicate DB queries).

## Production deployment notes

Run seeders after deploying:
- `php artisan db:seed --class=PermissionSeeder` (registers `edit_site_chrome` permission)
- `php artisan db:seed --class=SystemPageSeeder` (creates `_header` and `_footer` system pages + seeds widget types)

Assign `edit_site_chrome` permission to appropriate roles via the admin UI.
