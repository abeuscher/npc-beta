## Process rules for every session

Base-prompt process rules apply in full. Session-specific:

- **JSON-LD is operator-injected script content — treat it as untrusted, but it is NOT HTML.** `HtmlSanitizer` would strip `<script>` and is the wrong tool. The safe path: **validate the input parses as JSON** (reject/flag invalid), then emit inside `<script type="application/ld+json">` with `<`/`</` escaped (e.g. `<` → `<`) so the value cannot break out of the script element. Settle the exact validation + whether multiple graphs (an array) are allowed at the Open Gate. Security-reviewed; Pest cases for the breakout-escape.
- **Reuse before building** — if the per-page `head_snippet` already renders into `<head>`, per-page custom JSON-LD may already be servable; scope the per-page half to what's genuinely missing and focus the build on the **site-wide** slot. Confirm at the Open Gate.
- **Blockquote first, citation as the design call** — Quill's blockquote is a native toolbar format (cheap to expose). `<cite>` has no native Quill blot; settle the v1 citation shape (a custom blot, an attribution line, or defer citation) rather than over-building.
- **Builds:** `npm run build` after the Quill toolbar / Vue editor changes (blockquote control) and any `resources/js`/`resources/scss` change; `build:public` after any widget asset change. The JSON-LD + responsive-`sizes` work is Blade/PHP (no build) unless it touches a widget asset.
- **Settle the open questions at session start** (JSON-LD storage + injection-safety + multi-graph; citation shape; responsive-`sizes` derive-vs-override) — log them.
