**Date:** 2026-04-04

## What was built

### Phase 1 — Widget type registration

- Added `social_sharing` widget type to `WidgetTypeSeeder` with config fields: heading, platforms (checkboxes), alignment, icon_size, background_color, text_color, full_width, mastodon_instance.
- Introduced `checkboxes` config field type: new inspector partial at `resources/views/livewire/partials/inspector-fields/checkboxes.blade.php` with CSS grid layout driven by `columns` attribute.
- Added `toggleCheckbox()` Livewire method to `PageBuilderInspector` for toggling values in/out of a checkboxes config array.
- Added `checkboxes => []` fallback in `WidgetType::getDefaultConfig()`.

### Phase 2 — Blade template and SCSS

- Created `resources/views/widgets/social-sharing.blade.php` with share URLs for all six platforms, Alpine.js copy-link with clipboard API and "Copied!" tooltip, `target="_blank" rel="noopener"` on all external links.
- Created `resources/scss/widgets/_social-sharing.scss` with flexbox row, alignment modifier classes, icon sizing via `--icon-size` CSS custom property, copied tooltip positioning.
- Created six SVG icon partials in `resources/views/widgets/partials/share-icons/`: bluesky, mastodon, email (Lucide), copy_link (Lucide), linkedin, facebook. All monochrome using `currentColor`.
- Installed `lucide-static` npm package for future icon needs.

### Additional changes

- **Hero widget:** added `background_color` and `text_color` color picker config fields (side by side via `group`). Background color injected into `--hero-bg` CSS custom property shorthand so it composites with background images. Fixed `.hero-bg` div rendering to also appear when only background color is set (no image). Fixed `$hasBg` flag to include background color.
- **PostController:** added `overlap_nav` / `__navOverlap` check (was missing entirely — hero full-bleed mode didn't work on blog post pages).
