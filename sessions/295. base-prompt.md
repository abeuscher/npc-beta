## Process rules for every session

Base-prompt process rules apply in full. Session-specific notes:

- **The migration is the risk, not the math.** This install has configured, hand-tuned typography (285 touched it). The flat-`font.size` → `{xl,lg,md,sm}` shape change must preserve every existing value **byte-exact at `xl`** and derive `lg/md/sm` from the per-class ramp only. A lost or mutated tuned value is silent data loss — the G1/285 shape. This is the required, load-bearing regression test; write it first.
- **Per-class default ramp, not a flat step.** Use the stub's calibration table (display/section/body × sm/md/lg/xl/xxl). The original uniform 25%-per-step rule is **superseded** — it over-shrinks body ~6×. Body default is 1.0 across the board (no scaling) unless the user overrides; only display and section carry a real ramp.
- **Open design call — settle it in-session, surface it, don't silently pick.** The 3-site calibration mean leans slightly aggressive for NonprofitCRM's deliberately-restrained register (`references.md` names Attio, the gentlest at ~0.62 phone, as the closest match). Bias the display phone default toward ~0.6 rather than the raw ~0.54 mean — but present the chosen ramp for confirmation at the manual-test handoff; this is a judgment surface, not an objective one.
- **Line-height rides the size.** Store line-height as a unitless multiplier that scales with the per-breakpoint font-size; do not emit per-breakpoint line-height px. Confirmed across all three calibration sites.
- **Heading bottom-margin as an em multiple.** Default ≈ 0.4em (h1/display), ≈ 0.5em (h2–h3/section), em-relative so it auto-scales with the ramp. The value leans on convention + the one site that styles headings directly (clay) — the multi-site average was inconclusive because most sites offload that spacing to container `gap`, which a richtext CMS cannot rely on. User-overridable.
- **Compiler emission is the only place the fix can live.** `TypographyCompiler::compile()` is injected as a post-bundle inline `<style>`; a pure-SCSS type clamp is silently overridden on this configured install. The change lands in `elementDeclarations()` (xl + three `@media` blocks) — not in `resources/scss/`.
- **The oracle holds the floor; the user tunes the ramp.** Build an objective check (body never below a legible minimum, no text overflows its container, no horizontal scroll, a heading never renders smaller than the body beneath it, hierarchy monotonic per breakpoint) over the five public pages and the Design System preview. Drive it to zero. The ramp *values* are the user's tuning surface — pause for that judgment, don't automate it.
- **No contract change.** If the work appears to need a Fleet Manager contract or `/api/health` touch, stop and surface — scope drifted.
- **Run the fast suite when implementation is complete:** `docker compose exec app php artisan test --exclude-group=slow`. Expect the 2450 / 0 baseline plus the new 295 tests, all green.
- **Pause for manual testing when human judgment is required.** End-of-session: announce the oracle is at zero, surface the chosen ramp (with the restrained-vs-mean call called out) and before/after multi-breakpoint captures for the user's visual review, then stop and wait.
