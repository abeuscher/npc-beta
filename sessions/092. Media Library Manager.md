# 092. Media Library Manager

Add a basic admin page under the CMS group that surfaces all media stored via Spatie Media Library, allowing browsing, inspection, and deletion of files.

---

## Design decisions (resolved before starting)

- **Read-only browsing with delete.** This page lists and displays existing media. Uploading new media remains the responsibility of each widget/collection editor — this page does not introduce a new upload flow.
- **Single Filament page, not a resource.** Media records are owned by other models (PageWidget, CollectionItem, EmailTemplate). A standalone resource would imply full CRUD which is not appropriate. A custom Filament page with a table is the right fit.
- **Grouped by owner.** The table should show what each media item belongs to (model type and label/name) so the user understands context.
- **Conversion status visible.** Show whether WebP/responsive conversions have been generated, so the user can spot failed conversions at a glance.
- **Bulk delete not in v1.** Single-item delete only to keep risk low. Deletion removes the media record and its files from disk.

---

## Phase 1 — Media Library page

Build `app/Filament/Pages/MediaLibraryPage.php` under the CMS navigation group.

The page displays a Filament table (via `HasTable`) querying the `media` table directly with the following columns:

- **Thumbnail** — small preview image (use the media URL; fall back to a file-type icon for non-images).
- **File name** — `file_name` column.
- **Type** — `mime_type`, displayed as a short label (e.g. "JPEG", "PNG", "SVG", "WebP").
- **Size** — `size` in human-readable format (KB/MB).
- **Owner** — derived from `model_type` and `model_id`. Show the model's display name where possible (e.g. collection item title, widget label, email template name). Fall back to "Unknown" for orphaned records.
- **Collection** — `collection_name` column.
- **Conversions** — badge showing conversion status: green if all registered conversions completed, yellow if partial, gray if none.
- **Uploaded** — `created_at`, human-readable relative date.

Table features:

- Searchable by file name.
- Filterable by model type (PageWidget, CollectionItem, EmailTemplate) and by mime type group (Images, Documents, Other).
- Sortable by file name, size, and upload date.
- Default sort: newest first.

---

## Phase 2 — Delete action and storage summary

- **Delete action** on each row. Confirmation modal with the file name. Deleting removes the Spatie media record (which cascades to disk cleanup via Spatie's observer).
- **Storage summary stats** in the page header: total file count, total storage size, and count of media with incomplete conversions.

---

## Security checklist

Before closing:

- [ ] Delete action requires appropriate permission (use existing `manage_cms` or equivalent).
- [ ] No public route exposed — page is admin-only behind Filament auth.
- [ ] Deletion uses Spatie's `->delete()` method to ensure disk files are cleaned up.

---

## Out of scope

- Upload flow from the media library page (media is uploaded via widget/collection editors)
- Bulk actions (delete, re-process)
- Re-triggering failed conversions from the UI
- Drag-and-drop or gallery view
- Media reuse / "pick from library" in widget editors
