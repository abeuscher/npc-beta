# 148. Editor Canvas in Vue

## Design decisions (resolved before starting)

- **Preview HTML source:** The store's `preview_html` property on each root widget (populated from bootstrap data and refreshed via `GET widgets/{id}/preview`) is injected into the canvas via `v-html`. No client-side Blade rendering.
- **Overlay approach:** Each `PreviewRegion` renders a transparent overlay div on top of the widget HTML. The overlay captures clicks (for selection) and prevents interaction with links, forms, and carousels inside the preview. `pointer-events: none` on the inner HTML div, `pointer-events: auto` on the overlay.
- **Viewport zoom:** The canvas container uses CSS `zoom` (matching the existing `preview-manager.js` approach). The composable computes `zoomFactor = min(1, paneWidth / presetViewport)`.
- **Library loading:** Widget JS libraries (Swiper, Chart.js, jCalendar) are loaded into the document head on mount and re-initialised when preview HTML changes. Port the logic from `preview-manager.js` into a `useLibraryLoader` composable.
- **Alpine re-init:** After injecting new preview HTML, Alpine components inside widget previews need to be destroyed and re-initialised (same `reinitWidgetAlpine` pattern from `preview-manager.js`).
- **No drag-and-drop this session.** Drag-and-drop reorder moves to session 150 alongside the handles-mode block list.
- **Toolbar event bridge:** The "+ Add Block" and "Save as Template" buttons in the Vue toolbar dispatch browser `CustomEvent`s that the existing Livewire modals listen for. The widget picker returns its result via a `widget-created` event that the Vue store listens for. See migration plan Part 3, section 3.3.
- **Parallel operation continues:** Both the Livewire editor and Vue canvas are visible (Vue section below Livewire, same as session 147). The Livewire editor remains functional. Removal happens in session 151.

## Phase 1 — Preview canvas components

### 1a. `useViewport` composable

Create `resources/js/page-builder-vue/composables/useViewport.ts`:

- Reactive state: `presetViewport` (default 1920), `zoomFactor` (computed)
- `computeZoom(paneWidth: number)` — sets `zoomFactor = Math.min(1, paneWidth / presetViewport)`
- `setViewport(width: number)` — updates preset and triggers recompute
- Expose the three presets: `{ width: 1920, label: 'Desktop' }`, `{ width: 1024, label: 'Tablet' }`, `{ width: 375, label: 'Mobile' }`

### 1b. `useLibraryLoader` composable

Create `resources/js/page-builder-vue/composables/useLibraryLoader.ts`:

- Port the `loadLibs()` and `reinitWidgetAlpine()` logic from `preview-manager.js`
- `loadLibs(libs: string[])` — loads CSS/JS from `window.__widgetLibs` manifest, idempotent
- `reinitAlpine(scopeEl: HTMLElement)` — destroys and re-inits Alpine trees inside the scope element, handles Swiper/Chart.js cleanup
- Both methods should be async and handle errors gracefully (console.warn, don't throw)

### 1c. `PreviewRegion.vue`

Create `resources/js/page-builder-vue/components/PreviewRegion.vue`:

- Props: `widget` (Widget object)
- Renders the widget's `preview_html` via `v-html` inside a container div
- Transparent overlay div on top captures clicks → calls `store.selectBlock(widget.id)`
- CSS class `preview-region--selected` applied when this widget is selected
- Emits no events — selection is handled via the store

### 1d. `PreviewCanvas.vue`

Create `resources/js/page-builder-vue/components/PreviewCanvas.vue`:

- Uses `useViewport` composable for zoom state
- Viewport toggle buttons (Desktop/Tablet/Mobile) in a header bar
- Scrollable container with the viewport-width inner div (same CSS zoom approach as current Blade)
- Renders a `PreviewRegion` for each root widget from `store.rootWidgets`
- On mount: calls `useLibraryLoader.loadLibs()` with `store.requiredLibs`
- Watches for preview HTML changes → calls `reinitAlpine()` on the scope element
- Empty state: "No blocks yet" message when no widgets exist
- Links and form submits inside the preview scope are prevented (`@click` and `@submit` handlers on the scope)

## Phase 2 — Editor toolbar

### 2a. `EditorToolbar.vue`

Create `resources/js/page-builder-vue/components/EditorToolbar.vue`:

- Block count display
- Edit / Handles mode toggle (calls `store.setMode()`)
- "Save as Template" button (visible when blocks > 0) — dispatches `open-save-template-modal` CustomEvent
- "+ Add Block" button — dispatches `open-widget-picker` CustomEvent with no parent/column context

Style to match the existing Livewire toolbar (same Tailwind-style classes).

### 2b. Event bridge — Vue ↔ Livewire

In `App.vue`, set up event listeners on mount:

- Listen for `widget-created` CustomEvent → call `store.loadTree()` or replace tree from event payload. **Note:** The existing Livewire `createBlock()` method doesn't dispatch this event yet. Add a browser dispatch at the end of `PageBuilder::createBlock()` that sends the updated tree data so the Vue app can stay in sync. Same for `deleteBlock()`, `copyBlock()`, `updateOrder()`, and `saveAsTemplate()`.
- Listen for `template-saved` → no-op or show a notification

In `PageBuilder.php`, add `$this->dispatch('widget-tree-updated', tree: $this->getBootstrapData())` after mutations (createBlock, deleteBlock, copyBlock, updateOrder) so the Vue app can re-sync when Livewire modifies the widget tree.
