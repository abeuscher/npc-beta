## Process rules for every session

Base-prompt process rules apply in full (`sessions/template-base-prompt.md`). Session-specific notes:

- **All-or-nothing rollout.** `@layer` is safe only if EVERY host rule is layered in one pass. Any unlayered rule — a missed partial, the runtime typography emit, a colour rule, an operator's custom SCSS — beats every layered rule and silently flips precedence sitewide. There is no incremental version of this change: the whole compiled bundle adopts layers together, or not at all.
- **Behavior-preserving by construction, then verified.** The intended visible change is **zero** — precedence today already mostly resolves the "right" way via source order plus the workarounds; `@layer` makes that robust, not different. Produce a **before/after diff of the compiled CSS** and a visual gate on representative public pages. The proof the layering took: the EventMiniCalendar / EventsListing specificity workarounds revert to natural declarations and still render correctly.
- **Decide the layer order and the custom-SCSS slot explicitly — surface before going deep.** A sane order is `reset, base, components, widgets` (host base lowest, widget interiors highest). Load-bearing decisions: where the runtime typography/colour emit sits (low, so widget heading rules can win — which retires the workarounds), and whether operator `custom_scss` is layered (and where) or left unlayered so it wins over everything (operators likely expect their own CSS to win). These are architectural — bring a recommendation to the Open Gate; do not commit-and-unwind.
- **Preserve the deliberate reach-ins.** The inherited-text-colour path (widgets inherit `.np-site` text colour) and any intentional host→widget inheritance must survive. Custom properties (`--np-color-*`) are unaffected by layers (they cascade normally); rule-based colour must be placed so inheritance still works.
- **Front-end builds:** host SCSS / partials / `theme/public.scss` → `npm run build`; anything that changes the widget bundle or the runtime emit → `docker compose exec app php artisan build:public`. Both surfaces are in play; rebuild both and diff the compiled output before and after.
