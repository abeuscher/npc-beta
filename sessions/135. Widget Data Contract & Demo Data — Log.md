## What was built

### Phase 1 ‚Äî Config schema field grouping

- Migration backfills `group` key (`content` or `appearance`) on every field in every widget type's `config_schema`
- Old layout-specific group values (`colors`, `hero_colors`, `nav_colors`) replaced with the two-value semantic scheme
- Updated `WidgetTypeSeeder` with `group` keys on all fields for fresh installs

### Phase 2 ‚Äî Field subtype system

- Added `subtype` keys to fields where the type default would produce wrong-looking preview output (headings ‚Üí `title`, URL-holding text fields ‚Üí `url`)
- Backfilled in the same migration and seeder update

### Phase 3 ‚Äî DemoDataService

- Created `app/Services/DemoDataService.php` with:
  - `generateForWidget(WidgetType)` ‚Äî generates a config array from the schema's type/subtype map
  - `generateCollectionData(sourceType, count, collection?)` ‚Äî generates events, blog_posts, products, or custom collection items matching `WidgetDataResolver` output shapes
  - `generateFieldValue(type, subtype, fieldDef)` ‚Äî public method for individual field generation
- Declarative type/subtype resolution using match expressions
- Uses Laravel's `fake()` helper (Faker) for realistic content
- Persists nothing ‚Äî all generated data is ephemeral
- Image fields return `null` (LoremFlickr integration deferred to session 136)

### Phase 4 ‚Äî Refactor debug generator onto DemoDataService

- `DashboardDebugGeneratorWidget::generateBlogPosts()` now uses `DemoDataService::generateForWidget()` for blog post body content instead of inline Faker paragraph loops
- Relationship wiring and persistence logic stays in the debug widget

### Bug fixes discovered during session

- **Inspector 50% width bug:** Adding `group` to every field caused the inspector's layout-grouping logic to wrap all fields in a 2-column grid (it treated any non-null `group` as a layout signal). Fixed by making the inspector skip semantic groups (`content`, `appearance`) and only use layout-specific group names for the 2-column wrapper.
- **Quill dark mode:** The Quill Snow theme has no dark mode support. Added CSS overrides in `AdminPanelProvider` for toolbar icons (`.ql-stroke`, `.ql-fill`), picker dropdowns, placeholder text, and tooltips. Scoped under Filament's `.dark` class.
