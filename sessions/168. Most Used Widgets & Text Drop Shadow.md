# 168. Most Used Widgets & Text Drop Shadow

Two small, independent enhancements: a "Most Used" default category in the widget picker, and a text drop-shadow control in the Appearance panel.

---

## Design decisions (resolved before starting)

- **Most Used tracking** uses a simple counter column (`usage_count`) on `widget_types`. Incremented when a widget is placed on a page (in `PageBuilder::createBlock()`). No decrement on delete ‚Äî the count represents lifetime placement frequency, not current count. This is the lightest approach and avoids needing a separate analytics table.
- **Most Used threshold**: a widget appears in the "Most Used" category if its `usage_count >= 1`. The category shows the top widgets sorted by count descending. If no widgets have been placed yet (fresh install), "Most Used" is hidden and "All" is the default.
- **Most Used as default**: when at least one widget qualifies, "Most Used" is pre-selected as the `activeCategory` when the picker opens. "All" is always available as the first pill to show everything.
- **Text drop shadow** lives in the Appearance > Text panel alongside the existing color picker. It uses a single toggle (on/off) that applies a readable text-shadow preset (`0 1px 3px rgba(0,0,0,0.6)`). No full shadow editor ‚Äî this is a quick usability win for text over background images.
- The shadow value is stored in `appearance_config` at `text.shadow` as a boolean. The renderer emits inline `text-shadow` style when true.

## Phase 2 ‚Äî "Most Used" Widget Category

### Migration

Add `usage_count` (integer, not null, default 0) to the `widget_types` table.

### Backfill usage counts

Write a one-time backfill in the migration (or a post-migration command) that sets `usage_count` for each widget type to the count of `page_widgets` rows referencing it. This ensures existing installs have accurate counts from day one.

### Increment on placement

In `PageBuilder::createBlock()`, after the `PageWidget::create()` call, increment the widget type's `usage_count`:

```php
WidgetType::where('id', $widgetTypeId)->increment('usage_count');
```

### Update `forPicker()`

Add `usage_count` to the array returned by `WidgetType::forPicker()` so the picker modal has access to it.

### Widget picker modal changes

In `widget-picker-modal.blade.php`:

1. Compute `$hasMostUsed` ‚Äî true if any widget type in the set has `usage_count >= 1`.
2. Add a "Most Used" pill before the existing category pills (but after "All"). Only render it when `$hasMostUsed` is true.
3. Set the initial `activeCategory` to `'most_used'` when `$hasMostUsed` is true, otherwise `''` (All).
4. Update the Alpine `matchesCategory()` function: when `activeCategory === 'most_used'`, filter to the top N widgets sorted by `usage_count` desc. A reasonable N is 8‚Äì10 (one-and-a-half rows in the 6-column grid). Pass this threshold as a Blade variable so it's easy to tune.
5. The "Most Used" category is synthetic ‚Äî it is not stored in the `widget_types.category` jsonb array. It's computed from `usage_count` at render time.

### Update `CATEGORY_OPTIONS`

Do **not** add `most_used` to `CATEGORY_OPTIONS` ‚Äî it is not a real category. It exists only in the picker UI.

## Phase 3 ‚Äî Text Drop Shadow

### Appearance config

Add the `text.shadow` path to the `appearance_config` shape. Boolean, default absent (falsy). When `true`, the renderer emits `text-shadow: 0 1px 3px rgba(0,0,0,0.6)`.

### TextPanel.vue

Add a toggle below the existing color picker:

- Label: "Drop Shadow"
- Bound to `text.shadow` in `appearance_config`
- Uses `store.updateLocalAppearanceConfig(widget.id, 'text.shadow', value)`
- Same styling conventions as existing controls in the panel

### AppearanceStyleComposer

In the `compose()` method, read `text.shadow` from the appearance config. If truthy, append `text-shadow: 0 1px 3px rgba(0,0,0,0.6)` to the inline style string.

### Update schema docs

Add `text.shadow` to the appearance config shape documentation in `docs/schema/page_widgets.md`.

## Out of scope

- Full shadow editor (offset/blur/color controls) ‚Äî this session adds a simple on/off toggle only.
- Decrementing `usage_count` on widget deletion.
- Per-user or per-page "most used" tracking ‚Äî this is global per install.
- Any changes to widget categories in the admin (`WidgetTypeResource`) ‚Äî "Most Used" is picker-only.
