## Process rules for every session

Base-prompt process rules (`sessions/template-base-prompt.md`) apply in full. Session-specific notes:

- **Survey first — it is the scoping gate.** Run the ~30-minute widget-hardcoding survey before migrating anything. Surface the count, the **two-class breakdown** (a: widgets using `$color-*` SCSS vars — already transitively token-backed via 297's demotion, lower risk; b: widgets hardcoding raw hex — the true silent-failure class), and a proposed split (Rule 11) **before** touching widget SCSS. Do not migrate ahead of the surfaced split decision.
- **The token list is the agreed, fixed contract.** `docs/theme-color-tokens.md` is canonical. Migrate widgets *onto* it; do not re-derive, rename, or expand it. Map semantic colours to the right tier (an error accent → `--np-color-error`, not `--np-color-brand`).
- **Verify in a real browser, computed values, three surfaces.** Public (`.np-site` → widget colours resolve to the token), Filament admin (no `.np-site` → no leak), page-builder preview (`.np-site` → faithful). Never assert "it works" from emitted CSS or `curl` — the 295/297 rule, not a guideline. 296 drift guard green at start and close.
- **The objective oracle is a Pest guard.** A test that scans the widget SCSS corpus for raw hex literals and bare `$color-*` references and fails on any hit. Write it early: it defines "done", sizes remaining work, and is the permanent regression gate (mirrors the convention-drift Pest gate the Code-Review track established).
- **Scope fences — do not cross them:**
  - **No schemes / chrome / shared resolver.** Theme-defined Default + Inverse schemes, the per-template scheme selector, the header/footer page-shell + "No header/footer" model, and the single shared public+preview resolver are **session 299**. Building any of that here is scope drift.
  - **No token-list change and no palette redesign.** 297's contract is fixed.
  - **No `templates`-table / relocation rework** (297 done) and **no Fleet / `/api/health`** (non-boundary, v2.3.0).
  - **Colour tokens only.** rgba shadows, gradients, and other non-colour hardcoded values are **not** in the contract — log them, do not migrate them. Non-colour widget defects → housekeeping inbox.
- **Run the fast suite when implementation is complete:** `docker compose exec app php artisan test --exclude-group=slow`. Front-end build per the base-prompt rule: widget SCSS / `widget_types` assets → `docker compose exec app php artisan build:public`; `resources/scss/**` → `npm run build`. Then verify the three surfaces in a real browser and confirm the drift guard green.
- **Pause for the one human-judgment surface:** whether a given widget's hardcoded colour maps to a tier-1 token, a tier-2 token, or is a deliberate non-theme accent (rare) is occasionally a judgment call. Batch these and surface them with before/after multi-surface captures at handoff — do not silently re-map semantically.
