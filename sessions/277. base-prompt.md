## Style rules

The style rules in `sessions/template-base-prompt.md` apply. Session-specific:

- **Iteration commit messages describe the deliverable** — e.g. "Iteration /1 — Widget catalog doc + sortable table" or "Iteration /2 — 5 widget help docs (bar_chart, event_calendar, donation_form, web_form, event_registration)".
- **Each new markdown doc lands with valid frontmatter** — `title`, `description`, `tags`, `category`, `standalone: true`. `help:sync` will warn on missing frontmatter (line 53 of `HelpSync.php`); treat warnings as failures.
- **No docstrings, comments, or type annotations on code you did not write.**
- **Simple, correct, well-considered solutions over fast or clever ones.**
- **Looseness is a design principle for this session.** The catalog is hand-curated, not auto-generated. The widget docs are markdown, not a new admin surface. Resist tightening anything that doesn't have to be tight — the easier these are to keep current, the more likely they stay current.
