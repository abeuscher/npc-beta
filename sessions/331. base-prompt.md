## Process rules for every session

Base-prompt process rules apply in full (`sessions/template-base-prompt.md`). Session-specific notes:

- **This conversion is NOT behavior-preserving — and that's the point.** For a column-capable widget, switching `@media` (viewport) → `@container np-widget` (own width) changes *when* collapse fires: it now collapses when the widget's column is narrow, regardless of viewport. That is the mobile-appearance improvement the 330 framework was built to enable. The bar here is **"collapse fires correctly off the widget's own width AND looks right at each width, both full-width and in a narrow column"** — not byte-identical output.
- **Per-widget visual sign-off is the genuine pause.** Playwright can objectively confirm a collapse *fires* at the right container width (verify it yourself, per the verify-objective-outcomes rule — 330's real-wrapper-injection harness is the model). Whether it *looks right* across widths is the owner's judgment — surface each widget for a visual check before moving on.
- **Same lightest pattern as 330 — do not invent.** Add the handle to the `_layout.scss` `np-widget` rule; convert each `@media (max-width: X)` to `@container np-widget (max-width: #{X})`. Keep the shared `np-widget` container-name (330 verified nested-container resolution: a widget inside a column inside `.page-layout` resolves the named query to its own wrapper). No new vocabulary, no per-widget container names unless a real conflict appears.
- **`EventsListing` hardcoded literal:** `@media (max-width: 48rem)` is `$bp-md` (768px) expressed in rem. Normalize to `#{$bp-md}` as part of the conversion so the widget collapses at the published constant, no host/widget drift.
- **Enforcement in the same pass:** as each widget's `@media` lines convert, delete them from `WIDGET_SCSS_VIEWPORT_MEDIA_BASELINE` in `WidgetStyleBoundaryConsumptionTest`. If all four migrate, the baseline empties and the gate then asserts **zero** viewport `@media` in widget SCSS. Keep `DesignGroupIntegrityTest` in sync if the file set changes (it shouldn't — no new test file).
- **Front-end builds:** widget SCSS → `docker compose exec app php artisan build:public` (regenerates `public/build/widgets/manifest.json`). `resources/scss/_layout.scss` → `npm run build`. Both will run this session (you touch both surfaces). When a widget's runtime output could have changed, run `build:public`.
