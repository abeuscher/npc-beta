# 122. Widget — Video & Image Background

Add a `background_media` widget type: a section with either a looping background video or a background image, with text content overlaid. The admin chooses between video URL or uploaded image as the background source.

---

## Design decisions (resolved before starting)

- The background source is a toggle: video URL or uploaded image. Only one is active at a time.
- Video backgrounds use an HTML5 `<video>` element with `autoplay`, `muted`, `loop`, `playsinline` — standard pattern for ambient background video. No controls shown.
- Video URL accepts a direct video file URL (MP4, WebM). This is not a YouTube/Vimeo embed — it is a self-hosted or CDN-hosted video file. The existing `video_embed` widget handles YouTube/Vimeo.
- Falls back to the image if the video fails to load (progressive enhancement).
- Categorised as `media` with `allowed_page_types: null`.
- Uses the standard widget contract.

## Implementation

### Seeder entry

| Field | Value |
|---|---|
| handle | `background_media` |
| label | `Video / Image Background` |
| category | `media` |
| allowed_page_types | null |
| render_mode | server |

### Config schema

| Key | Label | Type | Default | Notes |
|---|---|---|---|---|
| source_type | Background Source | select | 'image' | Options: image, video |
| background_image | Background Image | image | null | Used when source_type is image, or as video fallback poster |
| video_url | Video URL | url | '' | Direct MP4/WebM URL. Used when source_type is video |
| heading | Heading | text | '' | |
| subheading | Subheading | textarea | '' | |
| content | Content | richtext | '' | Optional body text below heading |
| overlay_opacity | Overlay Opacity | number | 50 | 0–100 |
| min_height | Minimum Height | select | '24rem' | Options: 16rem, 24rem, 32rem, 40rem, 100vh |
| text_align | Text Alignment | select | 'center' | Options: left, center, right |

### Template

Renders a container with either a `<video>` or `<img>` background depending on `source_type`. The image is always present as a poster/fallback. Text content overlaid with configurable opacity overlay. Same structural pattern as the hero widgets.
