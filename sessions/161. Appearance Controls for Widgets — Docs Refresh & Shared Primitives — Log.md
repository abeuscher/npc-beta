### Phase 3 — NinePointAlignment primitive

- Created `resources/js/page-builder-vue/components/primitives/NinePointAlignment.vue`.
- 3rem-square inline SVG (48-unit viewBox), 9 selectable points on a faint grid, click and arrow-key navigation, focus ring, accessible `role="radiogroup"` with a live `aria-label` reflecting the current value.

### Phase 4 — ColorPicker primitive + theme palette wiring

Click-outside and Escape close the popover. Optional `<slot name="icon">` reserved for the session 164 text-colour variant. Stores hex strings only; emits `''` on the no-color swatch.
- Repointed `InspectorField.vue` (special-cased the `color` field type to use the new primitive without the `field`/`widget` prop fallthrough), `WidgetAppearanceControls.vue` (background + text colour), and `LayoutInspectorPanel.vue` (layout container background) to use `ColorPicker.vue`.
- Deleted `resources/js/page-builder-vue/components/fields/ColorPickerField.vue`.

### Phase 5 — GradientPicker primitive + composition helpers

- Created `app/Services/GradientComposer.php` with `compose(?array $value): string` and `blank(): string`. Strict sanitization: hex pattern `^#(?:[0-9a-fA-F]{3}|[0-9a-fA-F]{6})$`, integer angles in `[0, 360]` (out-of-range falls back to `180`), type allowlist (`linear`, `radial`), and a stricter CSS-override regex (`^(?:linear|radial)-gradient\(\s*[#0-9a-fA-F,\s%.deg-]+\)$`) that rejects `url()`, `expression()`, and anything outside the hex / digit / comma / space / percent / decimal / `deg` / minus allowlist. Multi-gradient stacks reverse the input array order so the last entry paints on top.
- Created `resources/js/page-builder-vue/helpers/gradient.ts` exporting `composeGradientCss()`, `GradientType`, `GradientLayer`, and `GradientValue`. Same sanitization rules, same stacking behaviour.
- Created `resources/js/page-builder-vue/components/primitives/GradientPicker.vue`. Inline-expanding panel (not a popover). Trigger swatch shows the live composed gradient. Empty state offers a "Create gradient" button that emits a default white→black 180° linear layer. Active state has a two-column grid: 8-preset row + Gradient 1 editor + optional Gradient 2 editor on the left, live preview swatch + monospace CSS readout on the right. Each gradient editor uses the new `ColorPicker` for from/to (the only consumer composing primitives this session). Type selector (linear/radial), conditional angle input, and a CSS override text input. "+ Add second gradient" appears only when one layer exists. Clear emits `null` (not `{ gradients: [] }`).
- Extended the `App.vue` verification harness with a GradientPicker section bound to a separate ref with a JSON readout. All harness lines tagged for cleanup.

## Security checklist — verified

- [x] Theme palette values exposed via bootstrap data are non-sensitive template colors only — no credentials, no tokens. The 6 fields in `Template::PALETTE_FIELDS` are all colour columns.
- [x] `GradientComposer::compose()` sanitizes all inputs. The CSS override path refuses anything that isn't a recognised CSS gradient expression with hex colors. The structured path refuses non-hex colors, non-integer angles, and unknown gradient types. All paths covered by Pest tests.
- [x] No raw user input is rendered into the editor preview without sanitization. The composer is the security boundary for the gradient string; the helper is identical in PHP and TS.
- [x] No new routes, permissions, or database writes introduced. The `theme_palette` field is read-only via existing bootstrap rendering. No migrations.
