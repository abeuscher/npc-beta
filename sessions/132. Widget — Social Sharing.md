# 132. Widget — Social Sharing

Add a `social_sharing` widget type: a row of share buttons for common social platforms. Each button opens the platform's share URL in a new window with the current page's URL and title pre-filled.

---

## Design decisions (resolved before starting)

- Share links use each platform's native share URL scheme (e.g. `https://twitter.com/intent/tweet?url=...&text=...`). No JavaScript SDK or third-party script loaded — pure anchor tags with constructed URLs.
- The page URL and title are passed into the template from the page context — the widget does not need the admin to type them in.
- Platform toggles let the admin choose which share buttons to show.
- Categorised as `content` with `allowed_page_types: null`.

---

## Before writing any code

Read the following:

- `app/Services/PageContext.php` — what page data is available at render time

- Understand how the widget template can access the current page's public URL and title. If not currently available in the render context, identify the minimal change to make it available — consistent with the widget contract (no special-casing).

### Template

A row (or column) of share buttons. Each platform button:

- Opens in a new window/tab (`target="_blank" rel="noopener noreferrer"`)
- Uses an inline SVG icon for the platform logo (simple, recognisable icons — no external icon library needed)
- Constructs the share URL using the current page's public URL and title

**Share URL patterns:**
- Facebook: `https://www.facebook.com/sharer/sharer.php?u={url}`
- Twitter/X: `https://twitter.com/intent/tweet?url={url}&text={title}`
- LinkedIn: `https://www.linkedin.com/sharing/share-offsite/?url={url}`
- Email: `mailto:?subject={title}&body={url}`
- Copy Link: Alpine.js `navigator.clipboard.writeText(url)` with a brief "Copied!" feedback

**Page URL and title:** investigate how to get the current page's public URL in the widget render context. The widget must access this through the standard rendering pipeline — not by reading request context directly (widgets render in the page builder preview too). If the render pipeline doesn't currently pass the page URL, add it as a standard variable available to all widget templates.
