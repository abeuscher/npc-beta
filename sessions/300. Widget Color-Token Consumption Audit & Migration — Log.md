## What was built

### Phase 1 — Survey + objective guard + split decision

- **Survey.** Enumerated the widget SCSS corpus. The `widget_types.css` DB column is empty on every widget — the entire migration surface is the 14 filesystem `app/Widgets/*/styles.scss` files (`widget_types.css` class is empty; scoping simplification noted as drift). 44 hardcoded-colour hit-lines, three classes:
  - **(a) 15 already token-backed** — bare `$color-*` SCSS vars (resolve via 297's `_variables.scss` demotion) or `var(--color-primary, …)` (the 297 legacy alias → brand). Low risk; migrate for contract clarity, zero rendered change.
  - **(b) 27 raw hex, no token backing** — the silent-failure class: `var(--color-X, #hex)` where `--color-X` is never emitted (always renders the literal), bare `#fff`/`#ffffff` fills, and widget-internal hex defaults. The real point of the session.
  - **(c) 2 scope-fenced** — `#000000` stops inside ProductCarousel gradients (out of the colour-token contract).
- **Objective guard written early.** `tests/Feature/WidgetColorTokenConsumptionTest.php` (design group) — scans the widget SCSS corpus, fails on any raw hex / bare `$color-*` outside an explicit reviewed allow-list. Mirrors the convention-drift gate shape. Added to `DesignGroupIntegrityTest`'s pinned list (count 17 → 18) in the same reviewed pass. Baselined to the 44-line survey, then shrunk to the 5 final exceptions.
- **Split decision surfaced and confirmed.** Two-class breakdown + per-widget hit list + the six semantic-mapping judgment calls presented to the user; one session, no split; judgment mappings approved as recommended.

### Phase 2 — Migration

- Migrated all 39 class-a + class-b lines across **10 widget SCSS files** (BlogListing, BlogPager, EventsListing, EventCalendar, BoardMembers, PricingChart, LogoGarden, Nav + the two `#fff` accents kept) to the canonical `var(--np-color-*)` contract (`docs/theme-color-tokens.md`, fixed — not re-derived).
- **Semantic-mapping judgment calls (user-confirmed):** `--color-text-secondary` → `--np-color-text`; `#fff` on a brand fill → `--np-color-brand-contrast`; `#fff` on a success fill / on rgba scrims (EventsListing badge, MapEmbed, SocialSharing) → kept as deliberate non-theme accents (documented allow-list entries); surface backgrounds `#ffffff` → `--np-color-bg`; emphasized pricing card `#f8f9fa` → `--np-color-surface` (intended visible shift); per-instance picker dead SCSS fallbacks (BoardMembers `--bm-*`, Nav `--nav-drop-*`, LogoGarden `--logo-container-bg`) → token fallback, zero rendered change (the blade always injects the concrete schema default; PHP schema defaults logged to the inbox, not changed here).
- Guard baseline shrunk to the **5 reviewed scope-fenced exceptions** only (EventsListing:96, MapEmbed:39, SocialSharing:62 deliberate `#fff` accents; ProductCarousel:25/30 gradient stops). Zero class-a/class-b remains — the silent-failure class is closed.
