## Deferred decisions

- **Re-ranking `unknown` above or below `yellow`.** Locked at `unknown ‚â° yellow` for v1.1.0. Operational experience may show that missing data should rank above yellow (more concerning) or below yellow (less concerning); the ranking may be revisited in a future bump. Flagged in the spec doc itself.
- **Other subchecks emitting `unknown`.** Only `last_backup_at` does in v1.1.0. The status enum is now available for any future subcheck that genuinely cannot determine its state (e.g., a future "external dependency reachable" check that times out before answering). Lands when needed; not a forward debt.
- **Top-level `unknown` emission.** Top-level `status` stays in `{green, yellow, red}` by design. Adding `unknown` at the top level would be a breaking change requiring `2.0.0`. Not on the roadmap.
- **FM-side catch-up.** Flagged at 240 close: the Fleet Manager repo needs its own boundary-touching session (refresh local cache to v1.1.0, align FM-side tests + contract validator against the new status enum) before FM Phase 2 (CRM-side Backup Pipeline) makes sense as a CRM-side session. Phase 2 stays queued until both (a) FM catch-up has happened *and* (b) a forcing function (second client install, FM v1 launch) fires. The 241 base prompt's close-gate Phase 1 directs Claude to verify FM-side state via `/home/al/fleetmanager/docs/imported/fleet-manager-agent-contract.md` before surfacing the 242 candidate menu.
