# Session 096 — Per-Page SEO & Header Snippets — Log

**Date:** 2026-03-30
**Branch:** session-096

---

## What was built

### Phase 1 — SEO meta fields on Edit Page

- Migration adding `og_image_path`, `noindex`, `head_snippet`, `body_snippet` columns to `pages` table.
- Updated Page model with new fillable fields and `noindex` boolean cast.
- Expanded SEO fieldset in `HasPageBuilderForm` with all four fields: `meta_title` (dynamic placeholder), `meta_description` (placeholder), `og_image_path` (FileUpload to `public/og-images/`), `noindex` toggle.

### Phase 2 — Per-page header & footer snippets

- Created `ValidHtmlSnippet` validation rule using `DOMDocument::loadHTML()`.
- Added `ActionGroup` (ellipsis menu) to `EditPage` header actions with "Edit Header & Footer Snippets" modal containing info block with CMS Settings link, monospace textareas for head/body snippets, HTML validation.
- New `edit_page_snippets` Spatie permission registered in `PermissionSeeder`, not assigned to any role by default.

### Phase 3 — Site-wide snippets in CMS Settings

- Added "Header & Footer Code Snippets" section to `CmsSettingsPage` with `site_head_snippet`, `site_body_open_snippet`, `site_body_snippet` (monospace textareas with HTML validation), favicon upload (.ico/.png/.svg), and default OG image upload (FileUpload to `public/site/`).
- Layout fix: contact email field now spans 8 columns, all three auto-publish toggles align in a row.

### Phase 4 — Public layout rendering

- Rewrote `layouts/public.blade.php` to render: `<title>`, `<meta description>`, OG tags (title, description, image, url, type), `<link rel="canonical">`, noindex meta, favicon, JSON-LD, site-wide head snippet, per-page head snippet, site-wide body-open snippet, site-wide body snippet, per-page body snippet — all in correct positions.
- Passed `$page` from `pages/show.blade.php` to the layout.

### Phase 5 — JSON-LD auto-generation

- Created `App\Services\SeoMetaGenerator` with `forPage()` returning title, description, OG image, canonical URL, OG type, and JSON-LD.
- Type mapping: `post` → `BlogPosting` (with author, publisher, dates), `event` → `Event` (with startDate/endDate from linked Event model), all others → `WebPage`.
- Fallback extraction: `extractDescription()` walks widget config fields for first ~160 chars of text; `extractFirstImage()` checks image config fields via Media Library then richtext `<img>` tags.

### Phase 6 — Sitemap and robots.txt

- Created `SitemapController` serving `/sitemap.xml` (cached XML, excludes system/member/noindex pages) and `/robots.txt`.
- Cache invalidation via `PageObserver` on create, update, delete, restore.

## Security checklist — verified

- [x] Snippets are rendered with `{!! !!}` (raw) — confirmed only users with `edit_page_snippets` (per-page) or `super_admin` (site-wide) can set snippet content.
- [x] `noindex` toggle does not leak through to pages where it shouldn't — default is false.
- [x] Sitemap excludes system and member pages (portal URLs are not indexed).
- [x] JSON-LD does not expose any PII (author name from `users` table only; no email or contact data).

## Production deployment notes

- Run `php artisan migrate` for the new pages columns.
- Run `php artisan db:seed --class=PermissionSeeder` to register the `edit_page_snippets` permission.
- Ensure `storage/app/public/og-images/` and `storage/app/public/site/` directories are writable (created automatically by FileUpload).
