# 137. Inline Text Editing

Enable in-place text editing within the widget edit mode built in session 136. When a widget is focused, its richtext and plain-text config fields become `contenteditable` regions inside the rendered preview. The user edits text directly in the widget's visual output rather than in a separate form.

---

## Design decisions (resolved before starting)

- **Inline editing is scoped to text fields only.** Richtext (`type: richtext`) and plain-text (`type: text`) config fields are editable in the rendered widget preview. Structured fields (dropdowns, toggles, images, colors, buttons) remain in the inspector panel.
- **Widget templates annotate editable regions.** Text output elements carry `data-config-key` attributes so the JS knows which element maps to which config field. E.g. `<h1 data-config-key="heading">{{ $config['heading'] }}</h1>`.
- **Richtext uses a minimal formatting toolbar.** Bold, italic, link, headings, lists ‚Äî matching the Quill capabilities already in use. The toolbar appears above/near the focused `contenteditable` region.
- **Plain-text fields are single-line.** Rendered as `contenteditable` spans with Enter suppressed.
- **Two-way sync with the inspector.** Inline edits update the inspector's text fields in real time. Inspector edits re-render the widget preview. Both write to the same `config` source via Livewire.
- **Persistence on blur.** Changes save when the user clicks away from the editable region (blur) or after a debounced interval during typing. No explicit "save" button needed.
- **CSS `zoom` compatibility.** The zoom-down container from session 136 means `contenteditable` operates at the zoomed scale. Verify cursor positioning, selection handles, and toolbar placement work correctly.

## Phase 1 ‚Äî Annotate widget templates with data-config-key

Add `data-config-key` attributes to text output elements in widget templates. Start with a representative set:

- `hero.blade.php` ‚Äî heading, subheading, content
- `text-block.blade.php` ‚Äî content
- `donation_form.blade.php` ‚Äî heading
- `events_listing.blade.php` ‚Äî heading

Establish the convention: any element that renders a `text` or `richtext` config value gets `data-config-key="{key}"`. Document the convention in a comment at the top of one template.

---

## Phase 2 ‚Äî contenteditable activation on focus

When a widget enters edit mode (focused/selected):

- Find all `[data-config-key]` elements inside the preview container
- Set `contenteditable="true"` on richtext fields, `contenteditable="plaintext-only"` on text fields
- Add visual affordance: subtle outline or background tint on editable regions to signal they're interactive
- Suppress Enter key on plain-text fields
- On blur of each editable region: read the innerHTML (richtext) or innerText (plain-text), call the inspector's `updateConfig(key, value)` Livewire method to persist

When the widget loses focus (deselected):

- Remove `contenteditable` from all regions
- Ensure the final state is saved

---

## Phase 3 ‚Äî Richtext formatting toolbar

Add a floating formatting toolbar for richtext `contenteditable` regions:

- Appears on focus of a richtext region, positioned above the element
- Controls: bold, italic, link (with URL input), heading level (H2/H3/H4), unordered list, ordered list
- Uses `document.execCommand` or the Selection API for applying formats
- The toolbar follows the zoom scale ‚Äî position it relative to the preview container, not the viewport
- Toolbar disappears when focus leaves the richtext region

---

## Phase 4 ‚Äî Two-way inspector sync

Wire bidirectional updates:

- **Inline ‚Üí Inspector:** When the user edits text inline, the corresponding field in the inspector panel updates in real time (or near-real-time via Livewire)
- **Inspector ‚Üí Inline:** When the user edits a text field in the inspector (e.g. via the Quill editor), the widget preview re-renders with the new content. The `widget-config-updated` event from session 136 already triggers this.
- Handle the case where both are being edited simultaneously ‚Äî last write wins, no conflict resolution needed
