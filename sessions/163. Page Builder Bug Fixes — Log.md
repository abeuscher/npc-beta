## What was built

### Phase 1 — Audit

- Traced every `dirtyWidgets` mutation site in [editor.ts](resources/js/page-builder-vue/stores/editor.ts). Confirmed the bit is added at six call sites (`updateWidget`, `updateLocalConfig`, `uploadImage`, `removeImage`, `updateLocalAppearanceConfig`, `updateLocalQueryConfig`) and cleared at exactly two — `refreshPreview` (only on a successful preview round trip) and `populateFromItems` (wholesale reset). No path clears the bit prematurely.
- Catalogued every direct reader of `widget.config[key]` outside the field components: `InspectorField.vue` (the on-mount block being deleted, plus a render-time computed that already falls back to `field.default`), `PreviewRegion.vue` `needsConfig` (walks `widget_type_required_config.keys`, treats `null|undefined|''` as missing — safe because required keys are always populated by `WidgetType::getDefaultConfig()` server-side), and `SelectField.vue` (`collection_handle` dependency for `options_from`, with a `?? ''` fallback — also safe).
- Confirmed the server-side `WidgetType::getDefaultConfig()` populates every schema-defined key when a widget is created via the API, so the on-mount default-write race only manifests against widgets persisted by seeders (which deliberately set only required values). This matched the mechanism trace in the prompt.
- Confirmed the update endpoint's `formatWidget` response intentionally omits `preview_html` (the preview is fetched separately by `refreshPreview`, and that round trip is what clears the dirty bit).
- Reviewed every existing test that touches `appearance_config` or the affected widgets and confirmed none would regress from the three fixes.

### Phase 2 — Apply the three fixes

- **Fix 1.1** — Deleted the `onMounted` defaults-write block at [InspectorField.vue:29-37](resources/js/page-builder-vue/components/InspectorField.vue) and removed the now-unused `onMounted` import. The render-time `fieldValue` computed already falls back to `field.default`, and the server-side widget templates apply matching `?? default` fallbacks per field, so visible behaviour is unchanged. This eliminates the bulk-defaults-on-mount race entirely.
- **Fix 1.2** — Replaced the unconditional `Object.assign` in `updateWidget` at [editor.ts](resources/js/page-builder-vue/stores/editor.ts) with a selective merge. When the widget is in `dirtyWidgets`, the response's user-mutable fields (`config`, `appearance_config`, `query_config`, `label`) are skipped — the local copy is by definition newer than the server's confirmation. Server-authoritative fields (`widget_type_*`, `layout_id`, `column_index`, `sort_order`, `is_active`, `is_required`, `image_urls`) always merge. The dirty bit is unchanged in `updateWidget` — only `refreshPreview` clears it on a successful round trip.
- **Fix 1.3** — Removed the 300 ms field-level Quill `emitTimeout` debounce in [RichTextField.vue](resources/js/page-builder-vue/components/fields/RichTextField.vue), the matching `clearTimeout` in `onUnmounted`, and the `emitTimeout` variable. The store's 350 ms `flushDebouncedSave` is now the single buffering layer for save batching.

### Optional polish (Option A)

- Made the [RichTextField.vue](resources/js/page-builder-vue/components/fields/RichTextField.vue) `modelValue` watcher skip its assignment entirely when the widget is dirty. The local Quill state is by definition newer than any `modelValue` arriving from the store, so a stale server snapshot can no longer overwrite the editor mid-typing.

### Phase 3 — Tests

- **`it renders schema defaults when widget config is empty`** appended to [tests/Feature/HeroWidgetTest.php](tests/Feature/HeroWidgetTest.php) — locks in the contract Fix 1.1 relies on. Constructs a hero with `config: []`, calls `WidgetRenderer::render`, and asserts the rendered HTML reflects every schema default (`hero--pos-center-center`, `hero--height-24`, `--hero-overlay: 0.5`) and the absence of the off-by-default classes (`hero--fullscreen`, `hero--overlap-nav`, `hero-scroll-indicator`).
- **`it update endpoint contract that the JS selective merge depends on`** appended to [tests/Feature/PageBuilderApiTest.php](tests/Feature/PageBuilderApiTest.php) — locks in the response shape Fix 1.2 relies on. PUTs label + config + appearance_config + query_config to `/widgets/{id}`, asserts user-mutable fields are echoed back exactly (including a `array_keys` canonical equality check on `config` to confirm no server-side merging), asserts the response includes the server-authoritative metadata fields the JS unconditionally merges, and asserts `preview_html` is NOT in the response.
