# 121. Widget — Full Size Hero

Add a `hero_fullsize` widget type: a viewport-height hero that resizes to fill the browser window. Same structure as the standard hero (heading, subheading, background image, CTA) but with `min-height: 100vh` and vertical centering.

---

## Design decisions (resolved before starting)

- This is a variant of the standard hero (session 120) with a different default height and layout behaviour. It is a separate widget type, not a config toggle on the base hero — users should see these as distinct choices in the picker.
- Uses the same widget contract: `config_schema`, server-rendered Blade, `configMedia`.
- Categorised as `content` with `allowed_page_types: null`.
- The template may share structure with the hero template. If the two templates are substantially identical, extract a shared Blade partial. If they diverge meaningfully, keep them separate. Use judgement — do not force a shared partial if it requires conditional complexity.

## Implementation

### Seeder entry

| Field | Value |
|---|---|
| handle | `hero_fullsize` |
| label | `Full Size Hero` |
| category | `content` |
| allowed_page_types | null |
| render_mode | server |

### Config schema

Same fields as the standard hero, except:

- `min_height` is removed — this widget is always `100vh`
- Add `scroll_indicator` (toggle, default true) — shows a subtle downward chevron/arrow at the bottom to hint that content continues below

### Template

Full-viewport hero. `min-height: 100vh` with flexbox vertical centering. Background image, overlay, heading, subheading, CTA — same rendering approach as the standard hero. Optional scroll indicator at the bottom (a CSS-animated chevron, hidden when toggled off).
