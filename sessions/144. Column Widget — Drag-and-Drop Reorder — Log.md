## What was built

### Phase 1 — Move-to actions for column widget block management

The original goal was cross-container drag-and-drop via connected SortableJS groups. After implementing and testing, we discovered a fundamental conflict: SortableJS physically moves DOM elements between containers, but those elements are Livewire component roots. Moving them across Livewire component boundaries breaks Livewire's DOM tracking, Alpine state, and causes widget duplication. After attempting a DOM-revert workaround (capture state, revert move, let Livewire re-render), we stepped back and chose a cleaner architecture: explicit "Move to" menu actions.

- **Root blocks** get menu items: "[Column Widget Label] → Col N" for each available column widget and slot on the page
- **Child blocks in column slots** get: "Move to Column N" (other columns in same widget) and "Move to Main List"
- Within-container drag reorder in the main list unchanged (single-container `x-sort` — no cross-boundary issues)
- Within-column-slot reorder via existing up/down buttons unchanged
- `PageBuilder::updateOrder()` rewritten with richer payload format `{id, parent_id, column_index, sort_order}`, page ownership validation, and column nesting prevention
- New methods: `PageBuilder::onMoveToMain()`, `PageBuilder::onMoveToColumn()`, `PageBuilderBlock::onChildMoveToColumn()`

### Phase 2 — Verify and polish existing column features

- **Column count decrease:** Added `relocateExcessChildren()` on PageBuilderBlock — when column count is reduced via the inspector, children in removed slots auto-relocate to the last remaining slot
- **Responsive stacking:** Added `@media (max-width: $bp-md) { grid-template-columns: 1fr !important; }` to `.widget-columns`
- **Carousel/logo garden in columns:** Added `min-width: 0` on grid children — standard CSS grid fix preventing Swiper and similar content-width-driven components from overflowing column boundaries
- **Demo seeder resilience:** Wrapped `addMedia()` calls in try/catch in LogoGardenDemoSeeder, BoardMembersDemoSeeder, and ProductDemoSeeder so disk access failures warn instead of aborting the seeder loop
