# 098. Additional Widget Types — Calendar, Chart & Video

Three new widget types for the public frontend: an event calendar powered by Calendar.js, a bar chart powered by Chart.js fed by collections, and a responsive video embed supporting YouTube and Vimeo.

---

## Design decisions (resolved before starting)

- **Calendar widget** uses `jcalendar.js` (Calendar.js v2.12.6, MIT, zero dependencies). Read-only display — no drag-and-drop, no editing. Events are fed from a lightweight JSON endpoint that queries published events. The library ships its own CSS; include it alongside the widget and override `:root` variables to align with the site theme where practical.
- **Chart widget** uses `chart.js` (MIT). Added to `package.json` as a frontend dependency. Initial implementation supports bar charts only — line/pie/doughnut deferred. The widget feeds off a collection using the same field-selector pattern as the carousel widget (collection handle → field key).
- **Video widget** supports YouTube and Vimeo in a single widget. The user pastes a URL; the widget auto-detects the provider by parsing the URL pattern. Rendered as a responsive `<iframe>` in a 16:9 aspect-ratio container. No autoplay. YouTube defaults to least-intrusive embed settings. Vimeo defaults to minimal branding.
- All three widget types follow the existing seeder pattern in `WidgetTypeSeeder` with config schemas, Blade templates in `resources/views/widgets/`, and CSS/JS fields where needed.
- Calendar.js and Chart.js assets are loaded only when the respective widget is present on a page — not globally. Use the widget's `js` and `css` fields or conditional `@push` stacks.

## Phase 1 — Install dependencies

- Add `jcalendar.js` and `chart.js` to `package.json` dependencies.
- Run `npm install` inside the app container.
- Verify both packages are importable. No Vite config changes should be needed — the calendar and chart instances will be initialised via inline widget JS using the globally available libraries, or imported in the widget templates.

## Phase 2 — Event calendar widget

### JSON endpoint

Create a lightweight public route (e.g. `/api/events.json`) that returns published, upcoming events as JSON. Fields: `id`, `title`, `from` (ISO datetime), `to` (ISO datetime), `description` (plain text excerpt), `url` (link to landing page if one exists). No authentication required — this is public event data only. Controller or closure route in `routes/web.php`.

### Widget type: `event_calendar`

- **Config schema:**
  - `heading` — text, optional, helper: "Heading displayed above the calendar"
  - `default_view` — select, options: `month` / `week`, default `month`, helper: "Initial calendar view"
- **Template:** renders a container `<div>` with a unique ID. Calendar.js is initialised in the widget's JS field, fetching events from the JSON endpoint and mapping them to Calendar.js event objects.
- **CSS:** include Calendar.js stylesheet via the widget's `css` field. Apply minimal `:root` overrides to align with site theme colours (use `var(--color-primary)` where possible).
- **JS:** initialise `calendarJs` on the container element with `manualEditingEnabled: false`. Fetch events from the JSON endpoint on mount. Map each event to `{ from, to, title, description }`. Set `views` config to honour the `default_view` setting. Disable editing, drag-and-drop, export, and search features — display only.

## Phase 3 — Bar chart widget

### Widget type: `bar_chart`

- **Config schema:**
  - `heading` — text, optional, helper: "Chart title"
  - `collection_handle` — select, options_from: `collections`, helper: "Data source collection"
  - `x_field` — select, options_from: `collection_fields:text`, depends_on: `collection_handle`, helper: "Field for X axis labels"
  - `y_field` — select, options_from: `collection_fields:text`, depends_on: `collection_handle`, helper: "Field for Y axis values (numeric)"
  - `x_label` — text, optional, helper: "X axis label"
  - `y_label` — text, optional, helper: "Y axis label"
  - `bar_color` — text, optional, helper: "Bar colour (hex or CSS variable, defaults to primary)"
- **Collections:** `['data']` (same pattern as carousel — declares the collection slot).
- **Template:** renders a `<canvas>` element inside a responsive container.
- **JS:** initialise a Chart.js bar chart on the canvas. Data comes from the resolved collection items — map `x_field` values to labels and `y_field` values to data points. Apply `heading` as the chart title, `x_label` / `y_label` as axis labels, `bar_color` as the dataset background colour (fall back to `var(--color-primary)` or `#0172ad`).

## Phase 4 — Video embed widget

### Widget type: `video_embed`

- **Config schema:**
  - `video_url` — text (URL), required, helper: "YouTube or Vimeo video URL"
  - `show_related` — toggle, default false, helper: "Show related videos at the end (YouTube only)"
  - `modest_branding` — toggle, default true, helper: "Reduce YouTube branding"
  - `show_controls` — toggle, default true, helper: "Show player controls"
- **Template:** parse `video_url` to detect provider and extract video ID:
  - **YouTube:** match `youtube.com/watch?v=`, `youtu.be/`, `youtube.com/embed/` patterns. Build embed URL: `https://www.youtube-nocookie.com/embed/{id}` with params: `rel` (from `show_related`), `modestbranding` (from `modest_branding`), `controls` (from `show_controls`), `autoplay=0`, `playsinline=1`.
  - **Vimeo:** match `vimeo.com/{id}` pattern. Build embed URL: `https://player.vimeo.com/video/{id}` with params: `byline=0`, `portrait=0`, `title=0`, `autoplay=0`.
  - If URL doesn't match either pattern, render a "Unsupported video URL" message instead of a broken iframe.
- **HTML structure:** `<div class="aspect-video"><iframe src="..." class="w-full h-full" frameborder="0" allow="fullscreen; picture-in-picture" allowfullscreen loading="lazy"></iframe></div>`
- Use `youtube-nocookie.com` domain for YouTube embeds (privacy-enhanced mode — no tracking cookies until playback).
- No JavaScript required — pure server-rendered HTML.

## Security checklist

Before closing:

- [ ] Event JSON endpoint returns only published event data — no draft, cancelled, or internal fields exposed.
- [ ] Video embed iframe uses `youtube-nocookie.com` for YouTube (privacy-enhanced mode).
- [ ] Video URL parsing rejects arbitrary URLs — only YouTube and Vimeo patterns produce an iframe.
- [ ] Chart widget escapes all collection-sourced text used in labels/titles.
- [ ] Calendar widget is read-only — no editing, no event creation from the public frontend.
