# 040. Tags — Unified Tag System — Log

## What was built

### Tag model consolidation
- Merged `Tag` (CRM) and `CmsTag` (CMS) into a single `tags` table with a `type` discriminator (`contact`, `page`, `post`, `event`, `collection`)
- Added `slug` column via Spatie Sluggable (`doNotGenerateSlugsOnUpdate` so label changes don't clobber custom handles)
- Changed unique constraint from `name` alone to composite `(name, type)`
- Migrated all `cms_tags` rows into `tags` with `type = 'collection'`
- Migrated all `cms_taggables` pivot rows into `taggables`
- Dropped `cms_taggables` and `cms_tags` tables
- Deleted `CmsTag` model, `CmsTagResource` and its Pages, `CmsTagPolicy`, `CmsTagFactory`

### Tag relationships
- `Tag` model: added `pages()`, `posts()`, `events()`, `collectionItems()` morphedByMany relationships
- `Page`, `Event`, `CollectionItem` models: added `tags()` MorphToMany relationships

### Tag pickers
Built `app/Forms/Components/TagSelect.php` — a static factory returning a `Group` with:
- `Select` (searchable, preloaded, scoped to the correct type via `->relationship()` third argument)
- `TextInput` with `+` suffix action for inline tag creation (`->dehydrated(false)`, uses `firstOrCreate`)

Applied to: Contact, Page, Post, Event edit forms, and Collection Item relation manager.

### Tag Manager
Rewrote `TagResource` as a unified manager under Tools navigation:
- Table: name, slug, type badge (colour-coded), attached record count (via `taggables` table)
- Filter by type
- Create: name + type (type hidden on edit)
- Edit: label (`name`) + handle (`slug`, unique validation), type read-only
- `EditTag::getTitle()` returns "Edit [Type] Tag"

### Mailing list fix
`MailingListQueryBuilder::applyFilter()` — `includes` and `not_includes` operators now scope tag queries to `type = 'contact'` to prevent cross-type name collisions.

### Permissions and tests
- Removed `cms_tag` from `PermissionSeeder`, replaced with `tag` permissions for `cms_editor`
- Updated `PermissionTest` to use `Tag::create(['name' => ..., 'type' => 'collection'])` instead of `CmsTagFactory`
- All 186 tests passing

### Help docs
- Created `resources/docs/tags.md` — unified Tag Manager documentation
- Updated `resources/docs/contacts.md`, `cms-pages.md`, `posts.md`, `events.md` — added Tags sections describing the Select + Create field UI

## Deferred decisions

- Tag colour picker: column exists, UI not surfaced
- Tag merging / bulk re-assignment
- Public-facing tag filtering pages
- Contact `source` field lock (moved to session 041)

## Key architectural notes

- The sentinel `__new__[name]` approach (originally used for inline creation) was replaced with the companion `TextInput` + `+` button pattern to avoid injecting UI elements into the dropdown list mid-flow
- `doNotGenerateSlugsOnUpdate()` was essential: without it, editing a tag's Label would silently regenerate and overwrite a custom Handle
- `saveRelationshipsUsing` is kept on the Select to call `$record->tags()->sync()` explicitly rather than relying on Filament's default relationship handling
