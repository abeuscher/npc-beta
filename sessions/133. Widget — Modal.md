# 132. Widget — Modal

Add a `modal_widget` widget type: a button that opens a modal overlay containing child widgets. The modal acts as a nesting container — like the column widget, but triggered by a click instead of always visible.

---

## Design decisions (resolved before starting)

- The modal widget is a **nesting container** — it uses `parent_widget_id` and child slots, following the same pattern as `column_widget`. It has one slot (column_index 0) that holds the modal's content widgets.
- The trigger is a configurable button rendered inline on the page. Clicking it opens the modal overlay.
- The modal uses Alpine.js for open/close state — same pattern as the help slide-over.
- Categorised as `layout` with `allowed_page_types: null`.
- **Widget contract:** the nesting behaviour must follow the same parent/child mechanics as the column widget. No special rendering pipeline — the children are rendered by the standard `WidgetRenderer` and passed as `$children` HTML, identical to how column widget children work.

## Implementation

### Seeder entry

| Field | Value |
|---|---|
| handle | `modal_widget` |
| label | `Modal` |
| category | `layout` |
| allowed_page_types | null |
| render_mode | server |
| default_open | true |

Set `default_open: true` so the page builder expands this widget to show the child slot for editing (same as column_widget).

### Config schema

| Key | Label | Type | Default | Notes |
|---|---|---|---|---|
| trigger_text | Button Text | text | 'Open' | Text on the trigger button |
| trigger_style | Button Style | select | 'solid' | Options: solid, outline, pill, link |
| modal_size | Modal Size | select | 'md' | Options: sm, md, lg, xl |
| show_close_button | Show Close Button | toggle | true | X button in the modal header |
| modal_title | Modal Title | text | '' | Optional title shown in the modal header |

### Template

**Trigger:** A styled button (matching CTA button patterns) that opens the modal via Alpine.js.

**Modal overlay:** Fixed-position overlay with backdrop. Contains:
- Optional header with title and close button
- Body area rendering `$children` (the nested widget content)
- Click-outside-to-close and Escape-to-close

**Nesting mechanics:** The modal declares one child slot (like a single-column `column_widget`). In the page builder admin, the modal widget expands to show an "add widget" interface for its content area — identical UX to adding widgets inside a column slot.

### Rendering

The `WidgetRenderer` already supports nesting — it renders children and passes them as `$children` HTML. The modal template wraps `$children` in the modal overlay structure. No changes to the rendering pipeline should be needed. If the renderer currently only supports nesting for `column_widget` specifically, generalise it to support any widget with children — consistent with the widget contract principle.
