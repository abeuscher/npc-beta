## What was built

### Data model

- **`forms` table** — `id`, `title`, `handle` (unique), `description`, `fields` (json), `settings` (json), `is_active`, timestamps
- **`form_submissions` table** — `id`, `form_id` (FK→forms, cascade), `contact_id` (nullable UUID FK→contacts, nullOnDelete), `data` (json), `ip_address`, `created_at` (no `updated_at` — immutable)
- **`app/Models/Form.php`** — casts `fields` and `settings` as array; `hasMany(FormSubmission)`; `fieldValidationRules()` method that builds Laravel validation rules from the field definition array including custom regex (with try/catch for malformed patterns)
- **`app/Models/FormSubmission.php`** — `const UPDATED_AT = null`; casts `data` as array; `belongsTo(Form)`, `belongsTo(Contact)`

### Field definition schema

Each field definition object:
```json
{
  "handle": "first_name",
  "type": "text",
  "label": "First Name",
  "placeholder": "",
  "required": true,
  "width": 6,
  "validation": "none",
  "validation_regex": null,
  "validation_message": null,
  "options": [],
  "contact_field": "first_name"
}
```

`contact_field` is new to this session — maps a form field to a Contact model column (standard columns or `custom_fields.{handle}`). Used by the contact sync logic.

Settings object:
```json
{
  "submit_label": "Submit",
  "success_message": "Thank you. Your message has been received.",
  "honeypot": true,
  "form_type": "general"
}
```

`form_type` values: `general` | `contact`. Contact forms trigger automatic contact creation/update on submission.

### Admin UI — FormResource

- Navigation: CMS group, sort 5, icon `heroicon-o-document-text`
- Filament Repeater-based field builder with all 11 field types, nested options repeater for select/radio, show/hide dependent fields via `->live()`
- `contact_field` select in each field row — lists standard Contact columns plus any `CustomFieldDef` records scoped to `contact` model
- `form_type` select in settings section with helper text warning when contact form lacks required mappings
- Table with submissions count (via `withCount`), Download JSON action (exports full field + settings definition as `.json`)
- `FormSubmissionsRelationManager` — read-only list with date, IP, and "Contact created" icon column; View action links to ContactResource edit page when `contact_id` is set, falls back to data modal when not; Delete action gated on `delete_form_submission` permission
- Page classes: `ListForms`, `CreateForm`, `EditForm` — all with `getBreadcrumbs()`

### Public rendering

- **`FormSubmissionController@store`** — honeypot check → PII scan (handles + values via `PiiScanner::scanHeaders()` and `scanCell()`) → validation → store submission → contact sync if `form_type=contact`
- **Contact sync logic** — finds or creates contact by email; updates non-empty mapped fields on collision; opt-in only for `mailing_list_opt_in` (never opts out); handles `custom_fields.*` mappings; writes a Note ("Contact created/updated via web form: {title}"); stores `contact_id` on the submission
- **Route** — `POST /forms/{handle}` with `throttle:10,1`, named `forms.submit`; registered before the catch-all slug route
- **`resources/views/components/public-form.blade.php`** — all 11 field types, pre-populated US state and country selects (US/CA/GB/AU at top), session flash success state, inline validation errors, honeypot hidden field, Blade `{{ }}` escaping throughout
- **`resources/scss/_layout.scss`** — added `.form-grid` (12-column CSS grid) and `.col-{1–12}` classes with mobile stack at 768px

## Decisions made during the session

- **Contact update on email collision** — non-destructive update: any non-empty submitted value overwrites existing contact fields; empty/blank submitted values are ignored; `mailing_list_opt_in` is one-directional (opt-in only on public forms — unchecked checkbox is treated as silence, not opt-out)
- **Duplicate email for non-members** — silently update and note; member collision handling deferred to Member Portal (noted in session-outlines.md)
- **PII rejection** — visible generic error ("One or more fields contain information that cannot be accepted"), not silent — user deserves feedback
- **JSON import** — deferred to session 049; download-only this session
- **Cloudflare / TrustProxies** — not applicable to current direct-served deployment; documented in session 048 security disclosure
- **`form_type=general`** — submissions stored only; no contact creation
