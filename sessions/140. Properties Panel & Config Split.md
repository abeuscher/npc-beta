# 140. Properties Panel & Config Split

Replace the page builder inspector's single scrolling form with a tabbed panel following the Adobe-style stacked toolbar pattern. Two tabs — Content and Appearance — split fields by their existing `group` key in `config_schema`. The Appearance tab also houses the spacing/style controls.

---

## Design decisions (resolved before starting)

- **Two tabs only.** Content and Appearance. Additional tabs (Layers, Query, etc.) may come later — this session establishes the tab infrastructure and the two-tab split.
- **Tab appearance modelled on Photopea-style panel tabs.** Reference files `panel-example.html` and `panel-example.css` in the project root show the pattern: inline-block tab items in a header bar, active tab gets full opacity + background color, inactive tabs are dimmed. Rounded top corners. Clean, minimal. We are borrowing the visual pattern only — not the HTML structure or the full CSS. These reference files can be deleted after the session.
- **Content tab:** all fields where `config_schema[].group === 'content'`. This includes text, images, collection handles, richtext — the "what" of the widget.
- **Appearance tab:** all fields where `config_schema[].group === 'appearance'`. This also includes the spacing panel (padding/margin from `style_config`), the full-width toggle, and the query settings section. These are the "how it looks and what data it pulls" controls.
- **Block label and Apply Changes button stay outside the tabs** — always visible at the top of the inspector regardless of which tab is active.
- **Tab state is Alpine-only.** No server round-trip to switch tabs. Default tab is Content.
- **Advanced/carousel settings accordion stays.** Fields with `'advanced' => true` remain in their collapsible section within whichever tab they belong to.
- **Dark mode support.** Tab styles must work in both light and dark Filament themes.

## Phase 1 — Tab infrastructure

Add a tabbed header to the inspector panel, above the config fields but below the block label and Apply Changes button.

**Tab bar:**
- Two tabs: "Content" and "Appearance"
- Alpine `x-data` with `activeTab: 'content'` on the tab container
- Active tab styling: full opacity, background colour, bottom border or underline to indicate selection
- Inactive tab: dimmed opacity, transparent background, hover highlight
- Rounded top corners on tabs, consistent with Filament's design language
- Clean transition on tab switch (opacity or background-color, 150–200ms)

**Tab content areas:**
- `x-show="activeTab === 'content'"` and `x-show="activeTab === 'appearance'"` on the two content divs
- No server round-trip — pure Alpine toggle

---

## Phase 2 — Content / Appearance split

Move fields into their respective tabs based on the `group` key in `config_schema`.

**Content tab contains:**
- All config fields where `field['group'] === 'content'` (or `group` is missing — default to content)
- The advanced/carousel settings accordion (only if it contains content-group fields)

**Appearance tab contains:**
- All config fields where `field['group'] === 'appearance'`
- The advanced/carousel settings accordion (only if it contains appearance-group fields)
- Query Settings section (collection handle, limit, direction, order, tag filters)
- Spacing panel (padding, margin, full-width toggle) — moved from the bottom of the inspector into this tab

**Edge cases:**
- If a widget has no content-group fields, the Content tab shows a "No content settings for this widget" placeholder
- If a widget has no appearance-group fields (unlikely), the Appearance tab still shows spacing controls
- The advanced accordion should only appear in a tab if that tab has advanced fields to show

---

## Phase 3 — Tab styling

Style the tab bar to match the Adobe/Photopea pattern adapted for Filament:

- Tab bar background: subtle separator from the content below (thin border or background shift)
- Active tab: `bg-white dark:bg-gray-800` with full opacity — visually connected to the panel body
- Inactive tab: `bg-transparent`, `opacity-60`, hover to `opacity-80`
- Tab text: `text-xs font-semibold uppercase tracking-wide` (matching existing inspector section headers)
- Rounded top corners (`rounded-t-md`) on each tab
- 1px bottom border on the tab bar, active tab's bottom border matches panel background (creating the "connected" look)
