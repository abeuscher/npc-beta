# Base Prompt — Session 239

We are about to begin a new session: **239. Donation Status Enum Alignment (Importer + Backfill)**.

This is a small, focused, single-bug session. The donations importer's `mapDonationStatus()` writes `'completed'` for `active` / `paid` / `succeeded` / blank / unrecognized inputs — a value that falls outside the canonical four-value enum (`pending`, `active`, `cancelled`, `past_due`) documented in `docs/schema/donations.md`, defaulted by `DonationFactory`, written by `StripeWebhookController`, observed by `DonationObserver`, and filtered on by `ContractResolver::resolveDonationList()`. Imported donations therefore land at a status that no consumer recognizes — they are absent from the Recent Donations widget, mishandled by the observer, and silently misrepresented in the admin UI.

The fix is a one-line mapper change plus a one-time backfill of existing `'completed'` rows. The bug was surfaced during session 234 manual testing and has carried forward unscheduled through 235 / 236 / 237 / 238.
