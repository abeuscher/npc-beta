## Session-specific deltas

- **The layout spec is canonical for layout interpretation; `copy.md` is canonical for words; `brief.md` system conventions remain in force.** Authority order per the spec's "Authority" section, mirroring 286 / 287.
- **The pricing-chart widget is the Band 2 carrier.** One widget instance covers the entire band: heading + subheading live in the widget's own config; three columns live in the widget's `columns` repeater; footnote lives in the widget's `footnote` config. Do **not** wrap the widget in a layout container with sibling Text widgets for heading / footnote — that's the old composite shape the widget supersedes.
- **Band 2's column content comes from `PricingChartDefinition::marketingSiteTiers()`.** The widget ships this helper as a public method returning the spec's three-column shape verbatim. When authoring `pricing.json`, copy the array literal that helper produces into the widget's `config.columns` field. (The widget's `demoConfig()` also uses this helper; reading the live `/dev/widgets/pricing_chart` rendered HTML is the fastest way to confirm the shape before writing it into `pricing.json`.) If the live spec at session-start time has drifted from the widget's helper output, prefer the spec — and decide whether to update the widget's helper for downstream customers (small system change, surface for user input) or just override per-instance for this page (local fix, just do it). Default to the local override unless the drift is structural.
- **Band 2's "Marketing site tiers preset" framing in the release-plan entry is approximate.** The widget does not ship a `presets()` entry for the columns — preset rules forbid content-group keys, and `columns` is content-group. The preset framing collapses to "use `marketingSiteTiers()` as the column-data source." Same outcome, different mechanism. Note this in the work log; no action.
- **Band 2's emphasis treatment is background-tint only at v1.** The widget's emphasized column gets `#f8f9fa` background; the other two stay white. No borders. This is the agreed v1 path per the spec's "Gap callout — Border styling" path 2 and 288's resolution. If the result reads too subtle in the live page, surface it as a follow-up gap (G-pricing-1 lift candidate). Do not patch the widget mid-session.
- **Hero-cell carrier defaults to Text widget + Image widget** per About Band 1's pattern (286 revision-pass + 287 application). Do not use the Hero widget for Band 1.
- **Footnote rendering.** The widget renders the footnote rich text as-is; the spec wants smaller-body type. The typography system has no caption row. Use italic + lighter weight inside Quill content via `<em>` (the widget's `demoConfig()` already demonstrates this pattern). Log as a continued gap if the live result reads too subtle.
