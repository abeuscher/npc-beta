## What was built

### Phase 1 — Unified layout shell

- Replaced Edit/Preview toggle with Edit/Handles toggle. Default mode is Edit.
- Edit mode: stacked widget preview on the left (all widgets rendered with public CSS via `WidgetRenderer`), persistent inspector panel on the right. Viewport width toggle (Desktop 1920 / Tablet 1024 / Mobile 375) with zoom scaling.
- Handles mode: existing block card list with drag handles, ellipsis menus, and reorder — unchanged from prior behavior except per-block preview rendering removed.
- Click-to-select on widget regions with visible selection border. Escape to deselect.
- Preview iframe removed entirely.
- Inspector always visible in both modes.

### Phase 3 — Library loading (page-level)

- On mount, scan all widget types present on the page and collect the union of all `libs` arrays (including children of column widgets).
- Load all required library JS and CSS once via the manifest at `window.__widgetLibs`, awaiting both JS `onload` and CSS `onload` before initializing Alpine trees.
- `x-ignore` wrapper on preview HTML prevents Alpine auto-init. Manual `Alpine.initTree()` called after libs load, with `_x_ignore` JS property lifted and restored around the call (Alpine checks the property, not just the attribute).
- Livewire morph bypassed entirely for the preview via `wire:ignore`. Content updates pushed via `preview-content-changed` browser event with full block HTML, applied manually by Alpine.
- Selection is instant (Alpine-only class toggle + lightweight server call for inspector mount). No preview re-render on selection change.

### Additional work

- **Apply Changes button:** Added to preview pane (sticky top-right). Preview only re-renders on explicit Apply click, widget add/delete/copy/reorder, or deselect — not on every inspector field change. Inspector field changes still persist to DB immediately.
- **Swiper CSS fix:** `AssetBuildService` library sources updated from non-existent `.scss` paths to actual `.css` files shipped by Swiper package. Added `effect-coverflow.css`.
- **Chart.js support:** Added `umd_js` path type to `AssetBuildService` for libraries that ship pre-built UMD bundles (build server returned 422 on chart.js ES module imports). Installed chart.js npm package.
- **Bar chart template rewrite:** Converted from raw `<script>` + `DOMContentLoaded` to Alpine `x-data`/`x-init` with JSON config block (avoids `@json()` breaking HTML attribute quotes). Canvas dimensions set from `getComputedStyle` pre-zoom width for correct sizing under CSS zoom.
- **Chart.js cleanup in reinit:** `Chart.getChart(canvas).destroy()` called before Alpine tree reinit to prevent "Canvas already in use" errors.
- **Widget hit area:** 10px vertical padding on selection regions. Border-based selection indicator (consistent on all sides).
- **Link/form interception:** `<a>` clicks get `preventDefault()`, form submits blocked inside preview.
- **Height pinning:** Widget region heights pinned during content refresh to prevent layout shift.
- **Viewport presets:** Tablet/Mobile views show scrollable device-frame container with centered viewport. Known limitation: CSS `vh` resolves to browser viewport, not simulated device height.
- **Session outlines updated:** Phase 2 (block controls overlay) and Phase 4 (inline text editing in preview) deferred to post-Beta 1.
