# 129. Embed Map Widget

Google Maps embed widget: admin pastes a share URL or iframe snippet, the widget extracts the embed URL and renders a responsive iframe. Scroll-zoom is disabled until the user clicks the map.

---

## Design decisions (resolved before starting)

- The widget accepts two input formats: a Google Maps share/place URL (`https://www.google.com/maps/place/...` or `https://maps.google.com/...`) and a raw embed iframe snippet (`<iframe src="https://www.google.com/maps/embed?..."`). A server-side parser extracts the embed URL from either format.
- Scroll-to-zoom hijacking is prevented by rendering the iframe with `pointer-events: none` inside an overlay container. An Alpine.js click handler removes the overlay and enables interaction. Clicking outside the map (or pressing Escape) re-enables the overlay. This is the standard pattern used across the web for embedded maps.
- The "View larger map" link is rendered by Google inside the iframe automatically and cannot be toggled — no config field needed.
- Responsive sizing uses CSS `aspect-ratio` with `min-height` and `max-height` constraints rather than a fixed pixel height. This adapts cleanly to all viewport widths.
- No API key required — the widget uses the standard `/maps/embed` URL which works without authentication.

## Phase 1 — Map URL parser

Create `App\Services\MapEmbedParser` with a single static method:

```php
public static function extractEmbedUrl(string $input): ?string
```

Behaviour:

- If `$input` contains an `<iframe` tag, extract the `src` attribute value.
- If `$input` is a Google Maps URL (`google.com/maps` or `maps.google.com`), convert it to an embed URL by replacing `/maps/place/` (or `/maps/@`) with `/maps/embed/v1/place?q=` and extracting the place query. For short-link formats (`goo.gl/maps/`, `maps.app.goo.gl/`), use the URL as-is in a `q=` parameter.
- If the input is already a `/maps/embed` URL, return it directly.
- Return `null` if the input cannot be parsed as a Google Maps reference.
- Sanitise the output: only allow URLs with host matching `*.google.com`.

## Phase 2a — Add `notice` config field type to the inspector

Add a new inspector field type `notice` that renders a non-interactive informational banner in the widget config panel. This is a display-only element — it stores no value in config.

**Field definition example:**

```php
['type' => 'notice', 'label' => 'Privacy note', 'content' => 'Google may use embedded maps to collect usage data. <a href="https://policies.google.com/privacy" target="_blank" rel="noopener">Google Privacy Policy</a>', 'variant' => 'warning']
```

**Implementation:**

- Create `resources/views/livewire/partials/inspector-fields/notice.blade.php`.
- Render a styled banner with the `content` value (rendered as HTML — this is admin-only, not user input).
- Support `variant` values: `info` (default, neutral), `warning` (amber/yellow). Style with a left border accent and small icon.
- The `PageBuilderInspector` already iterates `config_schema` to render fields — no changes needed there beyond adding the partial. Verify that non-input fields are skipped when saving config (they should be, since `notice` fields have no `key`).

**Use in the map embed widget:**

```php
['type' => 'notice', 'label' => 'Privacy', 'content' => 'Google may use embedded maps to collect visitor data. See <a href="https://policies.google.com/privacy" target="_blank" rel="noopener">Google\'s Privacy Policy</a>.', 'variant' => 'warning'],
```

## Phase 3 — Blade template and SCSS

Create `resources/views/widgets/map-embed.blade.php`:

- Extract embed URL via `MapEmbedParser::extractEmbedUrl($config['map_input'])`.
- If no valid URL, render nothing.
- Heading rendered as `<h2>` if present.
- Map container with inline `aspect-ratio`, `min-height`, `max-height` from config.
- Scroll-lock overlay: a transparent `<div>` absolutely positioned over the iframe with `pointer-events: all` and a subtle "Click to interact" label centred at the bottom.
- Alpine.js `x-data` manages an `active` state:
  - Click on overlay sets `active = true`, removes overlay, enables iframe pointer-events.
  - Click outside the widget or `Escape` key sets `active = false`, restores overlay.

### SCSS (`resources/scss/widgets/_map-embed.scss`)

- `.widget-map-embed`: basic wrapper
- `.map-embed__container`: `position: relative; overflow: hidden;` with `aspect-ratio`, `min-height`, `max-height` applied via inline style
- `.map-embed__iframe`: `width: 100%; height: 100%; border: 0;`
- `.map-embed__overlay`: absolute fill, transparent, cursor pointer, flex-centred hint label at bottom
- `.map-embed__overlay-hint`: small text label ("Click to interact with map"), semi-transparent background pill
- `.map-embed--active .map-embed__overlay`: hidden
