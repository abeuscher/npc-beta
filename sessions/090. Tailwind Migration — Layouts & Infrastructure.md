# 090. Tailwind Migration — Layouts & Infrastructure

Replace Pico CSS with Tailwind CSS across the public frontend. This session installs Tailwind, migrates the design tokens, rewrites the layout templates (header, footer, portal chrome), and converts structural components. Widget templates are deferred to session 091.

---

## Design decisions (resolved before starting)

- **Tailwind replaces Pico entirely.** Pico is removed from `package.json` and all imports. No hybrid period.
- **CSS layers stay.** The `@layer base, libraries;` pattern introduced in session 089 remains. Tailwind's base/components/utilities directives go in the appropriate layers. Third-party library CSS (Swiper, future packages) stays in the `libraries` layer.
- **Preflight is enabled.** Tailwind's reset (Preflight) replaces Pico's element-level resets.
- **Design tokens migrate to `tailwind.config.js`.** Current Pico variable overrides (`_theme.scss`) become Tailwind theme extensions — colors, fonts, spacing, border-radius.
- **Site theme editor writes CSS custom properties.** Runtime brand customization (colors, fonts set in CMS > Site Theme) is delivered as CSS custom properties on `:root`. The Tailwind config references these variables so utility classes respond to runtime changes without a rebuild.
- **Dark mode uses Tailwind's `class` strategy**, wired to the existing Alpine `theme` store and `[data-theme]` attribute.
- **The admin panel is untouched.** Filament ships its own styles. Tailwind is scoped to public-facing views only via the `content` paths in `tailwind.config.js`.
- **`_layout.scss` is largely replaced** by utility classes in templates. A small `app.scss` (or equivalent) file remains for the `@tailwind` directives, the `@layer` declarations for third-party CSS, and any custom CSS that doesn't fit utility classes (e.g. carousel widget styles).

---

## Phase 1 — Install & configure Tailwind

1. `npm install -D tailwindcss @tailwindcss/forms` (the forms plugin resets form elements to a clean, unstyled baseline — critical for widget forms).
2. Create `tailwind.config.js` with:
   - `content` paths covering `resources/views/**/*.blade.php` and `app/Livewire/**/*.php`.
   - `darkMode: 'class'` (will use `[data-theme="dark"]` selector — configure via `darkMode: ['class', '[data-theme="dark"]']`).
   - `theme.extend` with the current design tokens extracted from `_theme.scss` and Pico defaults: primary color, font family, spacing scale, border-radius.
   - CSS custom property references for runtime-configurable values: `primary: 'var(--color-primary, #1095c1)'` etc.
3. Replace `public.scss` contents:
   - `@tailwind base; @tailwind components; @tailwind utilities;`
   - `@layer libraries { ... }` block for Swiper CSS imports.
   - Import a small `_custom.scss` for any remaining hand-written CSS (carousel widget styles, etc.).
4. Remove Pico from `package.json` and delete the `@use` / `@import` of Pico.
5. Delete `_theme.scss` (tokens moved to Tailwind config). Keep `_layout.scss` temporarily — it will be emptied as styles migrate to templates.
6. Verify the Vite build succeeds and the output size is reasonable.

## Phase 2 — Public layout (`layouts/public.blade.php`)

Rewrite the public layout template with Tailwind utility classes:

- Page structure: `min-h-screen flex flex-col` body, `flex-1` main content area.
- Container: `max-w-7xl mx-auto px-4` (or similar — match current container width).
- Typography base: set default font, line-height, and color via Tailwind's `@layer base` or a `<body>` class.

---

## Phase 3 — Header & navigation (`components/site-header.blade.php`)

Rewrite the header component:

- Brand left, nav right grid → Tailwind `flex justify-between items-center`.
- Nav links: `flex items-center gap-1` with hover states.
- Dropdown menus: existing Alpine.js logic stays, Pico-based styling replaced with Tailwind (`absolute`, `bg-white`, `shadow-lg`, `rounded`, `border`, `dark:bg-gray-800`).
- Hamburger / mobile nav: existing Alpine toggle stays, `_layout.scss` mobile nav styles replaced with Tailwind responsive utilities (`hidden md:flex`, etc.).

---

## Phase 4 — Footer (`components/site-footer.blade.php`)

Rewrite the footer:

- `flex justify-between items-center` layout.
- Theme toggle: existing Alpine logic stays, Pico toggle styles replaced with Tailwind.

## Phase 5 — Portal layout & chrome (`layouts/portal.blade.php`, portal header)

Rewrite the portal layout:

- Portal header bar: distinct background, user info left, nav right.
- Dark mode variants using `dark:` prefix.
- Logout link styling.

## Phase 6 — Structural components

Convert remaining structural components to Tailwind:

- `components/page-widgets.blade.php` — widget wrapper with spacing.
- `components/picture.blade.php` — minimal changes (mostly passes through attributes).
- `components/public-form.blade.php` — form element styling via `@tailwindcss/forms` plugin + utility classes.
- `components/inline-svg.blade.php`, `components/svg-icon.blade.php` — likely minimal changes.
- `pages/show.blade.php` — page template wrapper.

---

## Phase 7 — Site theme editor integration

Update the site theme editor (`SiteThemePage`) to write CSS custom properties instead of SCSS variables:

- Brand color → `--color-primary`
- Font family → `--font-family`
- Any other runtime-configurable tokens.

These are injected as inline `<style>` on the public layout. The Tailwind config references them so utility classes like `text-primary`, `bg-primary` respond to the admin's choices.

---

## Phase 8 — Cleanup

- Remove Pico from `node_modules` / `package.json`.
- Delete `_theme.scss`.
- Empty or delete `_layout.scss` (move any remaining custom CSS to `_custom.scss`).
- Verify dark mode toggle works end-to-end.
- Run `php artisan test` and fix any failures.
- Visual review of all public pages (home, blog index, blog post, event page, portal login, portal account).
