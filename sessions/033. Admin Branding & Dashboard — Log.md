# 033. Admin Branding & Dashboard — Log

## What was built

### General Settings — Admin Panel section

Added a new "Admin Panel" section to `GeneralSettingsPage` with four fields:

- **Company Name** (`admin_brand_name`) — text input, hint reads "Appears in the admin header beside your logo."
- **Dashboard welcome message** (`dashboard_welcome`) — rich text editor, hidden on dashboard when empty
- **Current logo preview** — read-only `Placeholder` showing the stored logo image (or "No logo uploaded.") so the user can see the current logo without re-uploading
- **Upload new logo** (`admin_logo_upload`) — file upload to `public` disk, `site/` directory, accepts PNG/JPG/SVG

Logo save logic only writes `admin_logo_path` when a new file is uploaded, preventing saves of other fields from wiping the stored logo.

`GeneralSettingsPage::save()` now calls `$this->redirect(static::getUrl())` after saving, triggering a full page reload so header changes are immediately visible.

### `AdminPanelProvider` — dynamic branding

The panel provider reads `admin_brand_name`, `admin_logo_path` from `SiteSetting` at boot and applies them via `->brandLogo()`. Both the logo image and the company name are rendered together as a single `HtmlString` in the logo slot (Filament renders `brandName` as a fallback only when no logo is set — the two cannot coexist natively).

Logo falls back to `public/images/admin-logo.png` when no custom logo has been uploaded. The placeholder file must be placed there manually.

### `DashboardWelcomeWidget`

New auto-discovered widget at sort `-10` (top of dashboard). Hidden via `canView()` when `dashboard_welcome` is empty. Renders rich text via `{!! !!}`.

### Seeder defaults

Added to `DatabaseSeeder`:

| key | default |
|-----|---------|
| `admin_brand_name` | `NonprofitCRM` |
| `admin_logo_path` | null |
| `dashboard_welcome` | `''` |

## Deferred decisions

- **Logo field pre-fill** — Filament's `FileUpload` cannot reliably round-trip a stored disk-relative path back into the component. The current UX shows a read-only preview above the upload field instead. The field always shows empty on page load; uploading a new file replaces the stored logo.
- **Storage symlink** — `php artisan storage:link` must be run once per installation for uploaded logos to be publicly accessible. This is standard Laravel setup.
- **Placeholder logo** — the file `public/images/admin-logo.png` must be provided manually. No default asset is included in the repo.
- **Logo sizing** — width/height fields were explored and removed. Filament's header layout constrains the logo slot; custom sizing was disruptive to the panel layout. Left as a future consideration.
