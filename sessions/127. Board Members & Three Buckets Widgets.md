# 127. Board Members & Three Buckets Widgets

Two new widget types: a board members grid for displaying people profiles from a collection, and a three-column content block with headings, body text, and CTA buttons.

---

## Design decisions (resolved before starting)

- Board members widget is collection-backed (`collections: ['members']`). Three buckets is config-driven (not collection-backed).
- Board members uses flexbox layout with configurable items per row (1-6) and configurable last-row alignment (left, center, right) via `justify-content`.
- Three buckets uses flexbox with `justify-content: space-between` so left and right buckets align to the content edge.
- Three buckets reuses the shared `widgets.components.buttons` partial and the `buttons` config type from the hero widget.
- LinkedIn and GitHub profile links use inline SVG icons. **The user will supply icon SVGs before the session starts.** If the icons are not present at `resources/views/widgets/components/icon-linkedin.blade.php` and `resources/views/widgets/components/icon-github.blade.php`, ask the user before proceeding.
- A `$gutter` SCSS variable will be added to `resources/scss/_variables.scss` (set to `1.5rem`) for use as the default gap in three buckets and available to future widgets.

## Phase 1 ‚Äî Board Members widget type registration

Add a `board_members` entry to `WidgetTypeSeeder` with:

- **handle:** `board_members`
- **label:** Board Members
- **description:** People grid with photos, names, titles, and optional bios from a custom collection.
- **category:** `['content']`
- **collections:** `['members']`
- **assets:** SCSS at `resources/scss/widgets/_board-members.scss`
- **config_schema:**
  - `heading` ‚Äî text. Optional heading displayed above the grid in an `<h2>`.
  - `collection_handle` ‚Äî select, `options_from: collections`
  - `image_field` ‚Äî select, `options_from: collection_fields:image`, `depends_on: collection_handle`
  - `name_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`
  - `title_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`. Job title.
  - `department_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`
  - `description_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`. Rich text bio.
  - `linkedin_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`
  - `github_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`
  - `extra_url_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`. URL value.
  - `extra_url_label_field` ‚Äî select, `options_from: collection_fields:text`, `depends_on: collection_handle`. Link label.
  - `items_per_row` ‚Äî number, default: 3, min 1, max 6, advanced
  - `row_alignment` ‚Äî select, options: `left` / `center` / `right`, default: `center`, advanced. Controls `justify-content` on the flex container for the last row.
  - `image_shape` ‚Äî select, options: `circle` / `rectangle`, default: `circle`
  - `image_aspect_ratio` ‚Äî text, default: `1 / 1`, advanced. CSS `aspect-ratio` value for rectangle mode. Ignored when shape is circle.
  - `background_color` ‚Äî color, default: `#ffffff`. Section background.
  - `pane_color` ‚Äî color, default: `#ffffff`. Individual card background.
  - `border_color` ‚Äî color, default: `#cccccc`. Card border colour.
  - `border_radius` ‚Äî number, default: 5, advanced. Border radius in px for cards.

  Place the three colour controls (`background_color`, `pane_color`, `border_color`) adjacent in the schema so they render side by side. Use `group: 'colors'` on all three.

## Phase 2 ‚Äî Board Members Blade template and SCSS

Create `resources/views/widgets/board-members.blade.php`:

- If `heading` is set, render it in an `<h2>`.
- Flex container with `justify-content` driven by `row_alignment` config.
- Each member is an `<article>` element.
- Card width calculated from `items_per_row`, same flexbox pattern as logo garden.
- Responsive: reduce items per row at `$bp-md` (max 2) and `$bp-sm` (1).
- **Image:** rendered as a `<div>` with the image as CSS background (`background-image`, `background-size: cover`, `background-position: center`). Circle mode applies `border-radius: 50%` and enforces `aspect-ratio: 1 / 1`. Rectangle mode uses the configured `image_aspect_ratio`.
- **Name** in `<h3>`.
- **Job title** in `<p>` with a semantic class.
- **Department** in `<p>` with a semantic class.
- **Description** rendered with `{!! !!}` (it's rich text from the collection).
- **LinkedIn** ‚Äî if present, render as an `<a>` with the LinkedIn SVG icon partial.
- **GitHub** ‚Äî if present, render as an `<a>` with the GitHub SVG icon partial.
- **Extra URL** ‚Äî if both URL and label are present, render as a plain `<a>`.
- Social links grouped in a `<div>` with a semantic class.
- All optional fields only render when the value is non-empty.
- Section background colour applied to the outer container. Card background, border, and border-radius applied to each `<article>`.

### SCSS (`resources/scss/widgets/_board-members.scss`)
- `.widget-board-members` container: flexbox wrap
- `.board-member` card: border, background, border-radius via CSS custom properties
- `.board-member__image`: background-image container, aspect-ratio, circle vs rectangle
- `.board-member__name`, `__title`, `__department`, `__bio`, `__links`: semantic classes
- Responsive breakpoints at `$bp-md` and `$bp-sm`
- No decorative styling on cards ‚Äî semantic structure only

## Phase 4 ‚Äî Three Buckets Blade template and SCSS

Create `resources/views/widgets/three-buckets.blade.php`:

- Flex container with `justify-content: space-between`.
- Three child `<div>` elements, each containing:
  - Heading in `<h3>` (if non-empty), with `text-align` from `heading_alignment`.
  - Body rendered with `{!! !!}`, with `text-align` from `body_alignment`.
  - Buttons via `@include('widgets.components.buttons', ['buttons' => $ctas_N, 'alignment' => $buttonAlignment])`.
- Each bucket takes equal width: `calc((100% - 2 * var(--bucket-gap)) / 3)`.
- Responsive: stack vertically at `$bp-md` (each bucket full width).

### SCSS (`resources/scss/widgets/_three-buckets.scss`)
- Add `$gutter: 1.5rem;` to `resources/scss/_variables.scss`.
- `.widget-three-buckets`: flex, `justify-content: space-between`, `gap: var(--bucket-gap, $gutter)`
- `.three-buckets__bucket`: width calc, flex column layout
- Responsive: full width at `$bp-md`
