# 126. Widget — Three Buckets

Add a `three_buckets` widget type: three side-by-side content blocks, each with an icon or image, heading, body text, and optional CTA link. A common marketing pattern for highlighting three key features, services, or value propositions.

---

## Design decisions (resolved before starting)

- This is a config-driven widget, not collection-backed. The three buckets are defined directly in the widget's config schema — the admin fills in fields for each bucket in the inspector panel.
- Three is fixed. If someone needs two or four, they use a column widget with text blocks. This widget exists to make the common three-bucket pattern fast and consistent.
- Categorised as `content` with `allowed_page_types: null`.
- Icons are selected from a predefined set (Heroicons via SVG references) or omitted in favour of a small image. Investigate which approach fits the existing config schema field types — if neither icon pickers nor image arrays are supported, use a simple text field for an emoji or icon name with a documented list of supported values.
