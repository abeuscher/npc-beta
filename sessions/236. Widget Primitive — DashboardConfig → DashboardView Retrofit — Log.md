## Deferred decisions

- **`manage_dashboard_config` permission rename to `manage_dashboard_view`.** High blast radius (Spatie permissions, role assignments, the `Gate::before` admin bypass, every `manage_dashboard_config` literal in tests + middleware). Low product value. Named in the 237 prompt's "Out of scope" list; pick up only if surfaced by an admin-UX review.
- **Vue concept-name strings unchanged.** `'mode' => 'dashboard'` (bootstrap), `'owner_type' => 'dashboard_config'` (bootstrap), `ownerType: 'dashboard_config'` (JS event payload). All three are decorative — Vue's `widget-created` handler reads only `ownerId` (verified). Renaming would require coordinating Vue + PHP without behavior change. Deferred indefinitely.
- **Route URI segment `{dashboardConfig}` and controller parameter variable name `$dashboardConfig`.** Renaming would change the URL pattern (`/admin/api/dashboard-builder/configs/{dashboardConfig}/...`). Kept byte-equivalent — only the type-hint changed (`DashboardConfig` → `DashboardView`). Cosmetic asymmetry between variable name and type. Cleanup whenever convenient.
- **`DashboardConfigFactory` non-existence noted.** No factory file existed pre-session; the unit test creates rows via `DashboardView::create([...])` directly. Adding a factory is a forward hook only if a test surface needs it.
- **`ViewRegistry` integration for DashboardView.** Per Decision 6 in the prompt — registry is shaped against `RecordDetailView`'s `(record_type, handle)` key; per-role DashboardView doesn't fit. Forward hook.
- **`layout_config` JSONB column on dashboards.** No current consumer; deferred until something genuinely needs it.
