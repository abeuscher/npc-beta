## What was built

### Phase 1 — Data model

- New `page_layouts` table (uuid PK, page_id FK with cascade, label, display enum-like string, columns int, layout_config JSONB, sort_order, timestamps)
- Added `layout_id` UUID FK to `page_widgets` (cascade on delete)
- Migration walks existing `column_widget` instances and reparents their children: creates a `page_layout` per column widget, sets `layout_id` on every child, deletes the column widget row
- Dropped `parent_widget_id` from `page_widgets` after data migration
- Deleted the `column_widget` row from `widget_types` and removed the seeder entry
- New `App\Models\PageLayout` model: UUID PK, `layout_config` cast to array, `page()` belongsTo, `widgets()` hasMany ordered by column_index then sort_order
- Updated `App\Models\PageWidget`: replaced `parent_widget_id` with `layout_id` in fillable, replaced `parent()`/`children()` relationships with a single `layout()` belongsTo, rewrote `serializeStack()` and `serializeOne()` to emit a merged page flow with type-tagged items, added `serializeLayout()`, rewrote `copyBetweenPages()` to copy layouts and their child widgets
- Schema docs: new `docs/schema/page_layouts.md`, updated `docs/schema/page_widgets.md`, added entry to `docs/schema/README.md`

### Phase 2 — API & rendering

- Rewrote `PageController::renderPage()` to load root widgets and layouts, merge them by `sort_order` into a single page flow, render each item appropriately. New `renderLayoutBlock()` builds the CSS container styles from `display`, `columns`, and `layout_config` and emits a `<div class="page-layout">` wrapper with one `<div class="layout-column">` per slot
- Mirror changes in `PagePreviewController` and the `renderWidgetBlock` path used by admin preview
- `App\Http\Controllers\Admin\PageBuilderApiController` rewrote: `store`, `destroy`, `copy`, `reorder`, `buildTree`, `formatWidget`, `formatWidgetWithPreview`, `renderWidgetForPreview`, `collectLibs`, `preview`, all switched from `parent_widget_id`/`children` semantics to `layout_id`/`slots` semantics. Removed `formatChildren()` and `renderColumnChildren()`. Removed column-nesting prevention logic (no longer applicable)
- New `formatLayout()` helper returning the layout shape with `slots`
- New layout CRUD endpoints: `storeLayout`, `updateLayout`, `destroyLayout`. `updateLayout` sanitizes `layout_config` to a whitelist of CSS keys (`grid_template_columns`, `gap`, `align_items`, `justify_items`, `justify_content`, `grid_auto_rows`, `flex_wrap`, `flex_basis`)
- `reorder` endpoint accepts mixed widget+layout items via a `type` field, validates ownership for both, validates referenced `layout_id` values belong to the page, defaults missing `type` to `'widget'` for backward compatibility with the in-flight Vue store
- `buildTree` returns both a new `items` key (merged page flow with type-tagged entries and layout slots) and a legacy `widgets` key (root widgets only) so the in-flight Vue store keeps working through Phase 3
- `routes/admin-api.php`: added `POST {page}/layouts`, `PUT layouts/{layout}`, `DELETE layouts/{layout}`
- `App\Livewire\PageBuilder::getBootstrapData()` rewritten to emit the same merged-flow shape (root widgets first, layouts with slot widgets, sorted by `sort_order`) and now returns both `widgets` (legacy) and `items` (new) keys
- Other PHP that referenced the old schema fixed during this phase: `App\Services\Media\ChromeRenderer`, `App\Filament\Resources\PageResource\Pages\CreatePage::hydrateWidgets()` (now handles serialized layout entries with slots)
- `WidgetTypeSeeder` cleaned up — `column_widget` entry removed

### Phase 3 — Vue editor: store & types

- `resources/js/page-builder-vue/types.ts` rewritten: `Widget` replaces `parent_widget_id` with `layout_id`, drops `children`. New `PageLayout`, `WidgetItem`, `LayoutItem`, and `PageItem` (discriminated union) types. New `CreateLayoutPayload` and `UpdateLayoutPayload` types. `ReorderItem` becomes a discriminated union supporting both widget and layout reorder shapes. `BootstrapData` and `TreeResponse` switched from `widgets` to `items`
- `resources/js/page-builder-vue/api.ts`: response types updated, new `createLayout`, `updateLayout`, `deleteLayout` methods
- `resources/js/page-builder-vue/stores/editor.ts` rewritten to track the merged page flow:
  - New state: `pageItems`, `layouts` (flat map), `selectedItemId`, `selectedItemType`
  - `widgets` flat map now includes both root widgets and widgets inside layout slots for unified config lookups
  - New action `populateFromItems()` builds all three maps from a merged items array; normalizes slots so every column index 0..columns-1 has a real array and every layout reference is shared between `pageItems` and `layouts` (so mutations propagate to both)
  - New actions: `selectItem`, `createLayout`, `updateLayout`, `deleteLayout`, `updateLocalLayout` (immediate local mutation + debounced API save)
  - Backward-compat shims retained so existing components keep working: `selectBlock`, `selectedBlockId` (computed), `rootWidgets`, `rootOrder`, `childrenOf`, `columnTargets`
- `resources/js/page-builder-vue/components/PreviewCanvas.vue`: drag-end ReorderItem shape updated to use the new `type`/`layout_id` fields

### Phase 4 — Vue editor: preview & interaction

- New `LayoutRegion.vue` component:
  - Renders a layout container with column slots
  - Hover-revealed handle bar above the layout (the only drag-init surface for the layout in the root list)
  - Persistent left-side select affordance (arrow button)
  - Per-slot draggable lists for child widgets
  - Per-slot styling for flex's per-column flex-basis
  - "+ Add widget" button in each empty column slot, dispatching `open-widget-picker` with `layoutId` and `columnIndex`
- `PreviewCanvas.vue`:
  - Renders `store.pageItems` mixing `PreviewRegion` and `LayoutRegion`
  - Add Block dropdown gains `+ 2/3/4 columns` entries that call `store.createLayout()` and select the new layout
  - `buildReorderPayload()` walks the merged page flow and emits a complete reorder payload (one item per widget with `layout_id` + `column_index`, one item per layout)
  - Cross-boundary drag-and-drop via vuedraggable's shared `group: 'page-items'`. Slot draggables have a `put` filter that rejects layout items (no nesting)
  - Root draggable uses `:list="store.pageItems"` (direct mutation), `handle=".preview-region__overlay, .layout-region__handle"` (widgets stay grabbable from anywhere on their body, layouts are only grabbable by their hover handle bar — frees the layout body to receive incoming widget drops), `:fallback-on-body="true"`, `:swap-threshold="0.65"` (matches the SortableJS nested-sortables example so cursors can pass through to nested targets)
  - Slot draggables also use `:fallback-on-body="true"` and `:swap-threshold="0.65"`
  - Alpine reinit watch updated to react to all widgets in the flat map (not just root widgets), so preview HTML changes inside layout slots also trigger re-init
  - New "+ Add widget" button at the bottom of the page flow (2rem font for legibility under preview zoom)
- `InspectorPanel.vue`: switches between `LayoutInspectorPanel` (when a layout is selected) and the existing widget inspector

### Phase 5 — Column inspector

- New `LayoutInspectorPanel.vue` with all CSS controls:
  - Header: editable label with debounced save + delete button (footer)
  - Display toggle: `display: grid` | `display: flex` two-button toggle
  - Columns count: number input (1–12), auto-extends `grid_template_columns` or `flex_basis` array as columns increase
  - Grid section (when display=grid): `grid-template-columns` with preset/manual mode and 9 presets (1fr 1fr, 2fr 1fr, 1fr 2fr, 3fr 1fr, 1fr 3fr, 1fr 1fr 1fr, 1fr 2fr 1fr, 2fr 1fr 1fr, 1fr 1fr 1fr 1fr); `gap` text input + presets dropdown; `grid-auto-rows` text input; `align-items` and `justify-items` selects
  - Flex section (when display=flex): `justify-content`, `align-items`, `gap` (with presets), `flex-wrap` toggle, per-column `flex-basis` text inputs
  - All CSS property names visible to the user as labels
  - Brief explanatory hint under each control
  - Delete button with confirmation: "Deleting this column layout will also delete all widgets inside it. Drag widgets out first to keep them."
- Live preview wired through the new `updateLocalLayout` store action — local state mutates immediately so `LayoutRegion`'s `containerStyle` computed re-renders, then the change is debounced-saved to the API at 500ms

## Deferred decisions

- **Nav widget redesign** — session 153 produced the column system. Session 154 will introduce a `nav` widget by extracting the existing nav rendering from the `site_header` widget. The full nav redesign (modes, dropdown, hamburger collapse, stack collapse, accordion) is deferred to a dedicated future session after 154 isolates the widget.
- **Chrome layout rendering** — `ChromeRenderer::renderPage()` currently only loads root widgets (`whereNull('layout_id')`) and does not render `PageLayout` records. The bug is invisible today because no chrome page has a layout, but the instant a column layout is added to `_header` or `_footer` the rendered chrome would be empty. **This is intentionally deferred to session 154**, which will fix the renderer as part of introducing columns to the header/footer layouts.
- **Nested layouts** (layouts inside layouts) — out of scope; session prompt explicitly excluded.
- **Layout templates / presets saved for reuse** — out of scope.
- **Per-widget grid-column spanning** — out of scope.
- **Responsive breakpoint controls for layouts** — out of scope.
