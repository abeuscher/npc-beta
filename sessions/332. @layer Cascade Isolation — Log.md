### Phase 1 — @layer cascade isolation (commit `3fdf754`, the planned work)

Rolled all host CSS into CSS cascade layers in one coordinated pass so widget interiors win over host base by **layer order, not selector specificity**.

- **Layer order `@layer reset, host, widgets;`** declared once via a new `resources/scss/_layers.scss`, `@use`'d first in `public.scss` (module CSS emits in use-order, so the order statement lands ahead of any host block). Owner confirmed a **3-layer** order over the prompt's suggested 4 — collapsing all host partials into one `host` layer preserves today's host-vs-host cascade exactly, so the only precedence that moves is widgets-over-host.
- **reset/vendor** (modern-normalize + swiper) loaded into `@layer reset` via `meta.load-css()` (a Sass `@use` can't carry a layer).
- **Every host partial** (`_base`…`_custom`) wraps its body in `@layer host { }` — one edit covers both compile paths, since the Vite `public.scss` bundle and the build-server widget bundle read the same files (confirmed: host base rules ship in both).
- **`AssetBuildService::collectSources()`** prepends the order statement, wraps the runtime emit (typography / colour / button overrides) in `@layer host`, and wraps each widget SCSS source in `@layer widgets`.
- **`public.blade` nav-font-family inline `<style>`** wrapped in `@layer host`; operator `custom_scss` left **unlayered** by design (wins over all layers — the intended operator-wins exception).
- **Specificity workarounds retired** as positive proof: EventMiniCalendar `.emc-event` and EventsListing `.events-list__row` `display:block` → natural `list-style:none`; EventsListing filter `flex:0 0 14rem` → `width:auto` (owner call, the one accepted visible change).
- **Standing guard** `WidgetStyleLayerDisciplineTest` (design group 21→22): asserts every host CSS source emits into a declared layer, nothing escapes unlayered. Colour-token baseline line shifted `:122`→`:123`.
- **Verified behavior-preserving by construction:** a source-level diff of `collectSources()` before/after showed only layer wrappers + the three reverts — zero host-rule content changed; the Vite host bundle diff was clean (only a cosmetic swiper `A,B{}`→`A{} B{}` split from `meta.load-css`).

### Phase 2 — Widget container-query rollback (commit `33278b6`, owner-directed)

Live investigation (Playwright against the real local site + the admin editor with owner creds) showed the 330/331 `@media`→`@container` migration had regressed the tool: it keyed widget collapse to mobile **viewport** breakpoints (`$bp-md` 768 / `$bp-sm` 576) applied to **container** width. A column in a multi-column layout is below those at every viewport, so a column-placed widget permanently rendered its phone layout on desktop **and silently overrode the operator's configured count** (Logo Garden "logos per row", Board Members "items per row", Pricing Chart column count). Confirmed live: a Logo Garden set to N rendered the hardcoded collapse instead.

**Owner decision: preserve user control.** A widget renders its configured count at every width; a cramped result in a narrow column is the operator's own undoable choice, not the tool overriding them. Real per-widget mobile support is future widget-system work.

- Removed every `@container np-widget` block — LogoGarden, BoardMembers, PricingChart, EventsListing (featured-hero stack), ProductCarousel (slide width) — leaving each widget's existing config-driven base rule intact.
- Removed the `.widget--* { container-type; container-name: np-widget }` context from `resources/scss/_layout.scss` (kept `.page-layout`'s container — that's the separate user-toggled page column-collapse, out of scope).
- Removed LogoGarden carousel/smooth Swiper viewport `breakpoints {0,576,768}` ramp; `slidesPerView` now honors `logos_per_row` flat.
- `widget-development.md`: replaced the "use `@container`" guidance with "no width-based collapse; mobile is future widget-system work."
- No `@media` reintroduced, so the `WidgetStyleBoundaryConsumptionTest` `@media` ban stayed satisfied untouched.
- **Re-verified in a real browser:** the public home page renders cleanly (2-column layout, counts honor config, Blog Listing shows 3 comfortable cards with a working pager); editor counts match public.
