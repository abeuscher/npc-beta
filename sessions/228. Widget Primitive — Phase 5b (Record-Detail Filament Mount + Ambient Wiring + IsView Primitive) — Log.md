## What was built

### Phase 1 — `IsView` interface + `RecordDetailView` concrete

- [app/WidgetPrimitive/IsView.php](app/WidgetPrimitive/IsView.php) — new top-level interface (sibling to `Slot` / `DataContract` / `AmbientContext`). Declares the widget-composition contract: `handle()`, `slotHandle()`, `widgets()`, `layoutConfig()`. Plain interface, not abstract class — concrete impls free to be Eloquent models or PHP classes.
- [app/WidgetPrimitive/Views/RecordDetailView.php](app/WidgetPrimitive/Views/RecordDetailView.php) — first concrete `IsView`, plain final PHP class. Constructed with `(handle, recordType, widgets, layoutConfig=[])`. `slotHandle()` always returns `'record_detail_sidebar'`. Lifts to Eloquent model in 5c when the table-backing lands.

### Phase 2 — `RecordDetailAmbientContext` payload + slot body

- [app/WidgetPrimitive/AmbientContexts/RecordDetailAmbientContext.php](app/WidgetPrimitive/AmbientContexts/RecordDetailAmbientContext.php) — empty payload after 5a; this session adds `public readonly Model $record` constructor field.
- [app/WidgetPrimitive/Slots/RecordDetailSidebarSlot.php](app/WidgetPrimitive/Slots/RecordDetailSidebarSlot.php) — signature changed from `ambientContext(): SlotContext` (throws RuntimeException after 5a) to `ambientContext(Model $record): SlotContext`. Body returns `new SlotContext(new RecordDetailAmbientContext($record), publicSurface: false)`. RuntimeException import dropped.

### Phase 3 — `SOURCE_RECORD_CONTEXT` source + projector + tokens

- [app/WidgetPrimitive/DataContract.php](app/WidgetPrimitive/DataContract.php) — `public const SOURCE_RECORD_CONTEXT = 'record_context';` added alongside the three existing sources.
- [app/WidgetPrimitive/RecordContextTokens.php](app/WidgetPrimitive/RecordContextTokens.php) — closed token registry. `TOKENS = ['record_id', 'record_type']`. `values(?Model $record)` returns the closed map; null record yields empty strings (fail-closed). Mirror of `App\Services\PageContextTokens` — placed at the prescribed `App\WidgetPrimitive` namespace location (the namespace migration of `PageContextTokens` itself is deferred, captured in the Deferred Decisions section below).
- [app/WidgetPrimitive/Projectors/RecordContextProjector.php](app/WidgetPrimitive/Projectors/RecordContextProjector.php) — mirror of `PageContextProjector`. `project(DataContract, ?Model)` calls `RecordContextTokens::values()` and returns the full closed token map (per source-as-capability — contracts on this source declare no `fields`).
- [app/WidgetPrimitive/ContractResolver.php](app/WidgetPrimitive/ContractResolver.php) — constructor extended with `RecordContextProjector $recordContextProjector`. New `match` arm dispatches `SOURCE_RECORD_CONTEXT`. New private `recordFromContext(SlotContext): ?Model` returns `$context->ambient->record` only when ambient is `RecordDetailAmbientContext`; null otherwise (fail-closed cross-ambient gate).
- [app/Providers/WidgetServiceProvider.php](app/Providers/WidgetServiceProvider.php) — `RecordContextTokens` and `RecordContextProjector` registered as singletons in `register()` alongside the existing `PageContextTokens` binding.

### Phase 4 — `WidgetRenderer` record-aware

- [app/Services/WidgetRenderer.php](app/Services/WidgetRenderer.php) — added `?Model $record = null` as the 5th positional optional parameter to `render()`. Three-way slot dispatch: canvas (existing), `record_detail_sidebar` (calls `ambientContext($record)`), default (calls `ambientContext()`). Fail-closed empty render shape (`['html' => null, 'styles' => '', 'scripts' => '']`) when slot is `record_detail_sidebar` and record is null. Mirror of how session 212 added the `$slotHandle` parameter — same shape, same precedent. Existing canvas/dashboard/preview/chrome call sites all leave the new param at default null — byte-equivalent.

### Phase 5 — Filament hosting on `EditContact`

- [app/Filament/Widgets/RecordDetailViewWidget.php](app/Filament/Widgets/RecordDetailViewWidget.php) — Filament Widget mirror of `DashboardSlotGridWidget`. Receives the parent EditRecord page's record via the standard `$this->record` mechanism. For 5b, hardcodes a `RecordDetailView` for `Contact::class` in `view()` containing one in-memory `PageWidget` (the placeholder) eager-bound to its `WidgetType`. Fail-closed empty array when `$this->record` is null.
- [resources/views/filament/widgets/record-detail-view.blade.php](resources/views/filament/widgets/record-detail-view.blade.php) — Blade view. Mirror of `dashboard-slot-grid.blade.php`. - [app/Filament/Resources/ContactResource/Pages/EditContact.php](app/Filament/Resources/ContactResource/Pages/EditContact.php) — added `protected function getFooterWidgets(): array { return [RecordDetailViewWidget::class]; }`.

## Production deployment notes

**`db:seed --class=WidgetTypeSeeder` must run on the deploy server** to sync the new `record_detail_placeholder` row into `widget_types`. Without it, `RecordDetailViewWidget` mounts but renders no content (`$pw->widgetType` is null → renderer's empty-render-shape guard fires). The seeder reads from registered widget definitions, so no seeder code changes — just re-run after deploy.

Beyond that: standard `migrate` (no migrations this session). No `build:public` (no widget asset, no SCSS).
