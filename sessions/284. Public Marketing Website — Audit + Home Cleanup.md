## Project intent

We are designing a marketing site **for the product**, using a small set of well-known B2B SaaS marketing pages as **structural design guides**. Lift their basic layouts and section rhythm; assume standard asset delivery; use FPO (placeholder) images from the sample-image library; apply that design pattern to the supplied copy. The borrow is structure-only — section-as-slab layouts, alternating background bands, generous whitespace, restrained type scale, no animations. Surface decoration (custom illustration, marketing-bright saturation, abstract metaphor imagery) is **not** what this site looks like. Reference images at `sessions/public website/references/` are structural specs only. Real photography lands post-track.

This framing applies to every PMW phase. PMW1 (this session) is the audit + home cleanup that establishes what the system can express against the references; PMW2 / PMW3 author the remaining four pages against the same pattern; PMW4 captures everything for review.

Audit the current home against these and surface the delta to the user before applying changes.

Confirm the import path at session start (admin Tools → Importer? page-builder API? something else?) — the brief assumes import-without-errors but doesn't pin the mechanism.

## Phases

### Phase 1 — Read all canonical inputs

Read in order, no skipping:

1. `sessions/tracks/public-marketing-website.md` — track doc.
2. `sessions/public website/brief.md` end-to-end.
3. `sessions/public website/copy.md` — full read; the home section is the in-scope half this phase.
4. `sessions/public website/references.md` and the four reference PNGs.
5. `sessions/public website/existing-home-page-export.json` — internalize the JSON shape, identify the two-color-models drift the brief flags.

### Phase 2 — System audit

Produce `sessions/public website/audit-summary.md`. Subsections:

- **Type scales** — locate via `resources/scss/`, `DesignSystemPage`, the typography SiteSetting JSON. List the scales available with their h1–h6 / p / li sizes and where they're declared.
- **Button styles** — locate via `button_styles` SiteSetting JSON, primary / secondary / text-only. List variant names + colors + states. Flag any missing variant the brief's CTAs would need (e.g. ghost-on-dark for the dark "This is not a SaaS company" band).
- **Appearance config schema** — `appearance_config` shape per `AppearanceStyleComposer`. List the keys (`background.color`, `background.gradient`, `background.image`, `text.color`, `layout.padding.*`, `layout.full_width`, etc.) and which ones the brief's section-band / padding / color-model conventions touch.
- **Widget inventory** — the `widget_types` table + `app/Widgets/` discovery. List widget handles, what each does, whether it can live inside a layout slot, and whether any widget the brief implies (the "tabbed feature explainer" forecast in the track doc; the Form widget for PMW3) exists or surfaces a gap.
- **Sample image library** — locate via `App\Data\SampleLibrary` or wherever sample images live. List categories + count + suitability for marketing-page placeholders.

Surface the audit summary for user review. Do not start Phase 3 until the user has reviewed and acknowledged.

### Phase 3 — Home page cleanup

Read `existing-home-page-export.json` against the audit findings + brief's conventions. Apply structural cleanup:

- Standardize section-band backgrounds onto `type: "layout"` blocks (not inner widgets) per the brief.
- Normalize section padding to 150px / 150px for major bands and 25px / 50px for widgets inside column layouts.
- Normalize color model: `appearance_config.text.color` as source of truth; inline `style="color:rgb(...)"` only for inline emphasis.
- Verify the established background tones (`#ffffff`, `#d4d4f2`, `#373c44`) are being used and no others have crept in.
- Preserve all copy verbatim. The existing copy is canonical per the brief's authority hierarchy.

Write the cleaned result to `sessions/public website/home.json`.

### Phase 4 — Re-import + screenshot + gap report scaffold

- Re-import `home.json` into the local dev system. Confirm import succeeds with no errors.
- Capture a screenshot of the rendered home page (informal — browser screenshot is fine for PMW1; the proper harness lands at PMW4).
- Scaffold `sessions/public website/gap-report.md`. First rows: anything Phase 2's audit surfaced as missing (button variant gaps, missing widgets, missing config options) + anything the home cleanup hit that warranted a workaround. Format per brief: *what was attempted / what blocked it / workaround used / recommended action*.
- Surface for user review: the audit summary, the cleaned home page (live URL), the gap report scaffold.
