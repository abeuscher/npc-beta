# 126. Widget — Alternating Panels

Add an `alternating_panels` widget type: a vertical stack of image-plus-text panels that alternate the image side (left, right, left, right). Sourced from a custom collection. Common pattern for feature tours, service descriptions, and storytelling sections.

---

## Design decisions (resolved before starting)

- Panel data comes from a custom collection via `WidgetDataResolver`. Each collection item needs an image field, heading, body text, and optional CTA.
- The alternation is automatic — odd items have image on the left, even items have image on the right. No per-item config needed.
- Categorised as `content` with `allowed_page_types: null`.
- Standard collection-backed widget contract.

### Config schema

| Key | Label | Type | Default | Notes |
|---|---|---|---|---|
| collection_handle | Collection | select | '' | Source collection |
| image_field | Image Field | text | 'image' | Field key for panel image |
| heading_field | Heading Field | text | 'heading' | Field key for panel heading |
| body_field | Body Field | text | 'body' | Field key for panel body text |
| cta_text_field | CTA Text Field | text | 'cta_text' | Field key for link text (optional) |
| cta_url_field | CTA URL Field | text | 'cta_url' | Field key for link URL (optional) |
| heading | Section Heading | text | '' | Optional heading above all panels |

### Template

Vertical stack of panels. Each panel is a two-column grid: image on one side, text content (heading, body, CTA) on the other. Odd-indexed panels: image left, text right. Even-indexed panels: image right, text left. Achieved with CSS `order` or `flex-direction: row-reverse` on alternate items.

On mobile, all panels stack with image on top. Images use `object-fit: cover` with a fixed aspect ratio. Tailwind utilities.
