# A001. Content Import/Export — Extend Bundle Coverage to Events, Products, Navigation, Collections

> **Agentic session — runs autonomously in a worktree alongside regular session work.** Pure backend pattern-matching: extend the existing page/template/design export+import shape (sessions 309 + 310) to four more entity types. No UI work. No architectural calls — the patterns are fixed by the time this session runs. Pest round-trip per entity is the binary acceptance signal.

One sentence: replicate the page exporter / importer / analyzer shape across `events`, `products`, `navigation`, and `collections`, each landing as its own self-contained extension to `ContentExporter` + `ContentImporter`.

---

## Why this is agent-friendly

- The architecture is fixed. Session 309 establishes the opt-flag + analyzer pattern; session 310 establishes the rollup wrapper. This session does NOT relitigate either.
- Each of the four entities is a textbook copy of the page extension: same shape, different table set.
- The `import_id_maps` plumbing and `findByExternalId` patterns already exist (see how pages and events-via-`createPublishedEventWithTiers` already use them in the e2e helpers).
- Acceptance is binary: Pest round-trip per entity passes or it doesn't.

## Scope, per entity

### Events

Tables: `events`, `ticket_tiers`, `event_registrations`.

- Exporter: `ContentExporter::exportEvents(array $eventIds, array $opts = [])`. Emits `payload.events: [{event: {…}, tiers: [{…}], registrations: [{…}]}, …]`. Skip registrations by default (operator data, not template content); flag-gated via `$opts['with_registrations']`.
- Importer: `ContentImporter::importEvent($data, $log)`. Resolves contact references for registrations via the existing `import_id_maps` plumbing. Page-level event landing pages (events with a `landing_page_id`) round-trip alongside the event.
- Analyzer: extend `analyze()` to count events, surface slug-collision summary.
- Round-trip Pest test: create event with two tiers and three registrations → export → import into clean DB → assert event + tiers + registration rows match.

### Products

Tables: `products`, `product_prices`. Media via the existing media-reference plumbing.

- Exporter: `ContentExporter::exportProducts(array $productIds, array $opts = [])`. Emits `payload.products: [{product: {…}, prices: [{…}]}, …]`. Product images flow through the existing `media` payload (reference-mode) — no separate handling needed.
- Importer: `ContentImporter::importProduct($data, $log)`. Standard pattern, no cross-entity references beyond media.
- Analyzer: extend to count products + price rows.
- Round-trip Pest test: create product with two prices + one image → export → import into clean DB → assert match.

### Navigation

Storage: `SiteSetting` row(s) holding the navigation tree as JSON. Confirm exact storage shape at session start by reading the Navigation Filament resource — adjust the export shape to match.

- Exporter: `ContentExporter::exportNavigation()`. Emits `payload.navigation: {…}` — a single object, the nav tree.
- Importer: `ContentImporter::importNavigation($data, $log)`. Single-row write to `SiteSetting`. Mirrors `importDesign()`'s shape exactly — additive deep-merge over resolver defaults if a resolver exists; whole-row replacement otherwise (whichever matches the existing Navigation editor's write path).
- Analyzer: extend with `has_navigation: bool`.
- Round-trip Pest test: set a non-default navigation tree → export → import into clean DB → assert the navigation row equals the source.

### Collections

Tables: `collections`, `collection_items`. Tags via `taggables` (existing plumbing).

- Exporter: `ContentExporter::exportCollections(array $collectionIds, array $opts = [])`. Emits `payload.collections: [{collection: {…}, items: [{…}]}, …]`. The custom `data` JSON per item travels verbatim — no transformation, no field-shape policing (the collection schema is the source of truth for shape).
- Importer: `ContentImporter::importCollection($data, $log)`. Standard pattern.
- Analyzer: extend to count collections + their item counts.
- Round-trip Pest test: create collection with three items (one tagged) → export → import into clean DB → assert collection + items + taggables match.

## Phases

Four phases, one per entity. Each is independent; the agent can sequence them in any order. Per phase:

1. Add exporter method.
2. Extend analyzer.
3. Add importer method + integrate into `ContentImporter::import()`'s entity loop.
4. Add round-trip Pest test.
5. Run fast Pest; iterate until green.

Order recommendation (smallest blast radius first): Navigation → Products → Events → Collections. The agent may reorder if a different sequence reveals shared plumbing it wants to extract first.
