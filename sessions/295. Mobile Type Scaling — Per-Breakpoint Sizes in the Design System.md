## Phases

### Phase 1 — `SiteSetting` shape migration with byte-exact back-compat

Convert each element's `font.size` from `{value, unit}` to `{ xl:{value,unit}, lg, md, sm }` in `TypographyResolver`. Existing stored values copy **byte-exact into `xl`**; `lg/md/sm` derive from the chosen per-class ramp. Non-destructive (don't rewrite the stored `SiteSetting` row on read). Required regression test: a configured install with hand-tuned sizes (mirror the 285-tuned shape) round-trips with every `xl` value identical to what was stored — the G1/285 data-loss guard, written first.

### Phase 2 — Design System per-breakpoint fields + line-height + heading-margin

On `DesignSystemPage` Theme → Text Styles, add the per-breakpoint size fields beneath the existing size control (xl = the existing value; lg/md/sm prefilled from the ramp, each user-overridable). Line-height becomes a unitless multiplier that rides the size. Add the per-class heading bottom-margin (em). Keep the existing save/export path; extend its shape.

### Phase 3 — `TypographyCompiler` per-breakpoint emission

`elementDeclarations()` emits the `xl` declaration plus three `@media (max-width: $bp-lg/md/sm)` blocks per element (font-size + the proportionally-scaled line-height + heading margin-bottom where applicable). Verify the emitted block overrides correctly given it loads post-bundle inline; confirm `compileScoped()` stays consistent.

### Phase 4 — Objective oracle + tuning handoff

Build/adapt the 294 multi-breakpoint capture to our five public pages + the Design System preview. Objective checks: body never below a legible minimum, no text overflow, no horizontal scroll, a heading never smaller than the body beneath it, hierarchy monotonic per breakpoint. Drive to zero. Capture before/after at the breakpoints. Surface the chosen ramp + captures for the user's visual judgment.
