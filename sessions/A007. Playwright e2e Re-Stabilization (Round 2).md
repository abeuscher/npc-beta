## Phases

### Phase 1 — Evidence: establish the read channel, gather the actual digest

No product or test fixes in this phase. The deliverable is a single CI run that publishes a compact, readable result digest the agent can pull via MCP.

- Work on the session's branch. (`CLAUDE.md` Git Workflow governs branch naming; never commit on `main`.)
- Mirror A003's methodology: temporarily scope the workflow to Playwright-only (`if: false` on `fast` / `slow` / `tests`; drop `playwright`'s `needs: slow`), add an `on: push` exclusion for the `ci-logs/**` ref to prevent re-trigger loops, capture Playwright's JSON reporter + tee stdout + record rc, and publish a compact digest (failing spec list with first error lines, per-spec timings, slowest-N for signature detection) plus any in-container probes (e.g. `storage/logs/laravel.log` tail, `public/build` state, `public/build/widgets/manifest.json` presence, delivered HTML head of `/`, `/news`, `/admin/login`) to a `ci-logs/playwright` branch the agent reads via `mcp__github__get_file_contents`. All of this is **temporary plumbing**; mark each piece with a `DIAGNOSTIC (temporary, A007)` comment so the handback strip is mechanical.
- Push and read the digest. Do not propose fixes from inference before the digest lands.

### Phase 2 — Classify each residual

For each failing spec the digest names, decide which decision-tree branch from the open-questions list applies. Evidence first; reasoning recorded inline in the session log (one bullet per failing spec is enough). Ambiguous cases — including any that look architecturally significant (a real product regression in the E16/E17/325 surfaces; an axe violation in authored public markup; a contract change that affects a non-trivial number of specs; anything that would expand scope beyond stabilization) — get surfaced to the user via `AskUserQuestion` with the diagnosis + recommendation, **not** silently resolved.

### Phase 3 — Apply fixes

One commit per coherent class of fix. For each:

- **Spec adjusted to the shipped product contract** (decision-tree a): minimal edit; the new assertion captures what the contract actually delivers; reference the post-322/323/325 contract in the test comment so a future reader sees the *why*. A spec rewritten more than trivially is re-authored to `docs/testing/playwright-discipline.md` (local-first authoring discipline).
- **Spec deleted as CI-dishonest** (decision-tree b): remove the assertion (or the whole test if its existence depended on the deleted assertion); clean up any imports/helpers that go orphaned; add a `[test-integrity]` line to `sessions/housekeeping-inbox.md` summarising what was removed and why, with the coverage-tradeoff statement (consistent with the A003/A005 pattern). Do not "skip" as a default — `describe.skip` only when the test still has value but is provably environment-blocked, and even then prefer deletion-with-housekeeping-note unless the user steers otherwise (the A005 delete-by-default policy).
- **Product fix** (decision-tree c): only when evidence points to a real regression in product code (e.g. an axe-flagged authored-markup defect). Keep the fix minimal and surgical; matching tests if the fix changes behaviour. **Surface the architectural choice before going deep** — a product fix is non-trivial scope for a stabilization session and the user should bless it.
- **Surfaced to the user** (decision-tree d): clear PM-level summary + recommendation + the option(s) you'd take if redirected.

Verify with one CI run. If it goes green (rc=0, no `unexpected`; retry-recovered `flaky` is acceptable per the project's standard `retries: 1` model), proceed to Phase 4.

### Phase 4 — Handback

- Strip every `DIAGNOSTIC (temporary, A007)` block from `.github/workflows/tests.yml`: restore `on: push: branches: ["**"]`; remove the Pest-disabling `if: false` guards; restore `playwright`'s `needs: slow`; restore the simple `npm run test:e2e -- --project=chromium` run line (drop the JSON/tee/rc-capture wrapper); remove the digest-publish + reflect-result steps; remove any container-probe captures.
- Delete the digest helper if one was added (mirror of the prior `scripts/ci/pw-digest.mjs`).
- Confirm the final workflow diff vs `main` contains only the intended permanent deltas. Re-validate YAML structurally and confirm no `DIAGNOSTIC (temporary, ...)` markers, no `ci-logs/` references, and no `if: false` on the gating jobs.
- Push. If the user asks for a PR, title it `A007 — Playwright e2e Re-Stabilization (Round 2)` and mention any housekeeping inbox additions and the test-integrity disposition applied to each failing spec.

## Out of scope

- **Reverting E16 / E17 / the EventCalendar retirement.** These are deliberate shipped product decisions; if a spec asserts on retired/changed behaviour, the spec updates.
- **Adding a build server to the e2e stack.** The project's recorded posture is "no external build pipeline in CI"; if a failing spec requires a runtime widget bundle the isolated stack never builds, treat it as CI-dishonest (decision-tree b), not as a license to add `bundleserver` infra.
- **Re-quarantining the suite.** Adding entries to a `QUARANTINED_SPECS` list (or its equivalent) to chase green is forbidden — the original stabilization's permanent decision was to delete that pattern.
- **Reworking the un-parked-but-non-gating posture of the Playwright job.** It runs after `slow`, non-gating, as designed. Promoting it into `tests.needs` is a separate, deliberate decision and not in scope here.
- **Generalised flake-tuning.** Per-spec `waitForTimeout` extensions / retry bumps are last-resort fixes; evidence must show variance the retry model can't absorb before tuning, and even then the no-whack-a-mole rule applies.
- **A Test Audit cycle.** A007 is a stabilization. The Test Audit track's next cycle (D3 mutation slice / D4 Pest relevance) is unrelated and out of scope.
- **Numbered-session work.** The numbered cadence is out of A007's scope.
