# 131. Content Listing Widgets

Upgrade the blog listing and events listing widgets with images, pagination, search, sort, layout controls, and a reusable content template pattern. Add thumbnail and header image media collections to the Page and Event models.

---

## Design decisions (resolved before starting)

- **Client-side pagination, search, and sort:** all data is rendered into a JSON array in Alpine.js `x-data`. Pagination, text search, and sort are handled client-side with Alpine â€” no Livewire round-trips, no new API endpoints. For nonprofits with <500 posts/events this is fast and simple. If scale demands server-side pagination later, the existing `/api/events.json` pattern can be extended.
- **Shared pager partial:** a reusable `resources/views/widgets/partials/pager.blade.php` Blade partial that any listing widget can `@include`. Accepts the Alpine component's page state and renders previous/next + page number buttons.
- **Content template pattern:** a `content_template` richtext config field where the admin writes markup with `{{field_name}}` tokens (e.g. `{{title}}`, `{{date}}`, `{{image}}`, `{{excerpt}}`). Processed per-item using the same `str_replace` + regex cleanup pattern as carousel captions and email templates. This gives full layout control per card without building a visual card editor.
- **Layout controls:** `display_mode` select (list vs grid) and `columns` number for grid mode. CSS grid handles the layout; modifier classes handle alignment.
- **Images on content types:** Page model gets `HasMedia` with `post_thumbnail` and `post_header` media collections. Event model gets a new `event_thumbnail` collection alongside the existing photo collection. Both get WebP conversions and responsive breakpoints via the existing `ImageSizeProfile` pattern.
- **Default content templates:** each widget ships with a sensible default `content_template` so the widget looks good out of the box. The admin can customise it.

## Phase 3 â€” Upgrade blog listing widget

Update `blog_listing` in the seeder with new config fields:

- `heading` â€” text (existing)
- `content_template` â€” richtext, default template with `{{image}}`, `{{title}}`, `{{date}}`, `{{excerpt}}` tokens
- `display_mode` â€” select: `list`, `grid` (default: `list`)

- `show_search` â€” toggle, default: false
- `sort_default` â€” select: `newest`, `oldest`, `title_az`, `title_za` (default: `newest`)
- `background_color` â€” color
- `text_color` â€” color

Rewrite `resources/views/widgets/blog-listing.blade.php`:

- Fetch all posts (no limit â€” pagination handles display).
- Serialize posts into a JSON array for Alpine, including: title, slug, url, date (formatted + ISO), excerpt (first ~160 chars of content, stripped of HTML), thumbnail URL.
- Alpine `x-data` component with: `items`, `page`, `perPage`, `search`, `sortBy`, methods for filtering, sorting, paginating.
- Render items using the content template token replacement pattern.
- `@include` the shared pager partial.
- List mode: vertical stack. Grid mode: CSS grid with configurable columns.
