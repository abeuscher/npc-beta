# 123. Tailwind Removal & Build Server Integration

Remove all Tailwind utility classes from public-facing templates, connect the app to the external build server to produce one CSS bundle and one JS bundle for the public site, and document the integration.

**Prerequisite:** Build server is running locally via Docker (`http://bundleserver:8080` on the shared `nonprofitcrm` Docker network). API key for local dev: `sk_build_test123`.

---

## Design decisions (resolved before starting)

- The build server handles **public-facing CSS and JS only**. The admin panel (Filament theme) stays on Vite.
- The app retains Node/NPM for the admin build. The build server offloads the public bundle.
- Template `custom_scss` (per-instance theming) remains compiled at runtime via ScssPhp for now. It moves to the build server post-beta.
- Widget bundles are written to `public/build/widgets/` with content-hashed filenames.
- The layout reads from `public/build/widgets/manifest.json` to reference the current bundle — same pattern as Vite's manifest.
- The SCSS framework (`_variables.scss`, `_layout.scss`, `_buttons.scss`, `_custom.scss`) was established in session 122. The `.site-container` class and breakpoint variables are the foundation.
- See `docs/widget-conventions.md` for the markup and styling patterns all widgets must follow.

## Phase 2 — Connect the build server

### Environment and config

Add to `.env` and `.env.example`:

```env
BUILD_SERVER_URL=http://bundleserver:8080
BUILD_SERVER_API_KEY=sk_build_test123
```

Add to `config/services.php`:

```php
'build_server' => [
    'url' => env('BUILD_SERVER_URL'),
    'api_key' => env('BUILD_SERVER_API_KEY'),
],
```

### Asset build service

Create `app/Services/AssetBuildService.php`. This service:

1. Collects all SCSS partials and widget CSS/JS from widget type records
2. Builds the request payload with content-hashed output filenames
3. POSTs to the build server with Bearer auth
4. Writes the returned bundles to `public/build/widgets/`
5. Writes `public/build/widgets/manifest.json` with the current filenames
6. Cleans up previous bundles
7. Logs build time and sizes

### Artisan command

Create `app/Console/Commands/BuildPublicAssets.php`:

- Signature: `build:public {--debug}`
- Calls `AssetBuildService::build()`
- Reports success/failure and build time
- Outputs detailed errors from the build server when `--debug` is passed

### Layout changes

Replace the runtime SCSS compilation block in the layout with a `<link>` tag for the CSS bundle from the manifest. Replace the per-widget JS loop with a single `<script>` tag for the JS bundle. Keep template `custom_scss` runtime compilation for now.

### Verify

Run `php artisan build:public --debug` and confirm:
- The build server receives the request
- One CSS bundle and one JS bundle are written to `public/build/widgets/`
- Manifest is created
- Public pages load correctly with the bundle tags
