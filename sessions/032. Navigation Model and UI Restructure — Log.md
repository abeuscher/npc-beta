# 032. Navigation Model and UI Restructure — Log

## What was built

### Migration
- Created `navigation_menus` table (`id` uuid PK, `label`, `handle` unique, `timestamps`).
- Seeded one `NavigationMenu` row per distinct `menu_handle` value found in `navigation_items` (using the handle as both `handle` and `label`).
- Added `navigation_menu_id` (uuid, FK → `navigation_menus.id`, cascade delete) to `navigation_items`.
- Populated `navigation_menu_id` from the existing `menu_handle` values.
- Dropped the `menu_handle` column from `navigation_items`.

### `NavigationMenu` model
- New model at `app/Models/NavigationMenu.php` with `HasUuids`, `$fillable: ['label', 'handle']`, and `hasMany(NavigationItem::class)`.

### `NavigationItem` model
- Replaced `menu_handle` with `navigation_menu_id` in `$fillable`.
- Added `belongsTo(NavigationMenu::class)` relationship.

### `NavigationItemResource` removed
- Deleted `app/Filament/Resources/NavigationItemResource.php` and its `Pages/` subdirectory entirely.

### `NavigationMenuResource`
- New resource at `app/Filament/Resources/NavigationMenuResource.php` with three page classes in `Pages/`.
- **List page** — label (sortable, searchable), handle (sortable), items count (via `withCount`). Edit and Delete actions, bulk delete.
- **Form** — two sections: Menu details (label + handle, 2-col), and Links (reorderable Repeater).
- **Repeater** — top-level items are collapsible with item labels. Each item uses `Grid::make(3)` for label / link type / page-or-URL, and `Grid::make(2)` for the two toggles (open in new window, is visible).
- **Children Repeater** — nested inside each top-level item, same schema (no further nesting), collapsible with item labels, starts expanded so new children are visible on creation.
- **Manual state management** — no `->relationship()` on Repeaters. `mutateFormDataBeforeFill` (Edit page) loads items from DB into the Repeater array. `mutateFormDataBeforeCreate` / `mutateFormDataBeforeSave` strip items from data into a property; `afterCreate` / `afterSave` delete and re-insert all items with correct `sort_order` and `parent_id`.
- `link_type` / `internal_page` virtual field resolution ported from the old resource into static helpers `itemToFormArray`, `resolveItemData`, and `saveItems` on the resource.

### Blade components updated
- `site-header.blade.php` and `site-footer.blade.php` — replaced `NavigationItem::where('menu_handle', $handle)` queries with `NavigationMenu::where('handle', $handle)->first()` followed by a scoped `->items()` query. Components fall back to `collect()` if no menu with that handle exists.

### `BasePageSeeder` updated
- Now creates (or finds) a `NavigationMenu` with handle `primary` before inserting navigation items, and passes `navigation_menu_id` to each `firstOrCreate` call.

### Light-theme border visibility fix
- Expanded `public/css/admin.css` to add scoped overrides (`html:not(.dark)`) for section rings, table container ring, table row dividers, table header borders, and fieldset/repeater item rings — all set to `gray-300` / `gray-400`. Dark theme is unaffected.
