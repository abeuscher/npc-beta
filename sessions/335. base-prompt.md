## Process rules for every session

Base-prompt process rules apply in full (`sessions/template-base-prompt.md`). Session-specific notes:

- **Reproduce before you fix; observe, don't assert.** Drive the live local public site at tablet (~768) and mobile (~375) viewports (read-only Playwright / standalone script — never the DB-resetting specs) to see the excess whitespace before touching code, and re-drive to prove it's reduced. Desktop must look unchanged.
- **Host CSS, not widget SCSS.** The spacing primitive's consuming rule and the scaling `@media` belong in the host appearance layer (`resources/scss/**`, `host` layer), keyed off the viewport — this is the brief's sanctioned mechanism and sidesteps the `WidgetStyleBoundaryConsumptionTest` widget-SCSS `@media` ban entirely. Do **not** reach for `@container` or touch widget-interior SCSS.
- **Scale, never override layout.** The primitive only multiplies the operator's own vertical values down at narrower widths. It must not change counts, columns, or any operator-configured layout — that is the 332 line that must not be re-crossed.
- **Front-end builds:** host `resources/scss/**` → `npm run build`; if any change affects what the build-server widget bundle emits (host base rules ship in both compile paths per 332), also run `docker compose exec app php artisan build:public`. Rebuild the touched surface before re-driving the browser.
- **Stay scoped to vertical section spacing.** Do not expand into type ramps, widget-interior grids, carousels, horizontal spacing, or the per-breakpoint editor work. If the spacing fix unavoidably crosses the breakpoint single-sourcing precursor, flag it rather than absorbing it.
