## What was built

### Phase 1 — Paid Event Registration

- Migration: added `stripe_session_id` column and `pending` status value to `event_registrations`
- New `EventCheckoutController` (guest) and `Portal\EventCheckoutController` (member): validate event (published, open, not at capacity, price > 0), create pending registration, build Stripe Checkout Session in payment mode, redirect to Stripe
- Updated `EventController` and `Portal\EventRegistrationController` to redirect to checkout when event has a price instead of rejecting
- Updated `event-registration.blade.php` widget: shows price, conditionally routes to checkout for paid events, handles `?registration=success/cancelled` query params
- Updated `events/show.blade.php` legacy template with same conditional routing
- Extended `StripeWebhookController::handleCheckoutSessionCompleted` with `event_registration_id` metadata detection — completes registration, stores payment intent ID, creates Transaction, dispatches QB sync
- `Event::isAtCapacity()` now counts `pending` registrations toward capacity
- Routes: `events.checkout` and `portal.events.checkout`

### Phase 2 — Paid Membership Signup

- Migration: added `stripe_session_id` and `stripe_subscription_id` columns to `memberships`
- New `MembershipCheckoutController`: handles both new signups (creates contact + portal account) and existing portal users; uses `subscription` mode for monthly/annual tiers, `payment` mode for one_time/lifetime
- Updated `portal-signup.blade.php` widget with tier selection dropdown; Alpine.js switches form action between signup (free) and checkout (paid) based on selected tier price
- Updated `SignupController` to create complimentary memberships when a free tier is selected
- Extended webhook handler with `handleMembershipCheckout` (activates membership, sets dates, creates Transaction) and `handleMembershipInvoice` (subscription renewal — extends expiry, creates renewal Transaction)
- Route: `membership.checkout`

### Additional — Transaction list improvements (iteration 3)

- Replaced generic "Source" column with color-coded "Category" badge (Donation, Purchase, Event, Membership)
- Added "Description" column resolving each transaction's subject to a readable label (fund name, product name, event title, tier name)
- Updated subject type filter to include all four transaction categories
- Direction column hidden by default, toggleable
- Eager-loaded subject + contact to prevent N+1
