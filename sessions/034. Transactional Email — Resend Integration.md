# 034. Transactional Email — Resend Integration

Add a Mail Settings page that lets an administrator configure how the application sends email. The default driver is `log` (emails written to the Laravel log, nothing actually delivered), which works out of the box with no configuration. Resend is the first live sending option. Switching to Resend requires only an API key entered in the UI — no `.env` changes or server access.

---

## Background & decisions

`config/mail.php` already defines a `resend` mailer entry (`transport: resend`) and defaults to the `log` driver via `env('MAIL_MAILER', 'log')`. Laravel's built-in Resend transport requires the `resend/resend-php` package and reads its API key from `config('resend.api_key')`.

`AppServiceProvider::boot()` already reads `SiteSetting::all()` into a `config()` call inside a `try/catch`. Mail configuration follows the same pattern — add keys to that same block. No second DB query, no second try/catch.

The `resend` mailer transport entry in `config/mail.php` does not need to change. No `.env` variables are needed for mail after this session — everything is driven from `site_settings`.

---

## What to build

### 1. Install package

```
composer require resend/resend-php
```

### 2. `MailSettingsPage`

Create `app/Filament/Pages/Settings/MailSettingsPage.php`.

- `canAccess()`: `super_admin` only
- `$navigationGroup`: `'Settings'`
- `$navigationLabel`: `'Mail'`
- `$navigationIcon`: `'heroicon-o-envelope'`
- `$navigationSort`: `2`
- `$title`: `'Mail Settings'`
- View: `filament.pages.settings.mail-settings-page`

**Form — Section "Sending"** (columns: 2):

- `Select::make('mail_driver')` — label `'Mail driver'`, required, options:
  ```php
  ['log' => 'Local — log only (no sending)', 'resend' => 'Resend']
  ```
  helper text `'Local writes emails to the Laravel log. No messages are delivered. Switch to Resend to send real email.'`, `->live()`

- `TextInput::make('mail_from_name')` — label `'From name'`, required

- `TextInput::make('mail_from_address')` — label `'From address'`, `->email()`, required

**Form — Section "Resend"** — `->visible(fn (Get $get) => $get('mail_driver') === 'resend')`:

- `TextInput::make('resend_api_key')` — label `'API Key'`, `->password()`, `->revealable()`, nullable, helper text `'Your Resend API key. Starts with re_.'`

Import `Filament\Forms\Get` for the closure.

**`mount()`**:

```php
$this->form->fill([
    'mail_driver'        => SiteSetting::get('mail_driver', 'log'),
    'mail_from_name'     => SiteSetting::get('mail_from_name', ''),
    'mail_from_address'  => SiteSetting::get('mail_from_address', ''),
    'resend_api_key'     => SiteSetting::get('resend_api_key', ''),
]);
```

**`save()`**:

```php
SiteSetting::set('mail_driver',       $data['mail_driver']);
SiteSetting::set('mail_from_name',    trim($data['mail_from_name']));
SiteSetting::set('mail_from_address', trim($data['mail_from_address']));
SiteSetting::set('resend_api_key',    trim($data['resend_api_key'] ?? ''));
```

Show success notification. Redirect to `static::getUrl()` after saving (same pattern as `GeneralSettingsPage`).

Create the Blade view `resources/views/filament/pages/settings/mail-settings-page.blade.php` — identical boilerplate to `general-settings-page.blade.php`.

---

### 3. Update `AppServiceProvider::boot()`

Inside the existing `config([...])` call (same `try/catch` block, same `$settings` variable), add:

```php
'mail.default'      => $settings->get('mail_driver')?->value      ?? 'log',
'mail.from.address' => $settings->get('mail_from_address')?->value ?? 'hello@example.com',
'mail.from.name'    => $settings->get('mail_from_name')?->value    ?? config('app.name'),
'resend.api_key'    => $settings->get('resend_api_key')?->value    ?? '',
```

Do not change anything else in `AppServiceProvider`.

---

### 4. Seed defaults

Add to the `$siteSettingDefaults` array in `DatabaseSeeder`:

```php
['key' => 'mail_driver',        'value' => 'log',    'group' => 'mail', 'type' => 'string'],
['key' => 'mail_from_name',     'value' => '',       'group' => 'mail', 'type' => 'string'],
['key' => 'mail_from_address',  'value' => '',       'group' => 'mail', 'type' => 'string'],
['key' => 'resend_api_key',     'value' => '',       'group' => 'mail', 'type' => 'string'],
```

## Out of scope

- SMTP driver configuration
- Additional sending providers (SES, Postmark, Mailgun) — tracked as a future session
- Email queue configuration
- Bounce or delivery webhook handling
- Any change to the content of notification emails
