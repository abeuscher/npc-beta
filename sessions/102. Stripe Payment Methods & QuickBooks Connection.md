# 102. Stripe Payment Methods & QuickBooks Connection

Two finance-infrastructure items: configurable Stripe payment method types for checkout, and the QuickBooks OAuth connection flow (no transaction sync yet).

---

## Design decisions (resolved before starting)

- Stripe payment method configuration lives on the Finance Settings page, not a separate page.
- QuickBooks uses OAuth 2.0. Token storage uses the existing encrypted SiteSetting pattern. No new table for tokens — store `qb_access_token`, `qb_refresh_token`, `qb_realm_id`, `qb_token_expires_at` as encrypted site settings.
- QuickBooks SDK: install `quickbooks/v3-php-sdk` via Composer (Intuit's official PHP SDK). If the SDK is abandoned or problematic, fall back to direct HTTP with Guzzle.
- The QuickBooks section of Finance Settings shows a Connect/Disconnect flow, not raw key entry. The OAuth client ID and secret are stored in `.env` (not in the database) since they are deployment-level credentials.
- No transaction sync this session — that is session 103.

## Phase 1 — Stripe Payment Method Configuration

### Settings UI

Add a "Payment Methods" section to the Finance Settings page, between the existing Stripe keys section and the QuickBooks section.

- Display a checkbox list of Stripe-supported payment method types. Start with: `card`, `us_bank_account` (ACH), `link`, `cashapp`, `amazon_pay`.
- `card` is always checked and cannot be unchecked (enforced in the UI with a disabled checkbox + helper text: "Card payments cannot be disabled").
- Default state on fresh install: only `card` enabled.
- Store as a JSON array in `SiteSetting` key `stripe_payment_method_types`.
- The setting is gated to the same permission as the rest of Finance Settings (`manage_financial_settings`).

### Checkout integration

- In both `DonationCheckoutController` and `ProductCheckoutController`, read the `stripe_payment_method_types` setting and pass it as the `payment_method_types` parameter when creating Checkout Sessions.
- If the setting is empty or missing, default to `['card']`.
- For recurring donations (subscription mode), only pass method types that Stripe supports for subscriptions. Currently that means: `card`, `us_bank_account`, `link`. Filter out unsupported types automatically — do not error.

### Help doc update

Update `resources/docs/settings-finance.md` with a section explaining the Payment Methods configuration, including a note about which methods support recurring donations.

## Phase 2 — QuickBooks OAuth Connection

### Environment variables

Add to `.env.example`:

```
QB_CLIENT_ID=
QB_CLIENT_SECRET=
QB_REDIRECT_URI="${APP_URL}/admin/quickbooks/callback"
QB_ENVIRONMENT=sandbox
```

### QuickBooks service

Create `app/Services/QuickBooks/QuickBooksAuth.php` with:

- `getAuthorizationUrl(): string` — builds the Intuit OAuth authorization URL with required scopes (`com.intuit.quickbooks.accounting`).
- `exchangeCode(string $code, string $realmId): void` — exchanges the authorization code for access + refresh tokens, stores them as encrypted site settings.
- `refreshTokenIfNeeded(): bool` — checks `qb_token_expires_at`, refreshes if within 5 minutes of expiry, returns true if a valid token is available.
- `disconnect(): void` — clears all QB token settings.
- `isConnected(): bool` — returns true if `qb_access_token` and `qb_realm_id` are set and the token is refreshable.

Token storage keys: `qb_access_token`, `qb_refresh_token`, `qb_realm_id`, `qb_token_expires_at`. All encrypted via `SiteSetting::setEncrypted()` / `getEncrypted()` (same pattern as Stripe keys).

### Routes and controller

Create `app/Http/Controllers/QuickBooksCallbackController.php`:

- `GET /admin/quickbooks/connect` — redirects to Intuit OAuth URL. Requires auth + `manage_financial_settings` permission.
- `GET /admin/quickbooks/callback` — handles the OAuth callback, exchanges code, redirects to Finance Settings with a success notification.
- `POST /admin/quickbooks/disconnect` — clears tokens, redirects to Finance Settings. Requires confirmation.

Register routes in the admin web group.

### Finance Settings UI

Replace the placeholder QuickBooks section with:

- **Not connected state:** Show a "Connect to QuickBooks" button that links to the connect route. Below it, display helper text: "QuickBooks Online credentials (Client ID and Secret) must be configured in the server environment before connecting."
- **Connected state:** Show the connected Realm ID (company identifier), token expiry timestamp, and a "Disconnect" button (danger, requires confirmation). Show a green "Connected" badge.
- If `QB_CLIENT_ID` is not set in the environment, show an info banner: "QuickBooks integration requires server-level configuration. Contact your administrator." and hide the Connect button.

### Help doc update

Update `resources/docs/settings-finance.md` QuickBooks section with:

- What QuickBooks Online is used for (one-way transaction sync — coming in a future update).
- How to obtain OAuth credentials from the Intuit Developer portal.
- The Connect/Disconnect flow.
- Note that transaction sync is not yet active — connecting now prepares for it.
