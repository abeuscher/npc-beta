## Changes Made

### Contact Form Layout

**File:** `app/Filament/Resources/ContactResource.php`

- **Top row**: Prefix removed from the name row. First Name and Last Name now each span 6 columns (equal width, full row).
- **Bottom row**: Prefix demoted to a 3-column row alongside Date of Birth and Age — each field spans 4 columns.
- **Right column**: Settings and Portal Access sections wrapped in a `Group` with `columnSpan(1)`, ensuring Portal Access renders directly beneath Settings in the right column instead of wrapping to the start of a new row.

### CMS Navigation Sort Order

Reordered all CMS group resources and pages to: Blog Posts (1) · Pages (2) · Events (3) · Collections (4) · Forms (5) · Navigation (6) · Site Theme (7).

**Files changed:**
- `app/Filament/Resources/PostResource.php` — sort 2 → 1
- `app/Filament/Resources/PageResource.php` — sort 1 → 2
- `app/Filament/Resources/EventResource.php` — sort 5 → 3
- `app/Filament/Resources/ContentCollectionResource.php` — sort 3 → 4
- `app/Filament/Resources/NavigationMenuResource.php` — sort 4 → 6
- `app/Filament/Pages/SiteThemePage.php` — sort 6 → 7
- `app/Filament/Resources/FormResource.php` — sort unchanged at 5

### Events Settings Consolidation

The Events Settings page held a single toggle ("Auto-publish new events"). That setting was moved to the CMS Settings page, and the Events Settings page was hidden from navigation.

**`app/Filament/Pages/Settings/EventsSettingsPage.php`:**
- Added `protected static bool $shouldRegisterNavigation = false`

**`app/Filament/Pages/Settings/CmsSettingsPage.php`:**
- Section changed to `->columns(12)`
- Site Name: `columnSpan(6)`
- Timezone: `columnSpan(4)`
- Contact Email: `columnSpan(4)`
- `event_auto_publish` toggle added at `columnSpan(4)`
- `mount()` updated to load `event_auto_publish` from `SiteSetting`
- `save()` updated to persist `event_auto_publish`

### Help System — Auto-Sync on Seed

Root cause identified: after a `migrate:fresh --seed` the `help_articles` and `help_article_routes` tables exist (they are in the schema dump) but are empty because no seeder called `help:sync`. Running `help:sync` manually restored all 33 articles. Fixed by adding `Artisan::call('help:sync')` as the last step in `DatabaseSeeder::run()`.

**`database/seeders/DatabaseSeeder.php`:**
- Added `use Illuminate\Support\Facades\Artisan`
- Added `Artisan::call('help:sync')` at end of `run()`

### Session Outlines — Promote Contact to Member

Added "Promote Contact to Member" as a new future-work stub in the Member Portal — Next Steps group, immediately after the Member Event Registration entry, describing an inline Filament `Action` on the contact Settings panel to create a `Membership` record for the current contact.
