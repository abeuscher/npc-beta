# 350. SEO & Semantic-HTML Cleanup

The cleanup half of the SEO cluster: seven targeted fixes that make the public site's HTML crawlable, semantically correct, and faster to paint — no new features, no schema, no contract change.

## The seven items

1. **Personal email exposed in client output.** `window.__site.contactEmail` (in `resources/views/layouts/public.blade.php:~249`) emits the configured contact email into page source on **every** page. Pull it from the **public** contact-email setting (the operator-set address), or **remove it from client output entirely** if nothing on the page consumes `window.__site.contactEmail` (grep for consumers first). Privacy + SEO.

2. **`/home` and `/` both resolve 200.** Canonical mitigates it, but `/home` should **301 → `/`** so a single URL exists. A route-level redirect in `routes/web.php`.

3. **`og:title` per-page verification.** The title field now wires through on the homepage; confirm the **same field populates `og:title` on all page types** (page / post / event / member). Was previously stuck on "Welcome". Verify through `SeoMetaGenerator`; fix any page type that doesn't wire through; add a standing test so it can't regress.

4. **Rich text emits `<ol>` for bullet lists.** Quill v2 represents `<ul><li>` as `<ol><li data-list="bullet">` (see `useInlineEdit.ts:194-208`). Semantically wrong for crawlers. Transform bullet-`<ol>` → `<ul>` (and drop the now-redundant `data-list="bullet"`) so the public HTML is a real unordered list. **The inbox carries a `[test-integrity]` request for an OL/UL list-rendering Playwright guard** — author it local-first if this fix lands.

5. **Headings wrapped in `<strong>`.** The editor produces `<h2><strong>…</strong></h2>`. Strip `<strong>`/`<b>` when it wraps the **entire** heading content (a heading is already bold; the nested `<strong>` is noise and can confuse weight/semantics). Same transform layer as item 4.

6. **`<p>` wrappers + misplaced `role` in nav.** The Nav widget renders `<p><a>…</a></p>` inside `<span role="menuitem">` — the richtext wrapper is leaking into nav output, and `role="menuitem"` belongs on the **interactive element** (the anchor), not the span. Unwrap the `<p>`; move the role to the `<a>`. `app/Widgets/Nav/template.blade.php`.

7. **Above-the-fold images lazy-loaded.** Image widgets hardcode `loading="lazy"`, including the hero / first image, which delays LCP. Add an eager path: a per-instance eager/lazy toggle **and/or** auto-eager the first image on a page, plus `fetchpriority="high"` on the chosen LCP candidate. **Scope to `Image` + `EventImage`** (the real LCP candidates) — do not sweep all 15 `loading="lazy"` sites.

## Open questions to resolve at session start

1. **Rich-text transform placement (items 4 + 5).** Save-time (in or beside `HtmlSanitizer`, mutating stored HTML going forward) vs render-time (a non-destructive filter on `{!! !!}` output). Name the blast-radius tradeoff — this touches every rich-text surface. Default lean: render-time (non-destructive, no migration of existing content), unless the owner wants the stored HTML cleaned permanently.
2. **Hero-LCP strategy (item 7).** Per-instance eager/lazy toggle, auto-eager-first-image, or both. A toggle is explicit and simple; auto-first needs render-order awareness. Settle the v1 shape.
3. **`og:title` (item 3).** Confirm whether the per-page-type audit reveals a real gap to fix or only needs a verification + standing test.

## Phases

Build after each step where a build applies. Pest-cover each fix.

1. **Decide the open questions** (transform placement; hero-eager shape; og:title gap-or-verify). Log at session start.
2. **The two quick wins:** `/home` → `/` 301 (item 2) + the `contactEmail` fix (item 1). Pest: redirect status; meta/output assertion.
3. **`og:title` per-page** (item 3): verify/fix through `SeoMetaGenerator`; Pest across page types.
4. **The rich-text transform** (items 4 + 5): OL/UL→`<ul>` + heading-`<strong>` strip at the chosen layer; Pest at the transform boundary (bullet list → `<ul>`; `<h2><strong>x</strong></h2>` → `<h2>x</h2>`; ordered lists + mid-heading bold untouched). Playwright OL/UL guard if landing.
5. **Nav `<p>`/role** (item 6): unwrap + move role; Pest render-shape on the Nav widget.
6. **Image eager/`fetchpriority`** (item 7): the eager path on `Image`/`EventImage`; Pest render-shape (eager image emits `loading="eager"` + `fetchpriority="high"`; others stay lazy).
