# 124. Widget — Logo Garden

Add a `logo_garden` widget type: a grid of partner/sponsor logos sourced from a custom collection. Logos display in a responsive grid with optional grayscale treatment and configurable column count.

---

## Design decisions (resolved before starting)

- Logo data comes from a custom collection via the existing `WidgetDataResolver` pipeline. The admin creates a "Logos" collection with an image field and optional link URL, then points this widget at it.
- This is a standard collection-backed widget — same pattern as the carousel.
- Categorised as `media` with `allowed_page_types: null`.
- Grayscale is a CSS filter toggle — logos render in grayscale by default, full colour on hover. Togglable.

## Implementation

### Seeder entry

| Field | Value |
|---|---|
| handle | `logo_garden` |
| label | `Logo Garden` |
| category | `media` |
| allowed_page_types | null |
| render_mode | server |
| collections | ["logos"] |

### Config schema

| Key | Label | Type | Default | Notes |
|---|---|---|---|---|
| collection_handle | Collection | select | '' | Admin picks the source collection |
| image_field | Image Field | text | 'image' | Field key in the collection for the logo image |
| link_field | Link Field | text | 'url' | Field key for optional click-through URL |
| columns | Columns | number | 4 | Grid columns on desktop (2 on mobile) |
| grayscale | Grayscale Effect | toggle | true | Grayscale by default, colour on hover |
| heading | Heading | text | '' | Optional section heading above the grid |
| max_height | Logo Max Height | text | '4rem' | CSS max-height for each logo image |

### Template

Responsive CSS grid. Each cell contains a logo image (from `$collectionData`) with `object-fit: contain` and constrained max-height. If `link_field` value exists on the item, the image is wrapped in an anchor tag. Grayscale applied via `filter: grayscale(1)` with `hover: filter: grayscale(0)` transition.
