# 169. Navigation Widget

Build a full-featured navigation widget for the page builder: dropdowns, mobile hamburger, branding slot, and configurable styling via RTE templates.

---

## Design decisions (resolved before starting)

- **Data source**: the widget config includes a `navigation_menu_id` select field. The widget pulls its items from the existing `NavigationMenu` / `NavigationItem` models.
- **Branding slot**: the nav widget has an optional top-level branding element — logo image, icon, or text. Configured via widget config fields (`branding_type`: none / logo / icon / text, plus the corresponding image/text value). This is separate from the standalone Logo Block widget, which still exists for other contexts.
- **Text styling**: parent nav items and dropdown children are styled via separate RTE templates (mustache-style, like system email templates). This gives maximum flexibility for translating a designer's work. Presets will wrap this for normal users post-beta.
- **Depth**: 3 levels supported (top-level, dropdown, nested dropdown). 4th level deferred post-beta.
- **Dropdown panel styling**: border color, border thickness, fill color, and fill gradient (reusing the existing gradient picker component).
- **Animations**: drop menus support slide or fade (configurable per widget). Mobile menu reveal supports slide or fade.
- **Mobile**: hamburger icon with hamburger-to-X CSS animation. Menu collapses at a configurable breakpoint.
- **Alignment**: 9-point alignment tool positions nav content within the widget. Dropdown panels align relative to their parent item (left / center / right).
- **Active state**: auto-detected by matching the current URL path against nav item links. Styled via the RTE template system.
- **Accessibility**: arrow-key navigation in dropdowns, escape-to-close, proper aria attributes (aria-expanded, aria-haspopup, role="menubar"/"menu"/"menuitem").

## Phase 2 — Config Schema & Data Pipeline

### Widget type config_schema

Update the `navigation` widget type's `config_schema` to include:

- `navigation_menu_id` — select field, populated from `NavigationMenu` records
- `branding_type` — select: none, logo, icon, text
- `branding_image` — image field (for logo/icon types)
- `branding_text` — text field (for text type)
- `alignment` — 9-point alignment (top-left through bottom-right)
- `drop_animation` — select: slide, fade
- `drop_align` — select: left, center, right (dropdown alignment relative to parent)
- `drop_border_color` — text (hex color)
- `drop_border_width` — number (px)
- `drop_fill_color` — text (hex color)
- `drop_fill_gradient` — gradient config (reuse existing gradient shape)
- `mobile_animation` — select: slide, fade
- `mobile_breakpoint` — number (px, default 768)
- `parent_template` — richtext (RTE template for top-level items, mustache tokens: `{{label}}`, `{{url}}`, `{{active_class}}`)
- `child_template` — richtext (RTE template for dropdown items, same tokens)

### Data resolution

Ensure the widget receives the full navigation tree (3 levels deep) as template data. The `NavigationMenu` selected in config should be resolved through `WidgetDataResolver` or directly in the widget template via a Blade query, following the pattern of other data-driven widgets.

## Phase 3 — Widget Template & Markup

### Blade template

Rewrite `resources/views/widgets/navigation.blade.php` with:

- Semantic `<nav>` element with `aria-label`
- `role="menubar"` on the top-level `<ul>`
- Branding slot rendered first (logo image, icon, or text based on config)
- Recursive item rendering for 3 levels
- `aria-haspopup="true"` and `aria-expanded` on items with children
- `role="menu"` on dropdown `<ul>` elements, `role="menuitem"` on `<li>` items
- Active class applied by matching current URL path
- Mustache token replacement in parent/child RTE templates for item rendering
- Dropdown panel wrapper with configurable border/fill styles
- Mobile hamburger button (hidden above breakpoint) with `aria-controls` and `aria-expanded`
- Mobile menu container (hidden above breakpoint, shown on toggle)

### Alpine.js interactivity

- Desktop: dropdown open/close on hover with a small delay (prevent flicker on mouse traversal)
- Desktop: dropdown open on focus for keyboard users
- Keyboard: arrow keys navigate between items, escape closes dropdown, tab moves focus normally
- Mobile: hamburger toggles menu visibility with configured animation (slide/fade)
- Hamburger-to-X CSS transition on the button icon

## Phase 4 — SCSS & Animations

### Widget SCSS

Write `resources/scss/widgets/_navigation.scss` (or update existing):

- Base layout: flexbox for horizontal nav, branding + menu items
- Dropdown positioning: absolute, aligned per config (left/center/right)
- Dropdown border and fill from inline styles (set via config in the template)
- Slide animation: `transform: translateY()` with transition
- Fade animation: `opacity` with transition
- Mobile: hamburger icon with three-bar-to-X transition (CSS only, no extra elements)
- Mobile menu: full-width slide-down or fade-in based on config
- Responsive: items stack vertically below breakpoint, dropdowns become nested indented lists
- Nested dropdown (level 3): offset to the side on desktop, indented further on mobile

### Gradient support for dropdown fill

Reuse the `GradientComposer` service to generate the CSS gradient string from the drop_fill_gradient config, applied as inline style on the dropdown panel.
