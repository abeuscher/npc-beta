## What was built

### Phase 1 — Data model

- Migration: `invoice_number` (string, nullable) and `line_items` (jsonb, nullable) on `transactions`.
- Migration: `import_source_id`, `import_session_id`, `external_id` on `donations` with index on `(import_source_id, external_id)`.
- Migration: `import_source_id`, `import_session_id`, `external_id` on `memberships` with index on `(import_source_id, external_id)`.
- Migration: nine new columns on `import_sources` for donations/memberships/invoices-scoped field maps, custom field maps, and contact match keys.
- Model updates: `Transaction`, `Donation`, `Membership`, `ImportSource` — fillable, casts, relationships.
- Schema docs updated for `transactions.md`, `donations.md`, `memberships.md`, `import_sources.md`.

### Phase 2 — Field registries

- Three entity-specific registries: `DonationFieldRegistry`, `MembershipFieldRegistry`, `InvoiceDetailFieldRegistry`.
- Three aggregate import registries: `DonationImportFieldRegistry`, `MembershipImportFieldRegistry`, `InvoiceImportFieldRegistry`.
- Added `invoice_number` to `TransactionFieldRegistry`.

### Phase 3 & 4 — Donations importer

- `ImportDonationsPage` — three-step wizard (Upload → Map → Preview) with contact auto-create toggle, WCG heuristic guesses, source preset save/restore.
- `ImportDonationsProgressPage` — dry-run/commit processor. Creates Donation + linked Transaction per row. Transaction deduplication on `(import_source_id, external_id)`. Contact auto-create when toggled on.

### Phase 5 — Memberships importer

- `ImportMembershipsPage` — three-step wizard with contact auto-create toggle.
- `ImportMembershipsProgressPage` — membership tier auto-create by name (case-insensitive). WA status normalisation: Lapsed→expired, Active→active, Pending - Renewal→pending, Cancelled→cancelled.

### Phase 6 — Invoice Details importer

- `ImportInvoiceDetailsPage` — three-step wizard. Requires Invoice # column mapped.
- `ImportInvoiceDetailsProgressPage` — collapses multiple CSV rows sharing the same Invoice # into one Transaction with `line_items` JSON array. Fill-blanks-only enrichment on existing Transactions (from events or donations import). Commits synchronously (no chunked tick) because line items must be grouped.

### Phase 7 — UI surface

- Importer landing page: replaced stub "Financial Data" card with three active cards (Donations, Memberships, Invoice Details). Grid widened to accommodate six cards.
- Review queue: preview, approve, and rollback actions handle `donation`, `membership`, `invoice_detail` model types.
- Financial review preview blade template with entity counts.
- Model type filter extended in the review queue table.

### Bonus — Note splitting

- All four wizard pages (events, donations, memberships, invoices) gained a "Note splitting" sub-form on note-mapped columns: "Whole cell" / "Split by date prefix" / "Split by custom regex".
- Date prefix mode splits on patterns like `7 Apr 2018:` and captures the date as the note's `occurred_at`.
- Custom regex mode lets the user provide their own lookahead split pattern.

### Bonus — Null-value normalization & column-aware error messages

- `FieldMapper::normalizeValue()` treats `n/a`, `N/A`, `na`, `none`, `null`, `-`, `--`, `—`, `undefined` as null across all four progress pages. Prevents false ambiguous-match errors on placeholder values.
- Ambiguous contact match errors now include the source column name and index: `"Ambiguous contact match on email = n/a (from column 23: "Email address")"`.
