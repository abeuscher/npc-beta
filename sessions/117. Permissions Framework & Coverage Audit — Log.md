# Session 117 — Permissions Framework & Coverage Audit — Log

**Date:** 2026-04-02
**Branch:** session-117/1

---

## What was built

### Phase 1 — Inventory matrix

- Produced a comprehensive matrix of every admin left-nav item × every role, documenting current gate method, gate value, and per-role access.
- Identified: 1 ungated resource (NavigationMenuResource — resolved by Filament policy auto-discovery), 6 isSuperAdmin gates to convert, 17 redundant canViewAny overrides, orphaned `household` permissions (kept for future use), and the demo role for removal.

### Phase 2 — Developer role

- Added `developer` role to PermissionSeeder with: full CRM + CMS CRUD, read-only finance (view_any/view for donation, transaction, fund, campaign), all standalone capabilities (theme, settings, import, custom fields, email templates, CMS/mail settings, membership tiers, form submissions, advanced filters), no user/role management.

### Phase 3 — isSuperAdmin gate conversions

- **CollectionResource**: `hasRole('super_admin')` → `can('view_any_collection')`
- **CustomFieldDefResource**: `hasRole('super_admin')` → `can('manage_custom_fields')`
- **EmailTemplateResource**: `hasRole('super_admin')` → `can('manage_email_templates')`
- **MembershipTierResource**: `isSuperAdmin()` → `can('manage_membership_tiers')` (canAccess, canDelete, toggle_archive action)
- **CmsSettingsPage**: `hasRole('super_admin')` → `can('manage_cms_settings')`
- **MailSettingsPage**: `hasRole('super_admin')` → `can('manage_mail_settings')`
- Cleaned up DonorsPage, GeneralSettingsPage, FinanceSettingsPage — removed redundant `hasRole('super_admin') ||` prefix (Gate::before already handles this).
- DashboardTaxonomyWidget — changed from super_admin-only to all authenticated users.

### Phase 4 — Removed redundant resource overrides

Removed `canViewAny()` overrides from 17 resources that duplicated their registered policy check:
ContactResource, EventResource, PageResource, ProductResource, FundResource, FormResource, UserResource, WidgetTypeResource, TransactionResource, DonationResource, NoteResource, OrganizationResource, CampaignResource, MembershipResource, MailingListResource, TagResource, ContentCollectionResource.

Kept justified overrides: PostResource (checks `view_any_post`, policy would check `view_any_page`), MemberResource (checks `view_any_member`, not a policy permission), TemplateResource (no policy, composite permission check), CustomFieldDefResource (standalone permission), EmailTemplateResource (standalone permission).

### Phase 5 — Seeder alignment

- Added `navigation_menu` CRUD to cms_editor role.
- Removed demo role entirely — deleted from seeder, removed `User::isDemo()` method, removed UserResource query filter, removed DashboardDebugGeneratorWidget check, deleted `DemoRoleTest.php`.
- Added 5 new standalone permissions: manage_custom_fields, manage_email_templates, manage_cms_settings, manage_mail_settings, manage_membership_tiers.
- Updated RoleResource `standalonePermissions()` from 8 to 18 entries (added view_any_member, edit_site_chrome, edit_page_snippets, manage_financial_settings, manage_donations, and 5 new manage_* permissions).
- Added `product` to the CMS permission area in RoleResource grid.
