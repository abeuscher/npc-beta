## Production deployment notes

This is a squash, so a deployed DB needs a one-time clean replay rather than a normal `migrate`:

- **If the server's data is disposable** (the demo droplet, the auto-deploy server): `docker compose exec app php artisan migrate:fresh --seed`.
- **If data must be kept:** a manual `psql < database/schema/pgsql-schema.sql` replay against a wiped DB, then confirm the `migrations` table is populated from the dump and that public + admin surfaces resolve.

After the replay, future `migrate` runs behave normally against the new baseline. No widget asset rebuild required (no widget SCSS/CSS/JS touched). No FM-side handoff — non-boundary. **Bump `VERSION` before any merge-to-main deploy** (done — 0.348.01); the squash itself ships no application code.
