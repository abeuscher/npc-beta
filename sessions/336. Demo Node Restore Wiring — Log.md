## Security considerations

The session's load-bearing security property is the `demo:restore` `isDemoMode()` hard-gate: a command that wipes and replaces the entire database from an arbitrary local blob must never be runnable on a production node. The gate is checked first, before any blob read or wipe, and is covered by a standing Pest guard asserting it refuses (and wipes nothing) outside demo mode. The command reads a local file only â€” no outbound fetch â€” preserving the demo node's deny-outbound egress posture. No HTTP surface, no auth path, and no Fleet Manager contract surface was touched; the CRM contract stays v2.3.0.
