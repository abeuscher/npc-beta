# 227. Widget Primitive — Phase 5a (Typed Ambient Context Primitive)

Introduce a typed `AmbientContext` abstraction with three sealed-by-convention subtypes (page, dashboard, record-detail), refactor `SlotContext` to carry a typed `AmbientContext` instead of a `PageContext + ?Page` pair, and update the two live slots' `ambientContext()` bodies — without wiring `RecordDetailSidebarSlot`, without Filament hosting work, and without changing the `$pageContext` Blade variable that every widget template still reads. Pure plumbing session that lays the abstraction Phase 5b–5c–5d build on.

The framing is the migration doc's Phase 5 line: *"`PageContext` generalizes to `SlotContext` with subtypes (page, dashboard, record-detail)."* Today's `SlotContext` is monomorphic — its constructor encodes "ambient is a page" via the `?Page $currentPageOverride` field. That shape served the page-builder canvas and dashboard grid (where dashboard's "page" is null), but record-detail's ambient is *a Contact* or *an Event* — not a page-shaped thing. This session turns `SlotContext`'s ambient into a polymorphic typed payload so 5b's record-detail wiring has a place to plug.

---

## Design decisions (resolved before starting)

- **Abstract sealed-by-convention `AmbientContext` base + three final concrete subtypes.** Sibling to `Slot` / `DataContract` / `QuerySettings` in the namespace organization — empty abstract body, all payload on the subtypes. PHP doesn't have sealed classes; convention enforces (the abstract is in `app/WidgetPrimitive/`, subtypes are in `app/WidgetPrimitive/AmbientContexts/`, no public extension surface this session).
- **`PageAmbientContext` carries `public readonly ?Page $currentPage`.** That is the only payload existing slots needed — the page-builder canvas threads the current page through for token substitution. Folds `SlotContext`'s `?Page $currentPageOverride` into the typed ambient.
- **`DashboardAmbientContext` carries no payload this session.** Placeholder. Dashboard widgets read no per-record state today (per session 212). The class exists so all three subtypes are defined together; future per-dashboard state lands here without re-typing.
- **`RecordDetailAmbientContext` carries no payload this session either.** 5b plugs the body (Contact, Event, etc.). Class is defined now to lock the type before 5b wires the slot.
- **`SlotContext` becomes `(AmbientContext $ambient, bool $publicSurface)`.** The `pageContext + currentPageOverride` constructor pair retires. `publicSurface` stays on `SlotContext` because it's a slot/surface concern (filters SOURCE_WIDGET_CONTENT_TYPE collections by `is_public`), not an ambient-data-shape concern.
- **`SlotContext::currentPage(): ?Page` survives as a delegate.** Inside, asks the ambient: returns the Page when `$ambient instanceof PageAmbientContext`, null otherwise. Existing callers (`ContractResolver`, retrofit tests) keep working byte-equivalent — no instanceof checks at the call site.
- **`Slot::ambientContext(...)` stays not-declared on the abstract.** The per-slot **arg** shapes still differ (page-builder takes `(PageContext, ?Page)`; dashboard takes nothing). What standardizes is the **return** type: `: SlotContext` across all three slot implementations. `RecordDetailSidebarSlot::ambientContext(): SlotContext` (still throws — the return type is reachable in declaration only). Phase 5b can revisit the arg-shape question when the record-detail call site is real.
- **Templates keep reading `$pageContext` unchanged.** The container singleton `app(PageContext::class)` is still bound into widget Blade as `$pageContext`. Migrating templates from `$pageContext->currentPage` to slot-ambient lookups is a load-bearing change deferred to a later Phase 5 session — doing it here would balloon the blast radius. Today: `$pageContext` is the template-side accessor; `SlotContext::currentPage()` is the resolver-side accessor; both return the same Page. After 5a: same.
- **`PageContext` survives this session.** `currentPage`, `currentUser`, and `form()` + `formCache` all stay (the Forms widget hasn't retrofitted; no scheduled session). PageContext stops being a constructor argument to `SlotContext`, but the singleton remains in the container.
- **No widget contract changes this session.** Every contract-locked widget continues to declare the same `fields` / `formatHints` / `querySettings`. The typed ambient is a layer the resolver consumes; it doesn't surface in widget API.
- **`RecordDetailSidebarSlot::ambientContext()` continues to throw.** The body stays exactly the same `RuntimeException` from session 211 (with the same "lands with Phase 5" message — since 5a is *part of* Phase 5, the message could narrow to "lands with Phase 5b"; that copy edit is in scope).

## Phase 1 — Define the `AmbientContext` abstraction

1. Create `app/WidgetPrimitive/AmbientContext.php`. Final-by-convention abstract class. Empty body. Sibling to `DataContract` / `QuerySettings` in the namespace organization. Add a class-level docstring naming the closed subtype set: page / dashboard / record-detail.
2. Create `app/WidgetPrimitive/AmbientContexts/PageAmbientContext.php`. `final class PageAmbientContext extends AmbientContext` with `public readonly ?Page $currentPage` constructor field. Default `null`.
3. Create `app/WidgetPrimitive/AmbientContexts/DashboardAmbientContext.php`. `final class DashboardAmbientContext extends AmbientContext` — empty constructor, no fields. Class-level comment notes "no payload yet; future dashboard ambient state lands here."
4. Create `app/WidgetPrimitive/AmbientContexts/RecordDetailAmbientContext.php`. `final class RecordDetailAmbientContext extends AmbientContext` — empty constructor, no fields this session. Class-level comment notes "Phase 5b plugs the record payload (Contact / Event / etc.)."

## Phase 2 — Refactor `SlotContext` to carry typed ambient

1. `app/WidgetPrimitive/SlotContext.php` constructor signature changes from `(PageContext $pageContext, ?Page $currentPageOverride = null, bool $publicSurface = true)` to `(AmbientContext $ambient, bool $publicSurface = true)`.
2. The `currentPage(): ?Page` method body changes to:
   ```
   return $this->ambient instanceof PageAmbientContext
       ? $this->ambient->currentPage
       : null;
   ```
   The method's external contract is unchanged — same signature, same null-vs-Page return shape.
3. The `pageContext` property removes. If anything in the codebase reads `$slotContext->pageContext->...`, that read needs to migrate to the typed ambient or to the container singleton. Grep at session start to inventory.
4. The class-level docstring updates to reflect the new design — the surface is "ambient context for a render in this slot" and the field is typed.
5. PHPDoc the new constructor parameter: `$ambient` is the typed ambient payload; `$publicSurface` retains its session-214 meaning.

## Phase 3 — Update slot implementations to construct typed ambient

1. `app/WidgetPrimitive/Slots/PageBuilderCanvasSlot.php`:
   - Method signature unchanged: `ambientContext(PageContext $pageContext, ?Page $currentPageOverride = null): SlotContext`.
   - Body changes to: `return new SlotContext(new PageAmbientContext($currentPageOverride ?? $pageContext->currentPage));` (or equivalent — the ambient should carry the override-or-pageContext-page result, mirroring today's resolution logic).
   - The `PageContext` arg becomes purely a "what's the fallback page if no override?" input. Once consumed into the ambient, the `PageContext` itself doesn't ride on the SlotContext anymore.
2. `app/WidgetPrimitive/Slots/DashboardGridSlot.php`:
   - Method signature unchanged: `ambientContext(): SlotContext`.
   - Body changes to: `return new SlotContext(new DashboardAmbientContext(), publicSurface: false);`. The `app(PageContext::class)` lookup retires — dashboard doesn't read PageContext today, that was vestigial from the SlotContext shape.
3. `app/WidgetPrimitive/Slots/RecordDetailSidebarSlot.php`:
   - Method signature tightens: `ambientContext(): SlotContext` (was `: object`).
   - Body unchanged — still throws `RuntimeException`. Optional copy-edit: update the message from "lands with Phase 5" to "lands with Phase 5b" (since 5a is itself Phase 5).
4. `app/WidgetPrimitive/Slot.php`:
   - The class-level docstring's "intentionally not declared" comment updates: arg shapes still differ across slots, **but every slot now returns `SlotContext`**. Document this.
   - No abstract method added — return-type uniformity does not require abstract enforcement and PHP 8 still wouldn't accept a shared declaration with mismatched args.
