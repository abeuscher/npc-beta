# 351. SEO Features — JSON-LD, Blockquote & Responsive Images

The feature half of the SEO cluster: three small, independent features — a custom JSON-LD slot, a real `<blockquote>`/`<cite>` editor control, and responsive image `sizes` that match the column they render in.

## The three features

1. **Custom JSON-LD slot.** The schema emitter (`SeoMetaGenerator`) outputs only a generic `WebPage` block today. Add a slot for **custom / additional JSON-LD** — site-wide and/or per-page — injected as a **separate** `<script type="application/ld+json">` tag, so operators can add `SoftwareApplication` / `Organization` / `Person` graphs alongside the `WebPage`. **Check `head_snippet` first** — if the existing per-page head-snippet field already renders into `<head>`, per-page custom JSON-LD may already be servable, narrowing the real gap to a **site-wide** `SiteSetting` slot. Treat the input as **untrusted-but-not-HTML**: validate it parses as JSON, then emit with `<` escaped so it can't break out of the script element. (Do **not** route it through `HtmlSanitizer` — that strips `<script>`.)

2. **Blockquote + citation editor control.** Add a real `<blockquote>` control to the rich-text editor (admin + page-builder inspector) for SEO/semantics — Quill's blockquote is a native toolbar format, so exposing it is cheap. Citation (`<cite>`) is the design call: Quill has no native cite blot, so settle a v1 shape (a custom blot, a blockquote attribution line, or defer citation). `HtmlSanitizer` already allows `blockquote` + `cite`, so the sanitizer likely needs no change — confirm.

3. **Responsive `sizes` from column layout.** The shared responsive-image partial emits `<source sizes="100vw">` regardless of the actual rendered width, so an image in a `2fr`/`3fr` grid column downloads ~2× the pixels it needs. Derive `sizes` from the column's fraction of the layout (`grid_template_columns`), **or** expose an operator override. Pick the v1 shape — a derived value is the right fix; an override is the cheap floor.

## Open questions to resolve at session start

1. **JSON-LD storage + safety + multiplicity.** Site-wide via a `SiteSetting` key; per-page via `head_snippet`/`custom_fields` (say what `head_snippet` already covers). The injection-safety approach (JSON-validate + escape `<`; reject/flag invalid JSON). Whether multiple graphs (a JSON array, or multiple script tags) are supported in v1.
2. **Citation shape.** Native blockquote toolbar control is in regardless; settle whether `<cite>`/attribution ships in v1 (custom blot vs attribution line) or is deferred to keep the control minimal.
3. **Responsive `sizes`: derive vs override.** Derive from the column `grid_template_columns` fraction (the real fix; needs the column context plumbed to the image partial) vs an operator `sizes` override field (the cheap floor). Decide the v1.

## Phases

The three features are independent; build/commit each naturally.

1. **Decide the open questions** (JSON-LD storage/safety/multiplicity; citation shape; sizes derive-vs-override). Log at session start.
2. **JSON-LD slot:** the site-wide `SiteSetting` field + the per-page field (reusing `head_snippet`/`custom_fields` to whatever extent needed) + the render (a separate `ld+json` script tag in `<head>`, after the existing `WebPage` block) + JSON validation + the `<`-escape. Pest: site-wide graph renders; per-page graph renders; **a `</script>`-breakout attempt is escaped/neutralised**; invalid JSON is rejected/flagged, not emitted raw.
3. **Blockquote (+ citation):** wire the native blockquote control into the admin + inspector Quill toolbars; the v1 citation shape if in; render + sanitizer confirm. `npm run build`. Pest render-shape (`<blockquote>` survives the sanitizer + renders); optional Playwright authoring guard only if it's a worthwhile standing artifact.
4. **Responsive `sizes`:** the derived value (or override field) in the shared image partial + the render. Pest render-shape (an image in a `2fr`/`3fr` column emits the expected `sizes`, not `100vw`).
