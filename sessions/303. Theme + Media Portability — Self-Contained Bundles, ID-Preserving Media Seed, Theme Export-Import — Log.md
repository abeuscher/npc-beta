## What was built

### Phase 1 ‚Äî Self-contained content+media bundle (B-1) + theme/design export-import (A)

- **`App\Services\ImportExport\BundleArchive`** ‚Äî the zip container primitive. `build(envelope, zipPath)` writes `bundle.json` + `media/{id}/{file_name}` (bytes read disk-agnostically via each descriptor's disk); `extract(zipPath)` enforces a **zip-slip** guard (no `..`, absolute, or symlink entries; resolved path must stay within the temp dir) and a **zip-bomb** guard (per-entry count + total uncompressed ceiling, configurable for tests) before extracting, returns `(envelope, mediaRoot, tempDir)`.
- **`ContentImporter` zip-vs-JSON detection + archive-first resolution.** `import()` gained an optional `?string $mediaRoot` (transient instance state); when set, page/widget media resolve from the extracted archive first, falling back to the existing local-disk behaviour. The JSON path is behaviourally unchanged (the additive, ignorable `media_transport` envelope key is `reference` for JSON / `embedded` for zip; `validateEnvelope()` untouched).
- **Queued export.** `App\Jobs\ExportBundleJob` serializes the envelope, builds the zip to the private `local` disk at `exports/bundles/{uuid}/{name}.zip`, and sends a persistent database notification with a gated download action. `App\Http\Controllers\Admin\BundleExportDownloadController` (route `filament.admin.exports.bundle.download`, `update_page`-gated, uuid-validated) serves it via `Storage::disk('local')->download()`. No `streamDownload` on this path.
- **Queued import.** `ImportBundleAction` now accepts `.zip` + JSON, 512 MB cap, dispatches `App\Jobs\ImportBundleJob` (synchronous fallback removed). The job sniffs zip-vs-JSON by magic bytes, extracts archive-first when zip, runs `ContentImporter`, always deletes the upload + temp dir in `finally`, and notifies via the bell with the `ImportLog` summary.
- **Sibling zip export actions** at the six content call sites (PageResource bulk, EditPageDetails, PostResource bulk, EditPostDetails, EditPageTemplate, EditContentTemplate) ‚Äî each keeps its existing JSON streaming action verbatim and gains an "Export with media (zip)" sibling that dispatches the queued job.
- **(A) Theme/design export-import.** `ContentExporter::exportDesign()` ‚Üí `payload.design` (the three `design`-group `SiteSetting` rows ‚Äî `theme_colors`, `typography`, `button_styles` ‚Äî as stored). A new `ContentImporter` `payload.design` pass runs before templates/pages: each row **deep-merges over its resolver defaults and never sweeps** (absent keys keep the concrete default, never null ‚Äî the 295 lesson / concrete-values rule); after the transaction commits it triggers the existing `AssetBuildService::build()` (same persistent-warning fallback as `DesignSystemPage`). An **Export Theme / Import Theme** header-action pair on `DesignSystemPage`, gated by its existing `manage_cms_settings`. Confirmed (not rebuilt) the session-301 `scheme`/`no_header`/`no_footer` carry-through already present in `ContentExporter`/`ContentImporter`.
- **Notifications infrastructure (approved).** Standard Laravel `notifications` table migration; `->databaseNotifications()` + `->databaseNotificationsPolling('30s')` enabled on the admin panel.

### Phase 2 ‚Äî Standalone ID-preserving media seed + Media Library UI (B-2)

- **`ContentExporter::exportMedia(ids)` / `exportAllMedia()`** ‚Üí `payload.media` (full posture-B descriptors: id, uuid, model_type/id, collection, name, file_name, mime, disk, size, manipulations, custom_properties, responsive_images, order_column, path).
- **`ContentImporter::importMedia()`** (runs before templates/pages): raw explicit-id `media`-row insert; file placed at `{id}/{file_name}` on the target media disk; **collision policy** ‚Äî identical (matching uuid+file_name) ‚Üí idempotent skip; divergent id ‚Üí `ImportLog` warning naming the id, no clobber, skip; **orphan-owner policy** ‚Äî owner absent on target ‚Üí still create row+file, log "parked"; bytes resolved archive-first then source-disk; the Postgres `media_id_seq` is reset to `MAX(id)` after seeding; `App\Jobs\RegenerateMediaConversionsJob` dispatched per seeded id after commit (conversions regenerated, never shipped).
- **`MediaLibraryPage`** gained "Export selected media" (bulk) + "Export all media" + "Import media bundle" actions, all gated `update_page`; the page's own `canAccess` stays `view_any_page` (decision #7).
- **End-to-end proof** (Pest): export a page by-reference (JSON) + its media standalone (zip) ‚Üí wipe target rows+disk ‚Üí seed media ‚Üí import the JSON page bundle ‚Üí the image resolves off the seed.

### Folded in at user request (mid-session)

- **Postgres `notifications.data` must be `json`, not `text`.** Filament's bell query (`data->>'format' = 'filament'`) needs the `->>` operator, which Postgres only allows on `json`/`jsonb`; the stock Laravel `text` column 500'd the admin on load. Column changed to `json`; added a regression test that runs Filament's exact JSON-operator query.
- **Notification visibility.** Operators were missing queued-job completion alerts: the bell unread count is now a solid-red pill (white, bold, no ring) that pulses twice every 10s (`prefers-reduced-motion` respected), via a scoped inline-style render hook (no asset build); the export-ready notification title now leads with the filename.
- **e2e CI fix.** The Playwright job had no migrate step before the test run and its readiness probe (`/admin/login`) is DB-independent, so a fresh runner served `relation "site_settings" does not exist` 500s ‚Üí exit 1. Added an explicit `migrate:fresh --seed --force` step after the readiness gate. Latent structural gap, **not a 303 regression** (works locally only because the persistent e2e volumes are pre-migrated/warm).
- **Investigated a user-reported "imported colour doesn't revert."** Reproduced the exact flow through the real editor save path incl. a simulated reload: it round-trips correctly. **No code bug** ‚Äî the background import doesn't live-refresh an already-open editor (reload shows the reverted value), and "Export Theme" exports the last *saved* values. Kept the full save‚Üíexport‚Üíchange‚Üíimport‚Üíreload regression test.
- **Planning (separate from 303 code).** E8 ("UI/UX Sprint") retired and superseded by the rescued two-session **Page Builder Inline Editing** arc (Session A = 304, Session B = 305): drafts rescued from `claude/investigate-stale-stylesheet-ANZJt` (drafted 2026-05-17, never on main), renumbered, de-drafted, and recorded in `release-plan.md` (retired ¬ß E8 slot, position 38) + a canonical `session-outlines.md` arc stub.

## Security considerations

The new external-facing seams were the security focus. Export artifacts live on the private `local` disk and are served only through the `update_page`-gated, uuid-validated download controller ‚Äî no public URL, no `streamDownload`. `BundleArchive::extract` enforces zip-slip (path normalization, no `..`/absolute/symlink) and zip-bomb (entry-count + uncompressed-size ceiling) guards before any byte is written; the existing descriptor path-traversal re-guards in `rewire*Media` are preserved and extended to the archive root. The posture-B media seed never clobbers a colliding id (warn-and-skip on divergence, idempotent skip on identity). Theme actions are behind `manage_cms_settings`; media/content export-import behind `update_page`; the Media Library page itself stays `view_any_page`. Theme import deep-merges over defaults and never sweeps stored config (no accidental nulling). No Fleet / `/api/health` / response-schema surface touched; CRM contract unchanged at v2.3.0. The one new table (`notifications`) holds only Filament's own notification payloads for the authenticated operator.

## Production deployment notes

- Run `php artisan migrate` (creates the `notifications` table). If an environment already ran the *original* stock-`text` form of this migration, instead run `php artisan migrate:rollback --step=1 && php artisan migrate` so `data` becomes `json` (the table is empty ‚Äî safe).
- The **worker container must be running** (`php artisan queue:work`) for exports/imports to process ‚Äî both directions are queued; the action only confirms "queued," the bell confirms completion.
- No `npm run build` (no Vue/SCSS source touched ‚Äî the bell style is an inline render-hook string). No `build:public` (theme import triggers `AssetBuildService::build()` at runtime, not a source build).
