## What was built

### Phase 1 — Inspector shell components

- `InspectorPanel.vue` — shell with "select a block" placeholder, header, tabs, field groups, and apply button
- `InspectorHeader.vue` — widget type badge (uppercase label) and inline label editor (pencil icon, OK/Cancel, Enter/Escape)
- `InspectorTabs.vue` — Content/Appearance tab bar with active state styling matching the existing Blade tabs

### Phase 2 — Field infrastructure

- `InspectorFieldGroup.vue` — layout grouping (consecutive fields with same non-semantic `group` render in 2-column grid), advanced fields accordion ("Carousel Settings"), empty message support
- `InspectorField.vue` — conditional visibility (`shown_when`/`hidden_when` evaluated against widget config), default value population on mount (fixes Livewire bug where fields appeared blank despite schema defaults), type-based component dispatch

### Phase 3 — Field components

- `TextField.vue` — text and URL input types
- `TextareaField.vue` — 4-row textarea
- `NumberField.vue` — number input with optional min
- `SelectField.vue` — dropdown with static options, `options_from` API resolution, and `collection_fields:*` dependency re-resolution
- `ToggleField.vue` — checkbox with "Enabled" label
- `CheckboxesField.vue` — multi-select checkbox grid with configurable columns
- `NoticeField.vue` — read-only info/warning display with variant-based styling
- `RichTextField.vue` — Quill editor with same toolbar config as existing Alpine module, inline image upload via existing endpoint, debounced content sync, external value change guard

### Phase 4 — Store changes and wiring

- Added `updateLocalConfig()` to Pinia store — instant local UI update + 500ms debounced API save via `updateWidget`
- Added `inlineImageUploadUrl` tracking from bootstrap data
- Updated `App.vue` with 2fr/1fr grid layout (preview + inspector) in both edit and handles modes
- `ApplyChangesButton.vue` — sticky footer, disabled when no dirty widgets, triggers preview refresh
- Fallback messages for unsupported field types (image, video, color, buttons)

### Bonus — Add Block positioning and auto-select

- Moved "+ Add Block" button from EditorToolbar into PreviewCanvas viewport bar with dropdown: "Insert at bottom", "Insert above selected", "Insert below selected" (above/below disabled when nothing selected)
- Fixed Blade event bridge to pass `insertPosition` through to Livewire `openAddModal()`
- Fixed off-by-one in "insert at bottom" by computing `max(sort_order) + 1` from widget data instead of relying on `count($this->blocks)`
- Added `block-selected` event listener in App.vue so newly created blocks auto-select in the Vue inspector
- Added `scrollIntoView({ behavior: 'smooth', block: 'center' })` to scroll to newly selected blocks (note: currently scrolls to Livewire instance since it appears first in DOM — will resolve when Livewire inspector is removed in session 152)

### Bug fixes

- Fixed preview canvas overflow: added `min-width: 0` on grid cells and `overflow: hidden` on preview container to prevent 1920px viewport preset from blowing out the grid layout
- Fixed toggle fields showing only "Enabled" with no context: restored field label rendering for toggle type
