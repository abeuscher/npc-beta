## What was built

### Phase 1 ‚Äî QuickBooks Sync Service

- Created `app/Services/QuickBooks/QuickBooksClient.php` ‚Äî wraps the QB SDK's DataService, calls `refreshTokenIfNeeded()` before every request, and exposes `createSalesReceipt()`, `createRefundReceipt()`, and `getIncomeAccounts()`.
- Created `app/Jobs/SyncTransactionToQuickBooks.php` ‚Äî queued job that syncs a transaction to QuickBooks. Skips silently if QB is not connected, no income account is configured, or the transaction already has a `quickbooks_id`. Routes inbound transactions to Sales Receipts and outbound (refund) transactions to Refund Receipts. Stores the QB ID and timestamp on success; stores a redacted error message on failure.

### Phase 2 ‚Äî Schema & Webhook Integration

- Migration adding `qb_sync_error` (nullable text) and `qb_synced_at` (nullable timestamp) to the `transactions` table.
- Updated `Transaction` model with new fillable fields and `qb_synced_at` datetime cast.
- Dispatched `SyncTransactionToQuickBooks` from `StripeWebhookController` after every completed transaction creation: general checkout, donation checkout, invoice payment succeeded, product purchase, and refund created. Failed transactions are not dispatched.

### Phase 3 ‚Äî Finance Settings UI & Admin Visibility

- Added "QuickBooks ‚Äî Transaction Sync" section to Finance Settings with an income account dropdown populated from the QB Chart of Accounts (cached 1 hour), a "Refresh Accounts" button, and save logic for the `qb_income_account_id` site setting.
- Added a "QuickBooks" badge column to the Transactions list showing Synced/Error/Pending/N/A with tooltip details.
- Added a "Sync to QB" manual re-sync table action gated to `manage_financial_settings` permission, hidden when already synced or QB not connected.
- Updated `resources/docs/settings-finance.md` with transaction sync documentation: income account selection, sync statuses, manual re-sync, and how automatic sync works.

### Roadmap additions

- Added "Deploy-Server Integration Test Suite" stub to post-Beta 1 Infrastructure & Ops section.
- Wrote full session prompts and base prompts for sessions 104 (QuickBooks Customer Matching) and 105 (QuickBooks Per-Type Account Mapping) and added stubs to session-outlines.
