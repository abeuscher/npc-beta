# 123. Widget — Video & Image Background

**Status: Partially completed in session 122.** The hero widget now supports `background_video` (MP4/WebM upload via Spatie media library) alongside `background_image`. Video takes priority when both are set. The `video` inspector field type was also added in session 122.

**What remains (if still needed):** A standalone `background_media` widget — a simpler section with video/image background and text overlay, distinct from the hero. This would be useful if pages need a background media section without the hero's CTA buttons, position controls, and nav overlap features. Decide whether to build this or just use the hero widget for all background media needs.

---

## Original scope (for reference)

- Standalone `background_media` widget type with video URL or uploaded image toggle
- Heading, subheading, richtext content, overlay opacity, min-height, text alignment
- Video fallback to image poster

## Already built (session 122)

- `type: 'video'` inspector field (MP4/WebM upload)
- `background_video` config field on hero widget
- `<video autoplay muted loop playsinline>` rendering with `object-fit: cover`
- Video takes priority over image when both set
- Overlay works over video same as image
- WidgetRenderer and PageBuilderInspector handle video media fields
