## Open-question decisions (settled at the Open Gate)

1. **Rich-text transform placement (items 4 + 5): render-time, not save-time.** `HtmlSanitizer::sanitize` is the one universal *save* seam (11 callers — widgets, notes, events, email templates, collections, custom fields, importers), but a save-time transform is a permanent rewrite that leaves every already-published page serving the broken markup until re-saved (needs a backfill). Render-time fixes existing *and* new content immediately, is non-destructive/reversible, and doesn't disturb the Quill editor round-trip (the in-place editor seeds from raw config via the store, never from rendered output). Placed in `WidgetRenderer`'s richtext loop — the universal *public widget* render seam — as a DOM-based `RichTextSemantics::normalize()` helper.
2. **Hero-LCP (item 7): per-instance eager toggle, not auto-eager-first.** A `loading_priority` config field (default `lazy`) on `Image` + `EventImage`; auto-eager-first was rejected as render-order plumbing for a guess the operator can make directly.
3. **`og:title` (item 3): verify + one small hardening.** All public page types render through the shared layout, so `og:title` already wires through uniformly; the real gap was `??` (null-only) in the views vs `?:` in `SeoMetaGenerator`. Tightened to `?:` + a standing per-type test.

## What was built

### Item 1 — Personal email no longer leaked into client output

`window.__site.contactEmail` (`resources/views/layouts/public.blade.php`) emitted the operator contact email into page source on every page. Grep found **zero** consumers of `window.__site.contactEmail` (JS/Vue/Blade) — removed the line outright rather than swapping it to the public setting. Privacy + SEO.

### Item 2 — `/home` 301 → `/`

Added `Route::get('/home', fn () => redirect('/', 301))` before the catch-all `/{slug}` in `routes/web.php`, so the homepage has a single crawlable URL instead of resolving 200 at both `/` and `/home`.

### Item 3 — `og:title` per-page-type

`resources/views/pages/show.blade.php` and `posts/index.blade.php` passed `'title' => $page->meta_title ?? $page->title` (null-only). Per the concrete-values rule `meta_title` is stored as `''` when unset, so `??` would emit an empty `<title>`/`og:title`. Aligned both to `?:` (empty-aware, matching `SeoMetaGenerator`). Confirmed `og:title` wires uniformly across page / post / event via the shared `layouts.public`; member pages use `layouts.portal` (auth-gated, no OG tags — correctly off the public surface).

### Items 4 + 5 — Rich-text semantics (`App\Support\RichTextSemantics`)

New DOM-based render-time helper (mirrors `HtmlSanitizer`'s parse/serialize, including the `{{token}}` restore), called in `WidgetRenderer`'s richtext field loop alongside `InlineImageRenderer::process`:

- **Bullet `<ol>` → `<ul>`** (item 4). Quill v2 represents `<ul><li>` as `<ol><li data-list="bullet">`. An `<ol>` whose direct `<li>` are *all* bullets is rewritten to a real `<ul>` and the redundant `data-list="bullet"` dropped (recurses for nested lists). Genuine ordered lists (`data-list="ordered"`, or a plain `<ol>`) and mixed lists are left untouched.
- **Heading-wrapping bold → unwrapped** (item 5). `<h2><strong>…</strong></h2>` → `<h2>…</h2>` only when the `<strong>`/`<b>` is the heading's sole meaningful child. Partial/mid-heading bold and bold inside paragraphs are left untouched.

Non-destructive: stored HTML is unchanged; existing published pages are corrected at render with no backfill.

### Item 6 — Nav `<p>` wrapper + misplaced `role`

`app/Widgets/Nav/template.blade.php` + `NavDefinition.php`: the Quill-backed parent/child templates wrapped the link in `<p>` and the `role="menuitem"` sat on the wrapper `<span>`. The `$renderTemplate` helper now strips a wrapping `<p>` and injects `role="menuitem"` (+ `aria-current="page"` when active) onto the anchor — the interactive element. The dropdown disclosure ARIA (`aria-haspopup`, `:aria-expanded`) and Alpine hover/focus/keyboard handlers stay on the wrapper; the arrow-down keyboard selector was updated from `[role=menuitem] a` to `[role=menuitem]` for the new structure. The seeded `defaults()` parent/child templates were de-`<p>`'d to match the clean schema defaults.

### Item 7 — Above-the-fold image eager / `fetchpriority`

`Image` + `EventImage`: a per-instance `loading_priority` select (`lazy` default / `eager`) in the schema + defaults. Eager emits `loading="eager"` + `fetchpriority="high"`; lazy keeps `loading="lazy"` and no `fetchpriority`. The `Image` widget threads `:loading` / `:fetchpriority` through the `x-picture` component (verified the anonymous-component attribute wiring with an uploaded-media test); `EventImage` and the demo-URL path write the attributes directly. Scoped to the two LCP candidates — the other ~13 `loading="lazy"` sites were left as-is per the session fence.

## Security considerations

Net positive on the privacy axis: item 1 stops the operator contact email leaking into every page's source. `RichTextSemantics` runs on already-sanitized HTML at render time and introduces no new tags or attributes — it only renames `<ol>`→`<ul>`, removes `data-list`, and unwraps an existing `<strong>`; it reuses `HtmlSanitizer`'s DOM round-trip and `{{token}}`-restore, so it adds no new untrusted-input path. The Nav change is markup-only. No new injection surface.
