## What was built

### Phase 1 — Manual Transaction Form Cleanup

- Replaced type dropdown with Grant and Adjustment only (expense removed — system never creates outbound money)
- Direction is now auto-derived from type via hidden field, `afterStateUpdated`, and `mutateFormData` hooks on create/edit pages
- Added searchable, nullable contact selector for QB customer matching
- QuickBooks reference field: hidden on create, read-only on edit when set
- Unknown types (Stripe-originated records) render as disabled text input instead of blank select
- Updated section description to reflect inbound-only manual transactions

### Phase 2 — Transaction Seeder

- Created `TransactionSeeder` with 14 representative records: 6 donation transactions, 3 purchase transactions, 2 refunds (with `stripe_id`), 2 manual (grant + adjustment), 1 error-state
- 10 factory contacts with mix of QB-linked, email-less variants
- Creates supporting donation, product, product price, and purchase records
- Idempotent via `Transaction::exists()` sentinel
- Added to `DatabaseSeeder` chain

### Phase 3 — Factory Update

- Added state methods to `TransactionFactory`: `manual()`, `fromStripe()`, `synced()`, `syncError()`, `refund()`
