## Key decisions

**SiteSetting instead of GeneralSettings:** The session prompt assumed a `app/Settings/GeneralSettings.php` class (spatie/laravel-settings style). This app uses a `SiteSetting` key-value model throughout. Adapted all layout and settings code to use `SiteSetting::get()` / `SiteSetting::set()` directly, consistent with the rest of the application.

**npm on host, not in Docker:** The app container is PHP-FPM only and has no Node.js. npm is available on the host (Node 22), and the project directory is volume-mounted, so building on the host produces identical output. No Dockerfile changes made.

**Appearance setting keys:** `public_primary_color`, `public_heading_font`, `public_body_font` — prefixed with `public_` to namespace them from admin-side settings (e.g. `admin_primary_color`).

**Alpine hover dropdown:** `<details>`/`<summary>` was the initial approach but prevents linking the parent item. Replaced with an Alpine `@mouseenter`/`@mouseleave` pattern on a `.nav-dropdown` wrapper span. Parent `<a>` navigates normally; chevron `<button>` with `@click.stop` handles touch. No new dependencies.
