# 160. Page Builder & Admin Chrome Fixes

Five targeted fixes to the page builder inspector and the Filament admin chrome. No new features — all five items restore intended behaviour, fix small regressions, or relocate an existing control. The changes touch both the Vue page builder app and the session 120 Filament custom theme.

---

## Design decisions (resolved before starting)

### 1. Inspector panel fixed width

- The Vue inspector lives in `resources/js/page-builder-vue/App.vue` and is currently sized with `grid-template-columns: minmax(0, 1fr) min(32rem, 33%)` at approximately line 73. When the admin left nav collapses (via the topbar fullscreen toggle delivered in session 156, or by manual collapse), the available viewport widens and the inspector grows proportionally because of the `33%` clause.
- **Fix:** replace `min(32rem, 33%)` with a fixed `32rem`. The inspector should not grow or shrink in response to changes in canvas width.
- Retain the existing two-track grid pattern; only the inspector track changes from a responsive expression to an absolute size.
- Minimum body width on narrow viewports is not in scope — the inspector has always assumed desktop-width admin usage. If extreme narrowness becomes a real issue later, address it separately.

### 2. Filament left sidebar collapse must persist independently of the topbar fullscreen toggle

- Session 156 delivered the topbar fullscreen toggle at `resources/views/components/admin-fullscreen-toggle.blade.php`. Today it drives **two** state pieces in lockstep: the content full-width mode (the `np-fullscreen` class on the body) AND the Filament sidebar collapse state (via Filament's Alpine `$store.sidebar.close()` / `open()`).
- **Bug:** if a user has manually collapsed the Filament sidebar, clicking the fullscreen toggle on and then off restores the sidebar to expanded — clobbering the user's intent. Conversely, a user who has expanded the sidebar gets it force-collapsed every time fullscreen toggles.
- **Fix:** decouple the two state pieces. The fullscreen toggle drives only the content full-width mode. Filament's sidebar collapse state remains user-controlled and persists via whatever mechanism Filament uses today (a cookie or localStorage key — Phase 1 will identify which).
- The existing user-menu full-width toggle (also delivered/touched in session 156) should continue to mirror only the content full-width mode after the decoupling.
- Acceptance: collapse the Filament sidebar manually, click the topbar fullscreen toggle on and off three times, confirm the sidebar remains collapsed throughout. Repeat the test starting from an expanded sidebar — it should remain expanded.

### 3. Topbar fullscreen toggle: eye icon → arrows-in / arrows-out

- The current icon pair in `admin-fullscreen-toggle.blade.php` is `eye` / `eye-slash` (heroicons). Replace with `arrows-pointing-out` / `arrows-pointing-in`.
- **State mapping:**
  - Normal mode (chrome visible): show `arrows-pointing-out` — the affordance reads "click to expand into fullscreen".
  - Fullscreen mode (chrome hidden): show `arrows-pointing-in` — the affordance reads "click to contract back to normal".
- Confirm the heroicon names in use match the heroicons version installed in the project. If the names differ slightly (e.g. `arrow-top-right-on-square`), pick the closest pair that conveys the same in/out semantic.
- No behavioural change from the icon swap itself — this is a polish item.

### 4. Filament left sidebar should not scroll when collapsed

- When the Filament sidebar is in its icon-only collapsed state, the nav list still triggers an overflow-y scroll. The user can scroll the collapsed rail with the mouse wheel even though there is nothing useful to scroll past. This is a CSS regression in the session 120 admin theme.
- **Fix:** investigate the collapsed-state CSS in the custom Filament theme (likely under `resources/css/filament/admin/theme.css` or a related stylesheet — Phase 1 will confirm). The collapsed nav container should only scroll when its content height genuinely exceeds the viewport height. The current symptom suggests one of: (a) a min-height that forces extra height in the collapsed state, (b) a flex child whose width-based wrap calculation is still using the expanded width, (c) an `overflow-y: auto` rule applied unconditionally.
- **Acceptance:** load an admin page with a long nav list, collapse the sidebar via Filament's own collapse control, confirm no scrollbar and no wheel-scroll events on the collapsed rail. Expand the sidebar and confirm scrolling returns when the content genuinely overflows.

### 5. "Save as Template" button — smaller and relocated

- Currently rendered in `resources/js/page-builder-vue/components/EditorToolbar.vue` at approximately lines 24–31. Uses the `.editor-toolbar__btn--secondary` class with `0.5rem 1rem` padding and `0.875rem` font-size. Visually it sits at the same prominence as the primary editor actions and competes for attention.
- **Fix:** shrink the button (smaller padding, smaller font size, or icon-plus-label treatment) and relocate it to a less prominent position in the toolbar. The exact placement is a judgment call — reasonable options include the far right of the toolbar grouped with secondary actions, or behind a kebab/overflow menu. Pick something that feels right and doesn't add new infrastructure (no new dropdown component just for one button).
- **Constraint:** the button currently triggers a Livewire bridge event that opens the save-as-template modal handled by `app/Livewire/PageBuilder.php::openSaveTemplateModal()`. The event wiring must continue to work after the visual reshuffle. Verify by clicking the relocated button and confirming the modal opens.
