## What was built

### Phase 1 — `DateFormat` vocabulary

- New `App\Support\DateFormat` final class at [app/Support/DateFormat.php](app/Support/DateFormat.php). 9 named public constants (`LONG_DATE`, `MEDIUM_DATE`, `LONG_DATETIME`, `MEDIUM_DATETIME`, `EVENT_FULL`, `EVENT_COMPACT`, `TIME_OF_DAY`, `EVENT_LIST_DATE`, `TIME_SMART`) plus a static `format(?Carbon $date, string $format): string` helper.
- `TIME_SMART` constant value is the sentinel `'__time_smart__'`; the helper resolves the conditional (`'ga'` if `minute === 0`, else `'g:ia'`) at call time. Sentinel was chosen because no legitimate format string uses double-underscore framing; never reaches `Carbon::format()` directly.
- Helper is null-safe: a null `$date` returns `''` for every format, including `TIME_SMART`. Verified by two unit tests.
- 12 unit tests at [tests/Unit/Support/DateFormatTest.php](tests/Unit/Support/DateFormatTest.php), one per named format plus both halves of `TIME_SMART` and both null-safety cases. All assertions use the fixed Carbon instance `2026-04-25 17:00:00` (Saturday 5:00 PM) for canonical output strings.

### Phase 4 — Schema migration: `published_at` on events and products

- New migration [database/migrations/2026_04_26_200905_add_published_at_to_events_and_products.php](database/migrations/2026_04_26_200905_add_published_at_to_events_and_products.php). Adds `published_at` (nullable timestamp, after `status`) to both `events` and `products`. Idempotent backfill in `up()`: `UPDATE … SET published_at = created_at WHERE status = 'published' [AND is_archived = false] AND published_at IS NULL`. Idempotent because the `WHERE published_at IS NULL` clause guards re-runs. `down()` drops both columns.
- Migration applied locally via `docker compose exec app php artisan migrate`. Both columns verified present.
- [docs/schema/events.md](docs/schema/events.md) and [docs/schema/products.md](docs/schema/products.md) updated with the new column rows, documenting observer-driven set behaviour.
- [app/Models/Event.php](app/Models/Event.php) and [app/Models/Product.php](app/Models/Product.php) gain `'published_at' => 'datetime'` casts. Eloquent infers datetime from timestamp columns, but explicit is preferred.
- [EventObserver.php](app/Observers/EventObserver.php) extended with `creating` and `updating` methods that auto-set `published_at = now()` on first transition to `status = 'published'` if currently null. The existing `created`/`updated`/`deleted` activity-log methods are unchanged.
- New [ProductObserver.php](app/Observers/ProductObserver.php). Registered via `#[ObservedBy(ProductObserver::class)]` attribute on `Product` (mirror of `Event`'s registration). Single behaviour: `creating` and `updating` auto-set `published_at = now()` on first transition to `status = 'published' && !is_archived` if currently null. Future product lifecycle hooks can join the same observer.
- Both observers use direct attribute assignment (`$model->published_at = now();`), which bypasses `$fillable` mass-assignment guards. `published_at` is **not** added to `$fillable` on either model — observer-driven only.
- [EventsListingDefinition.php](app/Widgets/EventsListing/EventsListingDefinition.php), [ThisWeeksEventsDefinition.php](app/Widgets/ThisWeeksEvents/ThisWeeksEventsDefinition.php), [ProductCarouselDefinition.php](app/Widgets/ProductCarousel/ProductCarouselDefinition.php) — `'published_at' => 'Published'` added to `orderByOptions` map. The events QuerySettings panel's "newest first" sort now orders by publication time instead of admin-row-creation time, fixing the gap from session 221 (where the events allowlist fell back to `created_at` because `published_at` didn't exist).

## Deferred decisions

- **Per-widget date-format selection** (inspector "Date format" select on each widget): scoped into session 226 as the format-hint mechanism on `DataContract`. The vocabulary this session lands is the closed set the next session's hint picks from.
- **Filament form field for explicit `published_at` override** on events / products: deferred. Observer-driven auto-set covers the current need. A future session can expose admin-controlled publish scheduling (mirror of `pages.published_at` form field) if user-driven scheduling becomes a real requirement.
- **Site-setting "default date display"**: deferred to session 226 or later. Cheapest extension path the original 225 stub flagged; can land when there's a second axis (e.g., `time_format`) to back it up.
- **Locale-aware formatting** (24-hour time, European date order, language-specific names): deferred. The vocabulary makes this future swap cheap (one constant body change moves the whole app); no hooks exposed this session.
- **`deleted_at` symmetry on events and products**: deferred. `status='cancelled'` and `is_archived` continue to function as tombstones; migrating off them is a bigger refactor than this session covered.
- **EventDescription's same-day vs cross-day structural branching**: noted during the close-gate conversation as belonging *in the projector*, not the template. Session 226 removes the branching entirely (multi-day events not meaningfully supported) and pulls EventDescription's date assembly into the projector.
