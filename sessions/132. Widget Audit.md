# 132. Widget Audit

Systematic audit of all widget types: fix residual Tailwind, relocate misplaced inline CSS/JS, ensure color fields use pickers, and report any structural issues.

---

## Design decisions (resolved before starting)

- **No new features this session.** This is a cleanup and consistency pass only.
- **Fixes are applied in place** — no new files unless an existing widget is missing an SCSS partial it should have.
- **Report-first approach:** scan all widgets, compile a findings list, then fix. The user reviews the findings before fixes are applied.

---

## Before writing any code

Read the following:

- `resources/docs/widget-development.md` — widget system reference (asset pipeline, config field types, SCSS conventions)
- `database/seeders/WidgetTypeSeeder.php` — all widget definitions
- Every Blade template in `resources/views/widgets/` — scan for inline styles, Tailwind classes, and structural issues
- Every SCSS partial in `resources/scss/widgets/` — check for consistency
- `resources/js/public.js` — verify exposed globals match what widgets actually use

---

## Phase 1 — Audit and report

Scan every widget type and produce a findings table with columns:

| Widget handle | Issue type | Description | Severity |

Issue types to check for:

- **Tailwind residue:** any Tailwind utility classes (`text-`, `bg-`, `flex`, `p-`, `m-`, `rounded-`, etc.) in widget Blade templates that should be plain CSS/SCSS
- **Misplaced inline CSS:** styles in the `css` column or inline `style` attributes that belong in an SCSS partial
- **Misplaced inline JS:** JS in the `js` column that should be in the Blade template's `x-data` or in `public.js`
- **Color fields:** any color config field using `type: 'text'` instead of `type: 'color'`
- **Orphaned config fields:** fields defined in `config_schema` but never referenced in the template
- **Unused template variables:** variables extracted in the `@php` block but never used in the markup
- **Missing SCSS partial:** widget has inline CSS or style attributes but no external SCSS file
- **Inconsistent BEM naming:** class names that don't follow the `.widget-{handle}` / child element pattern
- **Accessibility gaps:** missing `aria-` attributes on interactive elements, missing `alt` on images

Present the findings table and wait for approval before proceeding.

---

## Phase 2 — Fix

Apply fixes for all approved findings:

- Move Tailwind classes to SCSS partials using the existing BEM convention
- Move inline CSS from the `css` column to SCSS partials; update `assets.scss` in the seeder
- Ensure all color-type config fields use `type: 'color'`
- Remove orphaned config fields or unused variables
- Run `build:public` after SCSS changes
