## Phases

### Phase 1 â€” Storage shape + Mailable + default template

- Define the `Form.settings.notifications` array shape (per scoping doc): `[{ to, subject, template, include_submission_data }]`. Backward-compatible â€” absent key = no notifications, existing forms untouched.
- `app/Mail/FormSubmissionMailable.php` â€” mirrors the existing Mailable pattern; accepts the `Form`, the `FormSubmission`, and one resolved notification config; renders the chosen template (v1: the default).
- `resources/views/mail/forms/submission-default.blade.php` â€” form title + submitted-field key/value table when `include_submission_data` is true. Plain, legible; this is a user-judgment surface at review.

### Phase 2 â€” Dispatch path + recipient resolution + SiteSetting

- `app/Observers/FormSubmissionObserver.php` (registered in the appropriate service provider) â€” `created()` reads `form.settings.notifications`, resolves each `to` (SiteSetting token or literal, sanitised/validated as an email), dispatches one `FormSubmissionMailable` per entry. Inline-in-controller is the documented alternative; prefer the observer (no controller surface change, mirrors the existing post-write side-effect pattern).
- New `site_owner_email` SiteSetting key + surface it on `app/Filament/Pages/Settings/MailSettingsPage.php` (next to the existing from-name / from-address fields) so the operator sets it without code. Validate as an email.

### Phase 3 â€” FormResource notifications UI

- Extend `app/Filament/Resources/FormResource.php` with a notifications repeater (a new tab/section beside the existing field-builder repeater): per-row `to`, `subject`, `include_submission_data` toggle. `template` defaults to the built-in and is not operator-exposed in v1 (or is a disabled/preset field â€” operator-legibility judgment, surface at review).

### Phase 4 â€” Wire the first consumer + verify end-to-end

- Populate the `contact-page` form's `settings.notifications` in `database/seeders/FormSeeder.php` with a single entry to `{{site_owner_email}}`. Re-seed.
- Set `site_owner_email` to a placeholder for local verification; submit `/contact` for real and confirm the notification renders + "sends" via the log mailer with the submitted data. Close this loop yourself.
