### Public-content indicator UX

The 32c success criterion includes "Public-content indicator visible on every record/widget surface that has potential to leak." Decide at session start what the indicator looks like:

- **Badge in list views** — a small "Published" / "Public" badge next to record titles. Cheapest to implement.
- **Banner in edit views** — a sticky banner at the top of the edit page: "This page is currently published — changes go live on save." More prominent, more friction. Useful for prevent-foot-shooting.
- **Column in list views** — explicit "Visibility" column showing Public / Draft. Most discoverable.

Recommendation: combination — column in list views (always there, easy to scan) + banner in edit views for currently-published records (prompts a pause before saving changes). Decide at session start; implementation work for the chosen shape lands as in-session fixes per Rule 2.
