## What was built

### Phase 1 — Color picker with swatches

- Added `editor_color_swatches` to bootstrap data (read from `SiteSetting`)
- Added `PUT /admin/api/page-builder/color-swatches` API endpoint with `update_page` permission
- Created `ColorPickerField.vue` — color wheel, hex input, theme palette preset, saved swatches with add/remove
- Color picker shows dimmed "?" indicator when no value is set, clear button when value exists
- Added `colorSwatches` ref and `saveColorSwatches` action to Pinia store
- Added `saveColorSwatches` to `api.ts`, `color_swatches` to `BootstrapData` type

### Phase 2 — Image/video upload and button list

- Created `ImageUploadField.vue` — handles both image and video types, thumbnail preview, upload/remove via store
- Created `ButtonListField.vue` — add/remove/reorder buttons with text/url/style fields (ported from Alpine `buttonListManager`)
- Added `image_urls` to Widget type, API responses, and bootstrap data for resolving media URLs
- Extended upload endpoint to accept video mimes (mp4, webm)
- Wired color, image, video, and buttons into `InspectorField.vue`, replacing all fallback messages

### Phase 3 — Spacing controls and query settings

- Created `SpacingControl.vue` — padding/margin with "All" computed shortcut, collapsible accordion (ported from Alpine `spacingControls`)
- Created `QuerySettings.vue` — limit, direction, order_by, include/exclude tags per collection handle
- Added both to InspectorPanel.vue Appearance tab

### Phase 4 — Universal appearance controls and store extensions

- Created `WidgetAppearanceControls.vue` — full-width toggle, background colour, text colour at top of Appearance tab
- Universal controls auto-hide when the widget's own config_schema already defines the same field (prevents duplicates)
- Defaults populated on mount: `#ffffff` background, `#000000` text
- Added `updateLocalStyleConfig` and `updateLocalQueryConfig` store actions with debounced save
- Added `background-color` and `color` to `buildInlineStyles` (both API controller and Livewire) so style_config colors render in preview
- Added `style_config.full_width` check alongside `config.full_width` in preview rendering
- Removed redundant `background_color`/`text_color` from events_listing and blog_listing config_schema
- Removed redundant `background_color`/`text_color`/`full_width` from social_sharing config_schema
- Removed redundant `full_width` from map_embed config_schema
- Renamed carousel `link_color`/`text_color` to `slide_link_color`/`slide_text_color` to avoid confusion with universal controls
- New widgets created with `style_config` defaults (`background_color: #ffffff`, `text_color: #000000`)

### Additional work — Widget defaults and error states

- Set explicit color defaults on 5 fields across bar_chart, carousel, social_sharing
- Set explicit color defaults on hero `background_color` (#ffffff) and `text_color` (#000000)
- Hero template: overlay only renders when background image/video is present (not background color alone)
- Hero template: removed `hero-content--on-image` CSS class that silently overrode text color
- Removed `.hero-content--on-image` CSS rule from `_custom.scss`
- Created shared `widget-placeholder` partial for unconfigured widgets
- Added placeholder error states to 8 widgets: Board Members, Logo Garden, Carousel, Event Description, Event Registration, Map Embed, Video Embed, Web Form
- Fixed `refreshPreview` to flush pending debounced saves before fetching preview from server
- Fixed `filesystems.disks.public.throw` from `false` to `true` — was causing Spatie media library to silently fail on file uploads
- Fixed 3 logo garden tests to use `Storage::fake('public')`
