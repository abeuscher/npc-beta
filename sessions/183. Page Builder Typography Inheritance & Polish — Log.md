## What was built

### Phase 1 — Inject theme typography into the page-builder preview

- New `TypographyCompiler::compileScoped(array $scopes, ?array $typography)` — same structured rules `compile()` produces, with each selector prefixed by the supplied scopes and grouped into a single rule. No user-CSS passthrough; reuses the same compiler the public layout uses.
- `resources/views/livewire/page-builder.blade.php` now resolves typography via `TypographyResolver::load()`, emits the Google Fonts preconnect + stylesheet, sets `--font-family-heading` / `--font-family-body` CSS vars on the two scopes, and renders the scoped `<style>` block.
- **Scoping approach chosen:** scoped-class prefix (not iframe / shadow-DOM). Both `.widget-preview-scope` (preview canvas) and `.ql-editor` (Quill rich-text) live under the same `.page-builder` Livewire root, so one pair of prefixes covers both. Filament chrome, inspector panel, and `.ql-toolbar` are unaffected. The existing `:not(nav ...)` exclusion stays — preview canvas does not render a `<nav>` element.

### Phase 2 — Quill editor inherits typography

- The same scoped CSS includes `.page-builder .ql-editor` as a second prefix, so headings/paragraphs/lists inside Quill render in the theme's fonts/sizes/spacing while the user is typing. `.ql-toolbar` retains its admin-chrome styling.

### Phase 3 — Theme editor UI polish

- Inherit button ("Use Heading" / "Use Body") moved out of `FontInput` into the right side of each element header in [TypographyPanel.vue](resources/js/theme-editor/TypographyPanel.vue). Visible whenever a bucket family is set; disabled when already matching; `@click.stop` so it doesn't toggle the drawer.
- [FontInput.vue](resources/js/page-builder-vue/components/primitives/FontInput.vue) reduced to a single 12-col grid with all eight fields on one row: Family(3) / Weight(2) / Case(2) / Size(1) / Unit(1) / Line ht.(1) / Letter sp.(1) / Unit(1). Inherit button props removed; component is now Theme-Editor-only and has no other consumers.
- Margin and Padding now sit side-by-side in `.theme-typography__spacing-row` (2-col grid) below FontInput.
- [SpacingInput.vue](resources/js/page-builder-vue/components/primitives/SpacingInput.vue) compacted: `minmax(0, 1fr)` tracks, hidden number-spinner buttons (Firefox + WebKit), `box-sizing: border-box` everywhere so two SpacingInputs fit in a half-width column without overflow.

### Side-channel work — preset / defaults bug fixes & the "concrete values" rule

Discovered mid-session, fixed before Phase 3:

- **Race condition** in the page-builder store: `saveDraftPreset` and "Copy as defaults()" hit the server while debounced spacing edits were still queued, so the server read stale `appearance_config`. Extracted `flushPendingSaves()` from `refreshPreview` and called it at the top of both flows.
- **Full wiring for `defaultAppearanceConfig()`** (option 2 from the design discussion): new method on `WidgetDefinition` returning a concrete zero-state shape (background `#ffffff`, text `#000000`, `layout.full_width: false`, padding/margin all sides = `0`). `PageBuilder::createBlock` seeds new widgets from `WidgetRegistry::find($handle)?->defaultAppearanceConfig()` instead of a hardcoded inline stub. `WidgetDefaultsController::export` now emits **two** methods to the clipboard — `defaults()` and `defaultAppearanceConfig()` — with the widget's current appearance deep-merged onto the type's concrete defaults. The deep-merge skips `null`/`''` overrides and coerces numeric strings to `int` when the base value is `int`.
- **Concrete values throughout the spacing pipeline:** `SpacingInput` now emits a `SpacingValue` of pure numbers (clearing a field produces `0`, not `''`/`null`). `SectionLayoutPanel.applySpacing` passes the numeric value through unchanged. `AppearanceStyleComposer` treats concrete `0` as "no inline override" so existing widget SCSS wrapper-padding (BlogListing, ProductCarousel, etc.) keeps rendering unchanged.
- **Rule recorded as "concrete":** the user wanted a single word besides "reactive" to invoke this principle in future sessions. Saved to `~/.claude/projects/-home-al-nonprofitcrm/memory/feedback_concrete_values.md` and indexed in `MEMORY.md`. Rule: every declared field carries a concrete value; no null/empty/missing as proxies for "default" or "unset."
