## What was built

### Iteration 1 ‚Äî Organizations importer auto-mapping helper

- Extracted `ImportOrganizationsPage::guessDestination()`'s inline `match()` into a dedicated `app/Services/Import/OrganizationFieldMapper.php`, modeled on the five sibling mappers (`FieldMapper` + Event/Note/Donation/Invoice/Membership).
- The mapper accepts the `?string $preset` the import wizard already passes via the `InteractsWithImportWizard` concern. The inline version declared no `$preset` parameter, so PHP silently dropped it ‚Äî Organizations was preset-blind (no Wild Apricot / Bloomerang support). The new mapper carries `generic` / `wild_apricot` / `bloomerang` preset maps like its siblings.
- Generic preset preserves every prior inline mapping and adds the standard spacing/underscore header variants plus `industry` and `ein` aliases (both valid `Organization` destinations the inline match omitted).
- `tests/Feature/OrganizationImportFieldMapperTest.php` ‚Äî 11 tests mirroring the sibling test shape.

### Iteration 2 ‚Äî Export-artifact TTL sweeper (`exports:clean`)

- New `app/Console/Commands/CleanExportBundlesCommand.php` (`exports:clean {--hours=48}`) reaps `exports/bundles/{token}/` directories on the local disk older than the TTL, and reaps empty token dirs. Scheduled daily at 02:00 in `bootstrap/app.php` alongside `backup:clean` / `media-library:clean`.
- Paired inbox item (downloaded-backup cleanup) **resolved by verification, no code:** the Fleet `BackupController::blob()` endpoint streams the newest backup straight off the backup disk (no server-side copy on download), and those backups are already reaped by `backup:clean`. Nothing accumulates; the Fleet controller was not touched (session is non-boundary-touching).
- `tests/Feature/CleanExportBundlesCommandTest.php` ‚Äî 4 tests (stale reaped, fresh kept, empty dir reaped, no-op when dir absent).

### Iteration 3 ‚Äî Rich-text toolbar button accessibility

- New `resources/js/admin/quill-toolbar-titles.js` (`applyToolbarTitles`) sets `title` + `aria-label` on each Quill toolbar control. Called after init in both `resources/js/admin/quill-editor.js` (admin Filament editor) and `resources/js/page-builder-vue/components/fields/RichTextField.vue` (inspector), which previously rendered icon-only controls with no accessible name or tooltip.
- `InlineFormatToolbar.vue` already carried reactive `aria-label`s; an `onMounted` block mirrors each into `title` for hover tooltips without disturbing the screen-reader labels.
- JS/template-only (runtime DOM attributes; no Pest seam). `npm run build` clean.

### Iteration 4 ‚Äî Ellipsis secondary-actions menu on the page-builder editor

- Extracted the page-details secondary-actions `ActionGroup` (Export Page / with media / with theme / with theme & media, Save Block Layout as Template, Edit Header & Footer Snippets, Delete) into `app/Filament/Resources/PageResource/Concerns/HasPageSecondaryActions.php`.
- Both `EditPageDetails` and the page-builder `EditPage` use the trait, so the menu is identical and can't drift. Operators no longer have to navigate to the detail view to reach those actions.
- Page-scoped per the inbox item; the parallel Post resource was left untouched.
- `tests/Feature/Session167Test.php` ‚Äî +2 tests (actions surface on `EditPage`; delete hidden for system pages on `EditPage`).

### Iteration 5 ‚Äî Column-layout `background_full_width` parity

- Widgets default to a full-bleed background (`widget_types` ships bg=true/content=false) but column layouts fell back to bg=false. Flipped the `AppearanceStyleComposer::resolveFullWidthForLayout` fallback to `true` (covers legacy layouts lacking the key) and, per the concrete-values rule, wrote concrete `background_full_width=true` / `content_full_width=false` at layout creation in both `PageBuilderApiController::storeLayout` and `RecordDetailViewBuilderApiController` (the shared resolver also serves record-detail views).
- Behavioural note: existing column layouts with no explicit full-width key now render full-bleed, matching widget behaviour.
- `tests/Feature/AppearanceBackgroundRenderTest.php` (+2, in the `design` group) and `tests/Feature/PageBuilderApiTest.php` (+1 assertion-set on the create path).
