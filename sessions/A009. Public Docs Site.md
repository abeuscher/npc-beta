# A009. Public Docs Site — /docs + Agent-Readable Markdown + AEO Wiring

> **Agentic session.** Expose the existing in-app help corpus (62 repo-canonical markdown articles in `resources/docs/`, already synced into `help_articles` via `help:sync`) on the public website at `/docs`, gated to the marketing instance only by a new `PUBLIC_WEBSITE` env flag. Every article serves twice — rendered HTML at `/docs/{slug}` and raw markdown at the sibling URL `/docs/{slug}.md` — and the existing AEO surfaces (`/llms.txt`, `/sitemap.xml`) extend to index it, plus a new `/llms-full.txt` whole-corpus fetch. One deploy-path fix rides along: `help:sync` joins `widgets:sync` in both deploy workflows so doc edits actually reach deployed servers.

One sentence: a visitor (or an AI crawler) on the marketing site gets a full, fresh, extraction-friendly docs library generated from the same repo markdown the in-app help system already renders — and no other instance of the product exposes any of it.

---

## Why this is agent-friendly

- **The hard half already exists.** Corpus, frontmatter convention, DB tables (`help_articles` + `help_article_routes`), sync command, markdown rendering path (`Str::markdown()` / league/commonmark), categories, hierarchy (`parent_slug`), related-articles query — all built and proven by the admin help system. This session adds a public *read surface* over existing rows, not a content system.
- **Acceptance is binary and self-verifiable in-container.** Routes return 200 with the flag on / 404 with it off; `.md` siblings carry `text/markdown`; `llms.txt` and `sitemap.xml` grow docs sections only when flagged. All Pest-assertable; no live-external dependency, no human visual judgment required to close (a styling polish pass can follow in a numeric session if the owner wants one).
- **No schema, no migration, no contract change.** Non-boundary; CRM contract stays v2.3.0. The only DB interaction is reading rows `help:sync` already writes.
- **Narrow touch surface.** One new controller (+ the two existing SEO controllers extended), routes, Blade views, one config key + helper, two workflow files, and a small link-normalization pass over `resources/docs/*.md`. The hard-no list keeps everything else out.

---

## Settled design decisions (locked at the planning session — do not relitigate)

1. **`/docs` is a hardcoded reserved prefix.** Registered in `routes/web.php` *before* the page-slug catchall (named routes win), like `/sitemap.xml`, `/robots.txt`, `/llms.txt`. **Not** a configurable `SiteSetting` prefix, **not** a `config/site.php` prefix entry. This feature is built for one site — the owner's.
2. **The gate is a dedicated env flag, not `APP_ENV` and not a SiteSetting.** `PUBLIC_WEBSITE=true` in the marketing node's `.env` (FM provisions it, same channel as `APP_ENV=demo` on the demo node). Read through config — `config('site.public_website')`, `env()` only inside `config/site.php` — so it survives `config:cache`. Helper `isPublicWebsite(): bool` in `app/helpers.php`, sibling to `isDemoMode()`. The marketing node stays `APP_ENV=production`; "public website" is an orthogonal property layered on a production node, not a replacement environment. Default `false` everywhere.
3. **Serve the markdown twice.** HTML at `/docs/{slug}`, raw markdown at `/docs/{slug}.md` (sibling *URLs*, dynamic from the DB — no emitted files, no static-site generator). The `.md` response is `text/markdown; charset=utf-8`.
4. **All synced articles are public in v1.** No per-article `public: false` opt-out (it would need a schema column). If a future article must stay internal, that's a deferred follow-up — note it in the log, do not build it.
5. **`help:sync` joins the deploy path.** Same gap-class as the session-341 `widgets:sync` fix: today doc edits never reach a deployed server unless someone runs `help:sync` by hand. Add it immediately after `widgets:sync` in both `.github/workflows/deploy.yml` and `.github/workflows/deploy-demo.yml`.

## Scope, per phase

### Phase 0 — Stack bring-up (cloud session prerequisite)

`vendor/` and `node_modules/` clone empty in the cloud container. Bring the stack up before any work: `docker compose up -d`, `composer install` (in-container via `./dev composer install`), `npm install`, `./dev artisan migrate:fresh --seed`, then `./dev artisan help:sync` so `help_articles` is populated with the real 62-article corpus to develop against.

### Phase 1 — Flag + helper

- `config/site.php` — new key: `'public_website' => env('PUBLIC_WEBSITE', false),` (plain `env()` read; no SiteSetting closure — this is deliberately *not* operator-configurable).
- `app/helpers.php` — add `isPublicWebsite(): bool` returning `(bool) config('site.public_website')`, mirroring `isDemoMode()`'s shape.
- `.env.example` — add `PUBLIC_WEBSITE=false` with a one-line comment ("marketing-site instance only — exposes /docs + AEO surfaces").

**Test note:** Pest exercises both states via `config(['site.public_website' => true|false])` — no environment juggling.

### Phase 2 — Public docs routes + controller + views

**Routes** (`routes/web.php`, registered with the other SEO/utility routes, before the catchall):

- `GET /docs` → `DocsController@index`
- `GET /docs/{slug}.md` → `DocsController@raw` (register **before** the HTML route, or constrain the HTML route's `{slug}` to exclude `.md` — verify Laravel's matching order handles the extension correctly and pick the unambiguous registration)
- `GET /docs/{slug}` → `DocsController@show`
- Every action opens with `abort_unless(isPublicWebsite(), 404);`

**Controller** — new `app/Http/Controllers/DocsController.php`:

- `index()` — all `HelpArticle` rows ordered by category then title, grouped by category for the view (category labels: crm / cms / finance / tools / settings / general — reuse whatever label map the admin `HelpIndexPage` uses).
- `show($slug)` — `HelpArticle::where('slug', $slug)->firstOrFail()`; build breadcrumb chain from `parent_slug` (the model's `parent()` accessor exists); pull related articles via the same tag-intersection query `HelpArticlePage` uses (extract to `HelpArticleService` if the duplication is non-trivial; copy if it's a few lines — judgment call, note it).
- `raw($slug)` — respond with the article's raw `content` column prefixed by a minimal plain-markdown header block: H1 title, the `description` line, `Last updated: {last_updated}`, and a `Canonical: {base_url}/docs/{slug}` line (read `base_url` from `SiteSetting` with the `config('app.url')` fallback, exactly as `LlmsTxtController` does). `Content-Type: text/markdown; charset=utf-8`.

**Views** — render inside the public site chrome so docs look like part of the site. **Mirror how `PostController` renders blog posts** (non-Page content inside the public layout + `ChromeRenderer` header/footer) — read it first and follow its shape exactly, including how it passes nav/chrome blocks and what the public layout expects when `$page` is absent (the global-`noindex` read in `layouts/public.blade.php` is independent of `$page`; verify nothing else NPEs).

- `resources/views/docs/index.blade.php` — docs landing: site chrome, intro line, category-grouped article list with titles + descriptions.
- `resources/views/docs/show.blade.php` — article page, **extraction-structured** per the AEO shape: breadcrumbs (Docs → parent → article), `<h1>` = article title, the frontmatter `description` as a one-sentence lede directly under the H1, visible "Last updated {date}", then the rendered markdown body (`Str::markdown()` — same call the admin surfaces use), then related-articles links, and a cross-link to `/docs` index. `<title>` pattern: `{Title} — {Site Name} Docs`. A "View as Markdown" link to the `.md` sibling is cheap and agent-friendly — include it.
- Styling: lightweight. If a small SCSS addition is needed for readable docs typography, add it to the public SCSS surface and run `npm run build` per the build decision rule. Do not touch widget SCSS or `build:public`.

### Phase 3 — AEO wiring (all conditional on the flag)

- **`LlmsTxtController`** — when `isPublicWebsite()`, append a `## Documentation` section after `## Pages`: one bullet per article, **linking the `.md` sibling URL** (`{base_url}/docs/{slug}.md`), title as link text, description after an em-dash. Keep the existing 1-hour cache (the cache key can stay; content is flag-stable per instance).
- **`SitemapController::buildSitemap()`** — when `isPublicWebsite()`, append `/docs` and every `/docs/{slug}` URL with `<lastmod>` from `last_updated` (fall back to `updated_at`). Same W3C date format the page entries use.
- **New `GET /llms-full.txt`** — flag-gated route (404 off-flag): the entire corpus concatenated — a site-identity header (same site name/description block `llms.txt` opens with), then each article as `# {title}` + canonical-URL line + body, separated by `---`. `Cache::remember` 1 hour, `text/plain; charset=utf-8`. Add it as a `LlmsTxtController@full` method (sibling to `index`) rather than a new controller.
- **robots.txt** — verify only: it already serves `Allow: /` for all user agents (no AI-crawler blocks) and references the sitemap. No change expected; assert in a test that the docs-enabled instance's robots output stays permissive.

### Phase 4 — Deploy-path fix + corpus link hygiene

- **`.github/workflows/deploy.yml` + `.github/workflows/deploy-demo.yml`** — add `docker compose -f docker-compose.prod.yml exec -T app php artisan help:sync` on the line immediately after the existing `widgets:sync` step in each.
- **Link normalization in `resources/docs/*.md`** (content-only edits; do not touch `HelpSync` parsing):
  - `](/admin/help/{slug})` absolute links → bare relative slug `]({slug})` (resolves correctly under both `/admin/help/` and `/docs/` because both surfaces serve articles at sibling URLs).
  - `]({slug}.md)` links → strip the `.md` (otherwise the public link lands on the raw-markdown sibling).
  - `](../../docs/schema/*.md)` repo-relative links → rewrite as plain text (the schema docs are not in the help corpus; don't link to nothing).
  - `](https://url)` placeholder links → fix or remove (judgment by surrounding prose; note each in the log).
  - Bare relative slug links (`](import-notes)`) are already correct on both surfaces — leave them.
  - Run `./dev artisan help:sync` after the edits and confirm Synced count is 62 / Skipped 0.

### Phase 5 — Tests (the close gate)

New Pest feature file(s), e.g. `tests/Feature/PublicDocsSiteTest.php` (+ a sitemap/llms addition to existing SEO tests if files for those exist — check first and follow convention):

1. **Gating:** with `config(['site.public_website' => false])` (the default), `/docs`, `/docs/{slug}`, `/docs/{slug}.md`, `/llms-full.txt` all return **404**. With it `true`, all return **200**.
2. **HTML article shape:** seeded/synced article renders with H1 = title, description present, last-updated visible, breadcrumb for a `parent_slug` article, `<title>` matches `{Title} — {Site Name} Docs`.
3. **Markdown sibling:** `Content-Type` is `text/markdown; charset=utf-8`; body contains the raw markdown (assert a known heading/fence survives unrendered) + the canonical-URL header line.
4. **llms.txt:** flag on → `## Documentation` section present with `.md` URLs; flag off → section absent (and the page-index behavior unchanged).
5. **sitemap.xml:** flag on → `/docs/{slug}` `<loc>` entries with `<lastmod>`; flag off → no docs entries.
6. **llms-full.txt:** flag on → contains at least two article titles and separators; flag off → 404.
7. **Catchall non-regression:** an existing published Page still renders at its slug; a Page cannot shadow `/docs` (route order).
8. **404 path:** `/docs/{nonexistent-slug}` → 404 even with the flag on.

Seed strategy: tests may either call `help:sync` against a temp fixture set or create `HelpArticle` rows via factory/`create()` directly — prefer direct row creation for speed (the sync command has its own existing behavior and isn't under test); add a `HelpArticle` factory if none exists.

Close-gate: full fast suite `./dev test` green (baseline 2812/0 at the 348 close; expect roughly +10–15). No Playwright spec — every acceptance signal here is server-rendered output Pest sees directly; a one-off browser spec would add nothing (per the Playwright-discipline heuristic).

## Hard-no list

- **No schema change, no migration.** The 348 squash baseline stands.
- **No Fleet Manager surface.** Nothing under `app/Http/Controllers/Api/Fleet/`, no contract-doc edit, contract stays v2.3.0. (FM-side will eventually set `PUBLIC_WEBSITE=true` when provisioning the marketing node — that is an FM-repo `.env`-template concern, not a contract surface; note it in the log's handoff line.)
- **No admin help-system changes.** `HelpSync` command parsing, `HelpArticleService::forRoute`, the `?` slide-over, `HelpSearch`, and the three Filament help pages are all out of scope (extracting the related-articles query into the service is the one sanctioned touch, and only if `show()` would otherwise duplicate it non-trivially).
- **No SiteSetting, no admin toggle, no configurable prefix** for any of this. Env-flag only, hardcoded `/docs`.
- **No widget / page-builder / Quill / SCSS-pipeline changes.** `build:public` should not need to run.
- **No JSON-LD / TechArticle schema** in v1 (deferred nice-to-have). No docs search UI. No marketing-page → docs cross-link content edits (that's CMS content work the owner does in the page builder, not repo work).
- **No content rewriting** of the 62 articles beyond the Phase-4 link normalization.

## Acceptance criteria

1. On a flag-off instance (the default, i.e. every customer/demo node): `/docs*`, `/llms-full.txt` are 404; `llms.txt` and `sitemap.xml` are byte-identical in behavior to today.
2. On the flag-on instance: `/docs` lists all 62 articles grouped by category; every article serves at both `/docs/{slug}` (HTML, in site chrome, extraction-structured) and `/docs/{slug}.md` (raw markdown + canonical header); `llms.txt` indexes the `.md` siblings; `sitemap.xml` carries the docs URLs; `/llms-full.txt` serves the concatenated corpus.
3. Both deploy workflows run `help:sync` after `widgets:sync`.
4. The corpus contains no `/admin/help/` links, no repo-relative links, no `.md`-suffixed internal links; `help:sync` reports 62 synced / 0 skipped.
5. Fast Pest suite green; new coverage per Phase 5.
