## Session-specific deltas

What the track doc defers to session time, plus concrete anchors mapped at A012:

- **The `SUSPENSION_STATE` flag.** Values `none` / `admin_locked` / `site_off`; absent = `none`. Read through a new `suspension` entry in `config/fleet.php` (`fleet.suspension.state` ← `env('SUSPENSION_STATE', 'none')`), beside the existing agent config — same env-derived grain as demo mode (`APP_ENV=demo` / `isDemoMode()`). An unrecognized value fails safe: treat as `none`, log a warning (a typo in a pushed key must not brick a paying client's admin).
- **The enforcement middleware** (one class, lean name like `EnforceSuspensionState`):
  - `admin_locked` → every admin-panel surface (Filament panel including login, plus the in-panel API route groups — `routes/admin-api.php` and siblings) renders a suspension notice instead. The notice carries reason-appropriate copy (reason codes: `delinquent` / `trial_expired` / `canceled` / `manual`, read from the billing-state document), the Stripe-hosted portal login link, and the billing contact on file — the lock screen is itself the path to self-cure. **Document missing → still locks**, generic copy. Public pages, donation/event/membership checkout, the member portal, backups, the scheduler, and all five FM `/api/*` endpoints are untouched.
  - `site_off` → all public routes render a static maintenance notice (503). FM `/api/*` endpoints stay up (a shut-off node is still monitored and recoverable). Manual-only state; the node just obeys the flag.
  - Registration: the Filament panel middleware stack (`AdminPanelProvider`) covers the panel + login; confirm at implementation time whether the admin-api groups ride the panel stack or need explicit registration — behavior, not wiring, is the requirement.
  - Orthogonal to demo mode — no `isDemoMode()` interplay (demo/internal nodes simply never get a suspension push).
- **The billing-state reader** (new, small, under `app/Services/Billing/`): reads `storage/app/private/fleet/billing-state.json`, validates the schema version, exposes typed accessors + `asOf()`, returns a null-object when the file is missing or malformed (malformed also logs). Display-only by contract — nothing it returns may influence enforcement. Cached per-request only (the file changes rarely; no cross-request cache to invalidate on push). Consumers this session: the lock-screen copy and the subcheck; CB2's Account page consumes it next.
- **The document schema** (contract text this session; FM starts writing it at FM-B2): schema version, `as_of`, plan (name / amount / currency / interval), status, next invoice (date / amount / line items), billing contact email, portal login URL, suspension (state / reason / since / grace-ends), trial (ends-at). Pushed over FM's existing SSH provisioning channel, atomic temp + rename. The path is deliberately inside `storage/app/private/fleet/` — excluded from backup blobs (same rationale as the backup success-record: per-node metadata must never restore onto another node) and untouched by `demo:restore`. Verify the exclusion holds and state it in the contract text.
- **The `suspension` subcheck** on `/api/health`: value `{state, billing_state_as_of}` — the *enforced* state from config plus the document's `as_of` (`null` when no document). Status `green` when `none`, `yellow` otherwise (an FM-side attention chip), **never red, excluded from the worst-of overall status** — a deliberately suspended node is not an unhealthy node. Mirrors the `data_hygiene` precedent exactly, including the worst-of-rule carve-out text. No email, no amounts — a state string and a timestamp, inside the contract's no-PII wire discipline. This is FM's read-back verification that a push took effect.
- **Contract doc revision:** new sections for the document (a pushed-artifact section in the § Demo-node-reset-coordination style), the flag, and the subcheck; subcheck table row; worst-of rule update; CHANGELOG entry; `Contract Version` → 2.6.0; `HealthController::CONTRACT_VERSION` via the shared trait (BackupController follows automatically). Cross-Repo block in `sessions/session-outlines.md`: version 2.6.0, last boundary-touching 366, FM absorption pending at FM-B2.
