# 096. Per-Page SEO & Header Snippets

Wire up per-page SEO meta fields, auto-generate JSON-LD structured data, add per-page and site-wide header/footer code snippet injection, auto-generate canonical URLs and a sitemap, and add a per-page noindex toggle.

---

## Design decisions (resolved before starting)

- `meta_title` and `meta_description` columns already exist on the `pages` table — wire them into the edit form and public layout. No migration needed for those two.
- New columns required on `pages`: `og_image_path` (string, nullable), `head_snippet` (text, nullable), `body_snippet` (text, nullable), `noindex` (boolean, default false).
- Site-wide snippets are stored as `SiteSetting` keys: `site_head_snippet`, `site_body_open_snippet`, `site_body_snippet`, `site_default_og_image`.
- OG image field: a simple text/URL input for now (path or external URL). Not a Spatie Media Library upload — keep it lightweight.
- Snippet code fields: use Filament `Textarea` with monospace font styling (`font-family: monospace; font-size: 0.85rem`). Do not install a code editor package.
- Snippet HTML validation: use PHP `DOMDocument::loadHTML()` wrapped in a custom validation rule. Catches malformed tags; does not lint JS inside `<script>` tags.
- Per-page snippet UI: use Filament `Actions\ActionGroup` (ellipsis menu) in the Edit Page header actions, positioned after the Delete button.
- New Spatie permission: `edit_page_snippets`. Not assigned to any role by default — power-user feature. Register in `PermissionSeeder`.
- JSON-LD is fully auto-generated from existing page data. No user-facing fields. Logic lives in a dedicated service class (`App\Services\SeoMetaGenerator` or similar), not inline in Blade.
- JSON-LD type mapping: `post` → `BlogPosting`, `event` → `Event`, all others → `WebPage`.
- No Twitter/X card meta tags.
- Canonical URLs are self-referencing, auto-generated from `base_url` + slug. No manual override field.
- CMS Settings page route for the modal info link: `CmsSettingsPage::getUrl()` (Filament auto-registers the route).
- Existing tests that touch the public layout and may need updating: `PageTest`, `PostTest`.

## Phase 1 — SEO meta fields on Edit Page

Wire up the existing `meta_title` and `meta_description` columns plus the new `og_image_path` and `noindex` columns in the page edit form.

- Add a "SEO" fieldset to the `PageResource` form (below the main content area) with:
  - `meta_title` — TextInput, placeholder showing the page title as fallback
  - `meta_description` — Textarea (~3 rows), placeholder indicating auto-extraction fallback
  - `og_image_path` — TextInput, helper text: "URL or path to the Open Graph image for social sharing"
  - `noindex` — Toggle, label: "Hide from search engines", helper text: "Adds a noindex meta tag. Use for thank-you pages, confirmation pages, etc."
- Migration: add `og_image_path`, `noindex`, `head_snippet`, `body_snippet` columns to `pages`.

## Phase 2 — Per-page header & footer snippets

- In `EditPage.php`, add an `Actions\ActionGroup` (ellipsis/dropdown menu) after the Delete action containing an action labelled "Edit Header & Footer Snippets".
- The action opens a modal with:
  - An info block above the fields: *"This control is for per-page code snippets only. If you are trying to install Google Tag Manager or any other site-wide scripts, please use the Site Header and Site Footer fields on the [CMS Settings Page](link)."* — make "CMS Settings Page" a clickable link to `CmsSettingsPage::getUrl()`.
  - `head_snippet` — Textarea, monospace, ~4 rows, label: "Head snippet (before `</head>`)"
  - `body_snippet` — Textarea, monospace, ~4 rows, label: "Body snippet (before `</body>`)"
- Validate both fields on save using `DOMDocument::loadHTML()` — reject if the HTML cannot be parsed.
- Gate the action behind the new `edit_page_snippets` permission.
- Register `edit_page_snippets` in `PermissionSeeder`. Do not assign it to any role by default.

---

## Phase 3 — Site-wide snippets in CMS Settings

- Add a new fieldset **"Header & Footer Code Snippets"** in `CmsSettingsPage`, positioned above the Image Sizes fieldset.
- Three fields:
  - `site_head_snippet` — Textarea, monospace, ~4 rows, label: "Site head snippet (before `</head>`)"
  - `site_body_open_snippet` — Textarea, monospace, ~4 rows, label: "Snippet below `<body>` tag" — helper text: "Use this for Google Tag Manager, Google Analytics, or similar scripts that must load immediately after the body tag opens."
  - `site_body_snippet` — Textarea, monospace, ~4 rows, label: "Site body snippet (before `</body>`)"
- Add a fourth field for the default fallback OG image:
  - `site_default_og_image` — TextInput, label: "Default Open Graph image", helper text: "Fallback image used when a page has no OG image set. URL or path."
- Same HTML validation rule on the three snippet fields.
- These are already gated behind `super_admin` (CMS Settings access control).

## Phase 4 — Public layout rendering

Update `resources/views/layouts/public.blade.php` to inject all SEO and snippet output.

**In `<head>`, just before `</head>`:**
1. `<title>` — use `meta_title` if set, else page `title`, else site name
2. `<meta name="description">` — use `meta_description` if set, else auto-extract first ~160 characters of visible widget text content, else site description from settings
3. `<meta property="og:title">` — same source as `<title>`
4. `<meta property="og:description">` — same source as description
5. `<meta property="og:image">` — use `og_image_path` if set, else first image from page widgets, else `site_default_og_image` from settings
6. `<meta property="og:url">` — canonical URL
7. `<meta property="og:type">` — `article` for posts, `website` for everything else
8. `<link rel="canonical">` — `base_url` + slug (self-referencing)
9. `<meta name="robots" content="noindex">` — only if `noindex` is true
10. JSON-LD `<script type="application/ld+json">` block (see Phase 5)
11. Site-wide head snippet (raw output)
12. Per-page head snippet (raw output)

**Immediately after `<body>` tag:**
13. Site-wide body-open snippet (raw output)

**Just before `</body>`:**
14. Site-wide body snippet (raw output)
15. Per-page body snippet (raw output)

## Phase 5 — JSON-LD auto-generation

Create `App\Services\SeoMetaGenerator` (or add to an existing service if appropriate):

- Method to generate JSON-LD array for a given Page model
- Type mapping:
  - `type = 'post'` → `BlogPosting` with `headline`, `description`, `image`, `datePublished`, `dateModified`, `author` (from `author_id` → User name), `publisher` (org name from `site_name` setting)
  - `type = 'event'` → `Event` with `name`, `description`, `image`, `startDate` (if available from linked event), `organizer`
  - All others → `WebPage` with `name`, `description`, `image`, `dateModified`
- Method to extract fallback meta description from page widget content (first ~160 chars of visible text)
- Method to find first image URL from page widgets (for OG image fallback)
- Output as JSON string for embedding in `<script type="application/ld+json">`

## Phase 6 — Sitemap and robots.txt

- Add a route at `/sitemap.xml` that returns an XML sitemap of all published pages where `noindex` is false, excluding `type = 'system'` and `type = 'member'`. Include `<lastmod>` from `updated_at`.
- Cache the sitemap output; invalidate on page publish/unpublish/update (use a simple cache key with a model observer or event listener).
- Add a route at `/robots.txt` that serves:
  ```
  User-agent: *
  Allow: /
  Sitemap: {base_url}/sitemap.xml
  ```

## Security checklist

Before closing:

- [ ] Snippets are rendered with `{!! !!}` (raw) — confirm this is intentional and that only users with `edit_page_snippets` (per-page) or `super_admin` (site-wide) can set snippet content.
- [ ] `noindex` toggle does not leak through to pages where it shouldn't — default is false.
- [ ] Sitemap excludes system and member pages (portal URLs must not be indexed).
- [ ] JSON-LD does not expose any PII (author name from `users` table is acceptable; no email or contact data).
