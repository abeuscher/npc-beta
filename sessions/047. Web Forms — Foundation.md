# 047. Web Forms ‚Äî Foundation

Build the foundation of the web forms system. Forms are defined as JSON field definitions stored in the database, rendered on the public site as Pico-styled HTML, and submitted to a dedicated endpoint that stores responses in a `form_submissions` table. This session covers the data model, the admin CRUD UI with a Filament Repeater-based field builder, public rendering, submission handling with honeypot spam protection, and a read-only submissions viewer in the admin. Actions (email notifications, contact creation) are deferred to session 048.

## Data model

### `forms` table

| Column | Type | Notes |
|---|---|---|
| `id` | bigint PK | |
| `title` | string | Admin-facing label |
| `handle` | string unique | Used to embed a form: `<x-public-form handle="contact">` |
| `description` | text nullable | Admin notes only, not shown publicly |
| `fields` | json | Array of field definition objects (see below) |
| `settings` | json | Form-level settings (see below) |
| `is_active` | boolean default true | Inactive forms return a 404 on submission |
| `timestamps` | | |

**Field definition object:**
```json
{
    "handle": "first_name",
    "type": "text",
    "label": "First Name",
    "placeholder": "",
    "required": true,
    "width": 6,
    "validation": "none",
    "validation_regex": null,
    "validation_message": null,
    "options": []
}
```

Field types: `text`, `email`, `tel`, `number`, `textarea`, `select`, `radio`, `checkbox`, `state`, `country`, `hidden`

Validation presets: `none`, `email`, `phone`, `zip`, `url`, `numbers_only`, `letters_only`, `custom_regex`

`options` is used only by `select` and `radio` ‚Äî array of `{ "value": "...", "label": "..." }` objects.

`hidden` fields have a `default_value` key instead of `placeholder` ‚Äî this value is written to the submission silently.

**`settings` object:**
```json
{
    "submit_label": "Submit",
    "success_message": "Thank you. Your message has been received.",
    "honeypot": true
}
```

### `form_submissions` table

| Column | Type | Notes |
|---|---|---|
| `id` | bigint PK | |
| `form_id` | bigint FK ‚Üí forms | |
| `data` | json | Key/value map of handle ‚Üí submitted value |
| `ip_address` | string nullable | |
| `created_at` | timestamp | No `updated_at` ‚Äî submissions are immutable |

No `updated_at` column ‚Äî add `public $timestamps = false` and set `created_at` manually, or use `const UPDATED_AT = null`.

## Part 1 ‚Äî Migration and models

### Migration

Create `forms` and `form_submissions` tables as described above. The `forms.fields` and `forms.settings` columns cast as `array`. The `form_submissions.data` column casts as `array`.

### `app/Models/Form.php`

Standard Eloquent model. Cast `fields` and `settings` as `array`. Has many `FormSubmission`. Add a `fieldValidationRules()` method that iterates `fields` and returns a Laravel validation rules array ‚Äî used by both the submission controller and the Filament preview. Validation preset map:

```php
'none'         => [],
'email'        => ['email'],
'phone'        => ['regex:/^[\d\s\-\(\)\+]{7,15}$/'],
'zip'          => ['regex:/^\d{5}(-\d{4})?$/'],
'url'          => ['url'],
'numbers_only' => ['regex:/^\d+$/'],
'letters_only' => ['regex:/^[a-zA-Z\s]+$/'],
'custom_regex' => [], // handled separately using validation_regex value
```

When `validation` is `custom_regex` and `validation_regex` is non-empty, add `regex:{value}` to the rules. Wrap in a try/catch ‚Äî if the regex is malformed, skip it silently (do not 500).

### `app/Models/FormSubmission.php`

Cast `data` as `array`. Belongs to `Form`. Set `const UPDATED_AT = null`.

## Part 3 ‚Äî Admin: Form Resource

### `app/Filament/Resources/FormResource.php`

Navigation: group `CMS`, label `Forms`, icon `heroicon-o-document-text`, sort `5`.

`canAccess()` requires `view_any_form`.

**Form schema** uses a Filament `Repeater` for the fields array. Each repeater item has:

- `Select` for `type` ‚Äî all 11 types listed above, `->live()` so dependent fields show/hide
- `TextInput` for `label` ‚Äî required
- `TextInput` for `handle` ‚Äî required, auto-populated from label using `->afterStateUpdated()` to slugify (lowercase, underscores). Editable.
- `TextInput` for `placeholder` ‚Äî hidden when type is `checkbox`, `radio`, `select`, `hidden`, `state`, `country`
- `TextInput` for `default_value` ‚Äî shown only when type is `hidden`
- `Toggle` for `required` ‚Äî hidden when type is `hidden`
- `Select` for `width` ‚Äî options 1‚Äì12, default 12
- `Select` for `validation` ‚Äî the 8 preset options listed above, `->live()`
- `TextInput` for `validation_regex` ‚Äî shown only when `validation` is `custom_regex`
- `TextInput` for `validation_message` ‚Äî shown only when `validation` is `custom_regex`
- `Repeater` for `options` (nested) ‚Äî shown only when type is `select` or `radio`. Each item has `value` (TextInput) and `label` (TextInput).

Form-level settings section (separate from the field repeater):

- `TextInput` for `settings.submit_label` ‚Äî default `'Submit'`
- `Textarea` for `settings.success_message` ‚Äî default `'Thank you. Your message has been received.'`
- `Toggle` for `settings.honeypot` ‚Äî label "Enable honeypot spam protection", default on

Top-level fields: `title` (required), `handle` (required, unique, auto-slugified), `description` (nullable Textarea), `is_active` (Toggle, default true).

**Table columns**: title, handle, is_active (badge), submissions count (using `withCount`), updated_at.

**Table actions**: edit, delete (with confirmation). Bulk delete.

### Submissions sub-resource or relation manager

Add a `FormSubmissionsRelationManager` to the Form resource. Read-only ‚Äî no create, no edit. Columns: `created_at`, `ip_address`, and one column per field handle rendered from `data`. Since field count is variable, show `created_at`, `ip_address`, and a "View" action that opens a modal displaying all submission key/value pairs. Delete action on each row (for GDPR removal).

## Part 4 ‚Äî Public rendering

### `app/Http/Controllers/FormSubmissionController.php`

One method: `store(Request $request, string $handle)`.

1. Find the form by handle where `is_active = true`. 404 if not found.
2. If honeypot is enabled in settings, check for a `_hp` field in the request. If it is non-empty, silently return the success response without saving ‚Äî do not reveal to the bot that it was caught.
3. Validate using `$form->fieldValidationRules()`. Return validation errors as JSON (for fetch submissions) or redirect back with errors (for standard form POST).
4. Store a `FormSubmission` with `data` = filtered request values (only keys matching field handles), `ip_address` from request.
5. Return the success message from `settings.success_message`.

### Route

```php
Route::post('/forms/{handle}', [FormSubmissionController::class, 'store'])
    ->name('forms.submit')
    ->middleware('throttle:10,1');
```

The `throttle:10,1` middleware limits to 10 submissions per minute per IP.

### `resources/views/components/public-form.blade.php`

Accepts a `handle` attribute. Looks up the form, returns nothing if not found or inactive.

Renders a `<form>` with `method="POST"` and `action="{{ route('forms.submit', $form->handle) }}"`. Includes `@csrf`.

If `settings.honeypot` is true, renders a hidden field:

```html
<div style="display:none" aria-hidden="true">
    <input type="text" name="_hp" value="" tabindex="-1" autocomplete="off">
</div>
```

Renders each field in a `.form-grid` div. Each field is wrapped in a `<div class="col-{{ $field['width'] }}">`. Field rendering by type:

- `text`, `email`, `tel`, `number` ‚Üí `<input type="{type}">` with label, placeholder, required, and `pattern` attribute if a regex is set
- `textarea` ‚Üí `<textarea>` with label
- `select` ‚Üí `<select>` with `<option>` for each option
- `radio` ‚Üí `<fieldset>` with a `<legend>` and one `<label><input type="radio">` per option
- `checkbox` ‚Üí `<label><input type="checkbox"></label>`
- `state` ‚Üí `<select>` pre-populated with US states (full name as label, 2-letter abbreviation as value)
- `country` ‚Üí `<select>` pre-populated with curated country list (US, CA, GB, AU at top, then full alphabetical list)
- `hidden` ‚Üí `<input type="hidden" value="{{ $field['default_value'] ?? '' }}">`

After the fields, renders the submit button using `settings.submit_label`.

After a successful submission (detect via session flash or Alpine state), renders the success message from `settings.success_message` in place of the form.

All inputs use Pico's native form styling ‚Äî no extra classes needed.

### SCSS

Add to `resources/scss/_layout.scss`:

```scss
.form-grid {
    display: grid;
    grid-template-columns: repeat(12, 1fr);
    gap: 1rem;
}

@for $i from 1 through 12 {
    .col-#{$i} { grid-column: span #{$i}; }
}

@media (max-width: 768px) {
    [class^="col-"] { grid-column: span 12; }
}
```
