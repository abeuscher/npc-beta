## What was built

### Phase 1 ŌĆö Media hygiene

- **`media:prune-orphans` ŌĆö closed the documented "wholesale-wipe gap"** (inbox `[0.336.01]`). Content-addressed media files under `storage/app/public/cas/<hash[0:2]>/<hash>/` are only removed by the refcounted `ContentAddressedFileRemover`, which fires on a model `->delete()`; bulk row clears that bypass Eloquent events ŌĆö `migrate:fresh --seed`, `demo:reset`, scrub regeneration, Playwright `resetDatabase()` ŌĆö truncate the `media` table without firing it, orphaning the files. `media-library:clean` does not cover this (it prunes orphan rows + stale conversions, not whole tree dirs with no row).
  - New `app/Console/Commands/PruneOrphanMediaCommand.php`: builds the set of `content_hash` values still referenced by a live `media` row, walks the `cas/` tree (on the `media-library.disk_name` disk, honoring the `media-library.prefix`), and targets only 64-char-hex hash directories in no live row. **Dry-run by default** (reports count + reclaimable bytes); the destructive pass requires `--force`. Empty shard dirs left by the sweep are removed; non-hash directories are left untouched. **On-demand only ŌĆö deliberately not wired into any deploy/demo path** (local-only growth; the demo droplet restores, never seeds; and it would be inert anyway per the documented scheduler-runner gap).
  - Pest `tests/Feature/MediaPruneOrphansTest.php` (+3): a planted orphan dir is removed under `--force` while a live file is never touched; the default dry-run deletes nothing; a tree of only live files is a clean no-op.
  - Verified end-to-end on the real local tree: dry-run reported **45 orphan directories / 17.1 MB** with nothing deleted. The actual `--force` sweep was left to the operator's discretion (the command is the deliverable).
  - `docs/app-reference.md` "Orphan media sweep" section rewritten from "gap remains open" to document the shipped command.

The grid switched to `repeat(7, minmax(0, 1fr))` so it can never exceed its container; dot sizes + grid gap moved to `em` so they ride the one font lever. `cqi` resolves against the widget's own box, never the page (the reason for the inner wrapper). The session-330 anti-morph guarantee (square `aspect-ratio: 1` cells, centred) is preserved.

## Security considerations

The only file-destructive surface is `media:prune-orphans`, and it is constrained: it deletes a directory only when its hash name is referenced by **no** live `media` row (the live set is built from the DB each run), it is **dry-run by default** with the destructive pass behind an explicit `--force`, it touches only well-formed hash directories under `cas/` (legacy id-based dirs and any non-hash directory are skipped), and it is on-demand local-only ŌĆö not wired into any deploy or demo path. The Random Data Generator Organizations arm reuses the existing super-admin authorization gate and the `scrub_data` source isolation, so the wipe still touches only synthetic rows and real data is untouched. The EventMiniCalendar change is presentation-only. The dashboard-builder change is test-only. No new auth surface, no boundary surface, no contract change.
