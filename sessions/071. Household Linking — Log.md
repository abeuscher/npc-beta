## What was built

### Phase 1 — Schema

- Migration drops the `households` table (after first removing the FK from `contacts`), adds a self-referential `household_id UUID nullable FK → contacts.id (nullOnDelete)`, and sets `household_id = id` for all existing contacts so each starts as their own head.
- `docs/schema.md` updated: `households` section removed, `contacts.household_id` note updated to reflect the self-referential FK and solo/head semantics.

### Phase 2 — Model and admin

- `Contact` model: `household()` belongsTo (old `Household` class) replaced with `head()` (self-referential BelongsTo), `householdMembers()` HasMany added, `householdName()` method added, `created` observer sets `household_id = id` for new contacts when no household is explicitly assigned.
- `Household` model, `HouseholdPolicy`, and `HouseholdResource` (all pages + relation manager) deleted.
- `AuthServiceProvider`: Household policy registration removed.
- `ContactResource`: new collapsible "Household" section in the right rail with a searchable Select (shows "Solo — no household" placeholder); "Affiliation" section also moved to the right rail and collapsed by default; table column updated to use computed `householdName()`; `in_household` filter fixed to `whereColumn('household_id', '!=', 'id')`; eager loads updated to use `head`.
- `EditContact`: `mutateFormDataBeforeFill` masks own-head as null for the Select display; `mutateFormDataBeforeSave` defaults null back to own ID; `afterSave` syncs head's address to member when household assignment changes; `getRedirectUrl` forces a full page reload after save so address changes are immediately visible in the form.

### Phase 3 — Portal

- `portal-account-dashboard` widget: shows computed household name (read-only) when the authenticated member is not the head.
- `portal-contact-edit` widget: non-head members see a two-choice prompt ("update just my address / leave household" or "update household address for everyone") when submitting an address change.
- `AccountController::updateAddress`: head's update cascades automatically to all members; non-head without a scope choice gets the prompt; `scope=mine` leaves household and updates address; `scope=household` updates head and cascades to all members.

### Patches

- **001** — Show email alongside name in the household head selector to disambiguate contacts with common names.
- **002** — Fix address sync to operate on `$this->record` directly rather than a `fresh()` instance, so the form reflects updated values after save.
- **003** — Household and Affiliation sections moved to right rail and collapsed by default; "Solo — no household" placeholder on the Household Select; full page redirect on save; New Membership button removed from memberships panel header; Notes button URL fixed to pass `getKey()` explicitly; `ContactNotes::mount` type hint updated to handle Livewire route model binding passing a full Contact model.
