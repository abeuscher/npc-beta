## What was built

### Phase 1 — Media collections for Page and Event models

- Added `HasMedia` / `InteractsWithMedia` to the Page model with `post_thumbnail` and `post_header` media collections (single file each), WebP conversions and responsive breakpoints via `ImageSizeProfile`.
- Added `registerMediaCollections()` to the Event model with `event_thumbnail` collection.
- Added `SpatieMediaLibraryFileUpload` fields to PostResource (thumbnail + header) and EventResource (thumbnail) Filament forms.

### Phase 2 — Shared pager partial

- Created `resources/views/widgets/partials/pager.blade.php` as a reusable Alpine.js pagination component (retained for future use, though the listing widgets ultimately moved to Swiper pagination).
- Created `resources/scss/widgets/_pager.scss` with shared pagination row styles, numbered bullet styling, and prev/next navigation button styles used by both listing widgets.

### Phase 3 — Blog listing widget upgrade

- Expanded `blog_listing` seeder config: `content_template` (richtext with `{{token}}` pattern), `columns` (select 1–6), `items_per_page`, `show_search`, `sort_default`, `effect` (slide/fade), `background_color`, `text_color`.
- Rewrote `blog-listing.blade.php` with Swiper-based pagination: each slide is a full page of cards in a CSS grid. Server-side token replacement for card rendering. Vanilla JS Swiper init in a `<script>` block (deferred via `window.load`). Search rebuilds slides by destroying and recreating the Swiper instance.
- Created `resources/scss/widgets/_blog-listing.scss`.

### Phase 4 — Events listing widget upgrade

- Same architecture as blog listing: Swiper page-grid slides, token replacement, search/sort, numbered pagination.
- Event-specific tokens: `{{location}}`, `{{price_badge}}`, `{{ends_at}}`.
- Created `resources/scss/widgets/_events-listing.scss`.

### Phase 5 — PageContext and WidgetDataResolver updates

- `PageContext::posts()` now eager-loads `media` for thumbnail access.
- `PageContext::upcomingEvents()` now eager-loads `media` and `landingPage`.
- `WidgetDataResolver::resolveBlogPosts()` includes `thumbnail_url`.
- `WidgetDataResolver::resolveEvents()` includes `thumbnail_url` and eager-loads `media`.

### Design evolution during session

- **List mode removed:** the list/grid toggle was replaced with a columns-per-row select (1–6), since a 1-column grid achieves the same result and keeps the content template WYSIWYG consistent.
- **Pagination moved from Alpine to Swiper:** equal-height cards across the entire set required Swiper's slide-based layout. Each slide contains a CSS grid "page" of cards. Swiper handles prev/next and numbered pagination dots via `renderBullet`. Search destroys and recreates the Swiper instance with filtered/chunked slides.
- **Pager nav row:** prev/next buttons pulled out of the Swiper container into a flex row alongside numbered pagination for natural flow positioning.
- **Content template defaults:** Quill-compatible HTML only (`<p>`, `<h3>`, `<h4>`, `<a>`) — no `<div>` wrappers or CSS classes, since Quill strips non-standard block elements. Templates include all available tokens so admins can see what's available and remove what they don't need.

## Production deployment notes

After deploying, run:

```bash
docker exec nonprofitcrm_app php artisan db:seed --class=WidgetTypeSeeder --force
docker exec nonprofitcrm_app php artisan build:public
```

Existing blog_listing and events_listing widget instances will use the hardcoded fallback template if their stored `content_template` is empty. To populate the default template into existing instances, delete and re-add the widgets on their pages.
