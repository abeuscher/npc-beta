### D3 — Mutation testing slice (Fleet Manager controllers)

- **Slice:** the four Fleet API controllers (`Health`, `Logs`, `Backup`, `Recovery`) + the `AdminAccountRecovery` / `AdminRecoveryResult` support services (~840 LOC), scoped via `infection.json5` `source.directories` + `excludes`. Chosen at the gate as the queue's most-grown, security-adjacent slice (v2.1.0 → v2.5.0). `timeout` raised to 120 in the committed config (the catcher-set run executes every covering test per mutant).
- **Result: 326 mutants, MSI 100% — 0 escaped, 0 uncovered.** The suite genuinely constrains the slice. (Across runs: 315–320 killed by tests, 4–9 timeouts — all the log-tailer's loop mutants, caught-by-timeout — and 2 tooling-artifact "errored" mutants where `IncrementInteger` turns `break 2` into an invalid break level.)
- **No deletions or consolidations.** Per-test catcher attribution (read from each killed mutant's `processOutput`, since `killedBy` isn't populated in this shim setup) proved too noisy to act on: Pest truncates failure-summary titles and attribution jitters run-to-run. No slice test is dead — the zero-catch entries are route/throttle-shape tests whose subject isn't mutated, slow-group tests excluded from the initial run, or ambiguous-attribution artifacts.
- **Applied: 10 `// guards:` markers** on the contract spec-mirror tests flagged fully-redundant in both trustworthy runs (5 in `HealthEndpointTest`'s envelope/version cluster, 1 `LogsControllerTest` envelope, 4 `BackupBlobEndpointTest` streaming clauses) — deliberate belt-and-suspenders keeps, marked so future passes skip them.
- **Two consolidation candidates flagged, not acted on** (recorded in the inbox paper trail): two `HealthEndpointTest` near-duplicate pairs (unknown→yellow at endpoint vs healthy-env; DB-failure red).
- **Workflow doc updated** (`docs/testing/mutation-audits.md`) with three operational findings the full-suite scale forced: narrow the initial coverage run to slice-consumer tests via `--test-framework-options` (the full-suite PCOV run now dies mid-run at ~3,000-test scale with a misleading "must be in a passing state"); raise the mutant `timeout` for catcher-set runs; and the catcher-attribution limits (title truncation not fixable via `--columns`/`COLUMNS`; jitter between runs → never delete on catcher evidence alone, require two-run agreement + a source read, prefer markers over deletions). The Fleet queue entry was struck as consumed.

- **Seven update-verdict files rewritten/trimmed**, all following the A005 / ContactExportTest precedent (route through the real production path, don't re-implement it inline):
  - **`StripePaymentMethodsTest`** — five inline-`array_intersect` reimplementation tests replaced by five routed through the real `StripeCheckoutService::buildParams()`; the two endpoint-validation tests and the SiteSetting round-trip pair kept.
