### Phase 1 â€” Image widget audit + aspect_ratio + max_width

- **Audit finding:** Image widget had no aspect-ratio enforcement at all. `template.blade.php` rendered the source image at its intrinsic ratio (`max-width: 100%; height: auto;`). `object_fit: cover/contain` was effectively inert because nothing constrained the container's ratio. All three home-layout-spec ratios (4:3, 1:1, 4:5) "rendered cleanly" only in the sense that they delivered whatever ratio the source asset carried.
- **User direction at audit handoff:** absorb the fix into 285 since the CSS solution is small.
- **Schema add:** `aspect_ratio` select with seven options â€” `auto` (default; preserves current intrinsic-ratio behavior) plus `1:1 / 4:3 / 3:2 / 16:9 / 4:5 / 3:4`. Plus a `max_width` text field accepting any CSS length (`400px / 60% / 20rem`); bare numbers are interpreted as pixels. Strict regex on the parsed value at render time rejects anything outside `px / % / rem / em / vw / vh`.
- **Template:** ratio class (`widget-image--ratio-{n}-{n}`) appended to the existing class list when set; `max-width` inline style applied when valid. `<x-picture>` component passes the class + style through to the inner `<img>`.
- **SCSS:** six new ratio classes in `app/Widgets/Image/styles.scss` using the CSS `aspect-ratio` property + `width: 100%`. `object_fit: cover/contain` becomes meaningful at the picked ratio (cover = crop to fit; contain = letterbox).
- **8 new Pest tests** covering schema registration, every ratio option, invalid ratio rejection, every valid CSS unit, bare-number-to-px coercion, invalid `max_width` silent rejection, and the no-style-attribute path.

## Security considerations

Two changes touched user-visible content paths:

- **TextBlock CTA URLs** route through the same `widget-shared.buttons` partial Hero uses. The partial `e()`-escapes `href` and detects external links via a host-comparison check â€” no new attack surface.
- **Image widget `max_width`** is rendered into an inline `style="max-width: â€¦"` attribute. The render-time regex (`/^[0-9]+(\.[0-9]+)?(px|%|rem|em|vw|vh)$/`) rejects any value that could carry a CSS expression, URL, or quote-escape; bare numbers coerce to pixels. A dedicated Pest case verifies `javascript:`, semicolon-suffixed values, and whitespace-bearing values all drop silently.

No schema, auth, or boundary-side change.
