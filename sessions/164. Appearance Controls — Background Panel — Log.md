## What was built

### Phase 1 — Composer extraction and renderer refactor

- Created `App\Services\AppearanceStyleComposer` with `compose(PageWidget $pw): array` returning `['inline_style', 'is_full_width']`. Handles background color, text color, padding, margin, full-width resolution (including column-child override).
- Added `registerMediaCollections()` to `PageWidget`, registering `appearance_background_image` as a single-file Spatie collection.
- Refactored all four renderers to delegate inline-style computation to the composer:
  - `PageBuilderApiController::renderWidgetForPreview()` — removed `buildInlineStyles()`.
  - `PageBuilder::renderWidgetForPreview()` (Livewire) — removed `buildInlineStyles()`.
  - `PageController::renderWidgetBlock()` and `renderLayoutBlock()`.
  - `PagePreviewController::renderWidgetBlock()` and `renderLayoutBlock()`.
- Block arrays now carry pre-computed `inline_style` instead of raw `appearance_config`. `page-widgets.blade.php` simplified to use it directly.

### Phase 2 — Background panel, gradient alpha, upload endpoints

- Extended `GradientComposer` with per-stop alpha (`from_alpha`/`to_alpha`, int 0..100, default 100). Emits `rgba(...)` when alpha < 100, preserves `#hex` when opaque. Hex-to-RGB supports both 3- and 6-digit hex. `CSS_OVERRIDE_PATTERN` stays hex-only.
- Mirrored the rgba logic in the TypeScript `composeGradientCss` helper (`gradient.ts`). Added `from_alpha`/`to_alpha` to the `GradientLayer` type.
- Extended `GradientPicker.vue` with per-layer From/To opacity sliders. All 7 presets default to opaque. 8th preset slot is a "remove gradient" button with a diagonal red line. Removed separate "Clear gradient" button. Removed the preview panel (trigger swatch and background panel preview serve as previews). Reorganized controls into paired rows: [From color + opacity], [To color + opacity], [Type + Angle], [CSS override]. Removed intermediate "create gradient" state — opening the picker shows presets immediately; selecting one creates the gradient.
- Extended `AppearanceStyleComposer` with gradient (via `GradientComposer`), image (via `getFirstMedia`), 9-point alignment lookup, fit allowlist, and `background-repeat: no-repeat`. Composes `background-image: GRADIENT, url(IMAGE)` shorthand with position and size only when at least one of gradient/image is present.
- Added `appearance_image_url` to both `formatWidget` and `formatWidgetWithPreview` responses (top-level, server-authoritative).
- Added `POST` and `DELETE` routes for `widgets/{widget}/appearance-image` in `routes/admin-api.php`. Controller methods mirror existing `uploadImage`/`removeImage` but write to `appearance_background_image` collection without mutating `appearance_config`. Uses `hasGeneratedConversion('webp')` fallback for immediate upload response.
- Added `uploadAppearanceImage` and `removeAppearanceImage` actions to the editor store and API module.
- Added `appearance_image_url` to the `Widget` TypeScript type.
- Created `BackgroundPanel.vue` under `components/appearance/`. Sections: Color (ColorPicker), Gradient (GradientPicker + helper line), Image (upload tile, thumbnail, remove, replace, NinePointAlignment, fit select). Image controls disabled when no image present. Mounted at the top of the Appearance tab in `InspectorPanel.vue`.

### UI/UX refinements (during UAT)

- **ColorPicker:** changed default placeholder from "No color set" to "Transparent". Reduced trigger text to 0.65rem, caret to 0.65rem with -8px margin-left. Added `flex-shrink: 0` to color wheel. Neutralized browser shadow DOM border on `<input type="color">` via `::-webkit-color-swatch` / `::-moz-color-swatch` pseudo-elements. Reduced wheel to 1.5rem (user change).
- **GradientPicker:** color picker popover stretches to full row width via `:deep()` override (`width: calc(200% + 0.5rem)`).
- **NinePointAlignment:** reduced control to 2.2rem.
- **BackgroundPanel:** reordered image controls to [Fit][Alignment] row.

### Shared inspector styles extraction

- Created `resources/js/page-builder-vue/styles/inspector.css` — single source of truth for shared form-control styles: `.inspector-label`, `.inspector-section-title`, `.inspector-control` (+ `--sm`, `--mono`), `.inspector-hint` (+ `--italic`), `.inspector-toggle-label`, `.inspector-checkbox`, `.inspector-range`.
- Imported once in `main.ts`. Updated 14 components to use shared classes, removing duplicated CSS. CSS bundle dropped ~4 KB.
- Added dark mode rules in `public/css/admin.css` for all shared classes, plus `bg-panel__heading`, color picker trigger/popover, gradient picker trigger/panel, `bg-panel__replace-btn`, and nine-point alignment SVG.

## Security checklist — verified

- [x] All color values pass through the hex pattern validator before being written to inline styles or rgba conversions.
- [x] All spacing values are cast to int before being written to inline styles.
- [x] Gradient alpha values are cast to int and clamped to `[0, 100]` before division.
- [x] Image URLs come exclusively from `Media::getUrl()` / `Media::getUrl('webp')` — never from user-controllable strings.
- [x] Background `fit` value is validated against the `{cover, contain}` allowlist.
- [x] 9-point `alignment` value is validated against the alignment allowlist.
- [x] Gradient CSS is composed exclusively via `App\Services\GradientComposer::compose()`.
- [x] The new upload endpoint requires `update_page` permission and validates `mimes` + `max` on the file.
- [x] The new delete endpoint requires `update_page` permission.
- [x] Column-child widgets (`widget.layout_id !== null`) cannot bypass the parent column's width by setting `appearance_config.layout.full_width = true`. The composer enforces this defensively.
- [x] `formatWidget`'s new `appearance_image_url` field does not leak any URL outside the `Media::getUrl()` path.
- [x] The `AppearanceStyleComposer` refactor does not regress any of the existing safety checks.
