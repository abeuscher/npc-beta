# NonProfitCRM

NonProfitCRM is an open source CRM and CMS platform built specifically for nonprofit organizations. It combines a Laravel application backend with Twill (Area 17) as the content management layer and Filament PHP as the CRM admin panel, providing a single, self-hosted installation per client with full contact management, membership tracking, donation and grant management, event registration, and content publishing — without WordPress, without multi-tenancy, and without shared infrastructure risk.

## Requirements

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) — the only local dependency. All other services (PHP, PostgreSQL, Redis, Nginx) run inside containers.

---

## Installation

```bash
git clone https://github.com/your-org/nonprofitcrm.git
cd nonprofitcrm
cp .env.example .env
# Fill in DB_PASSWORD and any API keys in .env
./dev up
./dev artisan key:generate
./dev artisan migrate
./dev artisan twill:superadmin
```

The app will be available at `http://localhost`.
The admin panel is at `http://localhost/admin`.

---

## Configuration

Copy `.env.example` to `.env` and set the required values. The minimum required before first run:

- `DB_PASSWORD` — PostgreSQL password (you choose this)
- `APP_KEY` — generated by `./dev artisan key:generate`

All other values (Stripe, Mailchimp, QuickBooks) are optional until those features are used.

## Development

```bash
./dev up                        # start all services
./dev down                      # stop all services
./dev artisan migrate           # run migrations
./dev artisan make:model Foo    # generate files
./dev composer require pkg/name # add a package
./dev shell                     # bash inside the container
./dev fresh                     # wipe DB and re-migrate
```
