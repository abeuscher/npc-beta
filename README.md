# NonprofitCRM

An open source CRM and CMS platform built specifically for small and mid-sized nonprofit organizations. Self-hosted, single-tenant, no recurring software fees.

Built on Laravel + Filament, with PostgreSQL and Docker. One installation per client. You own your data.

---

## What it does

**CRM**
- Contacts, households, and organizations — with a flexible type system and custom fields
- Membership tracking with status and renewal dates
- Notes attached to any record
- Tags for flexible grouping and filtering
- CSV import and export with field mapping and import history
- Importer enforces automatic rejection of files containing sensitive financial and personal identifiers (credit card numbers, SSNs, bank routing numbers, driver's licence numbers); a developer bypass flag (`IMPORTER_SKIP_PII_CHECK`) is available for local testing only

**Finance**
- Donations, transactions, funds, and campaigns
- Grant tracking and fund allocation

**Events**
- Event creation and management
- Public-facing registration pages
- Registrant export and email notifications

**CMS**
- Page builder with a block-based layout system
- Blog posts with rich text (Quill editor)
- Navigation management with slug control
- Content collections for grouping pages and posts
- Public routing unified under a single slug resolver

**Administration**
- Role-based access control with fine-grained permissions
- User management
- Context-sensitive help system (Markdown source, searchable via PostgreSQL)
- Custom field definitions for contacts, events, and pages

## Requirements

[Docker Desktop](https://www.docker.com/products/docker-desktop/) is the only local dependency. PHP, PostgreSQL, Redis, and Nginx all run inside containers.

## Installation

```bash
git clone https://github.com/your-org/nonprofitcrm.git
cd nonprofitcrm
cp .env.example .env
# Set DB_PASSWORD and any optional API keys in .env
./dev up
./dev artisan key:generate
./dev artisan migrate
./dev artisan db:seed
./dev artisan make:filament-user
```

The app will be at `http://localhost`.
The admin panel is at `http://localhost/admin`.

---

## Configuration

Minimum required values in `.env` before first run:

- `DB_PASSWORD` — PostgreSQL password (you choose this)
- `APP_KEY` — generated by `./dev artisan key:generate`

Everything else (Stripe, Mailchimp, QuickBooks) is optional and only needed when those integrations are used.

## Development

```bash
./dev up                        # start all services
./dev down                      # stop all services
./dev artisan migrate           # run migrations
./dev artisan make:model Foo    # generate files
./dev composer require pkg/name # add a package
./dev shell                     # bash shell inside the PHP container
./dev fresh                     # wipe the database and re-migrate
```

### Help system

Help articles live in `resources/docs/` as Markdown files with YAML frontmatter. After editing or adding files, sync them to the database:

```bash
./dev artisan help:sync
```

Frontmatter format:

```yaml
---
title: Managing Contacts
description: How to add, edit, import, and export contacts.
version: "0.24"
updated: 2026-03-16
tags: [contacts, import, export]
routes:
  - filament.admin.resources.contacts.index
  - filament.admin.resources.contacts.create
---
```

## Deployment

Production targets any Docker-compatible host (Fly.io, Render, VPS with Docker Compose). Stateful services (PostgreSQL, Redis) should use managed providers in production rather than containerized instances.

Deployment is handled by a GitHub Actions workflow triggered on push to `main`. Three secrets must be configured in the GitHub repository:

| Secret | Purpose |
|---|---|
| `DEPLOY_HOST` | IP address or hostname of the production server |
| `DEPLOY_USER` | SSH user on the production server |
| `DEPLOY_SSH_KEY` | Private SSH key for authentication |

All other configuration (app key, database credentials, API keys, mail settings) lives in a `.env` file on the server and is not managed through GitHub secrets.
